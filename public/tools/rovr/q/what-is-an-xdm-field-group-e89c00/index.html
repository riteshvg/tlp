What is an XDM field group?

What Is an XDM Field Group?

An XDM field group (also called a "schema field group" or "mixin") is a reusable collection of related fields that you can add to an Experience Data Model (XDM) schema. Field groups standardize how specific types of data are structured and named across Adobe Experience Platform and downstream services.

Core Purpose

Field groups allow you to:

  • Reuse common data structures — Instead of defining the same fields repeatedly, add a pre-built field group to any schema that needs those fields.
  • Maintain consistency — All schemas using the same field group represent data the same way, enabling better data integration and analysis across AEP services.
  • Speed up schema creation — Start with Adobe's pre-defined field groups rather than building from scratch.
  • Ensure compatibility — Use Adobe's recommended field groups (like AEP Web SDK ExperienceEvent and Consumer Experience Event) to ensure your data works seamlessly with Web SDK, Analytics, Target, and other Adobe services.

Pre-defined vs. Custom Field Groups

Pre-defined Field Groups — Adobe provides industry-standard field groups that cover common use cases (web events, commerce transactions, customer profiles, etc.). These are product-agnostic and follow naming conventions that work across all Adobe services.

Custom Field Groups — When your organization has data that doesn't fit into pre-defined groups, you can create your own field group to capture those unique attributes and behaviors.

Common Pre-defined Field Groups for Web Data

When setting up XDM schemas for web data collection, Adobe recommends adding these two field groups:

1. AEP Web SDK ExperienceEvent — Captures standard web interaction data (page views, clicks, scrolls, form submissions, etc.) in a format optimized for the Web SDK.

2. Consumer Experience Event — Provides additional commerce and behavioral fields for tracking customer interactions across touchpoints.

How to Add a Field Group to Your Schema

1. Open the Schema Editor in the Adobe Experience Platform UI.

2. Select your schema (or create a new one).

3. In the Field groups section, click Add.

4. Search for the field group by name (e.g., `AEP Web SDK ExperienceEvent`).

5. Check the box next to the field group.

6. Click Add field groups to include it in your schema.

Once added, all the fields within that field group become part of your schema and can be populated with data.

Key Takeaway

Field groups are the building blocks of XDM schemas. They bundle related fields together so that your customer experience data follows standardized structures—enabling consistent data collection, integration, and activation across Adobe Experience Platform and all downstream analytics, personalization, and orchestration tools.
