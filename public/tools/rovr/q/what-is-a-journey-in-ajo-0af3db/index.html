What is a Journey in Adobe Journey Optimizer?

A journey is an automated, multistep customer experience that orchestrates personalized interactions across channels in response to customer behavior, business events, or scheduled campaigns.

Core Definition

Journeys in Adobe Journey Optimizer empower you to create personalized, multistep customer journeys that adapt in real-time to your audience's behavior and needs. Using an intuitive drag-and-drop canvas, you can orchestrate messages and actions across multiple channels, leveraging contextual data and audience targeting for maximum impact.

How Journeys Work

1:1 Real-Time Orchestration

A journey is a canvas where each profile travels their own path at their own pace. AJO tracks where each person is in the flow and reacts in real time to their behavior — whether that's an action they take, a period of inactivity, or a change in their profile.

Example — Cart Abandonment Journey:

```

Customer A: Abandoned cart → Wait 2 hours → No purchase? → Send reminder → Purchased? → End

Customer B: Abandoned cart → Wait 2 hours → Already purchased → End immediately

```

Each customer experiences a different timeline based on what they actually do.

Key Journey Capabilities

  • Wait activities that create personalized timing between steps
  • Conditional branching that routes profiles to different paths
  • Frequency capping to control how often a customer receives messages
  • Test mode to validate logic before going live
  • A/B experiments that split profiles into percentage-based groups for testing
  • Multi-channel actions — email, SMS, push, in-app, web, code-based experiences, and more

Four Types of Journeys

The type of journey depends on what triggers it:

1. Unitary Event Journeys

  • Triggered by a specific profile behavior (e.g., a purchase, app sign-in)
  • Profiles enter one at a time when the event occurs
  • Best for: Behavior-triggered experiences

2. Read Audience Journeys

  • Start with an audience from Adobe Experience Platform
  • Send messages in batch to all profiles in that audience
  • Best for: Scheduled campaigns, newsletters, seasonal outreach

3. Audience Qualification Journeys

  • Triggered when profiles qualify for (or exit from) a specific audience segment
  • Profiles enter as they meet the audience criteria
  • Best for: Milestone-based experiences like loyalty tier changes

4. Business Event Journeys

  • Triggered by business events (e.g., stock updates, weather alerts) that affect multiple profiles simultaneously
  • Best for: System-wide events that impact many customers

Journey vs. Campaign

| Aspect | Journeys | Campaigns |

|--------|----------|-----------|

| Execution | 1:1 orchestration — each profile at their own pace | Batch execution — audience processed together |

| Logic | Complex, multi-step with conditions and waits | Simpler, more direct messaging |

| Timing | Real-time, event-driven | Scheduled or triggered by API |

| Use Case | Multi-touch customer lifecycle experiences | Scheduled activations or one-time sends |

Journey Designer Interface

The interface includes:

  • Palette — Activities and actions to add to your journey
  • Canvas — Visual representation of your journey flow
  • Properties pane — Configuration details for each element

Basic Journey Creation Steps

1. Define a journey entry point — An audience segment or a real-time event

2. Add message actions across channels — Email, push, SMS, in-app, web, and more

3. Test your journey with test profiles before activation

4. Publish your journey and monitor its performance
