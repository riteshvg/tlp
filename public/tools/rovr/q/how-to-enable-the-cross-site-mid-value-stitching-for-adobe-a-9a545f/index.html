Cross-Device Analytics (CDA) – Enabling Stitching in Adobe Analytics

Cross-Device Analytics (CDA) supports field-based stitching, which uses an Analytics variable (such as a prop or eVar) containing a person identifier to link devices together. Here is a complete guide to enabling it.

Prerequisites

Before enabling CDA, ensure the following are in place:

1. Existing customer eligibility – CDA is available to existing Adobe Analytics customers (not new contracts by default).

2. A cross-device report suite – You must have a report suite that collects data across devices.

3. Person identifier variable – A `prop` or `eVar` must be consistently populated with a person/user identifier (e.g., a hashed login ID) on your implementation.

4. Admin privileges – You must belong to a Product Profile Admin group or have full admin privileges in Adobe Analytics.

5. All prerequisites for your chosen stitching method must be met – Review the requirements for field-based stitching specifically.

Steps to Enable Cross-Device Analytics

Step 1 – Contact Adobe Customer Care

Open a support ticket with Adobe Customer Care to have CDA provisioned on your cross-device report suite. This is a required first step — CDA cannot be self-enabled without Adobe provisioning it on the backend.

Step 2 – Verify Your Implementation

Ensure the Analytics variable you intend to use as the person identifier (e.g., an eVar) is:

  • Consistently populated when users authenticate or are identifiable
  • Passing a stable, unique identifier (e.g., hashed email or CRM ID)

Step 3 – Configure a Virtual Report Suite

Once Adobe has provisioned CDA on your report suite:

1. Navigate to Analytics > Components > Virtual Report Suites

2. Click Create to create a new Virtual Report Suite

3. Select the base cross-device report suite

4. Enable the Cross-Device Analytics option within the Virtual Report Suite settings

5. Configure the stitching field by selecting the prop or eVar that contains your person identifier

6. Save the Virtual Report Suite

Step 4 – Validate Stitching

After the Virtual Report Suite is created:

  • Allow time for data processing (stitching is applied retroactively up to the lookback window allowed by your contract)
  • Use the Cross-Device panels and segments in Analysis Workspace to validate that devices are being correctly stitched together

Key Considerations

| Aspect | Detail |

|---|---|

| Stitching Method | Field-based stitching uses a prop/eVar as the person identifier |

| Recommended For | More transparency and predictability in visitor tracking |

| Retroactive Processing | Stitching can be applied retroactively within your contract's lookback window |

| Access | Analysis is done through a CDA-enabled Virtual Report Suite |

Important Notes

  • If all prerequisites are not met, the feature will not be available or will not function correctly.
  • Field-based stitching is the recommended approach as it offers greater transparency and predictability.
  • CDA stitching is separate from the stitching capabilities available in Customer Journey Analytics (CJA) — if you are on CJA, the stitching configuration process differs.
