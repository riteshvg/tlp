What are the steps required to connect AJO to AEP?

Connecting Adobe Journey Optimizer to Adobe Experience Platform

Adobe Journey Optimizer is natively built on top of Adobe Experience Platform — it is not a separate system that requires an explicit "connection." AJO uses AEP's underlying infrastructure (Profiles, Schemas, Datasets, Segments, Identity Service, and Datastreams) directly. However, there are several configuration steps required to ensure AJO can properly read from and write to AEP, and that data flows correctly between your data collection layer and both platforms.

Understanding the Relationship

| AEP Component | How AJO Uses It |

|---|---|

| XDM Schemas | Defines the structure of profile and event data used in journeys |

| Datasets | Stores ingested profile and event data that AJO reads |

| Real-Time Customer Profile | Powers journey personalization and eligibility |

| Identity Service | Resolves customer identities across touchpoints |

| Audiences / Segments | Used as journey entry conditions or action filters |

| Datastreams | Routes Web SDK / Mobile SDK data to AEP and AJO |

Key Configuration Areas

1. Schema and Dataset Setup in AEP

Before AJO can act on customer data, the underlying data structures must exist in AEP:

1. Navigate to Data Management → Schemas in AEP

2. Create or confirm existence of:

  • An XDM Individual Profile schema (for profile attributes)
  • An XDM Experience Event schema (for behavioral/event data)

3. Enable both schemas for Real-Time Customer Profile

4. Create corresponding Datasets and enable them for Profile ingestion

5. Ingest your customer data into these datasets via batch or streaming

2. Identity Namespace Configuration

AJO needs to resolve and stitch customer identities correctly:

1. Navigate to Customer → Identities in AEP

2. Confirm the identity namespaces you will use are configured (e.g., Email, ECID, CRM ID)

3. Ensure your schemas have the correct fields marked as Identity and that one is set as the Primary Identity

3. Datastream Configuration (for Web/Mobile Data Collection)

If you are collecting data from websites or mobile apps, a Datastream connects Adobe Data Collection to AEP and AJO:

1. Navigate to Data Collection → Datastreams in AEP

2. Click New Datastream

3. Select your XDM Event Schema

4. Enable the following services on the datastream:

  • Adobe Experience Platform — to send data to AEP datasets and Profile
  • Adobe Journey Optimizer — to enable AJO to receive inbound events and deliver offer decisioning responses

5. Save and note the Datastream ID for use in your Web SDK or Mobile SDK configuration

4. Web SDK / Mobile SDK Configuration

To stream real-time behavioral data from your digital properties:

1. Install the Adobe Experience Platform Web SDK extension via Adobe Experience Platform Tags (Data Collection UI)

2. Configure the extension with your Datastream ID from Step 3

3. Implement `sendEvent` calls on key interactions (page views, form submissions, purchases)

4. Ensure the event payloads conform to your XDM schemas

5. Journey Optimizer Channel Configuration

Within Journey Optimizer itself, configure the channels AJO will use to communicate with customers:

1. Navigate to Administration → Channels in Journey Optimizer

2. Configure the channels you need:

  • Email — set up sending domain, from address, IP pool
  • SMS — connect to an SMS provider (Sinch, Twilio, Infobip)
  • Push — connect to your mobile app credentials (APNs, FCM)

3. Ensure each channel configuration points to the correct profile field for the delivery address (e.g., `profile.personalEmail.address`)

6. Event Configuration in Journey Optimizer

For event-triggered journeys, configure custom events that listen for AEP streaming events:

1. Navigate to Administration → Configurations → Events in Journey Optimizer

2. Create a Unitary Event linked to your XDM Experience Event schema

3. Define the event condition that uniquely identifies this event type

4. This event will fire automatically when matching data is streamed into AEP via your Datastream

7. Audience / Segment Sharing

For audience-based journeys (Read Audience journeys):

1. Build audiences in AEP → Segmentation

2. These audiences are automatically available in Journey Optimizer under Journeys → Audiences

3. Use them as entry sources in Read Audience journey types or as conditions within journeys

Summary Flow

```

Website / App

↓

Web SDK / Mobile SDK

↓

Adobe Data Collection (Tags)

↓

Datastream (AEP + AJO services enabled)

↓

AEP (Schemas → Datasets → Real-Time Profile → Audiences)

↓

Journey Optimizer (Events → Journeys → Channels → Delivery)

```

Important Notes

  • No separate connector or API integration is needed between AJO and AEP — they share the same underlying platform
  • Sandbox alignment is critical — ensure AJO and all AEP components are configured within the same sandbox
  • Access control permissions must be configured in AEP → Administration → Permissions to ensure the correct roles have access to both AEP and AJO features
  • Changes to schemas, datasets, or profiles in AEP are immediately reflected in Journey Optimizer
