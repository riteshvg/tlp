Overview

A Datastream (also called a "configuration") is the server-side configuration for the Adobe Experience Platform Web and Mobile SDKs. It acts as the connection point between your website/app and Adobe services, controlling how data flows from the Edge Network to all downstream services.

Think of a datastream as the "rules engine" that determines:

  • Where data is sent (AEP, Analytics, Target, AJO, etc.)
  • How data is transformed and mapped
  • When data is processed
  • What configurations apply to each service

Key Definition

> A datastream is a server-side configuration that tells the Adobe Experience Platform Edge Network how to process, route, and deliver customer data to Adobe services and external systems.

The Critical Distinction

This is foundational to understanding AEP architecture:

| Component | Location | Purpose |

|---|---|---|

| Web SDK Configuration | Client-side (Browser) | Handles SDK initialization settings like `edgeDomain` |

| Datastream Configuration | Server-side (Edge Network) | Handles routing, transformation, service configuration |

```javascript

// CLIENT-SIDE: Web SDK Configuration

alloy("configure", {

datastreamId: "prod-datastream-123", // Points to datastream

orgId: "ADOBE_ORG_ID@AdobeOrg",

edgeDomain: "data.example.com" // Client-side setting

});

// SERVER-SIDE: Datastream Configuration

// Defines what happens AFTER Web SDK sends data to Edge Network

// Set in AEP UI (not in code)

```

What Problem Does a Datastream Solve?

Before Datastreams (Legacy AppMeasurement approach):

```

Every tracking change required:

1. Modify JavaScript code on website

2. Map variables to Analytics props/eVars manually

3. Deploy code change to production

4. Wait for development cycle completion

Result: Slow, error-prone, requires constant developer involvement

```

After Datastreams (Modern Web SDK approach):

```

Tracking changes made serverside:

1. Update datastream configuration in AEP UI

2. Changes apply immediately to all Web SDK implementations

3. No website code changes required

4. No deployment cycle needed

Result: Fast, centralized, self-service configuration

```

Creating a Datastream

Step 1: Access Datastreams

1. Log in to experience.adobe.com

2. Navigate to Data Collection → Datastreams

3. Click New Datastream

Step 2: Basic Configuration

```

Datastream Name: "Retail Production"

Description: "Production datastream for retail e-commerce"

Default Environment: Production

```

Step 3: Add Services

For each service you want to use, add a configuration:

Adobe Experience Platform Configuration

```

Service: Adobe Experience Platform

Enabled: ✅ ON

Settings:

├── Sandbox: prod (or your sandbox name)

├── Event Dataset: Web Events Dataset

├── Profile Dataset: Customer Profile Dataset

├── Identity Graph Private Edge: Enabled

├── Profile Stitching: Enabled

├── Segment Qualification Events: Enabled

├── Profile Lookup: Enabled

└── Option to enable Real-Time Customer Data Platform

```

What this does:

  • Routes all Web SDK events to the specified AEP sandbox
  • Stores raw events in the Event Dataset
  • Updates customer profiles in real-time
  • Enables profile-based personalization
  • Enables journey triggering
Adobe Analytics Configuration

```

Service: Adobe Analytics

Enabled: ✅ ON

Settings:

├── Report Suite ID: retail-prod

├── Tracking Server: metrics.retailstore.com

├── SSL Tracking Server: metrics.retailstore.com (SSL)

└── Visitor ID Namespace: (auto-populated)

```

What this does:

  • Routes Web SDK events to Analytics
  • Maps XDM fields to Analytics props/eVars
  • Enables real-time Analytics reporting
  • Maintains visitor ID continuity
Adobe Target Configuration

```

Service: Adobe Target

Enabled: ✅ ON

Settings:

├── Property Token: abc123def456

├── Environment ID: production

├── Namespace: (auto-populated)

├── Third-party ID Namespace: (optional)

└── On-device Decisioning: Enabled/Disabled

```

What this does:

  • Enables real-time Target personalization
  • Returns personalization decisions in Web SDK response
  • Enables A/B testing and multivariate testing
  • Renders personalized experiences within 100ms
Adobe Journey Optimizer Configuration

```

Service: Adobe Journey Optimizer

Enabled: ✅ ON

Settings:

└── (Enabled/Disabled toggle)

```

What this does:

  • Enables journey triggering based on Web SDK events
  • Allows AJO to respond to real-time customer interactions
  • Enables event-triggered journeys and campaigns
Adobe Audience Manager Configuration

```

Service: Adobe Audience Manager (AAM)

Enabled: ⚠️ (optional, if AAM is used)

Settings:

├── Cookie Destinations: (if applicable)

├── URL Destinations: (if applicable)

└── Audience Sharing: (enable/disable)

```
