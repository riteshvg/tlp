59. How do Data Governance and Consent Policies prevent activation to destinations?

How Data Governance and Consent Policies Prevent Activation to Destinations

The Two-Layer Enforcement System

AEP has two independent policy enforcement layers that can prevent activation to destinations. Both must pass for activation to succeed.

```

Activation to Destination Initiated

↓

╔═══════════════════════════════════════════════╗

║ LAYER 1: DATA GOVERNANCE POLICIES ║

║ (Data Usage Labels + Marketing Actions) ║

║ "Is this data type allowed for this use?" ║

╚═══════════════════════════════════════════════╝

↓

✅ PASS → Continue to Layer 2

❌ BLOCK → ACTIVATION BLOCKED (Policy Violation)

↓

╔═══════════════════════════════════════════════╗

║ LAYER 2: CONSENT POLICIES ║

║ (Customer Consent Preferences) ║

║ "Did the customer consent to this use?" ║

╚═══════════════════════════════════════════════╝

↓

✅ PASS → ACTIVATION PROCEEDS

❌ FAIL → FILTER OUT non-consenting profiles

```

Layer 1: Data Governance Policy Enforcement

Data governance policies prevent activation based on what the data is (data usage labels) and how you want to use it (marketing action).

How Data Governance Enforcement Works

```

Does the audience/field have a DATA USAGE LABEL?

↓

YES → Is there a POLICY restricting this label + marketing action combination?

↓

Policy Exists? → Is activation allowed for this label + action?

↓

❌ NOT ALLOWED → BLOCK ACTIVATION

✅ ALLOWED → Proceed to Consent Policy layer

```

Example 1: PII Cannot Be Used for Behavioral Advertising

Scenario: Marketing team wants to activate an audience to Facebook (behavioral advertising platform).

The Data:

```

Audience: "High-Value Customers"

Contains: Customer email addresses

Field Label: I1 ("Directly Identifiable data")

```

The Policy:

```

Policy Name: "No PII for Behavioral Advertising"

IF: Data has label I1 (Directly Identifiable)

AND Marketing action = Behavioral Advertising

THEN: BLOCK activation (no exceptions)

```

What Happens During Activation:

1. Marketer clicks "Activate to Facebook"

2. AEP system checks:

  • "Does this audience contain fields with I1 label?" → ✅ YES (email)
  • "Is there a policy blocking I1 + Behavioral Advertising?" → ✅ YES

3. Policy evaluation result: ❌ VIOLATION FOUND

4. Activation is BLOCKED

Error Message Shown:

```

⚠️ POLICY VIOLATION DETECTED

Activation Cannot Proceed

Policy Violated: "No PII for Behavioral Advertising"

Issue: Field 'personalEmail.address' has label 'I1' (Directly Identifiable)

Cannot be used for marketing action: 'Behavioral Advertising'

Resolution Options:

├── ✅ Remove the email field from activation mapping

├── ✅ Activate to a different destination type (non-advertising)

├── ✅ Request policy exception from Compliance team

└── ✅ Remove the I1 label (if legally allowed)

```

What the Marketer Must Do:

1. Option A: Remove email from the mapped fields

```

Before: [email, name, age] → Facebook

After: [age, interests] → Facebook (non-PII fields only)

```

2. Option B: Choose a different destination type

```

Instead of Facebook (behavioral ads) → Use Adobe Target (on-site personalization)

```

3. Option C: Request policy exception

```

Contact: Legal/Compliance team

Reason: "Need to sync email for customer service use case"

Approval: Legal approval required

```

Example 2: Contractual Export Restrictions

Scenario: Data analyst wants to export customer data to Snowflake (data warehouse).

The Data:

```

Dataset: Customer_Master

Contains: Customer profiles, transaction history, preferences

Dataset Label: C2 ("Data cannot be exported outside of AEP")

```

The Policy:

```

Policy Name: "No Data Export Outside AEP"

IF: Data has label C2

AND Marketing action = Data Export

THEN: BLOCK activation (contractual restriction)

```

What Happens During Activation:

1. Analyst clicks "Activate to Snowflake"

2. AEP system checks:

  • "Does this dataset have C2 label?" → ✅ YES
  • "Is there a policy blocking C2 + Data Export?" → ✅ YES

3. Policy evaluation result: ❌ VIOLATION FOUND

4. Activation is BLOCKED

Error Message Shown:

```

⚠️ CONTRACTUAL RESTRICTION

Data Export Blocked

Policy Violated: "No Data Export Outside AEP"

Issue: Dataset 'Customer_Master' has label C2

Contractually cannot be exported to external platforms

Reason: Your data processing agreement with [Vendor] prohibits

exporting customer data outside of Adobe systems

Resolution Options:

├── ✅ Use AEP's built-in analytics (Customer Journey Analytics)

├── ✅ Use Adobe Query Service for analysis within AEP

├── ✅ Contact Legal to negotiate contract modification

└── ✅ Use a different, unlabeled dataset

```

Why This Matters:

  • Protects against accidental data breaches
  • Ensures contract compliance with data controllers
  • Prevents regulatory violations (GDPR, CCPA, HIPAA)

Example 3: Combined Label Restrictions

Scenario: Marketer wants to use customer data for on-site personalization (Adobe Target).

The Data:

```

Audience: "Premium Customers"

Field 1: customer_name → Label: C1 ("Cannot combine with directly identifiable data")

Field 2: email → Label: I1 ("Directly Identifiable")

```

The Policy:

```

Policy Name: "C1 + I1 Cannot Be Combined for Personalization"

IF: Data has BOTH labels C1 AND I1

AND Marketing action = On-Site Personalization

THEN: BLOCK activation (conflicting restrictions)

```

What Happens During Activation:

1. Marketer clicks "Activate to Adobe Target"

2. AEP system checks:

  • "Does audience have both C1 and I1 labels?" → ✅ YES
  • "Is there a policy blocking C1+I1 + On-Site Personalization?" → ✅ YES

3. Policy evaluation result: ❌ VIOLATION FOUND

4. Activation is BLOCKED

Resolution Options:

```

Option 1: REMOVE CONFLICTING FIELD

├── Remove email (I1 label) from Target mapping

├── Keep only name for personalization

└── Result: Compliant activation ✅

Option 2: REMOVE RESTRICTED FIELD

├── Remove name (C1 label) from Target mapping

├── Keep email for user identification

└── Result: Compliant activation ✅

Option 3: CHANGE DESTINATION TYPE

├── Don't use Target (personalization)

├── Use Analytics (measurement) instead

└── Result: Different policy applies ✅

Option 4: REQUEST POLICY MODIFICATION

├── Contact Compliance/Legal team

├── Explain business use case

├── Get approval to modify or disable policy

└── Result: Activation allowed after approval ✅

```

Layer 2: Consent Policy Enforcement

Consent policies filter out profiles that haven't given consent for the intended use — they don't block the entire activation, but reduce the audience size.

How Consent Policy Enforcement Works

```

Does an active Consent Policy apply to this destination's marketing action?

↓

YES → Check each profile
