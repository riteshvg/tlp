Sandbox Environments

Sandboxes are virtual workspaces designed for:

  • Safe experimentation — Test features and configurations without affecting live systems
  • Development and testing — Build and validate journeys before deployment
  • Team collaboration — Configure shared content folders for seamless teamwork
  • Role assignment — Control user access and permissions within each sandbox
  • Object transfer — Copy journeys, campaigns, templates, and other objects between sandboxes

Key Sandbox Features

  • Isolated environments for different purposes (development, testing, etc.)
  • Ability to export and import objects (journeys, campaigns, custom actions, content templates, fragments)
  • Sandbox tooling for managing objects across environments
