How does identity management work in the Experience Platform Web SDK?

Identity Management in Experience Platform Web SDK

Based on the available documentation, here is a comprehensive overview of how identity management works in the Experience Platform Web SDK.

Overview

The Web SDK handles identity management through a combination of identity namespaces, identity maps, and integration with the Adobe Experience Platform Edge Network. It replaces several legacy identity libraries with a unified approach.

What the Web SDK Replaces

The Web SDK consolidates identity management from multiple legacy libraries into a single implementation:

| Legacy Library | Replaced By |

|---|---|

| `Visitor.js` (Experience Cloud ID Service) | Web SDK |

| `AppMeasurement.js` (Adobe Analytics) | Web SDK |

| `AT.js` (Adobe Target) | Web SDK |

| `DIL.js` (Adobe Audience Manager) | Web SDK |

Previously, multiple calls were needed for each data collection library to handle identities. Now, a single call through the Edge Network can:

  • Retrieve an ID
  • Fetch a Target experience
  • Send data to Audience Manager
  • Pass data to Adobe Experience Platform

Identity Namespaces

Identity namespaces are a core concept in how the Web SDK classifies and manages identities.

What They Are

  • Identity namespaces describe the type of a specific identity
  • For example, a namespace called `Email` identifies a value like `name@email.com` as an email address
  • They are configured in Adobe Experience Platform and referenced in Web SDK implementations

Types of Identity Namespaces

| Type | Description |

|---|---|

| Standard namespaces | Pre-built namespaces provided by Adobe (e.g., `Email`, `Phone`, `ECID`) |

| Custom namespaces | Namespaces you create for your own identity types (e.g., a loyalty ID or CRM ID) |

Configuring Identity Namespaces

Identity namespaces must be configured in AEP before they can be used in your Web SDK implementation. This is covered as part of the standard Web SDK implementation tutorial, which includes:

1. Setting up your datastream

2. Configuring your XDM schema

3. Defining your identity namespaces

Identity Map Data Element

The Identity Map data element in Adobe Data Collection (Tags) is the primary mechanism for capturing user IDs in a Web SDK implementation.

What It Does

  • Captures user identifiers (e.g., CRM IDs, email addresses, loyalty IDs) from your data layer
  • Maps those identifiers to the appropriate identity namespace
  • Sends the identity data to the Edge Network as part of XDM payloads

How to Configure It

Prerequisites:

  • A Tags property with the Web SDK extension installed
  • Identity namespaces configured in AEP
  • User IDs available in your website's data layer

Steps:

1. In Adobe Data Collection, open your Tags property

2. Navigate to Data Elements and click Add Data Element

3. Select Adobe Experience Platform Web SDK as the extension

4. Select Identity Map as the data element type

5. Define one or more identity entries, each with:

  • Namespace — the identity namespace (e.g., `Email`, `CRM_ID`)
  • ID Value — the actual identifier value pulled from your data layer
  • Authenticated State — one of:
  • `Ambiguous`
  • `Authenticated`
  • `Logged Out`
  • Primary — flag to designate whether this is the primary identity

6. Save the data element and reference it in your XDM object data element

How Identities Flow Through the Web SDK

Here is the end-to-end flow:

```

Website Data Layer

↓

Identity Map Data Element (Tags)

↓

XDM Object Data Element

↓

Web SDK sendEvent call

↓

Adobe Experience Platform Edge Network

↓

AEP Identity Service → Real-Time Customer Profile (Identity Graph)

```

Key Considerations

| Topic | Detail |

|---|---|

| ECID | The Experience Cloud ID (ECID) is automatically managed by the Web SDK — you do not need to manually configure it |

| Authenticated state | Always pass the correct authenticated state to ensure identity stitching works correctly |

| Primary identity | Only one identity per namespace should be marked as primary |

| Streaming destinations | At least one target identity namespace must be mapped when activating audiences to streaming destinations |

| File-based destinations | Identity namespace configuration is optional for file-based destinations |

Implementation Tutorial

Adobe provides a full end-to-end tutorial — "Implement Adobe Experience Cloud with Web SDK" — that walks through:

1. Configuring identity namespaces in AEP

2. Capturing identities using the Identity Map data element in Tags

3. Sending identity data via the Web SDK to the Edge Network

4. Validating identity resolution using Experience Platform Debugger and Assurance

The tutorial uses a sample retail website called Luma to demonstrate a realistic implementation across Adobe Experience Cloud applications including Analytics, Target, Audience Manager, and AEP.
