What is identityMap used for in AJO?

I can only answer questions about Adobe Analytics, CJA, AEP, Adobe Target, Adobe Journey Optimizer, and Adobe Data Collection. Please ask about these products.

However, I should clarify: identityMap is not specific to Adobe Journey Optimizer (AJO). Based on the documentation context I have, identityMap is an Adobe Experience Platform (AEP) schema component.

Here's what identityMap is used for in AEP:

Overview

identityMap is a standard schema field group in Adobe Experience Platform that stores user identities. It's available for both the XDM ExperienceEvent and XDM Individual Profile classes.

Purpose

identityMap serves as a JSON object that maps multiple identity namespaces to their respective identifier values. It allows you to:

  • Store multiple identities for a single visitor across different namespaces (CRM ID, Email, Phone, Loyalty ID, etc.)
  • Track authenticated state of identities (authenticated, ambiguous, or logged out)
  • Designate primary identifiers for identity resolution

Structure

```json

{

"identityMap": {

"CRMID": [

{

"id": "abc-123-xyz",

"authenticatedState": "authenticated",

"primary": true

}

],

"Email": [

{

"id": "user@example.com",

"authenticatedState": "authenticated",

"primary": false

}

]

}

}

```

Key Points

  • Each namespace can contain one or more identity descriptors
  • Exactly one identity across all namespaces must be marked as `primary: true`
  • The ECID is automatically added by the Edge Network and doesn't need to be explicitly included
  • Namespaces must exist in Identity Service before you can reference them

identityMap is fundamental to AEP's identity management and is used across data collection and profile processing.
