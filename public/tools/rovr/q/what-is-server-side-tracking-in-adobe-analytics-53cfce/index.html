What is server side tracking in Adobe Analytics?

I don't have information on server-side tracking specifically in Adobe Experience League documentation.

However, the retrieved documentation does cover server-side forwarding, which is a related but different concept. Server-side forwarding is designed for sharing data from Analytics to other Adobe Experience Cloud solutions in real time. Let me explain what that involves:

Server-Side Forwarding

Server-side forwarding allows Analytics to push data to other CX Enterprise solutions (like Adobe Audience Manager) and for those solutions to push data back to Analytics during the data collection process.

Benefits of Server-Side Forwarding:

  • Reduces page calls - For Audience Manager customers, it eliminates the need for separate DIL (Data Integration Library) data collection calls, improving page load times
  • Data sharing - Enables real-time data sharing among Adobe CX Enterprise solutions
  • Best practices - Aligns with recommended implementation approaches for Audience Manager and Analytics
  • GDPR compliance - Allows data controllers to restrict pre-consent data from being forwarded until consent is received

Current Support:

The documentation notes that legacy tracking server-based server-side forwarding will continue to be supported, but new implementations should use the report suite-based server-side forwarding to take advantage of bidirectional data sharing between Analytics and Audience Manager.
