How do I track specific commerce events like purchases or cart additions using productListItems?

Tracking Specific Commerce Events with `productListItems`

Overview

Commerce events are tracked by setting the appropriate action in the `commerce` object alongside the `productListItems` array. Each event type signals a specific user action in the shopping journey.

Commerce Event Types Reference

| UI Label | XDM Field | Description |

|---|---|---|

| Product Views | `productViews` | A view of an individual product occurred |

| Product List Opens | `productListOpens` | A new product list/cart was created |

| Product List Adds | `productListAdds` | A product was added to the cart |

| Product List Removals | `productListRemovals` | A product was removed from the cart |

| Product List Views | `productListViews` | The cart/product list was viewed |

| Product List Reopens | `productListReopens` | A previously abandoned cart was re-activated |

| Save For Laters | `saveForLaters` | A product was saved to a wish list |

| Checkouts | `checkouts` | Checkout was initiated |

| Purchases | `purchases` | An order was accepted/completed |

| In Store Purchase | `inStorePurchase` | An in-store purchase was made |

Detailed Event Implementations

1. Product View

Track when a user views a product detail page:

```javascript

alloy("sendEvent", {

"xdm": {

"commerce": {

"productViews": {

"value": 1

}

},

"productListItems": [

{

"SKU": "HT105",

"name": "The Big Floppy Hat",

"priceTotal": 29.99

}

]

}

});

```

6. Purchase (`purchases`)

The purchase event is the most critical commerce event. Key rules:

> ⚠️ The purchase event is the only required action in a commerce conversion, and it must have a product list referenced.

The purchase event also typically includes an `order` object with order-level details:

```javascript

alloy("sendEvent", {

"xdm": {

"commerce": {

"purchases": {

"value": 1

},

"order": {

"purchaseID": "a8g784hjq1mnp3",

"purchaseOrderNumber": "VAU3123",

"currencyCode": "USD",

"priceTotal": 39.98

}

},

"productListItems": [

{

"SKU": "HT105",

"name": "The Big Floppy Hat",

"quantity": 1,

"priceTotal": 29.99

},

{

"SKU": "HT104",

"name": "The Small Floppy Hat",

"quantity": 1,

"priceTotal": 9.99

}

]

}

});

```

Important Rules & Best Practices

  • One commerce action per event call — pair each `sendEvent` with the specific commerce action that occurred.
  • `purchases` requires a product list — always include at least one item in `productListItems` for purchase events.
  • `order` object belongs inside `commerce` — order-level metadata like `purchaseID`, `currencyCode`, and `priceTotal` go in `commerce.order`, not in `productListItems`.
  • `value: 1` — the `value` property on each commerce action should always be set to `1` to count the event occurrence.
  • `productListItems` is always an array — even for single-product events.
