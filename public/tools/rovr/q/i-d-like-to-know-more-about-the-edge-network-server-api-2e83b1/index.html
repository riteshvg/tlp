Adobe Experience Platform Edge Network Server API

Overview

The Edge Network Server API is a secure, authenticated server-side API that allows you to send data directly to the Adobe Experience Platform Edge Network from your servers. This enables data collection without relying on client-side SDKs deployed in browsers or mobile apps.

▶ Watch: Adobe Experience Platform Edge Network Server API overview

Key Use Cases

The Edge Network Server API is ideal for scenarios where client-side data collection isn't practical or possible:

Primary Use Cases:

  • IoT Devices — Smart devices, wearables, set-top boxes that can't run browser-based SDKs
  • Desktop Applications — Native desktop software that needs to send data to Experience Platform
  • Server-to-Server Integration — Backend systems collecting and forwarding user activity
  • Sensitive Data Collection — Data that requires secure, authenticated transport across the network

When to Use Edge Network Server API vs. Web SDK:

| Scenario | Use Edge Network Server API |

|---|---|

| Collecting from IoT devices, set-top boxes | ✓ Yes |

| Collecting from desktop applications | ✓ Yes |

| Collecting from backend/server systems | ✓ Yes |

| Require secure, authenticated data transmission | ✓ Yes |

| Collecting from web browsers | ✗ Use Web SDK instead |

Benefits

Secure, Authenticated Transport

  • Authentication layer protects sensitive data in transit
  • Data is encrypted and validated before reaching the Edge Network
  • Ideal for PII and regulated data

Server-Side Processing

  • No client-side exposure of tracking code
  • Reduces attack surface for sensitive implementations
  • Full control over data sent from your systems

Unified Data Collection

  • Same Edge Network infrastructure as Web SDK and Mobile SDK
  • Data flows to the same destinations (Analytics, Audience Manager, Real-Time CDP, etc.)
  • Consistent XDM schema usage across all collection methods

Real-Time Routing

  • Data is processed and routed to Adobe solutions in real time
  • Same benefits as Web SDK for personalization, audience activation, and analytics

High-Level Implementation Overview

Step 1: Define a Report Suite (for Analytics)

If sending data to Adobe Analytics, ensure you have a report suite defined.

Step 2: Set Up Schemas

Standardize your data using the Experience Data Model (XDM):

  • Create or leverage existing XDM schemas in the Schemas UI
  • Map your server-side data fields to XDM schema fields
  • This ensures your data is consistent with other Experience Platform implementations

Step 3: Configure a Datastream

A datastream represents the server-side configuration for the Edge Network Server API:

  • Defines which Adobe services receive your data (Analytics, AEP, Audience Manager, etc.)
  • Maps XDM data to solution-specific fields
  • Handles data routing rules and transformations
  • Sets up event forwarding destinations if needed

Step 4: Authenticate & Call the API

  • Set up authentication credentials (API key or OAuth tokens)
  • Make HTTP POST requests to the Edge Network Server API endpoint
  • Send XDM-formatted event data in the request payload

Authentication

The Edge Network Server API supports secure, authenticated requests to protect sensitive data:

Key Security Features:

  • Authentication required — All requests must include valid credentials
  • Encrypted transport — Data is encrypted in transit
  • Validation — Requests are validated before processing

For detailed authentication setup, refer to the Authentication documentation in the Server API guide.


Key Differences from Web SDK

| Aspect | Web SDK | Edge Network Server API |

|---|---|---|

| Deployment | Browser/client-side | Server/backend systems |

| Ideal For | Web & mobile apps | IoT, desktop apps, servers |

| Authentication | Implicit (datastream-based) | Explicit (API key/OAuth) |

| Use Case | Real-time web tracking | Device & system integration |

| Data Source | Browser events | Server systems, devices |
