Key Components and Features of Adobe Journey Optimizer

Core Components of AJO

Adobe Journey Optimizer is built on several foundational pillars that work together to deliver personalized customer experiences:

1. 🗺️ Journey Orchestration

Journeys are the core of AJO — they are automated, real-time, event-driven customer experiences that adapt to individual customer behavior.

What is a Journey?

A journey is a flow diagram that:

  • ✅ Starts with a trigger event (customer action)
  • ✅ Contains decision nodes (conditions, branching logic)
  • ✅ Includes message activities (email, SMS, push, in-app)
  • ✅ Has wait steps (pause for time periods)
  • ✅ Can include audience reads (segment membership checks)
  • ✅ Executes real-time personalization
  • ✅ Can end in multiple ways (exit conditions, timeouts)

Journey Canvas

The visual journey builder allows you to drag-and-drop activities:

```

START EVENT

↓

Audience Qualifying?

├─ YES → Evaluate Conditions

│ ├─ Condition A Met? → Send Email A

│ ├─ Condition B Met? → Send SMS

│ └─ No Conditions Met? → Send Default Message

│

├─ NO → Exit Journey

│

└─ Wait 2 Days

↓

Send Follow-up Email

↓

Wait for Click Event

├─ Clicked? → Send Thank You Email

└─ Not Clicked? → Send Reminder SMS

```

Journey Activities

| Activity Type | Description | Example |

|---|---|---|

| Start Event | Triggers the journey | "User abandons cart" |

| Audience Read | Check if profile is in audience | "Is user VIP member?" |

| Condition | If/then branching logic | "If purchase > $100, send VIP offer" |

| Action | Send message or trigger custom action | Send email, SMS, or webhook |

| Wait | Pause for time period or event | Wait 3 days or until user logs in |

| Jump | Skip to another point in journey | Jump to "End Journey" if opted-out |

| Orchestration | Manage overlapping journeys | Apply conflict resolution rules |

2. 📣 Campaign Management

Campaigns in AJO are scheduled or API-triggered messages sent to audiences in bulk — distinct from journeys which are individual, event-driven.

Types of Campaigns

Scheduled Campaigns

```

Define Audience

↓

Create Message Content

↓

Set Send Schedule (specific date/time)

↓

Execute Send

↓

Measure Results

```

API-Triggered Campaigns

```

External System sends API request

↓

Campaign triggered with payload data

↓

Send message to specified profile

↓

Record interaction

```

Orchestrated Campaigns

```

Scheduled send to audience

↓

Wait for engagement

↓

Send follow-up based on behavior

↓

A/B test different follow-ups

↓

Final send based on performance

```

Campaign Features

  • ✅ Audience targeting — Use AEP segments or build inline audiences
  • ✅ Multi-channel — Email, SMS, push, in-app, web, direct mail
  • ✅ Content variants — A/B testing, multivariate testing
  • ✅ Personalization — Dynamic content using profile attributes
  • ✅ Scheduling — One-time or recurring sends
  • ✅ Real-time reporting — Monitor performance as campaign executes
  • ✅ Frequency capping — Limit message volume to customers

3. 🎯 Decision Management (Offer Decisioning)

Decision Management is AJO's AI-powered offer engine — it determines which offers, content, or actions each customer should receive.

Core Concepts

Offers

```

Offer Components:

├── Offer Content (email creative, SMS text, etc.)

├── Eligibility Rules (who can receive this offer)

├── Constraints (frequency cap, budget cap)

├── Priority (ranking when multiple offers apply)

└── Fallback Offer (if no offers are eligible)

```

Example Offer:

```

Name: "20% Off First Purchase"

Eligibility:

├── User is in "New Customers" segment

├── Has NOT purchased in last 30 days

└── Email is confirmed

Constraints:

├── Max 5 times per customer

├── Max $50,000 budget total

└── Active Jan 1 - Jan 31

Priority: High

Fallback: "15% Off General Offer"

```

Decision Rules

```

Decision = Set of Offers + Rules for Selection

Example:

IF customer_tier = "Gold"

THEN show [Offer A, Offer B, Offer C]

ELSE IF customer_tier = "Silver"

THEN show [Offer B, Offer D]

ELSE

THEN show [Offer D, Offer E]

```

AI Ranking

```

Multiple offers eligible for customer

↓

AI Model evaluates:

├── Offer propensity score

├── Historical conversion rates

├── Customer preference signals

├── Time of day patterns

└── Seasonal trends

↓

AI ranks offers by predicted conversion probability

↓

Top-ranked offer is delivered

```

Decision Delivery Methods

1. Via Journey Activity

```

Journey includes "Decision" activity

↓

Decision engine selects best offer per profile

↓

Offer delivered in message (email, SMS, etc.)

```

2. Via Campaign

```

Campaign includes decision logic

↓

Each profile in audience receives personalized offer

```

3. Via API

```

External system calls AJO Decision API

↓

Real-time decision returned

↓

External system delivers offer to customer

```
