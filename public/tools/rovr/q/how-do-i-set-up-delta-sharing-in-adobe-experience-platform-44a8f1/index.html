How do I set up Delta Sharing in Adobe Experience Platform?

Setting Up Delta Sharing in Adobe Experience Platform

>[!NOTE]

> Delta Sharing is currently in limited beta and will only be available until July 15, 2026. Contact your Adobe account team to request access before proceeding.

What Is Delta Sharing?

The Delta Sharing source connector allows you to securely connect to your Databricks Delta Sharing source and create virtual datasets in Adobe Experience Platform. Key characteristics:

  • Shared data is not physically ingested or copied into the Experience Platform data lake
  • Data is represented through relational schemas as virtual datasets
  • You can query and work with external tables through Experience Platform services (Query Service / Data Distiller)
  • Reduces duplicate data storage, lowers storage costs, and simplifies data management

Prerequisites

Before connecting, ensure the following requirements are met:

Databricks Provider Prerequisites

  • The Databricks Delta Sharing source must be configured and accessible
  • Shared tables you want to use must be available through Delta Sharing
  • Shared tables must be representable through relational schemas in Experience Platform

Experience Platform Knowledge Requirements

Ensure you have a working understanding of the following Platform components:

| Component | Purpose |

|---|---|

| Sources | Create connections and dataflows for supported external data sources |

| XDM Schemas | Shared tables are represented in Experience Platform through relational schemas |

| Datasets | Shared data is represented as virtual datasets |

| Query Service / Data Distiller | Used to query and work with virtual datasets |

Setup Steps in the UI

Step 1: Access the Sources Catalog

1. Log in to your Adobe Experience Platform instance

2. Navigate to the Sources section from the left navigation

3. Browse the sources catalog to locate the Delta Sharing connector

Step 2: Review Schema Structure

>[!IMPORTANT]

> Carefully review your schema structure before proceeding. Once you move forward, you will no longer be able to make changes to your schema.

  • Ensure your shared tables map correctly to the relational schema representation in Experience Platform
  • Confirm the schema aligns with your downstream use cases in Query Service or Data Distiller

Step 3: Connect Your Databricks Delta Sharing Account

1. Select the Delta Sharing source connector from the catalog

2. Provide your Databricks Delta Sharing connection credentials

3. Validate the connection to confirm access to your shared tables

Step 4: Configure Virtual Datasets

  • Select the shared tables you want to represent as virtual datasets in Experience Platform
  • Map the tables to the appropriate XDM relational schemas
  • Virtual datasets will be created without physically copying source data into Platform

Step 5: Query with Query Service or Data Distiller

Once configured, use Query Service or Data Distiller to:

  • Query virtual datasets as if they were native Platform datasets
  • Join shared external data with other Experience Platform data sources
  • Build downstream workflows, segments, or analytics use cases

Key Limitations to Plan Around

| Limitation | Detail |

|---|---|

| Beta availability | Only available until July 15, 2026 |

| No physical ingestion | Data is not copied into the Platform data lake |

| Schema immutability | Schema cannot be changed after initial setup |

| Relational schema requirement | Shared tables must be representable as relational schemas |

| Access requirement | Must contact Adobe account team for beta access |
