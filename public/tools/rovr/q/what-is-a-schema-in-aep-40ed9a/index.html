What is a schema in AEP?

Schema in Adobe Experience Platform (AEP)

A schema is a set of rules that represents and validates the structure and format of data in Adobe Experience Platform.

Core Definition

At a high level, schemas provide an abstract definition of a real-world object (such as a person) and outline what data should be included in each instance of that object (such as first name, last name, birthday, and so on).

In addition to describing data structure, schemas:

  • Apply constraints and expectations to data so it can be validated as it moves between systems
  • Enable semantic normalization — allowing data to be interpreted consistently regardless of origin
  • Remove the need for translation across applications
  • Support data reusability across your organization without conflicts

Why Schemas Matter

Standardization

Experience Platform uses schemas to describe the structure of data in a consistent and reusable way. By defining data consistently across systems, it becomes easier to retain meaning and gain value from data.

Before Data Ingestion

Before data can be ingested into Experience Platform, a schema must be composed to:

  • Describe the data's structure
  • Provide constraints on the type of data that can be contained within each field

Cross-Organizational Sharing

Schemas are the standard way of describing data in Experience Platform, allowing all data that conforms to schemas to be:

  • Reused across an organization without conflicts
  • Shared between multiple organizations

Schema Building Blocks

A schema is composed of:

1. A Base Class (required)

  • Examples: XDM Individual Profile, XDM ExperienceEvent

2. Zero or More Schema Field Groups (optional)

  • Reusable sets of fields that extend the schema

3. Custom Fields (optional)

  • Organization-specific additions

Schema Content

Schemas may include:

  • Behavioral attributes
  • Timestamps
  • Identities
  • Attribute definitions
  • Relationships
  • Data type constraints

Real-World Example

A customer loyalty program schema might include:

  • Class: XDM Individual Profile
  • Field Groups: XDM Business Person Details, IdentityMap
  • Custom Fields: Loyalty points, membership tier, preferred language

This defines what data about loyalty members can be stored and how it's structured.

Key Capabilities

Schema Library

The Schema Library contains:

  • Industry-standard XDM resources made available by Adobe
  • Custom resources defined by your organization

Schema Registry

The Schema Registry provides:

  • A user interface for viewing and managing schemas
  • A RESTful API for programmatic schema management

Important Note

XDM schemas are extremely customizable, allowing you to tailor them to your specific business needs while maintaining consistency with AEP's standardized framework.

For more detailed guidance, refer to the basics of schema composition documentation.
