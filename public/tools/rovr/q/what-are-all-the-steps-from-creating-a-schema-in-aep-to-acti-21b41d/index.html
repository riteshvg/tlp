Steps from Creating a Schema to Activating a Dataset to a Destination in AEP (with Screenshots)

Here's a comprehensive walkthrough with visual examples:

1. Create an Experience Data Model (XDM) Schema

Step 1: Open the Schema Editor

In the Schemas workspace, select [!UICONTROL Create schema] in the top-right corner. A dropdown menu appears with options for:

  • [!UICONTROL Standard] schemas
  • [!UICONTROL Relational] schemas

Step 2: Choose Your Schema Type and Define the Structure

Once you select your schema type, the Schema Editor opens with an interactive visual canvas. Here's an example of a loyalty program schema:

In this editor, you can:

  • View your schema structure on the canvas
  • Add field groups to include pre-built fields
  • Configure individual fields and their properties
  • Define relationships between data elements

Step 3: Add Field Groups

Click the [!UICONTROL Add] button to add schema field groups (reusable collections of fields):

Field groups provide pre-configured fields for common data types. For example, demographic field groups include structured fields for personal information:

Step 4: Configure Your Schema

The complete schema dialog shows all your selected components:

Once your schema is configured:

  • Define your class (foundation of the schema)
  • Add additional field groups as needed
  • Customize individual fields
  • Configure data types and relationships
  • Save your completed schema

2. Ingest Data into Datasets

Once your schema is created, data ingestion occurs:

  • Your data is mapped to the XDM schema you created
  • Data is successfully ingested and stored as datasets in the Data Lake
  • Datasets are structured as tables with:
  • Columns (schema fields)
  • Rows (individual records)
  • Metadata (descriptive information about the data)

3. Prepare Data for Activation

Your ingested data flows through Experience Platform services:

  • [!DNL Identity Service] - resolves and manages identities across your data
  • [!DNL Real-Time Customer Profile] - creates unified, real-time consumer profiles by aggregating data from multiple sources

4. Create and Configure a Destination Connection

Navigate to Destinations:

In the Experience Platform UI, go to the Destinations workspace and:

1. Select a destination from the catalog

2. Configure the connection with:

  • Authentication credentials
  • Connection details specific to your destination platform
  • Destination name and description

3. Enable alerts (optional) - Subscribe to notifications about the status of your dataflow to receive updates on successful activations or errors

4. Complete the setup by selecting [!UICONTROL Next] after providing all connection details

5. Export and Activate Your Dataset

Step 1: Select Datasets to Activate

After configuring your destination connection, you'll reach the [!UICONTROL Export datasets] step:

  • Choose which datasets you want to activate to the destination
  • Select from the available datasets in your organization

Step 2: Map Fields

Configure how fields from your dataset map to the destination's expected format:

  • Identify source fields from your dataset
  • Map them to corresponding destination fields
  • Ensure data compatibility

Step 3: Schedule Activation

Set up the activation schedule:

  • Choose activation frequency (one-time, daily, scheduled intervals)
  • Configure any destination-specific scheduling options
  • Confirm your activation settings

Step 4: Review and Activate

Review all your configuration:

  • Destination details
  • Selected datasets
  • Field mappings
  • Activation schedule

Then activate by selecting the final confirmation button.

6. Monitor Dataflows

After Activation:

Use the Destinations workspace to monitor your dataflows:

  • Track dataflow status - See which dataflows are active, paused, or have errors
  • View flow statistics - Monitor:
  • Records successfully processed
  • Records that encountered errors
  • Data delivery status
  • Check execution details - Review individual batch runs and their outcomes

Monitor dataflows for destinations in the UI to ensure your data is being successfully activated to external partners.

7. Validate Data Export

Confirm successful delivery:

  • Verify that your dataset has successfully reached your destination platform
  • For example, if your destination is a Snowflake data warehouse, check that data appears in the expected tables
  • Validate record counts and data integrity
  • Check that data formatting matches your requirements

Key Prerequisites Throughout the Process

✓ Understanding of XDM - Knowledge of how schemas are composed and structured

✓ Required Permissions:

  • View Destinations
  • View Datasets
  • Manage and Activate Dataset Destinations

✓ Sandbox access - Work in appropriate development or production sandboxes

✓ Destination compatibility - Ensure your chosen destination supports dataset export
