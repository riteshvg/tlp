Overview

Debugging identity graph issues in Adobe Experience Platform requires understanding how identities are linked, visualizing graph structures, and using the available tools to diagnose problems. AEP provides several dedicated tools for identity graph debugging and validation.

1. Identity Graph Viewer

The Identity Graph Viewer is the primary tool for visualizing and debugging identity graph issues in Adobe Experience Platform.

How to Access:

1. Navigate to Adobe Experience Platform

2. Select Identities in the left navigation

3. Select Identity Graph from the list of tabs in the header

How to View a Specific Identity Graph:

1. Select your identity namespace (e.g., Email, Phone, ECID, CRM ID)

2. Enter the corresponding identity value for the profile you want to inspect

3. Click View to load the identity graph

> Tip: Select the table icon to see a panel with a list of all identity namespaces available in your organization. You can use any identity namespace as long as you have a valid identity value connected to it.

What You Can See in the Graph Viewer:

  • Identity nodes – Each unique identity value as a node
  • Identity links – Connections between different namespaces
  • Namespace labels – Which namespace each identity belongs to
  • Graph relationships – How identities are stitched together
  • Linked profiles – All profiles connected through shared identities

2. Identity Dashboard

The Identity Dashboard provides high-level insights into the state of your identity graphs across your entire organization.

How to Access:

1. Navigate to Adobe Experience Platform

2. Select Identities in the left navigation

3. Select the Dashboard tab

What You Can Monitor:

  • Overall identity count – Total number of identities in your organization
  • Graph count trends – How identity graphs are growing over time
  • Identity count by namespace – Distribution of identities across namespaces
  • Graph count by graph size – Breakdown of single vs. multi-identity graphs
  • Graphs with two or more identities – Trends organized by namespace

Checking for Collapsed Graphs:

1. Select the ellipses (...) next to any metric

2. Select View more for further information

3. Validate that there are no collapsed graphs

> In the example above, both email and phone are marked as unique namespace, so therefore there are no collapsed graphs in the sandbox.

3. Graph Simulation Tool

The Graph Simulation tool allows you to test identity linking rules and simulate graph behavior before applying them to production.

▶ Watch: Graph Simulation Deep Dive

How to Access:

1. Navigate to Adobe Experience Platform

2. Go to Identities → Graph Simulation

How to Load an Existing Graph:

1. Click Load

2. Select Existing graph

3. In the dialog, enter a namespace and identity value that belong to the graph you want to inspect

4. When the load succeeds, Graph Simulation shows the graph with its:

  • Activities – All identity linking events
  • Algorithm configuration – Current identity optimization settings
  • Graph view – Visual representation of the loaded graph

Key Uses for Debugging:

  • Simulate rule changes – Test how new identity linking rules would affect existing graphs
  • Validate algorithm settings – Confirm identity optimization algorithm is working correctly
  • Preview graph changes – See impact of configuration changes before going live
  • Test unique namespace settings – Verify namespace priority and uniqueness rules
  • Experiment with "unique per graph" settings – Test different scenarios to verify rules needed for your business

4. Common Identity Graph Issues & Solutions

Issue 1: Graph Collapse (Too Many Identities Merged)

Symptoms:

  • One identity graph contains too many unrelated profiles
  • Customer profiles are incorrectly merged together
  • Shared devices causing identity merging issues

Causes:

  • Shared device identifiers being used as linking identities
  • Missing unique namespace configuration
  • Incorrect identity priority settings

Solutions:

  • Configure unique namespaces – Mark authenticated identifiers (email, phone) as unique
  • Review namespace priority – Ensure high-priority namespaces take precedence
  • Use Identity Optimization Algorithm – Configure rules to prevent over-merging
  • Check Graph Simulation – Test rule changes before applying to production

Issue 2: Identities Not Being Linked

Symptoms:

  • Profiles that should be merged remain separate
  • Customer journey shows fragmented profile history
  • Audience membership not reflecting full customer history

Causes:

  • Missing identity namespace in event data
  • Identity values not matching across datasets
  • Incorrect XDM field mapping for identity fields
  • Namespace not configured as identity in schema

Solutions:

  • Verify event data – Check that identity fields are populated in ingested events
  • Review schema configuration – Ensure identity fields are correctly marked in XDM schema
  • Check namespace mapping – Verify identity namespaces match across all datasets
  • Use Query Service – Query datasets to verify identity values are consistent

Issue 3: Dangling LoginID Scenario

Symptoms:

  • LoginID (e.g., CRM ID) exists in graph without being linked to a primary identifier
  • Profile data associated with LoginID not accessible via primary identity
  • Incomplete customer profile data

Causes:

  • LoginID was ingested without a corresponding primary identifier (e.g., ECID)
  • Event data missing required identity fields
  • Incorrect identity linking order

Solutions:

  • Review ingestion order – Ensure primary identifiers are ingested before or alongside LoginIDs
  • Check event payloads – Verify all required identity fields are present in each event
  • Use Graph Simulation – Simulate the dangling LoginID scenario to understand impact
  • Implement identity settings – Configure identity settings to handle LoginID scenarios correctly

Issue 4: Incorrect Namespace Priority

Symptoms:

  • Wrong identity being used as primary identifier in merged profiles
  • Profiles merging based on lower-priority identifiers
  • Identity resolution not matching expected behavior

Solutions:

  • Review namespace priority settings – Check current priority configuration in Identity Settings UI
  • Reconfigure namespace priority – Adjust priority to reflect your identity resolution strategy
  • Test with Graph Simulation – Validate priority changes before applying to production

5. Identity Graph Linking Rules

Identity Graph Linking Rules help data architects maintain accurate customer profiles and prevent graph collapse.

Key Concepts:

| Concept | Description |

|---|---|

| Unique per graph | Ensures only one identity of a specific namespace exists per graph |

| Namespace priority | Defines which namespace takes precedence during identity resolution |

| Identity Optimization Algorithm | Applies rules to prevent graph collapse and over-merging |

| Linking rules | Define how different namespaces should be connected |

Troubleshooting with Linking Rules:

1. Review existing rules in the Identity Settings UI

2. Load your graph in Graph Simulation

3. Experiment with rule changes to see impact

4. Validate results match expected behavior

5. Apply rules to production only after thorough testing

6. Identity Settings UI

Identity Settings allow you to configure how identities are linked and prioritized across your organization.

How to Access:

1. Navigate to Adobe Experience Platform

2. Go to Identities → Identity Settings

Key Configuration Options:

  • Unique namespaces – Mark namespaces that should only have one identity per graph
  • Namespace priority – Set the order of importance for identity resolution
  • Identity Optimization Algorithm – Configure rules for preventing graph collapse

7. Debugging with Query Service

Use Query Service to programmatically investigate identity issues:

Check identity distribution across namespaces:

```sql

SELECT

namespace,

COUNT(*) as identity_count

FROM

identity_dataset

GROUP BY

namespace

ORDER BY

identity_count DESC

```

Find profiles with multiple identities:

```sql

SELECT

identityMap,

COUNT(*) as linked_identities

FROM

your_profile_dataset

GROUP BY

identityMap

HAVING

COUNT(*) > 5

ORDER BY

linked_identities DESC

```

Check for missing identity values:

```sql

SELECT

COUNT(*) as records_without_identity

FROM

your_event_dataset

WHERE

identityMap IS NULL

OR identityMap = '{}'

```

Best Practices

✓ Use Graph Simulation first – Always test rule changes in simulation before applying to production

✓ Configure unique namespaces – Prevent graph collapse by marking authenticated IDs as unique

✓ Monitor the Identity Dashboard – Regularly check for collapsed graphs and anomalies

✓ Set correct namespace priority – Ensure your most reliable identifiers have highest priority

✓ Validate event data – Use Assurance to verify identity fields in SDK events

✓ Test with real graphs – Load existing graphs in Graph Simulation to validate configuration

✓ Document identity strategy – Maintain clear records of your identity linking rules
