Overview

Adobe Tags is built around three core building blocks:

1. Extensions – Pre-built functionality packages (like "apps" for your tag manager)

2. Data Elements – Variables that capture and store values from your website

3. Rules – Logic that defines WHEN something happens, IF certain conditions are met, and THEN what actions to execute

Think of it like a simple programming model:

```

Extension = Library/Toolkit providing functionality

Data Element = Variable holding a value

Rule = IF/THEN logic that uses data elements to trigger actions

```

Understanding the Concepts

Simple Real-World Analogy

Imagine a Retail Store with an Automated Discount System:

```

Extension = Cash Register System

(Pre-built system that knows how to apply discounts)

Data Element = "Customer Loyalty Tier"

(Variable: captures whether customer is Bronze, Silver, Gold, Platinum)

Rule = "Apply 20% Discount IF Customer is Gold Member THEN Show Discount"

(Logic: If loyalty tier = Gold, trigger discount action)

```

Layer 1: Extensions

What is an Extension?

An Extension is a pre-built package of functionality that adds capabilities to your Tags property. Extensions are like "apps" for Adobe Tags — they provide pre-built features without requiring custom code.

Each extension provides:

  • Pre-built actions you can use in rules
  • Pre-built conditions you can use to evaluate data
  • Data element types you can create
  • Settings and configuration for that functionality

Core Adobe Extensions

| Extension | Purpose | Provides |

|---|---|---|

| Core | Fundamental tag management | Events (page load, clicks), Conditions, Custom code, Data storage |

| Web SDK | Adobe Experience Platform data collection | sendEvent, setConsent, getIdentity actions; XDM data element types |

| Adobe Analytics | Send data to Adobe Analytics | Set variables, Send beacons, Analytics-specific events |

| Adobe Target | Trigger Target A/B tests | Send mbox requests, Adobe Target-specific logic |

| Adobe Audience Manager | Send data to AAM | Signal actions, segment data |

| Adobe Journey Optimizer | Trigger AJO decisioning | Send events to AJO journeys |

| Adobe Client Data Layer | Manage website data layer | Push events, Listen to data layer changes |

| Experience Cloud ID Service | Legacy identity management | Set/get visitor ID (being phased out by Web SDK) |

| Google Analytics | Send data to Google Analytics | Google Analytics events, page tracking |

| Facebook Pixel | Trigger Facebook conversions | Facebook conversion events, audience tracking |

Installing an Extension

1. Log in to experience.adobe.com

2. Navigate to Data Collection → Tags

3. Open your Tag Property

4. Go to Extensions → Catalog

5. Search for the extension (e.g., "Adobe Experience Platform Web SDK")

6. Click Install

7. Configure the extension settings

8. Click Save

Creating Custom Extensions

For proprietary functionality not available in the extension catalog, you can build custom extensions:

```javascript

// Example custom extension for a proprietary analytics platform

module.exports = {

name: "Custom Analytics Platform",

version: "1.0.0",

// Extension configuration UI fields

getSettings: function() {

return [{

name: "accountId",

label: "Account ID",

required: true

}];

},

// Actions provided by this extension

getActions: function(settings) {

return [{

name: "track-event",

label: "Track Custom Event",

perform: function(xdmData) {

// Send data to custom platform

fetch("https://analytics.customplatform.com/track", {

method: "POST",

body: JSON.stringify({

accountId: settings.accountId,

event: xdmData

})

});

}

}];

}

};

```

Layer 2: Data Elements

What is a Data Element?

A Data Element is a named variable that captures and stores a value from your website. Instead of hard-coding values in rules, you create data elements that dynamically retrieve values from your data layer, cookies, JavaScript, or other sources.

Why Data Elements Matter

```

Without Data Elements (Hard-Coded):

─────────────────────────────────

Rule Action: "Send event with product name: shoes"

Problem: If product name changes, need to edit rule code

With Data Elements (Dynamic):

──────────────────────────────

Data Element: "Product Name" → reads from: digitalData.product.name

Rule Action: "Send event with product name: %Product Name%"

Benefit: If page changes, data element automatically captures new value

No rule editing needed

```

Data Element Types Available

| Data Element Type | Source | Example Use Case |

|---|---|---|

| JavaScript Variable | Any JavaScript object/variable | Custom data layer values |

| Data Layer Variable | Adobe Client Data Layer | Structured event data |

| Cookie | Browser cookies | User preferences, authentication tokens |

| Query String Parameter | URL query string | Campaign IDs, search keywords |

| DOM Attribute | HTML element attributes/content | Button labels, product IDs from DOM |

| Local Storage | Browser local storage | Saved user preferences |

| Session Storage | Browser session storage | Current session state |

| Constants | Hard-coded static value | Fixed strings, IDs |

| Regular Expression | Pattern matching on other data | Extract portions of strings |

| Custom Code | JavaScript code you write | Any custom logic |

| XDM Object (Web SDK) | XDM schema structure | Complex nested objects for AEP |

| Computed Attribute (Web SDK) | Pre-calculated profile value | Customer lifetime value, segments |

Creating a Data Element: Step-by-Step

Example 1: JavaScript Variable Data Element

```

1. Navigate to Data Elements tab

2. Click "Add Data Element"

Configuration:

├─ Name: "Page Name"

├─ Extension: Core

├─ Data Element Type: JavaScript Variable

├─ JavaScript Variable Path: digitalData.page.pageInfo.pageName

├─ Storage Duration: Visitor

└─ Click Save

```

Result: When you reference `%Page Name%` in a rule, it retrieves:

```javascript

digitalData.page.pageInfo.pageName

// Returns: "Product Detail Page", "Homepage", etc.

```
