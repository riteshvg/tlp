What is the Edge Network Server API and when should I use it instead of the Web SDK?

Edge Network Server API vs. Web SDK

What is the Edge Network Server API?

The Adobe Experience Platform Edge Network Server API is a server-side API that allows you to send data directly to the Adobe Experience Platform Edge Network from your backend systems. Unlike the Web SDK (which runs in the browser), the Server API enables secure, authenticated server-to-server communication with the Edge Network.

When to Use the Server API Instead of Web SDK

Use the Edge Network Server API in these scenarios:

1. Server-Side Data Collection

  • Collecting data from backend systems rather than client-side
  • Capturing events that originate on your servers rather than user browsers

2. IoT and Non-Browser Devices

  • Set-top boxes
  • Desktop applications
  • IoT devices
  • Any non-web environment where the JavaScript Web SDK cannot run

3. Sensitive Data & Authentication

  • You need to collect sensitive data securely across the network
  • Authentication is required to validate data sources
  • The Server API supports built-in authentication mechanisms that protect data in transit

4. First-Party Data Control

  • You want to keep sensitive customer data on your servers
  • You prefer to validate and authenticate data before sending it to Adobe

Key Architectural Differences

| Aspect | Web SDK | Server API |

|--------|---------|-----------|

| Runtime | Client-side (browser) | Server-side (backend) |

| Language | JavaScript | Language-agnostic (HTTP) |

| Use Cases | Web/mobile tracking | Backend systems, IoT, authenticated flows |

| Authentication | Implicit (via datastream) | Explicit server API authentication |

| Data Sensitivity | Lower (browser context) | Higher (secure backend) |

Implementation Overview

A high-level workflow using the Server API includes:

1. Define a report suite — Set up your Analytics configuration (if using Analytics)

2. Setup schemas — Create XDM (Experience Data Model) schemas to standardize your data structure

3. Configure a datastream — A datastream represents your server-side configuration when using the Edge Network Server API; it defines how data flows to your destinations

Web SDK and Server API: Complementary, Not Competing

Both the Web SDK and Server API send data to the same Adobe Experience Platform Edge Network. They can work together:

  • Use Web SDK for client-side, browser-based data collection
  • Use Server API for server-side, authenticated, or non-browser data collection
  • Both feed into the same unified Edge Network for centralized processing and activation

The choice depends on your data source and security requirements, not on which solution is "better" — they're designed for different use cases within the same ecosystem.
