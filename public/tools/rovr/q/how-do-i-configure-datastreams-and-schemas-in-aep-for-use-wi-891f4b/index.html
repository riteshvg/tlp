How do I configure datastreams and schemas in AEP for use with AJO?

Configuring Datastreams and Schemas in AEP for Use with AJO

This guide walks through the complete setup of XDM schemas and datastreams in Adobe Experience Platform so that data flows correctly into Journey Optimizer for journey triggering, personalization, and profile-based targeting.

Prerequisites

  • Access to Adobe Experience Platform and Journey Optimizer
  • Admin or data engineer permissions in AEP
  • Adobe Experience Platform Web SDK or Mobile SDK implemented (or planned)
  • Understanding of your data model (what customer attributes and events you need to capture)

Part 1: Schema Configuration

Schemas define the structure of all data flowing into AEP and subsequently into AJO. You need two core schema types.

Schema Type 1: XDM Individual Profile Schema

This schema stores persistent customer attributes (name, email, preferences, loyalty status, etc.) used for personalization in AJO.

1. Navigate to Data Management → Schemas in the AEP UI

2. Click Create Schema

3. Select XDM Individual Profile as the base class

4. Click Add field groups to add pre-built field groups relevant to your use case:

  • Demographic Details — name, date of birth, gender
  • Personal Contact Details — email address, phone number
  • Consent and Preferences — for managing marketing opt-ins
  • Loyalty Details — if applicable to your use case

5. For any custom attributes not covered by standard field groups, create a custom field group:

  • Click Create new field group
  • Add your custom fields with the appropriate data types

6. Mark the primary identity field (e.g., CRM ID or Email):

  • Select the field
  • Toggle Identity on
  • Select the appropriate Identity Namespace
  • Check Primary Identity

7. Enable the schema for Real-Time Customer Profile:

  • Toggle Profile on in the schema properties panel
  • Confirm the enablement dialog

8. Click Save

Schema Type 2: XDM Experience Event Schema

This schema captures behavioral and interaction events (page views, form submissions, purchases, app interactions) that trigger journeys in AJO.

1. Navigate to Data Management → Schemas

2. Click Create Schema

3. Select XDM ExperienceEvent as the base class

4. Add relevant field groups:

  • AEP Web SDK ExperienceEvent — if using Web SDK for data collection
  • Consumer Experience Event — for commerce interactions
  • Environment Details — device, browser, OS information
  • Any custom field groups specific to your events (e.g., form submission fields)

5. Mark the identity field to link events to profiles:

  • Typically ECID from the IdentityMap field group
  • Add IdentityMap field group if not already present

6. Enable the schema for Real-Time Customer Profile:

  • Toggle Profile on
  • This ensures events are stitched to the correct customer profile

7. Click Save

Schema Best Practices for AJO

| Best Practice | Reason |

|---|---|

| Always enable schemas for Profile | AJO reads from Real-Time Customer Profile |

| Use IdentityMap on event schemas | Enables identity stitching across devices |

| Set a clear Primary Identity on profile schemas | Ensures correct profile resolution in journeys |

| Use standard XDM field groups where possible | Ensures compatibility with AJO out-of-the-box features |

| Keep event schemas aligned to journey trigger conditions | Simplifies custom event configuration in AJO |

Part 2: Dataset Configuration

Each schema needs a corresponding dataset to store the actual data records.

Create a Profile Dataset

1. Navigate to Data Management → Datasets

2. Click Create Dataset

3. Select Create Dataset from Schema

4. Select your XDM Individual Profile schema

5. Give the dataset a meaningful name (e.g., `CRM Profile Dataset`)

6. Enable the dataset for Real-Time Customer Profile:

  • Toggle Profile on in the dataset detail view

7. Click Finish

Create an Event Dataset

1. Repeat the same process selecting your XDM Experience Event schema

2. Name it meaningfully (e.g., `Web Behavioral Events Dataset`)

3. Enable for Real-Time Customer Profile

4. Click Finish

Part 3: Datastream Configuration

Datastreams represent the server-side routing configuration that tells the Edge Network where to send data collected by the Web SDK or Mobile SDK.

Create a New Datastream

1. Navigate to Data Collection → Datastreams in the AEP or Data Collection UI

2. Click New Datastream

3. Provide a Name and optional Description

4. Select your XDM Experience Event schema in the Event Schema field

5. Click Save

Configure AEP Service on the Datastream

This routes incoming data to your AEP datasets and Real-Time Customer Profile.

1. On your new datastream, click Add Service

2. Select Adobe Experience Platform from the service dropdown

3. Configure the following:

  • Event Dataset: Select the XDM Experience Event dataset created in Part 2
  • Profile Dataset: Select the XDM Individual Profile dataset created in Part 2
  • Edge Segmentation: Toggle on if you need same-page or next-page personalization
  • Personalization Destinations: Toggle on if using AJO or Target for real-time personalization responses
  • Offer Decisioning: Toggle on if using AJO Decision Management for offer delivery

4. Click Save

Enable Adobe Journey Optimizer Service on the Datastream

This enables AJO to receive inbound events via the Edge Network and return personalization/offer responses.

1. On the same datastream, click Add Service again

2. Select Adobe Journey Optimizer from the service dropdown

3. Ensure it is enabled

4. Click Save

Advanced Datastream Settings

Additional optional settings available on your datastream:

  • Geo Lookup: Automatically appends geographic information (city, country, latitude/longitude) to incoming events
  • Device Lookup: Appends device and browser metadata to events
  • Bot Detection: Filters out non-human traffic before it reaches AEP and AJO
  • First Party ID Cookie: Enables first-party device ID persistence for better identity resolution

Part 4: Connecting It All Together

Once schemas, datasets, and datastreams are configured, reference the datastream in your Web SDK implementation:

```javascript

alloy("configure", {

datastreamId: "YOUR_DATASTREAM_ID",

orgId: "YOUR_ORG_ID@AdobeOrg"

});

```

Then send events conforming to your XDM Experience Event schema:

```javascript

alloy("sendEvent", {

xdm: {

eventType: "web.formFilledOut",

_yourTenant: {

formName: "Contact Us",

assetClassSelected: "Equities"

},

identityMap: {

ECID: [{

id: "YOUR_ECID_VALUE",

primary: true

}]

}

}

});

```

Following this complete setup ensures that all data collected from your digital properties flows correctly through the Edge Network into AEP profiles and datasets, and is immediately available to Adobe Journey Optimizer for journey triggering, audience evaluation, and personalized communications.
