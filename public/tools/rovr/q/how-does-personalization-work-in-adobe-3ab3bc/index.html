How does personalization work in Adobe?

Personalization in Adobe Journey Optimizer

Personalization in AJO empowers you to craft tailored, impactful customer experiences by dynamically customizing content based on individual profiles, behaviors, and contextual data. Whether you're creating personalized emails, notifications, or offers, the tools and techniques provided make it easy to connect the right message to the right person at the right time.

Foundation: Profile Data

Personalization is based on profile data managed by the XDM Individual Profile schema defined in Adobe Experience Platform. This is the only schema you can use to personalize content in Journey Optimizer.

You can also leverage:

  • Computed attributes — Summarize individual behavioral events into computed profile attributes available on AEP
  • AEP datasets — Leverage data from Adobe Experience Platform for lookup personalization (currently in beta — datasets must first be enabled via API call)

The Personalization Editor

The personalization editor is the centerpiece of personalization in Journey Optimizer. It is available in every context where you need to define personalization, such as emails, push notifications, and offers.

Using the editor, you can:

  • Select data from available personalization sources
  • Arrange and combine multiple data points
  • Customize expressions using Handlebars syntax and helper functions
  • Validate your expressions before publishing

Personalization Sources

| Source | Syntax | Description |

|--------|--------|-------------|

| Profile attributes | `profile.*` | Individual profile fields from AEP XDM Individual Profile schema |

| Audiences | `inAudience()` | Audience membership checks |

| Offer decisions | Decision management | Personalized offers from Offer Decisioning |

| Target attributes | Orchestrated campaigns only | Attributes calculated on the campaign canvas, including enrichment collection arrays |

| Tokens | `context.token` | Session or authentication tokens |

Personalization Syntax

Message Content — Handlebars

In message content (emails, push, etc.), personalization uses Handlebars syntax with `{{#each}}` loops for iterating over arrays and dynamic content rendering.

Journey Expressions — Expression Editor

When configuring journey activities such as conditions or custom action parameters, you use the Journey Expression Editor with functions like `first`, `all`, and `serializeList`.

> ⚠️ These are two different syntaxes — Handlebars for message content, and Journey expressions for journey configuration.

Key Personalization Features

  • Expression Fragments — Reusable content blocks that can be used across multiple messages
  • Helper Functions — Advanced functions to unlock deeper personalization possibilities
  • AI Assistant for Personalization Expressions — AI-powered assistance for building personalization expressions
  • Personalization Playground — An interactive tool to learn and experiment with personalization using sample data without requiring live datasets

Personalization in Web/App Channels (Web SDK)

For web and app personalization, common implementation topics include:

  • Prevent flicker — Hide and reveal containers during personalization loading
  • Track displayed content — Record display events for rendered personalized content
  • Top/bottom of page events — Request decisions early, then include measurement metrics later

Adobe also maintains a repository of Web SDK sample applications covering:

  • Client-side personalization
  • Server-side personalization
  • Hybrid personalization
  • Personalization in single-page applications (SPAs)

Personalization Schema Types

When working with personalization decisions delivered via the Edge Network, several schema types govern how content is handled:

| Schema | Purpose |

|--------|---------|

| `https://ns.adobe.com/personalization/measurement` | Tracks clicks and interactions on personalized offers |

| `https://ns.adobe.com/personalization/default-content-item` | Fallback/default content when no personalized offer is eligible |

| `https://ns.adobe.com/personalization/eventHistoryOperation` | Tracks which experiences have been delivered or blocked |
