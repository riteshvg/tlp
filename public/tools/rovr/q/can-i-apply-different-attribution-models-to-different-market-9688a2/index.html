Applying Attribution Models in Customer Journey Analytics

What the Documentation Confirms

CJA's attribution capabilities are highly flexible, and here is what is explicitly supported:

1. Unlimited Attribution Model Comparison

You can dynamically compare as many attribution models as you want simultaneously. This means you can:

  • Place multiple versions of the same metric (each with a different attribution model) side by side in a Freeform table
  • Use the Attribution panel to visualize how different models distribute credit across all your channels at once
2. Attribution via Calculated Metrics

You can build calculated metrics using any number of attribution allocation methods, including:

  • First Touch — 100% credit to first touchpoint
  • Last Touch — 100% credit to last touchpoint
  • Linear — equal credit across all touchpoints
  • Time Decay — dimension values closest in time to conversion get the most credit
  • U-Shaped — weighted credit to first and last touchpoints
  • Inverse J — 60% credit to the first dimension value, 20% to the last, 20% shared by the middle
  • Custom/Position-Based — define your own position-based attribution weighting
  • Algorithmic — credit is dynamically determined by a statistical algorithm

3. Attribution Applied Across Cross-Channel Data

CJA supports sophisticated attribution across:

  • Web revenue
  • Mobile app revenue
  • Product returns (removable from calculations)
  • Any custom dimension or metric — not just marketing campaigns

Recommended Workaround

Based on what is documented, the practical approach would be to:

1. Create separate calculated metrics — one per attribution model you want to apply

2. Add multiple metric columns to a single Freeform table — each column using a different attribution model

3. Use segments/filters to isolate specific channel subsets per metric column if needed

This gives you a highly flexible, side-by-side comparison across channels and models, even if it is not a single-column per-channel configuration.
