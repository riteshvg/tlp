Meeting Regulatory Requirements for Journey Step Metadata Capture & Downstream Sharing

Based on the Adobe documentation, there are two complementary mechanisms you can leverage to capture detailed metadata at each journey step and share it with downstream systems.

1. Automatic Journey Step Event Streaming to AEP

Journey Optimizer automatically streams journey step event data to Adobe Experience Platform for every node traversal in a journey — no manual configuration required.

How It Works

  • A step event is generated each time an individual moves from one node to another in a journey
  • For example, a journey with one event, one condition, and one action generates three step events sent to AEP
  • The schemas and datasets created for step events are provisioned automatically and are read-only (protecting data integrity for audit purposes)

What This Gives You for Regulatory Purposes

  • A complete, immutable record of every profile's progression through a journey
  • Data is stored in AEP datasets and can be combined with other digital or offline data sources
  • Analyzable in Customer Journey Analytics (CJA) for advanced attribution modeling, cross-channel journey visualization, and predictive analytics

2. `executionMetadata` Helper for Custom Key-Value Metadata

For attaching custom regulatory or compliance context to specific delivery actions, use the `executionMetadata` helper in the personalization editor.

Syntax

```

{{executionMetadata key="your_key" value="your_value"}}

```

How to Implement

1. Open any channel content inside a campaign or journey

2. Navigate to the personalization editor for a specific content element

3. Add the `executionMetadata` helper with your custom key-value pairs (e.g., consent version, regulation identifier, processing basis)

4. Upon runtime, the metadata is appended to the existing Message Feedback Event Dataset under this schema path:

```json

"_experience": {

"customerJourneyManagement": {

"messageExecution": {

"metadata": {

"your_key": "your_value"

}

}

}

}

```

Regulatory Use Cases

| Metadata Key | Example Value | Purpose |

|---|---|---|

| `consent_version` | `v2.3` | Track which consent version was active at send time |

| `legal_basis` | `legitimate_interest` | Record GDPR processing basis |

| `data_residency_region` | `EU` | Tag geographic processing context |

| `campaign_regulation_id` | `CCPA-2024-001` | Link to a specific regulatory requirement |

3. Key Limitations to Plan Around

  • `executionMetadata` is not supported in custom actions or inbound channels (Web, Code-based experience, In-App Message, Content Cards)
  • There is a 2KB upper limit on key-value pairs per action — if exceeded, the message still delivers but key-value pairs may be truncated
  • The metadata is not visible when content is previewed or displayed

4. Downstream Sharing Architecture

```

Journey Step Events (auto) ──► AEP Datasets ──► CJA (analysis & reporting)

│

executionMetadata (custom) ──► Message Feedback Event Dataset ──► External Reporting/Compliance Systems

```

  • Both data streams land in AEP datasets, making them accessible via AEP Query Service for extraction, transformation, and export
  • CJA can be configured to consume Journey Optimizer datasets for advanced attribution and cross-channel journey visualization
  • External compliance systems can be fed via AEP data export capabilities

Summary Recommendation

For full regulatory coverage, combine both approaches:

1. Rely on automatic step event streaming for a tamper-proof, system-generated audit trail of all journey traversals

2. Use `executionMetadata` to enrich outbound message events with specific compliance context (consent version, legal basis, etc.) for reconciliation with external reporting or DMP/consent management platforms
