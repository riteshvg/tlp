Overview

Adobe Experience Platform Tags (formerly known as Adobe Launch, and before that Adobe Dynamic Tag Manager) is a next-generation tag management system built natively into Adobe Experience Platform. It allows organizations to manage and deploy marketing, analytics, and personalization technologies on their websites and mobile apps — all without requiring developers to manually update code for every change.

Key Definition

> Adobe Experience Platform Tags is a unified data collection platform that enables you to deploy, manage, and govern all your marketing and analytics tags from a single interface — reducing reliance on development teams and accelerating time to value.

Core Architecture of Tags

Tags is built around five core concepts:

1. Properties

A Property is the container that holds all your Tags configuration for a specific digital property.

| Property Type | Use Case | Example |

|---|---|---|

| Web Property | Websites and SPAs | www.retailstore.com |

| Mobile Property | iOS and Android apps | Retail Store iOS App |

| Edge Property | Server-side collection | API-based data collection |

Creating a Web Property:

1. Log in to experience.adobe.com

2. Navigate to Data Collection → Tags

3. Click New Property

4. Enter Name (e.g., "Retail Store - Production")

5. Select Web as the platform

6. Enter your domain(s)

7. Click Save

2. Extensions

Extensions are pre-built integrations that add functionality to your Tags property. They are the equivalent of "apps" for your tag management system.

| Extension Category | Examples |

|---|---|

| Adobe Native | Adobe Experience Platform Web SDK, Adobe Analytics, Adobe Target, Adobe Audience Manager |

| Identity | Experience Cloud ID Service, Customer Privacy |

| Marketing | Google Analytics, Facebook Pixel, LinkedIn Insight Tag |

| Utility | Core Extension, Adobe Client Data Layer, Common Analytics Plugins |

Key Extensions for AEP Implementation:

  • Adobe Experience Platform Web SDK – Primary extension for sending data to Edge Network
  • Adobe Client Data Layer – Manages a standardized data layer on your website
  • Core – Provides fundamental event types, conditions, and actions
  • Adobe Experience Cloud ID Service – Legacy identity management (replaced by Web SDK)

3. Data Elements

Data Elements are variables that capture and store values from your website's data layer, cookies, JavaScript objects, or other sources.

```

Data Element Examples:

─────────────────────

Name: "Page Name"

Type: JavaScript Variable

Variable: digitalData.page.pageInfo.pageName

Value: "Homepage" (captured at runtime)

Name: "Product SKU"

Type: Data Layer Variable

Layer Variable: ecommerce.detail.products[0].id

Value: "SHOE-RUN-001" (captured when product viewed)

Name: "Customer Email"

Type: Cookie

Cookie Name: "user_email"

Value: "customer@email.com" (if logged in)

Name: "ECID"

Type: AEP Web SDK (Identity)

Value: "98765432109876543210" (auto-managed by Web SDK)

```

Creating a Data Element:

1. In your Tags property, navigate to Data Elements

2. Click Add Data Element

3. Enter a Name (e.g., "Product Name")

4. Select the Extension (e.g., Core)

5. Select the Data Element Type (e.g., JavaScript Variable)

6. Configure the variable path (e.g., `digitalData.product.name`)

7. Set storage duration and default value

8. Click Save

4. Rules

Rules define the logic for when, what conditions must be met, and what actions to take for tag firing.

Rules have three components:

```

RULE STRUCTURE:

───────────────

Events (WHEN):

  • Page Loaded
  • Button Clicked
  • Form Submitted
  • Custom Event Fired
  • DOM Ready

Conditions (IF):

  • Page URL contains "/products"
  • Data Element equals "Gold Member"
  • Cookie exists "user_authenticated"
  • Browser is Chrome/Firefox/Safari

Actions (THEN):

  • Send Event (Web SDK)
  • Set Variables (Analytics)
  • Fire Beacon (Analytics)
  • Send Beacon (Analytics)
  • Custom Code

```

Example Rule: Track Product Page Views

```

Rule Name: "Product Page View"

EVENT:

Type: Page Loaded (DOM Ready)

CONDITION:

Type: Value Comparison

Left: %Page Type% (Data Element)

Operator: Equals

Right: "product-detail"

ACTION 1: Adobe Client Data Layer - Push Event

Event: {

"event": "product-view",

"productInfo": {

"sku": "%Product SKU%",

"name": "%Product Name%",

"price": "%Product Price%"

}

}

ACTION 2: Adobe Experience Platform Web SDK - Send Event

Event Type: commerce.productViews

XDM Data: %XDM Product View Object%

```

5. Environments & Libraries

Tags uses three environments to manage the deployment lifecycle:

| Environment | Purpose | Who Uses It |

|---|---|---|

| Development | Build and test new changes | Developers, QA |

| Staging | Pre-production validation | QA, Stakeholders |

| Production | Live customer-facing site | End customers |

A Library is a collection of rules, data elements, and extensions that gets built and deployed to an environment.

Library Publishing Workflow:

```

1. Make changes (add rules, data elements, extensions)

↓

2. Save changes to Development library

↓

3. Build Development library

↓

4. Test in Development environment

↓

5. Submit library for Staging

↓

6. Test in Staging environment

↓

7. Approve and publish to Production

↓

8. Production library goes live

```

How Tags Works End-to-End

```

┌─────────────────────────────────────────────────────────────┐

│ TAGS WORKFLOW │

│ │

│ 1. CONFIGURE │

│ • Install extensions (Web SDK, Analytics, Target) │

│ • Create data elements (page name, product ID, etc.) │

│ • Build rules (when/if/then logic) │

│ │

│ 2. BUILD │

│ • Compile rules, data elements, extensions │

│ • Generate JavaScript library file │

│ • Deploy to chosen environment │

│ │

│ 3. DEPLOY │

│ • Embed Tags script on website: │

│ <script src="//assets.adobedtm.com/abc123.js"> │

│ • Tags library loads on page │

│ │

│ 4. EXECUTE │

│ • Rules evaluate on every page load/interaction │

│ • Data elements capture values dynamically │

│ • Actions fire (send events, set variables, etc.) │

│ │

│ 5. COLLECT │

│ • Web SDK sends XDM events to Edge Network │

│ • Analytics sends beacons to Analytics servers │

│ • Target requests personalization decisions │

└─────────────────────────────────────────────────────────────┘

```

Tags Embed Code

Once a Tags property is created and published, it's deployed to your website via a single embed code snippet:

```html

<!-- Adobe Experience Platform Tags - Head Embed Code -->

<!-- Development Environment -->

<script src="//assets.adobedtm.com/launch-dev-abc123.min.js" async></script>

<!-- Staging Environment -->

<script src="//assets.adobedtm.com/launch-staging-abc123.min.js" async></script>

<!-- Production Environment -->

<script src="//assets.adobedtm.com/launch-abc123.min.js" async></script>

```

This single script tag:

  • Loads the entire Tags library
  • Initializes all configured extensions
  • Evaluates rules on page load and user interactions
  • Fires appropriate actions based on rule conditions

Key Capabilities

1. Self-Service Tag Management

  • Add, modify, and remove tags without developer involvement
  • Test changes safely in Development before going live
  • Roll back changes instantly if issues arise

2. Governed Publishing Workflow

  • Role-based permissions – Control who can create, approve, and publish
  • Approval workflows – Require sign-off before production deployment
  • Audit logs – Track every change made by every user

3. Extension Ecosystem

  • 200+ pre-built extensions in the Adobe Exchange marketplace
  • Custom extensions – Build your own extensions for proprietary systems
  • Automatic updates – Extension vendors can push updates to the marketplace

4. Advanced Debugging

  • Tags Debugger – Chrome extension for real-time rule monitoring
  • Assurance integration – Deep debugging for mobile implementations
  • Test mode – Validate rules before publishing to production

5. API-First Architecture

  • Full REST API – Automate Tags operations programmatically
  • CI/CD integration – Integrate Tags publishing into your deployment pipeline
  • Bulk operations – Manage multiple properties at scale

Tags in the AEP Ecosystem

```

Adobe Experience Platform Tags

↓

↓

Web SDK sends XDM events

↓

Edge Network receives events

↓

Datastream routes to AEP

↓

AEP processes events:

• Data Lake storage

• Identity resolution

• Profile update

• Segment evaluation

↓

AJO triggers journeys

Analytics reports behavior

```

Tags sits at the very beginning of the AEP data collection pipeline — it is the mechanism that deploys the Web SDK and ensures data flows correctly from the browser to all downstream Adobe services.

Required Permissions for Tags

To work with Tags, users need the following permissions:

| Permission | Description |

|---|---|

| Manage Properties | Create and configure tag properties |

| Develop | Create and edit rules, data elements, extensions |

| Manage Environments | Create and manage deployment environments |

| Publish | Deploy libraries to production |

| Manage Extensions | Install and configure extensions |

Prerequisites for Getting Started

Before setting up Adobe Experience Platform Tags:

  • Access to experience.adobe.com with Adobe ID credentials
  • Property rights: Develop, Manage Extensions
  • AEP permissions: Data Modeling, Identities
  • An XDM schema created in AEP (for Web SDK implementation)
  • A datastream configured in AEP (to route data to Adobe services)

Getting Started: Quick Steps

1. Log in to experience.adobe.com

2. Navigate to Data Collection → Tags

3. Click New Property → Enter name, select Web platform

4. Install extensions:

  • Adobe Experience Platform Web SDK
  • Adobe Client Data Layer
  • Core (auto-installed)

5. Configure Web SDK extension with your datastream ID

6. Create data elements for your data layer variables

7. Create rules for page views, interactions, and events

8. Build and test in Development environment

9. Publish to Staging then Production
