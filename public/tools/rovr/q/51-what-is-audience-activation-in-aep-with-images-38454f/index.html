Audience Activation in Adobe Experience Platform (AEP)

What is Audience Activation?

Audience Activation is the process of taking the audiences (segments) you have built in Adobe Experience Platform and delivering them to external destinations so they can be used in real-world marketing campaigns, advertising, personalization, and suppression use cases.

> Collect Data → Unify Profiles → Build Audiences → ACTIVATE

Activation is the final step of the Real-Time CDP workflow — without it, your audiences stay inside AEP and produce no business value.

The Big Picture: Activation Flow

This diagram illustrates the end-to-end activation architecture:

  • Left side — Data sources feeding into AEP
  • Center — AEP processing (profiles, segments, governance)
  • Right side — Destinations receiving activated audiences

Types of Activation Methods

AEP supports multiple ways to activate audiences:

| Method | Description | Best For |

|---|---|---|

| Streaming Activation | Real-time, event-driven API exports | Ad suppression, real-time personalization |

| Batch Activation | Scheduled file exports (CSV, Parquet) | CRM, email, offline campaigns |

| Ad-Hoc Activation | On-demand API-triggered export | Backfills, missed exports, testing |

| Edge Activation | Activation via Edge Network for same-page personalization | Web/app personalization |

| Expanded Activation | Audience Manager segments activated via AEP | AAM customers without full RTCDP |

Step-by-Step: Audience Activation Workflow

Prerequisites

Before activating, ensure you have:

  • ✅ Access to Real-Time CDP or AEP with destinations enabled
  • ✅ The View Destinations, Activate Destinations, View Profiles, and View Segments access control permissions
  • ✅ At least one audience built in the Audience Portal
  • ✅ A destination connection configured
  • ✅ For identity exports: the View Identity Graph permission

Step 1: Open the Destinations Catalog

1. Log in to Adobe Experience Platform

2. In the left navigation, click Destinations

3. Click the Catalog tab

4. Browse destinations by category:

  • Advertising — Google, Meta, Amazon Ads, The Trade Desk
  • Social — Facebook, LinkedIn, TikTok, Pinterest
  • Cloud Storage — Amazon S3, Azure Blob, SFTP, Google Cloud
  • Email Marketing — Adobe Campaign, Mailchimp, Oracle Eloqua
  • CRM — Salesforce, HubSpot, Microsoft Dynamics
  • Personalization — Adobe Target, Custom Personalization
  • Data Warehouse — Snowflake, BigQuery, Azure Data Lake

Step 2: Configure the Destination Connection

1. Click Set up or Activate audiences on your chosen destination card

2. Provide authentication credentials (API keys, OAuth, S3 bucket details, etc.)

3. Name your connection and assign marketing use case tags for governance

4. Click Next

Step 3: Select Audiences

1. In the Select audiences step, browse your Audience Portal

2. You will see all available audiences:

  • 🟦 Platform-generated — built with Segment Builder or Audience Composition
  • 🟩 Custom uploads — externally generated CSV audiences
  • 🟧 Audience Manager segments — imported via Expanded Activation

3. Check the audiences you want to activate

4. Click Next

> Each audience has a unique Audience ID used to reference it in API calls and dataflow configurations.

Step 4: Identity & Attribute Mapping

This step defines which fields are exported to the destination.

For Streaming Destinations:
  • Map identity namespaces (Email, GAID, IDFA, etc.) to destination identity fields
  • Map profile attributes (first name, email, phone) to destination field names
  • At least one mandatory identity field must always be mapped
For Batch (File-Based) Destinations:
  • Map XDM schema fields to destination column names
  • Add calculated fields for transformations (e.g., lowercase email, hash functions)
  • Optionally include segment membership status as a column

Example Mapping Table:

| Source Field (AEP XDM) | Destination Field |

|---|---|

| `person.name.firstName` | `first_name` |

| `personalEmail.address` | `email` |

| `mobilePhone.number` | `phone` |

| `segmentMembership.status` | `audience_status` |

Step 5: Configure Scheduling

Batch Destinations:
  • Export frequency: Once / Daily / Weekly
  • Export type:
  • Full file — All qualifying profiles every run
  • Incremental file — Only profiles that changed since last export
  • Set start date and time
  • Configure file naming conventions
Streaming Destinations:
  • No manual schedule needed — exports are event-driven
  • A profile export is triggered when:
  • A profile qualifies for a mapped segment (`realized`)
  • A profile exits a mapped segment (`exited`)
  • A mapped attribute value changes

Step 6: Review & Finish

1. Review the summary screen:

  • Destination name and type
  • Selected audiences
  • Mapped fields
  • Export schedule

2. Click Finish

3. Destination status moves to Active ✅

Audience Membership States

Every profile exported to a destination carries a segment membership status:

| Status | Meaning | Action at Destination |

|---|---|---|

| `realized` | Profile just qualified for the segment | Include / Add |

| `existing` | Profile was already a member | Keep / Continue |

| `exited` | Profile left the segment | Remove / Suppress |

> Streaming destinations receive both `realized` and `exited` events in real-time.

> Batch destinations export `realized` + `existing` profiles as active members and exclude `exited` profiles.

Ad-Hoc (On-Demand) Activation

AEP supports immediate, on-demand activation outside of the normal schedule via the Ad-Hoc Activation API.

When to Use Ad-Hoc Activation:

  • 🔄 Backfill a destination with a large initial audience
  • 🛠️ Re-send data after a missed or failed scheduled export
  • 🧪 Test an activation flow without waiting for the next scheduled run

Required API Headers:

```

Authorization: Bearer {ACCESS_TOKEN}

x-api-key: {API_KEY}

x-gw-ims-org-id: {ORG_ID}

x-sandbox-name: {SANDBOX_NAME}

Content-Type: application/json

```

Audience Manager Expanded Activation

For organizations using Adobe Audience Manager, the Expanded Activation feature allows AAM segments to be activated to social and advertising destinations — without requiring a full Real-Time CDP license.

Expanded Activation Workflow:

```

Audience Manager Segments

↓

Imported as Custom Uploads into AEP Audience Portal

↓

Configure Advertising / Social Destination

↓

Map & Activate Audiences

↓

Facebook / Google / LinkedIn / etc.

```

Key Requirements:

  • ✅ Audience Manager audiences must be based on hashed email addresses
  • ✅ Only Advertising and Social destination connectors are supported
  • ✅ Audiences appear in AEP as custom uploads (externally generated)

Data Governance During Activation

AEP automatically enforces data governance throughout the activation process:

```

Audience Selected for Activation

↓

Data Usage Labels Evaluated (C1, C2, C3, C4...)

↓

Marketing Action Policies Checked

↓

✅ No Violation → Activation Proceeds

❌ Violation Detected → Activation BLOCKED

```

How Governance Works:

  • Every dataset and field can carry data usage labels
  • Marketing action policies define what can be done with labeled data
  • If a violation is detected, AEP shows a policy violation warning and blocks the activation
  • You must resolve the conflict by:
  • Removing the violating field from mapping
  • Changing the marketing action assigned to the destination
  • Updating data usage labels on the source data

Account Audience Activation

AEP also supports activating Account Audiences (B2B use case) — where the audience consists of companies/accounts rather than individual people.

Activating an Account Audience:

1. Navigate to Audiences → Account Audiences

2. Select the account audience

3. Click Activate to destination

> ⚠️ Only a limited number of destinations support account audiences — verify destination compatibility before proceeding.

Creating & Activating External Audiences via API

You can also create audiences externally (outside AEP) and activate them using the AEP APIs:

High-Level API Flow:

1. Authenticate — Obtain access token via Adobe Developer Console

2. Create the external audience — Use the Segmentation Service API

3. Import audience members — Batch ingest profile data linked to the audience

4. Configure destination — Set up destination via Destinations API

5. Create activation dataflow — Map audience to destination via API

6. Trigger ad-hoc export — Use ad-hoc activation API for immediate delivery

Complete Summary

| Concept | Description |

|---|---|

| Destination | External system or platform receiving your audience data |

| Activation | The process of exporting and delivering audiences to destinations |

| Streaming Activation | Real-time, event-driven exports via API |

| Batch Activation | Scheduled, file-based exports (daily, weekly, etc.) |

| Ad-Hoc Activation | On-demand API-triggered immediate export |

| Segment Membership Status | `realized`, `existing`, `exited` |

| Data Governance | Automatic policy enforcement during activation |

| Expanded Activation | AAM segments activated to social/advertising via AEP |

| Account Audiences | B2B audiences of companies/accounts (not individuals) |

| Identity Export | Requires `View Identity Graph` permission |

Audience Activation is the bridge between your data platform and your marketing execution layer — transforming unified customer intelligence into real, actionable campaigns across every channel and platform.
