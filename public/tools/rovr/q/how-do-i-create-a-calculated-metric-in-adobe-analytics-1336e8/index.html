How do I create a calculated metric in Adobe Analytics?

Creating a Calculated Metric in Adobe Analytics

Calculated metrics let you create new metrics from existing data without changing your implementation. They are available across Analysis Workspace projects and can use mathematical operators, statistical functions, segments, and more.

▶ Watch: Calculated Metrics - Implementation-less Metrics

Three Ways to Create a Calculated Metric

Method 1: Calculated Metric Builder (Available Across All Projects)

The Calculated Metric Builder is the most full-featured option and makes the metric available organization-wide.

Steps:

1. In Adobe Analytics, navigate to Components > Calculated Metrics.

2. Click the + Add button to open the Calculated Metric Builder.

3. Enter a Name for your metric (e.g., `Revenue per Visit`).

4. Optionally add a Description to help other users understand the metric's purpose.

5. Set the Format (Decimal, Time, Percent, or Currency) and number of Decimal Places.

6. Choose a Polarity (whether an upward trend is good or bad).

7. Drag and drop Metrics, Dimensions, Segments, and Functions from the left panel onto the Definition canvas.

8. Use operators (add, subtract, multiply, divide) between components to build your formula.

9. Preview the metric output in the Preview panel.

10. Click Save when finished.

Method 2: Quick Calculated Metric (Single Project Only)

This is the fastest way to create a calculated metric when you only need it for the current project.

1. In an Analysis Workspace project, right-click on a metric column header in a freeform table.

2. Select Create metric from selection.

3. Configure the formula using the selected metrics.

4. Click Apply — the metric is created and applied immediately but is only available within that project.

Method 3: Via the API

Use this method to programmatically create calculated metrics.

Prerequisites: Adobe Developer Console access with a valid token and your organization's Global Company ID.

1. Set the request URL to:

```

https://analytics.adobe.io/api/{GLOBALCOMPANYID}/calculatedmetrics

```

2. Set the request type to POST.

3. Set Authorization to use your desired token.

4. Add the `x-api-key` header with your Client ID from Adobe Developer Console.

5. In the Body, set to Raw / JSON and paste your definition. Example — a metric that returns `Page Views + 1`:

```json

{

"rsid": "examplersid",

"name": "Example calculated metric",

"definition": {

"formula": {

"func": "add",

"col1": {

"func": "metric",

"name": "metrics/pageviews",

"description": "Page Views"

},

"col2": 1

},

"func": "calc-metric",

"version": [1, 0, 0]

}

}

```

6. Submit the request. The API returns a JSON object with the new metric's `id`.

7. Verify by navigating to Components > Calculated Metrics in the Analytics UI — your new metric will appear there.

8. To delete it later, call:

```

DELETE https://analytics.adobe.io/api/{GLOBALCOMPANYID}/calculatedmetrics/{ID}

```

Key Benefits of Calculated Metrics

  • No implementation changes required — build on data you already collect.
  • Always current — metrics update dynamically, unlike exported spreadsheets.
  • Scalable — apply the same metric across large datasets (e.g., conversion rate across 100,000 products) without manual calculation.
  • Reusable — once created in the Builder, available to your entire organization across all Workspace projects.
