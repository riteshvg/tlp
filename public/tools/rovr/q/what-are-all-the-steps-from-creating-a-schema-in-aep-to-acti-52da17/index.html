What are all the steps from creating a schema in AEP to activating a dataset to a destination?

Creating a Schema and Activating a Dataset to a Destination in AEP

Here's the complete workflow from schema creation through dataset activation:

1. Create a Schema

Open the Schemas workspace

In the Adobe Experience Platform UI, navigate to the [!UICONTROL Schemas] workspace.

Select schema type

Click [!UICONTROL Create schema] in the top-right corner. You'll be presented with options to create:

  • Standard schemas
  • Relational schemas

Choose the appropriate type based on your data structure and use case.

Define your schema

  • Set up your schema class (the foundational structure)
  • Add schema field groups that contain reusable field definitions
  • Add data types for specific field configurations
  • Customize individual fields as needed

>[!NOTE]

>If you're ingesting CSV data, you can use AI-generated recommendations to map that data to an XDM schema without manually creating the schema yourself.

2. Create a Dataset

Once your schema is defined, create a dataset based on that schema:

  • The dataset will be stored in the Data Lake
  • It acts as a storage and management construct for your data
  • Datasets contain the schema (columns) and data (rows), plus metadata

3. Ingest Data into the Dataset

  • Move data from source connectors into your dataset
  • Data is persisted within the Data Lake as part of the dataset

4. Data is Processed by Platform Services

The ingested data flows through:

  • [!DNL Identity Service]: Manages identity resolution and identity graphs
  • [!DNL Real-Time Customer Profile]: Creates unified, real-time customer profiles from aggregated data across sources

5. Activate Data to Destinations

Understand dataflows

Dataflows are representations of data jobs that move data across Experience Platform. They connect your datasets to destination platforms for activation.

Choose a destination category

Experience Platform supports activation to multiple destination categories:

  • Adobe destinations
  • Advertising destinations
  • Analytics destinations
  • Cloud storage destinations
  • Data Management Platform (DMP) destinations
  • Email marketing destinations
  • Mobile engagement destinations
  • Personalization destinations
  • Social destinations
  • Streaming destinations (HTTP)
  • Survey destinations
  • Voice of the Customer destinations

Configure the destination connection

  • Set up authentication and connection details for your chosen destination
  • Map dataset fields to destination fields
  • Define activation schedules and parameters

Activate the dataset

  • Select the dataset to export
  • Configure which audiences or segments to activate (if applicable)
  • Specify scheduling and frequency
  • Review and confirm the activation

6. Monitor the Dataflow

Once activated, monitor your dataflow:

  • Track data movement from your dataset to the destination
  • Verify successful exports
  • Monitor for any errors or issues in the activation process
  • Use the Destinations UI to view dataflow status and metrics

Required Permissions

To complete this workflow, ensure you have:

  • [!UICONTROL View Destinations]
  • [!UICONTROL View Datasets]
  • [!UICONTROL Manage and Activate Dataset Destinations]

These access control permissions may need to be granted by your product administrator.

Key Prerequisites

  • Working understanding of Experience Data Model (XDM)
  • Knowledge of schema composition basics
  • Understanding of your data structure and use case
  • Access to appropriate sandboxes for development and testing

This end-to-end workflow enables you to organize your data with a schema, capture it in a dataset, and then seamlessly activate it to external partners for cross-channel marketing campaigns, email campaigns, targeted advertising, and other use cases.
