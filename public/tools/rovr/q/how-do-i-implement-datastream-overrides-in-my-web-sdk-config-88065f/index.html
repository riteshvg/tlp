How do I implement datastream overrides in my Web SDK configuration?

Implementing Datastream Overrides in Web SDK

Prerequisites

Before you begin, ensure:

  • You have access to Adobe Experience Platform or Data Collection UI
  • You have an existing datastream configured
  • Your Web SDK tag extension is installed on your tag property
  • Your implementation does not conflict with Dynamic Datastream Configuration rules (overrides take precedence and will silently bypass those rules)

Step 1: Define Overrides in the Datastreams UI

You must first configure the override options at the datastream level before they can be used in the SDK.

1. Log in to experience.adobe.com

2. Navigate to Data Collection > Datastreams in the left navigation

3. Select the datastream you want to configure overrides for

4. Define your override values (e.g., alternate Analytics report suites, Target workspaces, Platform datasets)

5. Save your datastream configuration

Step 2: Send Overrides to the Edge Network

Once overrides are defined in the Datastreams UI, you can send them to the Edge Network using one of the following methods:

Option A: Via the Web SDK Tag Extension (Recommended for Tags users)

1. Navigate to Data Collection > Tags

2. Select your desired tag property

3. Navigate to Extensions, then select Configure on the Adobe Experience Platform Web SDK card

4. Scroll down to the Datastream configuration overrides section

5. Configure overrides for each environment (development, staging, production) separately

> You can copy override settings between environments to save time.

Option B: Via the `sendEvent` Web SDK Command

You can pass overrides directly in a `sendEvent` call using the `edgeConfigOverrides` parameter:

```javascript

alloy("sendEvent", {

xdm: { / your XDM data / },

edgeConfigOverrides: {

com_adobe_analytics: {

reportSuites: ["alternate-report-suite"]

},

com_adobe_target: {

propertyToken: "your-alternate-property-token"

}

}

});

```

Option C: Via the `configure` Web SDK Command

To apply overrides globally across all events, include `edgeConfigOverrides` in the `configure` command:

```javascript

alloy("configure", {

datastreamId: "your-datastream-id",

orgId: "your-org-id",

edgeConfigOverrides: {

com_adobe_analytics: {

reportSuites: ["alternate-report-suite"]

}

}

});

```

Important Considerations

| Consideration | Detail |

|---|---|

| Override priority | Client-side datastream overrides take precedence over server-side Dynamic Datastream Configuration rules |

| Silent bypass | Events with overrides bypass Dynamic Datastream Configuration rules with no error or warning |

| Edge Network API | Edge Network API integrations do not currently support datastream overrides |

| Avoid for consent/personalization | Do not use datastream overrides for personalization use cases or consent data |

| Environment-specific | Overrides must be configured separately for development, staging, and production environments |

Conflict Check Before Going Live

Before enabling overrides, verify that:

  • Your SDK implementation does not send `edgeConfigOverrides` in `sendEvent` or `configure` calls for events that Dynamic Datastream Configuration rules should handle
  • For each event type moved to Dynamic Datastream Configuration rules, remove the corresponding `edgeConfigOverrides` from your SDK code first
