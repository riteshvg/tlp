How do I trigger a journey when a form is submitted in AJO?

Triggering an AJO Journey on Form Submission

This use case demonstrates how a customer form submission triggers an Adobe Journey Optimizer journey in real time, enabling immediate, personalized follow-up communications based on the information provided by the user.

Prerequisites

  • Access to Adobe Experience Platform
  • Basic understanding of AEP concepts (Profiles, Audiences, Datasets)
  • Familiarity with Journey Optimizer
  • Email channel configured in Journey Optimizer

High-Level Implementation Steps

1. Define an event schema in Adobe Experience Platform

2. Configure a custom event in Adobe Journey Optimizer

3. Configure the email channel

4. Build a journey triggered by the custom event

5. Send form submission data to Adobe Experience Platform

6. Configure dataset lookup to include reference data in the email

Step 1: Define an Event Schema in Adobe Experience Platform

Before configuring anything in Journey Optimizer, you need a schema that represents the form submission event.

  • Log in to Adobe Experience Platform
  • Navigate to Data Management → Schemas
  • Create an XDM Experience Event schema that captures the fields from your form (e.g., name, email, asset class selected, etc.)
  • Enable the schema for Profile and Experience Event ingestion

▶ Watch: Use Case Introduction and Event Schema Creation

Step 2: Configure a Custom Event in Adobe Journey Optimizer

1. Log in to Journey Optimizer

2. Navigate to Administration → Configurations

3. Click Manage on the Events card, then click Create Event

4. Specify the event details:

  • Name: Give it a meaningful name (e.g., `AssetClassSelected`)
  • Type: Select Unitary (for single profile-triggered events)
  • Event ID Type: Select Rule-based or System-generated based on your needs
  • Schema: Select the XDM schema you created in Step 1

5. Define the condition that identifies this event (e.g., check whether the event type equals your custom event name)

6. Save the event

Step 3: Configure the Email Channel

▶ Watch: Creating Email Channel Configuration

1. In Journey Optimizer, navigate to Administration → Channels → Create channel configuration

2. Select Email from the channel list

3. Provide a meaningful name and description

4. Fill out the email settings (from address, reply-to, etc.)

5. Specify execution details — define which profile field holds the recipient's email address

6. Activate the email channel configuration

Step 4: Build the Journey

▶ Watch: Create a Journey

1. In Journey Optimizer, navigate to Journeys → Create Journey

2. Add an Event as the entry trigger:

  • Select the custom event you created in Step 2 (e.g., `AssetClassSelected`)
  • The journey will start whenever this event is received for a profile

3. Add an Email action node:

  • Select your email channel configuration from Step 3
  • Design your email content using the Email Designer, personalizing it with attributes from the event payload (e.g., the form fields submitted)

4. Optionally add:

  • Wait nodes for delayed follow-ups
  • Condition nodes to branch logic based on form values
  • Lookup dataset references to enrich the email with additional reference data

5. Publish the journey

Step 5: Send Form Submission Data to Adobe Experience Platform

When a user submits the form on your website or application, you need to stream that data into AEP to trigger the journey.

  • Option A – HTTP Streaming API: Configure an HTTP Streaming inlet in AEP and POST the form data as an XDM Experience Event directly to the streaming endpoint
  • Option B – Adobe Web SDK: Use the AEP Web SDK (configured via Adobe Experience Platform Tags) to send the form submission event from the browser

The event payload must match the XDM schema defined in Step 1 and include the correct event type so Journey Optimizer identifies it as the custom event configured in Step 2.

Step 6: Configure Dataset Lookup (Optional)

If your email needs to reference additional data not captured in the form (e.g., product details, pricing tiers), you can configure a lookup dataset:

  • Create a dataset in AEP containing the reference data
  • Configure the dataset lookup in your journey to join on a key from the event payload (e.g., asset class ID)
  • Use the looked-up fields as personalization tokens in your email content
