52. What are Data Usage Labels? Why are they important?

Data Usage Labels in Adobe Experience Platform

What are Data Usage Labels?

Data Usage Labels are metadata tags applied to datasets and individual fields (columns) in AEP that categorize data according to governance restrictions and compliance regulations. They define how data can and cannot be used within your organization.

Labels work in conjunction with governance policies to create a complete data governance framework — labels identify what the data is, and policies enforce what can be done with it.

Simple Analogy:

Think of labels as warning stickers on hazardous materials:

  • 🏷️ Label = "This data contains PII"
  • 📋 Policy = "PII data cannot be used for behavioral advertising"
  • ✅ Enforcement = The system automatically blocks attempts to use PII for behavioral ads

Control Access to Sensitive and Protected Data

Data usage labels give you the ability to:

> "Control access to specific data attributes and/or segments, allowing you to design flexible workflows for the various personas and teams operating Experience Platform use cases."

This means you can:

  • ✅ Allow marketers to use certain audience segments
  • ❌ Block engineers from accessing PII fields
  • ✅ Permit data analysts to export financial data to data warehouses
  • ❌ Prevent that same data from being used for social media advertising
  • ✅ Enable compliance teams to audit data usage across the organization

Data Governance Framework in AEP

The complete data governance framework consists of:

```

Data Usage Labels (What is the data?)

↓

Marketing Action Policies (What can we do with it?)

↓

Activation / API Calls (Trying to use the data)

↓

Policy Enforcement (Allowed? Blocked?)

```

The Three Pillars:

| Pillar | Definition | Example |

|---|---|---|

| Labels | Categorize data by sensitivity/type | `C1` (Contractual), `I1` (Identifiable), `S1` (Sensitive) |

| Policies | Define rules about what marketing actions are allowed | "Cannot use PII for behavioral advertising" |

| Enforcement | Automatically check policies during data usage | Block activation if policy violated |

Core Data Usage Labels

AEP provides out-of-the-box labels that cover common data governance scenarios. Labels are organized by category:

Contract (C) Labels — Legal/Business Restrictions

| Label | Name | Description |

|---|---|---|

| C1 | Data cannot be combined with directly identifiable information | PII must stay separate from behavioral data |

| C2 | Data cannot be exported outside of AEP | Contractually bound — cannot leave the system |

| C3 | Data cannot be combined with on-site data | Cannot merge with website behavioral tracking |

| C4 | Data cannot be used for on-site personalization | Cannot personalize web/app experiences |

| C5 | Data cannot be used for cross-site targeting | Cannot use for retargeting across sites |

| C6 | Data cannot be used for on-site ad targeting | Cannot target ads on the website |

| C7 | Data cannot be used for on-site content personalization | Cannot personalize email/SMS/other content |

| C8 | No measurement | Cannot be used for analytics on websites or apps |

| C9 | Data cannot be used in ML/AI models | Cannot use for machine learning |

| C10 | Data cannot be used for geofencing | Cannot use for location-based targeting |

| C11 | Data cannot be used for interest-based marketing | Cannot use for behavioral/interest targeting |

| C12 | No general data export | Cannot be exported or downloaded from AEP |

Identity (I) Labels — PII Types

| Label | Name | Description |

|---|---|---|

| I1 | Directly identifiable data | Can directly identify a person (name, SSN, email) |

| I2 | Indirectly identifiable data | Can identify with other data (device ID + location) |

Sensitive (S) Labels — Sensitive Data Categories

| Label | Name | Description |

|---|---|---|

| S1 | Sensitive financial data | Bank accounts, credit card numbers, financial records |

| S2 | Sensitive health data | Medical records, health conditions, prescriptions |

Special Labels

| Label | Name | Description |

|---|---|---|

| RHD | Contains Recorded Heightened Data | Data flagged under regulatory or business requirements |

| LDI | Limited Data Use | Data subject to limited use agreements |

Where Labels Are Applied

1. Dataset-Level Labels

Labels applied at the dataset level are automatically propagated to all fields within that dataset.

```

Dataset: Customer_Master

├── Applied Labels: [C2, I1, S1]

├── Field: email

│ └── Inherits: [C2, I1, S1]

├── Field: phone

│ └── Inherits: [C2, I1, S1]

└── Field: address

└── Inherits: [C2, I1, S1]

```

2. Field-Level Labels

Labels can also be applied directly to individual fields without propagation to the entire dataset.

```

Dataset: Customer_Master

├── Field: email

│ └── Labels: [C2, I1] ← Field-specific label

├── Field: phone

│ └── No labels (inherited from dataset if any)

└── Field: preferences

└── Labels: [C11] ← Different label than other fields

```

This allows granular control — some fields in a dataset might be highly sensitive while others are less restricted.

Label Inheritance for Audiences

When you create an audience segment, it automatically inherits labels from the underlying datasets and fields used to build it.

```

Audience: "High-Value Customers"

Built from:

├── Dataset A (Labels: C2, I1)

├── Dataset B (Labels: C11)

└── Dataset C (No labels)

Result: Audience inherits [C2, I1, C11]

```

When this audience is activated to a destination, the inherited labels are validated against policies to ensure compliance.

Why Data Usage Labels Are Important

1. Regulatory Compliance

Data usage labels help you comply with privacy regulations:

  • 🇪🇺 GDPR — Identifies personally identifiable information (I1, I2 labels)
  • 🇨🇦 PIPEDA — Canadian privacy law requiring PII protection
  • 🇺🇸 CCPA/CPRA — California privacy laws with consumer rights
  • 🏥 HIPAA — Health information protection (S2 label)
  • 💳 PCI-DSS — Payment card data security (S1 label)

2. Risk Mitigation

Labels reduce the risk of accidental data misuse:

```

Scenario: Marketer tries to export customer email addresses to a 3rd party

Without Labels:

❌ No controls → Export succeeds → Data breach → Lawsuits

With Labels:

📋 Email field has [C2, I1] labels

📋 Policy: "C2 data cannot be exported"

🛡️ System blocks export automatically

✅ Breach prevented

```

3. Data Accountability

Labels create an audit trail of data usage:

  • 📊 Track which fields have been used for which purposes
  • 👥 Identify who accessed sensitive data and when
  • 📋 Demonstrate compliance to regulators and auditors
  • 🔍 Answer questions like: "Is this data being used for PII-restricted purposes?"

4. Governance at Scale

In large organizations with hundreds of datasets, labels enforce consistent governance
