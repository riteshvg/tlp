How do I migrate projects and components from Adobe Analytics to Customer Journey Analytics?

Migrating Projects and Components from Adobe Analytics to Customer Journey Analytics

Overview

The migration process involves two main activities:

  • Re-creating Adobe Analytics projects in Customer Journey Analytics
  • Mapping dimensions and metrics from Adobe Analytics report suites to dimensions and metrics in Customer Journey Analytics data views

Segments are also migrated but do not need to be manually mapped. All migrated components appear in a migration summary when the migration completes.

Prerequisites

Before migrating, you must complete the following preparation steps:

1. Ingest Data into Adobe Experience Platform

You need to get your Adobe Analytics report suite data into CJA using one of two methods:

Option A — Adobe Analytics Source Connector:

1. Set up report suites for ingestion into Adobe Experience Platform and Customer Journey Analytics

2. Ingest and use the data via the Analytics source connector

Option B — Web SDK:

1. Set up report suites for ingestion into Adobe Experience Platform and Customer Journey Analytics

2. Ingest data via the Adobe Experience Platform Web SDK

> ⚠️ When using the Web SDK, all schema fields must be manually mapped

2. Create a Connection and Data View in CJA

You will need a Customer Journey Analytics data view ID ready before migration, as it is required to map components from your Adobe Analytics report suite.

3. Conduct an Analytics Inventory

Use the Adobe Analytics Inventory tool (available to Analytics administrators) to understand what projects, components, and assets exist before migrating.

Migration Steps (UI Method)

1. Log in as an Adobe Analytics Administrator — migration is an admin-only function.

2. Navigate to the migration tool in the Adobe Analytics admin interface.

3. Select the projects you want to migrate to Customer Journey Analytics.

4. Map dimensions and metrics:

  • Some dimensions and metrics are automatically mapped
  • Others require manual mapping to their corresponding CJA data view dimensions and metrics

5. Review the migration summary — once the migration completes, all migrated components are listed here.

6. Verify the migrated projects in Customer Journey Analytics Analysis Workspace.

Migration via API (Alternative Method)

You can also perform the migration using the Adobe Analytics 2.0 APIs. The two migration API endpoints support:

  • Segments
  • Calculated metrics
  • Date ranges
  • Dimensions and metrics (if mapped beforehand)

Required identifiers for API migration:

  • Adobe Analytics project ID
  • Adobe Analytics report suite ID
  • Customer Journey Analytics data view ID

> ⚠️ If you plan to migrate dimensions and metrics via API, use the mappings APIs first so the project ID can be associated with those mappings before running the project migration.

What Gets Migrated

| Component | Migrated? | Requires Mapping? |

|---|---|---|

| Projects (Analysis Workspace) | ✅ Yes | No (re-created automatically) |

| Dimensions & Metrics | ✅ Yes | Yes (some auto, some manual) |

| Segments | ✅ Yes | No |

| Calculated Metrics | ✅ Yes | No |

| Date Ranges | ✅ Yes | No |

Post-Migration: User Onboarding

Plan 3–6 months for users to become familiar with the key differences between Analysis Workspace in Adobe Analytics and Customer Journey Analytics. While Analysis Workspace is the primary tool in both products, there are meaningful differences in how it works in CJA, particularly around cross-channel data and the connection/data view model.
