Setting Up Customer Journey Analytics (CJA) in Your Enterprise

Here is a comprehensive overview of the end-to-end setup process for Customer Journey Analytics.

Prerequisites

Before you begin, confirm the following:

  • Your organization is provisioned for Adobe Experience Platform (AEP).
  • You have purchased the Customer Journey Analytics SKU (either as an add-on to Adobe Analytics Select, Prime, or Ultimate, or as a standalone product).
  • You have appropriate access control permissions within your organization.

High-Level Architecture

CJA works by pulling datasets from Adobe Experience Platform into Connections, defining Data Views on top of those connections, and then analyzing data in Analysis Workspace.

Step-by-Step Setup Workflow

Step 1 — Architect and Create Your Schema

1. Define a Solution Design Reference — document every field you plan to collect (dimensions, metrics, identifiers).

2. Navigate to Adobe Experience Platform and create an Experience Data Model (XDM) schema that maps to your data sources.

3. If migrating from Adobe Analytics, decide whether to use your existing schema or create a new one optimized for CJA.

Step 2 — Create a Datastream

1. In Adobe Experience Platform Data Collection, create a Datastream.

2. Add Adobe Experience Platform as a service within the datastream so that data flows into AEP datasets.

Step 3 — Create Datasets

1. Create one or more datasets in Adobe Experience Platform based on your XDM schema.

2. If you use classifications, create lookup datasets to support them.

3. Monitor dataset ingestion to ensure data is arriving correctly.

Step 4 — Implement Data Collection (Web SDK Recommended)

1. Create a Tag property in Adobe Data Collection.

2. Add the Web SDK extension to your tag property.

3. Implement the loader tag for the Web SDK extension on your website or app.

4. Configure the tag to send data to the datastream you created in Step 2.

> If you are an existing Adobe Analytics customer, you can also use the Adobe Analytics Source Connector to ingest historical and ongoing report suite data into AEP — without re-implementing data collection immediately.

Step 5 — Set Up a Connection in CJA

1. In Customer Journey Analytics, navigate to Connections.

2. Create a new Connection and select the AEP dataset(s) you want to include.

3. Configure connection settings such as the Person ID (the identity field that stitches data across channels) and data retention settings.

Step 6 — Create a Data View

1. In CJA, create a Data View on top of your connection.

2. Define which dimensions and metrics from your dataset fields are available for reporting.

3. Configure attribution models, persistence settings, and value filters for each component as needed.

Step 7 — Build Projects in Analysis Workspace

1. Navigate to the Workspace tab in CJA.

2. Create projects using the drag-and-drop interface.

3. Build cross-channel reports and visualizations using the dimensions and metrics defined in your Data View — no SQL required.

Bringing in Historical Adobe Analytics Data

If you are migrating from Adobe Analytics:

1. Create an XDM schema for the Analytics Source Connector.

2. Set up the Adobe Analytics Source Connector in AEP, mapping report suite fields to XDM.

3. Add the Source Connector dataset to your CJA Connection.

4. Compare data between your old Adobe Analytics implementation and CJA to understand any differences.

User Onboarding Recommendations

  • Provide hands-on training focused on differences between Adobe Analytics and CJA (e.g., data models, reporting paradigms).
  • Offer live workshops and on-demand resources (video tutorials, internal documentation).
  • Allow users 3–6 months to become comfortable with Analysis Workspace in CJA.
  • Direct users to Customer Journey Analytics tutorials on Experience League for self-service learning.

Access the Guided Upgrade Tool

If you are upgrading from Adobe Analytics, CJA includes a built-in upgrade guide:

1. In CJA, select the Workspace tab.

2. In the left panel, select Upgrade to Customer Journey Analytics.

3. Follow the on-screen instructions for a dynamically generated, step-by-step upgrade path tailored to your organization.
