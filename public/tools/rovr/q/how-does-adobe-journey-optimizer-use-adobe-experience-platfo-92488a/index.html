How Adobe Journey Optimizer Uses AEP Profiles and Audiences to Personalise Journeys

The Foundation: AEP as the Central Data Layer

Adobe Journey Optimizer is built directly on top of Adobe Experience Platform — it does not maintain a separate, isolated data store. Instead, it uses the same shared data foundation as other Adobe CX Enterprise applications.

> Think of Adobe Experience Platform as your central data layer, and Journey Optimizer as an application that orchestrates journeys and messages using that shared data foundation.

Here's how the architecture works:

  • Schemas and datasets live in Adobe Experience Platform
  • Identities and the Real-Time Customer Profile are managed by AEP's Identity Service and Profile Service
  • Journey Optimizer reads profile and event data from AEP to evaluate journey conditions, personalise messages, and select offers
  • Journey Optimizer writes interaction data back into AEP datasets (send, open, click, bounce events, and journey step events)
  • It can also look up additional datasets at runtime without copying that data into the profile

Real-Time Customer Profiles

Journey Optimizer enables brands to create a unified, 360-degree view of each customer through real-time customer profiles. Key capabilities include:

  • Real-time data ingestion — as data streams in or batch uploads occur, datasets update dynamically
  • Identity graph creation — resolving identities across channels and devices
  • Customer engagement tools — profiles engaged through journeys, campaigns, or decisioning count toward the Engageable Profiles licence metric

Audiences

Audiences are collections of people who share similar behaviours and/or characteristics. They are:

  • Centrally configured and maintained in AEP using the Adobe Experience Platform Segmentation Service
  • Readily accessible within Journey Optimizer to be activated in journeys and campaigns

Dynamic Audience Membership

As real-time data streams or batch uploads occur, Journey Optimizer dynamically moves individuals in and out of audiences and journeys in real time — ensuring that targeting always reflects the most current customer state.

Advanced Segmentation with Real-Time CDP

When combined with Adobe Real-Time Customer Data Platform, Journey Optimizer lets you:

  • Layer in audiences for more complex segmentation
  • Bidirectionally share audiences with other Adobe CX Enterprise solutions

How Personalisation Works in Practice

Journey Optimizer uses profile and audience data across three core activation areas:

| Area | How Profile/Audience Data Is Used |

|---|---|

| Journeys | Journey conditions are evaluated against real-time profile data; individuals enter, progress, or exit based on live behavioural triggers and audience membership |

| Campaigns | One-time messages are sent to all individuals belonging to a selected audience, with content personalised using profile attributes |

| Decisioning | Offers are selected based on profile data and eligibility rules defined against audience segments |

The Continuous Data-to-Experience Cycle

The overall flow is continuous:

1. Data is collected — client-side experience data is sent to the AEP Edge Network

2. Profiles are built and updated — Identity Service and Profile Service maintain a real-time unified profile

3. Audiences are evaluated — Segmentation Service determines audience membership dynamically

4. Journey Optimizer acts — reads profile/audience data to orchestrate personalised journeys and messages

5. Interaction data is written back — engagement events (opens, clicks, etc.) flow back into AEP datasets, enriching the profile further for future personalisation

This closed loop ensures that each customer interaction informs the next, enabling data-driven personalisation at scale.
