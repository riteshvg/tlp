What the Documentation Confirms About Derived Fields

Derived Fields Capabilities

The documentation explicitly covers derived fields for marketing channel setup:

  • Non-destructive — original source data is never modified
  • Retroactive — logic applies to historical data automatically at query time
  • Flexible rule builder — customizable logic using conditions, functions, and templates
  • Marketing Channel function template — a pre-built template to quickly configure channel logic
  • Report-time processing — channels evaluated at query time, not collection time
  • Complex transformations — supports concatenation, classification, URL parsing, and more

Derived Fields Best Practices

The documentation also notes that derived fields should be used with discipline to avoid:

  • Performance problems — overly complex logic can slow query execution
  • Maintenance overhead — too many derived fields can become difficult to manage
  • Logical errors — common mistakes in classification, attribution, and data transformation

Recommended Approach Based on Available Documentation

Based on what is documented, Derived Fields are the clearly recommended approach for marketing channel setup in CJA because:

1. They are explicitly supported via the Marketing Channels function template

2. They support retroactive, non-destructive channel definition

3. They align with CJA's report-time processing architecture

4. They work natively with Web SDK implementations

5. They provide full flexibility to replicate or improve upon Adobe Analytics channel logic
