What is a datastream override?

Datastream Overrides

A datastream override allows you to define additional configurations for your datastreams that conditionally trigger different datastream behaviors without requiring you to create a new datastream or modify your existing settings.

Key Characteristics

Datastream overrides help you:

  • Conditionally trigger different datastream behaviors based on specific conditions
  • Avoid creating multiple datastreams for different scenarios
  • Update configurations dynamically without changing your website or app code
  • Pass additional configurations to the Edge Network via the Web SDK

What You Can Override

Datastream configuration overrides let you modify the following datastream configurations:

  • Experience Platform event datasets
  • Adobe Target property tokens
  • Audience Manager ID sync containers
  • Adobe Analytics report suites

Common Use Cases

Multi-country operations: A company with different websites for different countries can dynamically switch data flow to country-specific datastreams, and also send data to a global datastream when customers perform important actions like placing an order or updating their profile.

Multiple business units: Companies with separate business units can use datastream overrides to ensure each business unit has its own datastream to receive data, instead of sending everything to a default datastream.

How It Works

Datastream overrides operate as a two-step process:

1. Define overrides in the Datastreams UI - Configure the override settings in the Adobe Experience Platform Datastreams configuration

2. Configure in your tag extension or Web SDK - Set up the overrides in your Web SDK tag extension or pass them via a Web SDK send event action

Overrides must be configured on a per-environment basis (development, staging, and production each have separate overrides), and you can copy override settings between environments as needed.
