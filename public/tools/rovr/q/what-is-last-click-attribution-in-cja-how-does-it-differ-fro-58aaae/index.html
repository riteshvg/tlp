Last Touch Attribution in Customer Journey Analytics

What is Last Touch Attribution?

Last Touch attribution gives 100% credit to the touch point occurring most recently before a conversion. It is the most basic and common attribution model, frequently used for:

  • Conversions with a short consideration cycle
  • Teams managing search marketing
  • Analyzing internal search keywords

How Last Touch Differs from First Touch

| | Last Touch | First Touch |

|---|---|---|

| Credit assignment | 100% to the most recent touchpoint before conversion | 100% to the first touchpoint seen in the lookback window |

| Best for | Short consideration cycles, search marketing, keyword analysis | Brand awareness campaigns, customer acquisition analysis |

| Common users | Search marketing teams | Display, social marketing teams |

In short:

  • Last Touch answers: "What was the final thing the customer interacted with before converting?"
  • First Touch answers: "What first brought the customer to us?"

> Important note on Adobe Analytics vs. CJA: Unlike Adobe Analytics, which collects Last Touch attribution in the report suite at hit time, CJA calculates attribution at report run time. This means attribution models in CJA apply retroactively to all historical data without reprocessing.

How to Configure Last Touch Attribution in CJA

There are two places to configure attribution in Customer Journey Analytics:

Option 1: Data View Settings (Default for all reports)

Set Last Touch as the default attribution model for a metric component so it applies across all reports using that data view:

1. Navigate to Data Views in CJA

2. Open your data view and go to the Components tab

3. Select the metric you want to configure

4. Under Attribution model, choose Last Touch

5. Set the desired lookback window

6. Save the data view

> You can also add multiple copies of the same metric, each with a different attribution model, for side-by-side comparisons directly in the data view.

Option 2: Component-Level Override in Analysis Workspace (Ad hoc)

Override attribution for a single instance within a Freeform table without changing the data view default:

1. Open a project in Analysis Workspace

2. Drag a metric (e.g., Revenue) into a Freeform table

3. Right-click the metric column header

4. Select Use non-default attribution model

5. Choose Last Touch and set your lookback window

6. Click Apply

You can also drag the same metric into the table multiple times, assigning a different attribution model to each instance for a direct comparison.

Option 3: Use the Attribution Panel

For a structured, visual comparison of Last Touch vs. other models side by side:

▶ Watch: Build the Attribution Panel

The Attribution panel lets you define success metrics, select and compare multiple attribution models (including Last Touch and First Touch), and visualize results with bar charts, Venn diagrams, and channel flow views.

Quick Tip: Aligning with GA4

If you're comparing CJA to GA4 (which defaults to data-driven attribution), setting GA4 to last-click and configuring a matching Last Touch model in CJA is the most reliable way to establish a like-for-like baseline for your metrics.
