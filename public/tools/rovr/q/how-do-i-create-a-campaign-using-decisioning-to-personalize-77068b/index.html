Creating a Campaign Using Decisioning to Personalize Web Offers in Adobe Journey Optimizer

Adobe Journey Optimizer (AJO) campaigns connect audiences, decision policies, and channels to deliver personalized offers in real time. Here's a complete guide to setting this up.

Overview

When a logged-in user visits your site, a personalization request is sent to AJO. Based on the user's profile attributes (like ZIP code or annual income), the decision policy evaluates available offers, applies selection strategy and ranking logic, and returns the best-matched offer as content displayed on the webpage.

Prerequisites

Before creating the campaign, ensure you have the following ready:

  • Access to Adobe Journey Optimizer and Adobe Experience Platform
  • Audiences already defined in AEP (e.g., segmented by user preferences or attributes)
  • A channel configuration (e.g., Web channel or Code-based experience) already created
  • Offer items, eligibility rules, and a ranking formula defined in Decisioning
  • A placement ID and decision ID if delivering via API
  • Basic understanding of AEP profiles, audiences, and datasets

Step-by-Step: Create a Campaign with Decisioning

Step 1: Navigate to Campaigns

1. Log in to Adobe Journey Optimizer.

2. In the left navigation, go to Journey Management > Campaigns.

3. Click Create campaign.

4. Select Schedule Marketing as the campaign type.

Step 2: Configure Campaign Details

5. Provide a meaningful name for the campaign.

6. Set your audience — select the AEP audience you want to target (e.g., users interested in Stocks, Bonds, or CDs).

7. Define the schedule — set start/end dates or trigger conditions for when the campaign should run.

Step 3: Configure the Action (Channel)

8. Navigate to the Action tab.

9. Select Code-based experience (or Web channel if configuring web offers) as the action type.

10. Select the channel configuration you created in the prerequisite steps.

11. Click Edit content.

Step 4: Add a Decision Policy

12. In the content editor, click Edit code to open the personalization editor — a non-visual, code-based interface for authoring your experience.

13. Insert a decision policy into the code. The decision policy controls how offers are selected and includes:

  • A collection of offer items (e.g., filtered by ZIP code or income level)
  • Eligibility rules that determine which offers apply to a given user
  • A ranking formula that scores and prioritizes the most relevant offers

Step 5: Define the Selection Strategy

14. Within the decision policy, configure your selection strategy:

  • Map it to the appropriate offer collection
  • Apply the relevant eligibility rules
  • Assign your ranking formula to prioritize offers

Step 6: Personalize and Preview Content

15. Use the personalization editor to write the code (e.g., HTML) that will render the offers on your webpage.

16. Reference profile attributes (e.g., `{{profile.zipCode}}`, `{{profile.annualIncome}}`) to drive dynamic content.

17. Preview how the offer will appear to different audience segments.

Step 7: Activate the Campaign

18. Return to the campaign summary screen.

19. Review all settings — audience, action, schedule, and decision policy.

20. Click Activate to publish the campaign.

How It Works at Runtime

Once live, the flow works as follows:

1. A user visits the website and a personalization request is sent to AJO via the AEP Web SDK (`Alloy.js`).

2. AJO uses the user's stitched identity and profile attributes to evaluate the decision policy.

3. The selection strategy and ranking logic determine the best-matched offer.

4. The offer is returned as HTML content and rendered in the user-facing experience (e.g., a carousel on the webpage).

Key Components Summary

| Component | Purpose |

|---|---|

| Campaign | Container orchestrating the entire experience delivery |

| Audience | Defines who receives the personalized offers |

| Code-based / Web Channel | The delivery surface for the offer content |

| Decision Policy | Controls offer selection logic |

| Selection Strategy | Combines collection + eligibility rules + ranking |

| Ranking Formula | Scores offers to surface the most relevant one |

| AEP Web SDK | Sends personalization requests from the browser to AJO |
