Getting Started with Customer Journey Analytics (CJA) — Step-by-Step Setup

Welcome! Here is a structured overview of how CJA works and how to set it up from scratch.

What is CJA?

CJA is Adobe's next-generation analytics solution built on Adobe Experience Platform (AEP). It unifies online and offline data — websites, mobile apps, call centers, CRM, in-store systems — into a single person-centric view, analyzed through Analysis Workspace.

▶ Watch: What is Customer Journey Analytics?

Prerequisites

Before you begin, ensure you have:

  • Access to Adobe Experience Platform (AEP)
  • Data already ingested into AEP datasets (event, profile, or lookup data)
  • A CJA license provisioned for your organization
  • Admin or Developer permissions in both AEP and CJA

High-Level Architecture

CJA sits on top of AEP and follows this flow:

AEP Datasets → CJA Connection → CJA Data View → Analysis Workspace

Step-by-Step Setup

Step 1 — Set Up Permissions

1. In Adobe Admin Console, assign users to the appropriate CJA product profiles.

2. Grant access to the relevant AEP sandboxes and datasets.

3. Assign roles such as Admin, Editor, or Viewer as appropriate.

Step 2 — Prepare Your Data in AEP

1. Ensure your data is ingested into AEP datasets using batch ingestion, streaming, or a source connector.

2. Confirm your datasets have a consistent Person ID (e.g., CRM ID, email hash, ECID) that will be used to stitch cross-channel data.

3. Plan your data model — identify which datasets are event, profile (lookup), or summary data.

Step 3 — Create a Connection

A Connection links your AEP datasets to CJA.

1. In CJA, navigate to Connections in the top menu.

2. Click Create new connection.

3. Give your connection a name and select the AEP sandbox.

4. Click Add datasets and select the relevant datasets (event, profile, lookup).

5. For each dataset, configure:

  • Person ID — the field used to identify individuals across datasets
  • Timestamp — the event timestamp field
  • Data source type — event, lookup, or profile

6. Set the connection rolling window (how much historical data to include).

7. Click Save.

Step 4 — Configure a Data View

A Data View defines how the data in your connection is interpreted for reporting.

1. In CJA, navigate to Data Views.

2. Click Create new data view and select your connection.

3. Under Settings, configure:

  • Name and description
  • Time zone
  • Session timeout rules

4. Under Components, drag dimensions and metrics from your schema fields into the view.

5. Configure component settings such as attribution models, value bucketing, substring logic, and include/exclude filters.

6. Click Save and continue.

Step 5 — Validate Identity & Stitching (Optional but Recommended)

1. Review how CJA is using your Person ID for identity resolution.

2. If you have anonymous and authenticated events to connect, enable Stitching to stitch identities across sessions.

3. Use the stitching validation preview to confirm data quality before full enablement.

Step 6 — Build Reports in Analysis Workspace

1. Navigate to Workspace in CJA.

2. Click Create project (or start from a template).

3. Select your Data View from the dropdown.

4. Use the drag-and-drop interface to build freeform tables, flow diagrams, fallout funnels, attribution panels, and more.

5. Apply filters (segments), date ranges, and calculated metrics as needed.

Step 7 — (Optional) Publish Audiences

  • Once you have meaningful segments, use Audience Publishing to send CJA-defined audiences back to AEP Real-Time CDP for activation.
