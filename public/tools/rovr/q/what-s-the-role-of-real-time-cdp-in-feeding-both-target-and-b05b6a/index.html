What's the role of Real-Time CDP in feeding both Target and AJO with audiences?

Real-Time CDP as the Audience Engine for Adobe Target and Adobe Journey Optimizer

Real-Time CDP serves as the centralized audience creation and activation hub that feeds both Adobe Target and Adobe Journey Optimizer with rich, unified customer profiles and segments. Here's how this works end to end.

What Real-Time CDP Provides at Its Core

Real-Time CDP is built on Adobe Experience Platform and brings together known and anonymous data from multiple enterprise sources to create unified customer profiles. These profiles power personalization and engagement across downstream applications like Target and AJO.

Its four core pillars that make this possible are:

  • Data unification — Combining enterprise data sources into single customer profiles
  • Governance & privacy — Trusted controls for known and anonymous identifiers
  • AI/ML-powered segmentation — Building actionable audiences at scale
  • Real-time activation — Delivering audiences across all channels and destinations

How Real-Time CDP Feeds Adobe Target

Real-Time CDP activates audiences to Adobe Target primarily through the Edge Network, enabling real-time on-site and in-app personalization. The flow works as follows:

1. Profiles are built in AEP by ingesting data from CRM systems, web/mobile behavior, call center data, and other enterprise sources

2. Audiences are defined in Real-Time CDP using AEP's segmentation tools (batch, streaming, or edge segmentation)

3. Target is configured as a destination in Real-Time CDP

4. Audience membership is shared to Target via the Edge Network, where it is available for experience decisioning in real time

5. Target uses the audience to serve personalized experiences — on-site offers, recommendations, A/B test variants — based on AEP profile attributes and segment membership

Key benefit: Because this runs through the Edge Network, the audience data is available at the moment a visitor hits your site, enabling truly real-time personalization without latency.

How Real-Time CDP Feeds Adobe Journey Optimizer

AJO is natively built on Adobe Experience Platform, which means it shares the same profile store and segmentation engine as Real-Time CDP. The integration is therefore deeply native:

1. Unified profiles built in AEP/Real-Time CDP are directly accessible within AJO — no separate export is needed

2. Audiences created via Audience Composition (a Real-Time CDP capability) can be used directly in:

  • AJO Campaigns
  • AJO Journeys (via Read Audience entry)

3. Real-time profile updates triggered by events flow into AJO journeys that use event-based triggers, allowing AJO to act on the most current profile data

4. Enrichment attributes from Audience Composition can accompany audience members into AJO for use in personalization within messages

Key Guardrails to Be Aware Of

| Guardrail | Detail |

|---|---|

| Max published Audience Compositions | 50 per sandbox (planned to increase) |

| Audience Composition block limit | Max 1 Enrich block, 1 Rank or Split block per composition |

| Split block data types | Only String supported — no date or boolean |

| Audience availability in AJO | Compositions must be published before they appear in AJO Campaigns or Journeys |

Summary

Real-Time CDP acts as the single source of truth for audience definition and profile enrichment, while Adobe Target and AJO act as the activation layers — Target for real-time on-site personalization and AJO for orchestrated cross-channel engagement. The shared foundation of Adobe Experience Platform is what makes this a cohesive, low-latency ecosystem rather than a series of point-to-point integrations.
