How to Test Personalization Before Sending

Adobe Journey Optimizer provides multiple tools and workflows to validate that your personalization renders correctly before sending messages to your audience.

1. Use the Personalization Playground

Before building your content:

  • Access the personalization playground to learn syntax and test expressions with sample data
  • Experiment with personalization logic in a safe environment
  • Validate that your expressions work as expected before applying them to your message

2. Preview with Test Profiles

During content creation:

  • Use test profiles to preview how your personalized content renders for specific customers
  • Test multiple scenarios by selecting different test profiles
  • Verify that personalization variables (names, attributes, conditional logic) display correctly

3. Test Multiple Variations with Sample Input Data

  • Use sample input data to test up to 30 variants without creating individual test profiles
  • Upload CSV/JSON files containing different customer attributes
  • Add values manually or auto-generate variants with AI
  • Preview all variations at once to see how content adapts to different profiles

Real-World Testing Scenarios

Scenario 1: Multi-segment campaign with personalized emails

→ Upload a CSV with different customer attributes and preview up to 30 variations simultaneously instead of creating individual test profiles

Scenario 2: Email-specific validation

→ Run Litmus tests to check display across email clients, then check the spam report to ensure inbox placement

Scenario 3: Getting stakeholder approval

→ Send proofs to internal reviewers with test profile data so they see exactly what customers will receive

Recommended Testing Sequence

1. Test expressions – Use the personalization playground

2. Preview personalization – Use test profiles to validate rendering

3. Test variations – Use sample input data for 10+ customer scenarios

4. Check deliverability – Run Litmus tests and spam reports for email

5. Send proofs – Share with stakeholders for final sign-off

6. Final check – Review approval workflows before sending

Key Takeaways

  • Test profiles are essential for previewing personalized content
  • Sample input data is the time-saver for testing multiple variations efficiently
  • Email-specific tools include rendering tests, spam reports, and proofs
  • Start early – Test personalization before building the full message content
