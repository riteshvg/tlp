What is Data Prep in Datastreams?

Data Prep is a feature within datastreams that allows you to transform, map, and validate data as it flows from Web SDK to Adobe services — without requiring code changes.

Data Prep sits between the Edge Network and your downstream services, allowing you to:

```

Web SDK XDM Data

↓

┌──────────────────────────────┐

│ DATA PREP │

│ │

│ • Map XDM fields │

│ • Transform values │

│ • Create calculated fields │

│ • Validate data │

│ • Enrich data │

└──────────────────────────────┘

↓

AEP / Analytics / Target / etc.

```

Data Prep Capabilities

1. Field Mapping

Map XDM fields to solution-specific field names:

```

XDM Field → Target Field

──────────────────────────────────────────────

commerce.productViews.value → Analytics event1

productListItems[0].SKU → Analytics eVar10

web.webPageDetails.name → Analytics pageName

personalEmail.address → Analytics visitorID

```

2. Data Transformation

Transform field values on the fly:

```

Original Value: 2024-01-15T10:30:00Z

Function Applied: Extract date only

Transformed Value: 2024-01-15

Original Value: jane.smith@email.com

Function Applied: Lowercase

Transformed Value: jane.smith@email.com (already lowercase)

Original Value: "running shoes"

Function Applied: Capitalize

Transformed Value: "Running Shoes"

Original Value: 129.99

Function Applied: Round to nearest integer

Transformed Value: 130

```

3. Calculated Fields

Create new fields based on existing data:

```

New Field: "Total Order Value"

Formula: commerce.order.priceTotal * (1 + commerce.order.taxAmount)

Result: Complete order value including tax

New Field: "Product Category Path"

Formula: CONCATENATE(productListItems[0].category, " > ",

productListItems[0].subcategory)

Result: "Running > Shoes"

New Field: "Days Since Last Purchase"

Formula: NOW() - DATEDIFF(lastPurchaseDate)

Result: 15 (days)

```

4. Conditional Logic

Apply different transformations based on conditions:

```

IF eventType == "commerce.purchases"

THEN create field "order_value" = commerce.order.priceTotal

ELSE IF eventType == "commerce.productViews"

THEN create field "product_viewed" = productListItems[0].SKU

ELSE

set field "other_event" = true

```

5. Data Validation

Validate data before sending to downstream services:

```

Field: email

Rule: Must match email regex pattern

Invalid Example: "not-an-email"

Action: Log error, set field to null

Field: price

Rule: Must be numeric and > 0

Invalid Example: "free"

Action: Flag for review, use default value

Field: age

Rule: Must be between 18 and 120

Invalid Example: 250

Action: Reject record, log error

```

Accessing Data Prep in a Datastream

Method 1: Via Adobe Experience Platform Service

When configuring AEP in a datastream:

1. Navigate to Data Collection → Datastreams

2. Select your datastream

3. Click Edit on the Adobe Experience Platform service

4. Scroll down to Data Prep for Data Collection

5. Click Configure

6. Click Add Mapping to create field mappings

Method 2: Via Data Prep UI

1. Navigate to Data Collection → Data Prep

2. Select Mapping Sets

3. Click Create New Mapping Set

4. Configure field mappings

5. Reference the mapping set in your datastream

Data Prep in Datastream Example

Scenario: Map E-Commerce Events to Analytics

Requirement:

  • Web SDK sends XDM commerce events
  • Need to map to Adobe Analytics eVars and events for real-time reporting

Data Prep Configuration:

```

Mapping 1:

Source XDM Field: commerce.productViews.value

Target Field: c.a.x.event1

Transform: None (direct pass-through)

Purpose: Track product views as Analytics event

Mapping 2:

Source XDM Field: productListItems[0].SKU

Target Field: c.a.x.eVar10

Transform: Uppercase

Purpose: Track product SKU in uppercase format

Mapping 3:

Source XDM Field: productListItems[0].priceTotal

Target Field: c.a.x.event23

Transform: Round to nearest integer

Purpose: Track product price as numeric event

Mapping 4:

Source XDM Field: web.webPageDetails.name

Target Field: c.a.x.pageName

Transform: None (direct pass-through)

Purpose: Track page name for site analysis

Mapping 5:

Source XDM Field: commerce.order.priceTotal

Target Field: c.a.x.currencyCode

Transform: Extract currency code from commerce.order object

Purpose: Track currency for international reporting

```

Data Prep vs. Web SDK Configuration

| Aspect | Data Prep (Datastream) | Web SDK Config |

|---|---|---|

| Location | Server-side (Edge Network) | Client-side (Browser) |

| Purpose | Transform & map data | Initialize SDK settings |

| Who manages | AEP Administrator | Developer |

| Examples | Field mapping, calculations | edgeDomain, orgId |

| Changes require | No website code update | Code deployment |

| Real-time updates | ✅ Yes (immediate) | ❌ No (requires deploy) |

| Visibility | All traffic sees transforms | Only affects new sessions |

Common Data Prep Use Cases in Datastreams

Use Case 1: Analytics Field Mapping

Problem: Web SDK sends XDM format, but Analytics needs its own field structure

Solution:

```

XDM: commerce.productViews.value = 1

Data Prep Maps To: c.a.x.event1 = 1

Result: Analytics receives event1 populated correctly

```

Use Case 2: Multi-Currency Support

Problem: Different countries use different currency codes

Solution:

```

Calculated Field: normalized_currency

IF geolocation.country == "US"

THEN USD

ELSE IF geolocation.country == "FR"

THEN EUR

ELSE IF geolocation.country == "JP"

THEN JPY

```

Use Case 3: PII Hashing (Compliance)

Problem: Need to hash email addresses for privacy compliance

Solution:

```

Source Field: personalEmail.address

Transform Function: SHA256(LOWERCASE(email))

Result: jane.smith@email.com →

8d969eef6ecad3c29a3a873fba5d2b1d

Target Field: hashed_email

```

Use Case 4: Default Values

Problem: Some fields are optional and may be missing

Solution:

```

Source Field: device.model

Default Value: "Unknown"

Result: If device.model is null, "Unknown" is used instead

```
