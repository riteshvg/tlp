What is Auto-Target in Adobe Target and how does it use machine learning?

Auto-Target in Adobe Target

What Is Auto-Target?

[!UICONTROL Auto-Target] is an advanced Adobe Target activity type that uses machine learning to automatically select and serve the most tailored experience to each visitor. Rather than requiring you to manually allocate traffic to winning variations, Auto-Target learns from visitor behavior in real-time and personalizes content dynamically.

Auto-Target serves the most tailored experience to each visitor based on:

  • The individual visitor's customer profile
  • The behavior of previous visitors with similar profiles

How Machine Learning Powers Auto-Target

Auto-Target uses advanced machine learning algorithms that:

1. Learn from visitor behavior — The system analyzes how different visitor segments respond to your marketer-defined experiences over time.

2. Build predictive models — Machine learning creates models that predict which experience will perform best for visitors with specific attributes and behavioral patterns.

3. Match visitors to experiences — When a new visitor arrives, their profile is compared against the learned patterns, and the system predicts which of your defined experiences will drive the highest conversion probability for them.

4. Continuously optimize — As more visitors interact with the activity, the machine learning models refine and improve their predictions, becoming increasingly accurate over time.

5. Account for similarity — The algorithm recognizes visitors with similar profiles to past high-performers and allocates them to winning experiences accordingly.

Key Capabilities

  • Personalization at scale — Handles multiple high-performing, marketer-defined experiences simultaneously without requiring manual traffic allocation.
  • Real-time adaptation — Continuously learns and adjusts which experiences to serve based on ongoing visitor interactions.
  • Profile-based targeting — Uses visitor attributes (demographics, behavior, past interactions) to predict the best experience match.
  • Exploration vs. exploitation balance — The system maintains some traffic allocation to all experiences to continue discovering performance patterns while primarily directing traffic to winners.

Real-World Impact

A major clothing retailer used an Auto-Target activity with ten product category-based experiences (plus a randomized control) to deliver personalized content. Key results:

  • Average lift: 29.09% across targeted experiences
  • Revenue impact: $1,700,000+ in lift generated in just 10 days
  • Personalization rate: 90% after models matured

Requirements & Availability

[!UICONTROL Auto-Target] is available only with [!DNL Target Premium] — it is not available in Target Standard without a Premium license.

[!UICONTROL Analytics for Target] (A4T) support — Auto-Target activities are fully compatible with A4T, allowing you to use Adobe Analytics metrics as your optimization goal. See A4T support for Auto-Allocate and Auto-Target activities for detailed guidance.

Auto-Target vs. Auto-Allocate

While related, these are distinct features:

  • [!UICONTROL Auto-Target] — Machine learning selects the best experience for each individual visitor based on their unique profile and behavior patterns.
  • [!UICONTROL Auto-Allocate] — Automatically shifts traffic to higher-performing experiences overall, but all visitors see the same winning experience; no individual-level personalization.

Getting Started

To create and configure an Auto-Target activity, you define:

1. Multiple high-performing experiences you want to test

2. The optimization goal (conversion metric)

3. Traffic allocation (the system handles the rest through machine learning)

4. Audience targeting (optional, to segment your visitors)

Once launched, Auto-Target's algorithms begin learning immediately, and traffic shifts automatically toward experiences that deliver better results for different visitor segments.
