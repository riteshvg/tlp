CTAS: Create Table As Select

What It Is

A SQL command that creates a completely new dataset by executing a SELECT query and storing all results into a new table in the data lake.

Basic Syntax

```sql

CREATE TABLE [IF NOT EXISTS] {new_dataset_name} AS

SELECT

col1,

col2,

col3

FROM source_dataset

WHERE condition;

```

Key Characteristics

| Aspect | Detail |

|---|---|

| Creates | Brand new dataset |

| First Run | Populates with all query results |

| Subsequent Runs | Overwrites entire dataset with new results |

| Use Case | One-time creation or full refresh |

| Data Loss Risk | Yes — previous data is replaced |

| Idempotency | `IF NOT EXISTS` prevents errors on re-run |

CTAS: Real-World Examples

Example 1: Create Validated Audience Dataset

Scenario: You want to create a clean, validated dataset of Gold tier members for activation

```sql

CREATE TABLE IF NOT EXISTS gold_tier_validated_audience AS

SELECT

identityMap['ECID'][0].id as ecid,

identityMap['Email'][0].address as email,

person.name.firstName,

person.name.lastName,

homeAddress.city,

homeAddress.country,

loyaltyStatus,

totalPurchaseValue,

segmentMembership['ups']['gold-tier-id'].status as segment_status,

segmentMembership['ups']['gold-tier-id'].lastQualificationTime as qualified_date,

CURRENT_TIMESTAMP as validation_timestamp

FROM profile_dataset

WHERE

segmentMembership['ups']['gold-tier-id'].status IN ('realized', 'existing')

AND homeAddress.country = 'US'

AND totalPurchaseValue >= 5000

AND loyaltyStatus = 'gold';

```

Result:

  • New dataset `gold_tier_validated_audience` created
  • Contains 85,000 validated profiles
  • Ready for activation to email system
  • Contains both identities and enriched attributes

Example 2: Create RFM Score Dataset

Scenario: Calculate Recency, Frequency, Monetary (RFM) scores for customer segmentation

```sql

CREATE TABLE IF NOT EXISTS customer_rfm_scores AS

SELECT

identityMap['ECID'][0].id as ecid,

email,

-- Recency: Days since last purchase

DATEDIFF(CURRENT_DATE, MAX(DATE(purchaseTimestamp))) as recency_days,

-- Frequency: Number of purchases

COUNT(DISTINCT purchaseId) as purchase_frequency,

-- Monetary: Total spend

SUM(purchaseAmount) as total_monetary_value,

-- RFM Score (simplified: R+F+M/3)

ROUND(

(

(5 - LEAST(DATEDIFF(CURRENT_DATE, MAX(DATE(purchaseTimestamp))), 365) / 73) +

LEAST(COUNT(DISTINCT purchaseId), 5) +

LEAST(SUM(purchaseAmount) / 1000, 5)

) / 3

) as rfm_score,

CASE

WHEN ROUND(

(

(5 - LEAST(DATEDIFF(CURRENT_DATE, MAX(DATE(purchaseTimestamp))), 365) / 73) +

LEAST(COUNT(DISTINCT purchaseId), 5) +

LEAST(SUM(purchaseAmount) / 1000, 5)

) / 3

) >= 4 THEN 'High Value'

WHEN ROUND(

(

(5 - LEAST(DATEDIFF(CURRENT_DATE, MAX(DATE(purchaseTimestamp))), 365) / 73) +

LEAST(COUNT(DISTINCT purchaseId), 5) +

LEAST(SUM(purchaseAmount) / 1000, 5)

) / 3

) >= 2 THEN 'Medium Value'

ELSE 'Low Value'

END as value_segment,

CURRENT_TIMESTAMP as calculated_date

FROM purchase_events

WHERE purchaseTimestamp >= CURRENT_DATE - INTERVAL '365 days'

GROUP BY ecid, email;

```

Result:

  • New dataset with RFM calculations
  • Can be used for ML model training
  • Powers personalization strategies

Example 3: Create De-Duplicated Dataset

Scenario: Clean up duplicate records from a poorly ingested dataset

```sql

CREATE TABLE IF NOT EXISTS clean_customer_data AS

SELECT DISTINCT

email,

firstName,

lastName,

country,

createdDate,

MAX(lastUpdated) as lastUpdated

FROM raw_customer_import

WHERE

email IS NOT NULL

AND email NOT LIKE '%test%'

GROUP BY

email,

firstName,

lastName,

country,

createdDate;

```

Result:

  • Deduplicated, clean dataset
  • Single source of truth for customer records
  • Bad test records filtered out

ITAS: Insert Table As Select

What It Is

A SQL command that appends query results to an existing dataset without overwriting previous data. The table must already exist.

Basic Syntax

```sql

INSERT INTO {existing_dataset_name}

SELECT

col1,

col2,

col3

FROM source_dataset

WHERE condition;

```

Key Characteristics

| Aspect | Detail |

|---|---|

| Requires | Table must already exist |

| First Run | Appends results to existing data |

| Subsequent Runs | Appends MORE rows (cumulative) |

| Use Case | Building audit logs, incremental loads |

| Data Loss Risk | No — previous data is preserved |

| Best With | Scheduled queries |
