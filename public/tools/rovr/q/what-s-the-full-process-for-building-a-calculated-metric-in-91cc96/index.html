What's the full process for building a calculated metric in Analytics and then using it in a CJA connection?

Building Calculated Metrics in Adobe Analytics and Using Them in Customer Journey Analytics

There are two distinct paths here worth clarifying: creating calculated metrics natively in CJA and migrating existing Adobe Analytics calculated metrics into CJA. Here is the full picture:

Understanding Calculated Metrics in CJA

Calculated metrics in Customer Journey Analytics are custom metrics built from existing metrics. They offer a flexible way to build, manage, and curate custom metrics without changing your data collection implementation.

▶ Watch: Introduction to Calculated Metrics in Customer Journey Analytics

Phase 1: Plan Your Calculated Metric

Before building, especially for metrics intended to be officially approved and shared widely, carefully plan:

  • What existing metrics will be combined or transformed?
  • What formula or logic is needed?
  • Who needs access to it?
  • Should it be an approved, official metric for your organization?

Phase 2: Build the Calculated Metric

Use the Calculated Metric Builder in CJA:

1. Navigate to Components > Calculated Metrics in Customer Journey Analytics.

2. Click + Add (or the Create button) to open the Calculated Metric Builder.

3. Give the metric a Name and optional Description.

4. Drag existing metrics and/or dimensions into the Definition canvas.

5. Apply operators, functions, or filters as needed to define your formula.

6. Set the Format (number, percent, currency, etc.) and decimal places.

7. Click Save to store the metric.

> If you are migrating from Adobe Analytics, note that some calculated metric functions or segment-based logic may need to be re-created in CJA terms, since CJA uses its own component model (e.g., Filters instead of Segments).

Phase 3: Apply the Calculated Metric in a Workspace Project

Once built, calculated metrics can be used directly in Analysis Workspace:

1. Open or create a Workspace project connected to a CJA Data View.

2. In the left rail, go to Components > Metrics.

3. Locate your calculated metric (use the search bar or browse).

4. Drag and drop it onto a panel, freeform table, or visualization.

Phase 4: Manage Your Calculated Metrics

From the Calculated Metrics Manager (Components > Calculated Metrics), you can perform the following actions on one or more metrics:

| Action | Description |

|---|---|

| Tag | Organize metrics with labels for easier filtering |

| Approve / Unapprove | Mark metrics as officially endorsed for your org |

| Filter | Filter the list by tag, owner, approval status, etc. |

| Favorite / Unfavorite | Pin frequently used metrics for quick access |

| Share | Share with other CJA users or groups |

| Copy | Duplicate a metric to use as a starting point |

Migrating Adobe Analytics Calculated Metrics to CJA

If you have existing calculated metrics in Adobe Analytics that you want to re-create in CJA, the process involves manual reconstruction:

▶ Watch: Moving Calculated Metrics from Adobe Analytics to CJA

Key tips for migration:

  • Segments → Filters: Adobe Analytics segments used inside calculated metrics must be rebuilt as CJA Filters.
  • eVar/prop references → Dimensions: Replace legacy variable references with the appropriate dimension components mapped in your CJA Data View.
  • Metric names may differ: Verify that the underlying metrics referenced in your AA calculated metric exist and are named correctly in your CJA Data View.
  • Adobe Template metrics: CJA provides pre-built calculated metric templates (identifiable by the small Adobe logo). Filter for them using ![Label] Adobe Template in the Components filter to avoid re-creating common metrics unnecessarily.

Quick Reference: Full Workflow Summary

| Phase | Task |

|---|---|

| Plan | Define logic, identify source metrics, determine sharing/approval needs |

| Build | Use the Calculated Metric Builder in CJA Components |

| Apply | Drag into Workspace projects tied to a Data View |

| Manage | Tag, approve, share, and copy via the Calculated Metrics Manager |
