What is datastream overrride?

Datastream Overrides

Datastream overrides allow you to define additional configurations for your datastreams that get passed to the Edge Network via the Web SDK. This feature helps you conditionally trigger different datastream behaviors without creating a new datastream or modifying your existing settings.

Key Differences from Dynamic Datastream Configurations

It's important to note that datastream overrides and dynamic datastream configurations are mutually exclusive — you must choose one or the other:

  • If you enable both, configuration overrides take precedence and the system ignores the dynamic datastream configuration rules.

What You Can Override

Datastream configuration overrides let you modify the following datastream configurations:

  • Experience Platform event datasets
  • Adobe Target property tokens
  • Audience Manager ID sync containers
  • Adobe Analytics report suites

How to Configure Datastream Overrides

Datastream overrides require a two-step process:

Step 1: Define overrides in the Datastreams UI

First, you must define your datastream configuration override when configuring a datastream in the Datastreams UI.

Step 2: Configure in the Web SDK tag extension

1. Log in to CX Enterprise using your Adobe ID credentials

2. Navigate to Data Collection > Tags

3. Select your desired tag property

4. Navigate to Extensions, then select Configure on the Adobe Experience Platform Web SDK card

5. Scroll down to the Datastream configuration overrides section

Environment-Specific Configuration

Datastream overrides must be configured on a per-environment basis. The development, staging, and production environments all have separate overrides. You can copy the override settings between any desired environments.

Common Use Cases

  • Multi-country operations: Dynamically switch data flow to country-specific datastreams based on user location
  • Multiple business units: Route data to different Experience Platform sandboxes for each business unit
  • Conditional routing: Send data to different datastreams based on user actions (like order placement or profile updates)
