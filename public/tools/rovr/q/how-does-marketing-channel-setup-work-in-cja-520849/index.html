How does marketing channel setup work in CJA?

Marketing Channel Setup in Customer Journey Analytics

Overview

Unlike traditional Adobe Analytics, CJA has no built-in marketing channel groupings. You must define and configure marketing channels explicitly using one of the available approaches. Channels are configured at the Data View level, not at the data collection level, which gives you powerful retroactive flexibility.

Two Primary Setup Approaches

1. Derived Fields (Recommended for Web SDK Implementations)

Derived fields are the recommended approach for setting up marketing channels in CJA, especially when using the Experience Platform Web SDK.

Step-by-Step Setup

Prerequisites: You must have an existing Data View configured in CJA.

1. In Customer Journey Analytics, select the Data View where you want to add marketing channels

2. In the data view, select the [!UICONTROL Components] tab

3. Select [!UICONTROL Create derived field] in the left rail

4. In the [!UICONTROL Create derived field] dialog box, select [!UICONTROL Function templates] from the drop-down menu

5. Drag the [!UICONTROL Marketing channels] template onto the blank canvas

6. Customize the logic for each marketing channel to match the logic you use in your Adobe Analytics environment

7. Modify output channel names or add logic for additional channels specific to your organization

8. In the right column, specify a name and description for the marketing channel

9. Select [!UICONTROL Save]

Key Benefits of Derived Fields
  • Non-destructive — original data is never modified
  • Retroactive — changes apply to historical data automatically
  • Flexible — customize logic to match your exact business needs
  • Report-time processing — channels are evaluated at query time, not collection time

2. Analytics Source Connector (Migration Path)

If you are ingesting data via the Analytics source connector, marketing channel data flows into CJA through that connector, carrying over rules from your traditional Adobe Analytics implementation.

However, be aware of these important limitations:

  • "Is First Page of Visit" rules are not supported — any rules using this criteria will be ignored
  • Override Last-Touch Channel setting is ignored — this can cause channels like "Direct" or "Internal" to attribute toward metrics in undesired ways
  • Some processing rules may behave differently due to the report-time processing nature of CJA
Recommended Fixes for Analytics Source Connector

To address the Override Last-Touch Channel issue, Adobe recommends one of these approaches:

1. Remove channels where Override Last-Touch Channel is unchecked in the Marketing Channel Manager

2. Delete the 'Direct' marketing channel in the Marketing Channel Manager, then rely on CJA's 'No value' dimension item, which can be renamed to 'Direct' or excluded entirely in the data view configuration

3. Create a marketing channel classification, classifying each value to itself except for channels you want to exclude, then use this classification dimension when creating a data view instead of `channel.typeAtSource`

Available Marketing Channel Templates in CJA

Once configured, CJA provides the following built-in templates for marketing channel analysis:

| Template | Purpose |

| --- | --- |

| Channel Overview Report | Shows how visitors arrive on your site using custom attribution |

| First Touch Channel | Views the first marketing channel a visitor matches during their engagement period (30 days default) |

| First Touch Channel Detail | Provides detail about the first marketing channel match (e.g., which search engine or keyword) |

| Last Touch Channel | Views the most recent channel a visitor matched before converting |

Key Dimension Mappings

When setting up marketing channels, the following dimensions are commonly used:

  • Marketing Channel → Define as a derived field using the Marketing Channels function template
  • Referring Domain and Referrer Type → Use for source/medium analysis
  • Tracking Code → Use for campaign values when UTM parameters are mapped to XDM fields
  • UTM Parameters → Must be manually mapped to XDM fields during implementation before they appear as dimensions

Important Behavioral Differences from Adobe Analytics

| Behavior | Adobe Analytics | Customer Journey Analytics |

| --- | --- | --- |

| Processing time | At data collection | At report time |

| Channel modification | Requires reprocessing | Retroactive, no reprocessing needed |

| "Is First Page of Visit" rule | Supported | Not supported |

| Built-in channel groupings | Yes | No — must define via derived fields |

| Override Last-Touch Channel | Honored | Ignored |

| Session determination | At collection time | At query time |

Best Practices Summary

1. Use derived fields for all new Web SDK implementations

2. Re-evaluate "Is First Page of Visit" rules before migrating from Adobe Analytics

3. Address Override Last-Touch Channel issues when using the Analytics source connector

4. Leverage report-time processing to test and refine channel definitions without data loss

5. Follow Adobe's updated Marketing Channel implementation best practices to maximize advanced attribution capabilities in CJA
