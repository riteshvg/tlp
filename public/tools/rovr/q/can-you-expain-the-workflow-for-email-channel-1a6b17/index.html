STEP 1: Subdomain Delegation

What is it?

Before sending any emails, you must delegate a subdomain to Adobe so that AJO can manage sending infrastructure, tracking URLs, and mirror page hosting on your behalf.

Why is it needed?

  • ✅ Establishes sender reputation with email providers (Gmail, Outlook, Yahoo)
  • ✅ Enables DMARC, DKIM, SPF email authentication records
  • ✅ Powers tracking URLs for click and open tracking
  • ✅ Hosts mirror page URLs for email preview
  • ✅ Protects your main domain reputation by isolating marketing email sending

Types of Subdomain Delegation

| Type | Description | Best For |

|---|---|---|

| Full Delegation | Adobe fully manages the subdomain DNS | Recommended — simplest approach |

| CNAME Delegation | You keep DNS control, point to Adobe via CNAME | Organizations with strict DNS policies |

| Custom Delegation | Partially delegated with custom setup | Advanced use cases |

How to Delegate a Subdomain

Prerequisites:

  • System Administrator role in AJO
  • Access to your organization's DNS records

Steps:

1. In AJO left navigation, go to Administration → Channels → Subdomains

2. Click Set up subdomain

3. Choose delegation type: Full delegation (recommended)

4. Enter your subdomain name (e.g., `email.yourbrand.com`)

5. AJO generates the DNS records required

6. Copy the DNS records and add them to your DNS provider (GoDaddy, Cloudflare, Route53, etc.)

7. Click Submit and wait for DNS propagation

8. AJO verifies the subdomain automatically

9. Status changes to ✅ Success

What Gets Set Up Automatically:

```

After Subdomain Delegation:

├── SPF Record (Sender Policy Framework)

├── DKIM Record (DomainKeys Identified Mail)

├── DMARC Record (Domain-based Message Authentication)

├── Tracking Subdomain (click.email.yourbrand.com)

├── Mirror Page Subdomain (view.email.yourbrand.com)

└── Bounce Handling (bounces.email.yourbrand.com)

```

STEP 2: IP Pool Creation

What is it?

An IP Pool is a group of dedicated IP addresses that are used to send emails on behalf of your organization.

Why is it needed?

  • ✅ Reputation isolation — Separate IPs for marketing vs. transactional email
  • ✅ Volume distribution — Spread sending load across multiple IPs
  • ✅ IP warmup — Gradually increase sending volume on new IPs
  • ✅ Brand separation — Different IP pools for different brands or regions

IP Pool Strategy

```

IP Pool: Marketing Emails

├── IP: 203.0.113.10 (warm)

├── IP: 203.0.113.11 (warm)

└── IP: 203.0.113.12 (warm)

IP Pool: Transactional Emails

├── IP: 203.0.113.20 (dedicated)

└── IP: 203.0.113.21 (dedicated)

IP Pool: EMEA Region

├── IP: 203.0.113.30

└── IP: 203.0.113.31

```

How to Create an IP Pool

Steps:

1. Go to Administration → Channels → IP Pools

2. Click Create IP Pool

3. Enter a Name (e.g., "Marketing Email Pool")

4. Select IP addresses to include in the pool

5. Click Submit

IP Warmup Plan

When using new IP addresses, you must gradually increase sending volume to build sender reputation:

```

Week 1: Send to 500 most-engaged subscribers/day

Week 2: Send to 2,000 most-engaged subscribers/day

Week 3: Send to 10,000 subscribers/day

Week 4: Send to 50,000 subscribers/day

Week 5+: Full volume (millions/day)

```

Why warmup matters:

  • Cold IPs with sudden high volume = ❌ Spam filters triggered
  • Gradually warmed IPs = ✅ Trusted by email providers

STEP 3: Channel Configuration (Surface Setup)

What is it?

A Channel Configuration (previously called a "channel surface" or "message preset") is a pre-configured set of technical parameters that defines HOW emails are sent — the sending infrastructure, authentication, tracking, and compliance settings.

Think of it as the "profile" for sending emails that gets reused across multiple journeys and campaigns.

How to Create an Email Channel Configuration

Prerequisites:

  • Subdomain delegated ✅
  • IP Pool created ✅
  • Manage Messages Presets permission ✅

Steps:

1. Go to Administration → Channels → Channel Configurations

2. Click Create channel configuration

3. Enter a Name (e.g., "Marketing Email - US")

4. Select Email as the channel type

5. Configure the following technical parameters:

Email Configuration Parameters

Sender Information

```

From Name: "Luma Brand" (shown in inbox)

From Email: marketing@email.yourbrand.com

Reply-to Email: support@yourbrand.com

Error Email: bounces@email.yourbrand.com

```

Subdomain & IP Pool Assignment

```

Subdomain: email.yourbrand.com ← select delegated subdomain

IP Pool: Marketing Email Pool ← select IP pool

```

Email Type

```

Email Type:

├── Marketing → Subject to frequency capping and unsubscribe

└── Transactional → Bypasses frequency caps (for order confirmations, etc.)

```

Unsubscribe Management

```

Unsubscribe Options:

├── One-click unsubscribe (List-Unsubscribe header) → ✅ Enable (recommended)

├── Unsubscribe link in footer → ✅ Required for CAN-SPAM/GDPR

└── Opt-out landing page URL → Configure custom unsubscribe page

```

Retry Settings

```

Retry Period: 3.5 days (default)

Retry Frequency: Every hour

Max Retries: 84 attempts

```

Why retry matters:

  • Email delivery sometimes fails temporarily (inbox full, server down)
  • AJO automatically retries within the configured period
  • After max retries, email is marked as bounced
Tracking Parameters

```

URL Tracking:

├── Enable click tracking: ✅ YES

├── Enable open tracking: ✅ YES

├── Custom UTM parameters:

│ ├── utm_source = AJO

│ ├── utm_medium = email

│ └── utm_campaign = {{campaign.name}}

└── Tracking subdomain: click.email.yourbrand.com

```

BCC (Blind Carbon Copy)

```

BCC Email:

├── Enable BCC: ✅ YES (optional - for compliance/archiving)

└── BCC Address: archive@yourbrand.com

```

Seed List

```

Seed List (optional):

├── Add internal email addresses

├── These receive a copy of every send

└── Used for QA and compliance monitoring

```

6. Assign Marketing Action for data governance:

7. Click Submit

8. Configuration status moves to ✅ Active

STEP 4: Create Journey or Campaign

Once the channel configuration is active, you create the delivery vehicle — either a Journey or a Campaign.

Option A: Journey (Event-Triggered Email)

Use when: Email should be sent in response to a customer event

```

Example: Cart Abandonment Journey

Event: User adds items to cart but doesn't purchase

↓

Wait: 1 hour

↓

Condition: Cart still abandoned?

├── YES → Send "Cart Abandonment Email"

│ ↓

│ Wait 24 hours

│ ↓

│ Purchased?

│ ├── YES → Exit Journey (success)

│ └── NO → Send "Final Reminder Email"

│

└── NO → Exit Journey (purchased)

```

How to create a Journey:

1. Go to Journey Management → Journeys

2. Click Create Journey

3. Name the journey (e.g., "Cart Abandonment - Email")

4. Set journey properties:

  • Timeout: Maximum journey duration
  • Re-entry: Allow profiles to re-enter? YES/NO
  • Re-entry wait period: If yes, how long to wait

5. Add Event trigger (e.g., `cartAbandoned`)

6. Drag and drop Email action onto canvas

7. Select the Channel Configuration created in Step 3

8. Create or select email content

Option B: Campaign (Scheduled Email)

Use when: Email should be sent to an audience at a scheduled time

```

Example: Monthly Newsletter Campaign

Define Audience: "All Newsletter Subscribers"

↓

Create Email Content: Monthly newsletter template

↓

Schedule: First Monday of every month, 9:00 AM local time

↓

Execute: AJO sends to all qualifying profiles at scheduled time

↓

Measure: Opens, clicks, conversions

```

How to create a Campaign:

1. Go to Journey Management → Campaigns

2. Click Create Campaign

3. Select campaign type:

  • Scheduled — for newsletter/promotional
  • API-Triggered — for transactional messages

4. Select Email as the action

5. Select the Channel Configuration from Step 3

6. Define Audience (AEP segment or inline audience)

7. Set Schedule (date, time, timezone, recurrence)

8. Create email content

STEP 6: Personalization & Dynamic Content

Expression Editor

AJO uses the Expression Editor for personalization using profile attributes, event data, and decision management offers.

Personalization Syntax

```

Hello {{profile.person.name.firstName}}!

Your order {{event.commerceOrder.orderNumber}} has been confirmed.

You have {{profile.loyaltyPoints}} points in your account.

```

Dynamic Content Blocks

Create content variations based on conditions:

```

Dynamic Content Block: Hero Banner

Condition 1: IF profile.loyaltyTier = "Gold"

→ Show: Gold Member Banner Image

→ Text: "Exclusive Gold Member Offer: 30% Off"

Condition 2: IF profile.loyaltyTier = "Silver"

→ Show: Silver Member Banner Image

→ Text: "Silver Member Offer: 20% Off"

Default: (No tier)

→ Show: Generic Banner Image

→ Text: "Special Offer: 10% Off"

```

Offer Integration (Decision Management)

```

Email Template includes Decision Activity

↓

At send time, Decision Engine evaluates:

├── Profile eligibility for each offer

├── AI ranking score for eligible offers

└── Frequency cap status for each offer

↓

Best offer is dynamically inserted into email

↓

Each recipient sees their personalized offer

```
