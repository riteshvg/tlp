How do I use Adobe Experience Platform audiences in Adobe Target for personalisation?

Using Adobe Experience Platform Audiences in Adobe Target for Personalisation

This integration allows you to share AEP audiences and profile attributes with Adobe Target, enabling powerful edge-based personalisation experiences.

Prerequisites

Before you begin, ensure you have:

  • Access to Adobe Experience Platform with the ability to create and activate audiences
  • An Adobe Target instance connected as a destination in AEP
  • A configured Datastream with Adobe Target enabled on the Edge Network
  • A sandbox environment in AEP

High-Level Workflow

The end-to-end process follows these stages:

1. Build your audience in AEP

2. Activate the audience to Adobe Target as a destination

3. Select profile attributes to share alongside the audience

4. Use the audience and attributes inside a Target activity for personalisation

Step-by-Step Guide

Step 1 – Build Your Audience in AEP

1. Navigate to Audiences in the AEP left rail.

2. Click Create Audience and define your segment rules using profile, behavioural, or event-based criteria.

3. Save and publish the audience.

Step 2 – Activate the Audience to Adobe Target

1. Go to Destinations in AEP.

2. Find and select your Adobe Target destination.

3. Click Activate Audiences.

4. Select the audience(s) you want to send to Target.

5. During activation, select any profile attributes you want to make available for personalisation — for example, `person.name.firstName`.

6. Complete the activation flow.

Step 3 – Create a Target Activity Using the AEP Audience

1. Log in to Adobe Target.

2. Create a new activity — for example, an Experience Targeting (XT) activity using the Form-Based Experience Composer.

3. Name your activity clearly (e.g., `RTCDP - XT (Form)`).

4. Select the AEP-activated audience as the target audience for the experience.

Step 4 – Insert AEP Profile Attribute Tokens for Personalisation

Once inside the activity editor, you can use shared profile attributes as personalisation tokens:

1. Locate the content area you want to personalise (e.g., an `h1` heading).

2. In the code, position your cursor where you want to insert the dynamic value:

```

document.querySelector("h1").innerHTML = "Hi there ";

```

3. Add the profile attribute token by selecting:

  • Source: Adobe Experience Platform
  • Sandbox: your sandbox name
  • Attribute: `person.name.firstName`

4. Click Add. The code will update to include the token:

```

document.querySelector("h1").innerHTML = "Hi there ${aep.person.name.firstName}";

```

5. Click Next to proceed through the activity setup.

Step 5 – Set Goals and Activate the Activity

1. On the Goals & Settings page, go to Goal Metrics.

2. Set the Primary Goal — for example, Engagement → Time On Site.

3. Click Save & Close.

4. On the Activity Overview page, click the Inactive dropdown and select Activate.

5. You will receive a visual confirmation that the activity is now live.

Key Points to Remember

  • Profile attributes must be explicitly selected during the audience activation step in AEP — only then are they available as tokens in Target.
  • The integration works via the Edge Network, meaning personalisation decisions happen at the edge for low latency.
  • AEP audiences appear in Target automatically once activation is complete — no manual import is needed.
  • This approach does not rely on third-party cookies, making it resilient to browser privacy changes.

Once live, your Target activity will deliver personalised experiences to visitors who match your AEP audience, with content dynamically populated using real-time profile attributes from AEP.
