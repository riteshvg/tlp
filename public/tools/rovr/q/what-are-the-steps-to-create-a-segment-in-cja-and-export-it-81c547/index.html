Creating a Segment in CJA and Exporting It to Adobe Target

Based on the available documentation, here is what I can directly confirm about segments in Customer Journey Analytics (CJA).

Creating a Segment in CJA

Option 1: Quick Segment (Simple Criteria)

Quick segments allow you to create segments directly within a CJA project without opening the full Segment Builder.

Prerequisites:

  • Access to a Customer Journey Analytics project
  • Appropriate component permissions

Steps:

1. Open your Customer Journey Analytics project in the UI.

2. Click the + Add button in the segment drop zone at the top of your project, or right-click a dimension/metric value directly in a visualization.

3. Define your segment criteria using the simplified interface — you can add up to 3 rules using simple include/exclude conditions.

4. Click Apply to apply the segment immediately to your project and see results in real time.

▶ Watch: Create a Quick Segment

> Note: "Filters" have been renamed "Segments" in the Customer Journey Analytics interface.

Option 2: Full Segment Builder (Complex Criteria)

For more advanced segmentation logic, use the full Segment Builder:

1. Navigate to Components > Segments in CJA.

2. Click + Create to open the Segment Builder.

3. Define your segment logic using dimensions, metrics, and events across multiple containers (Event, Session, Person).

4. Click Save to save the segment for reuse across projects.

Exporting CJA Audiences to Adobe Target

Exporting CJA-built segments directly to Adobe Target involves a workflow through Adobe Experience Platform (AEP). Here is the general approach based on available documentation:

Step 1: Identify Your Audiences in AEP

Before ingesting or exporting, identify the audiences you want to work with:

1. In the Experience Platform UI, select Customer > Audiences.

2. Select Browse and search for the audiences you want to use.

3. Note the Audience ID for each audience for later use.

Step 2: Create a Profile-Enabled Dataset

You need a dataset based on the XDM Individual Profile schema to hold exported audience members.

Use the Catalog Service API to create the dataset (this cannot be done via the UI for this schema type):

```shell

curl -X POST \

'https://platform.adobe.io/data/foundation/catalog/dataSets?requestDataSource=true' \

-H 'Authorization: Bearer {ACCESS_TOKEN}' \

-H 'Content-Type: application/json' \

-H 'x-api-key: {API_KEY}' \

-H 'x-gw-ims-org-id: {ORG_ID}' \

-H 'x-sandbox-name: {SANDBOX_NAME}' \

-d '{

"name": "Segment Export Job Dataset for CJA",

"schemaRef": {

"id": "_xdm.context.profile__union",

"contentType": "application/vnd.adobe.xed+json;version=1"

},

"fileDescription": {

"persistet": true,

"containerFormat": "parquet",

"format": "parquet"

}

}'

```

The response will return a `DATASET_ID` you will need in the next step.

Step 3: Export the Audience Using the Segmentation API

Use the `/export/jobs` endpoint of the AEP Segmentation Service API to persist audience members to the dataset created above:

  • Export jobs are asynchronous processes
  • You can programmatically retrieve, create, and cancel export jobs via the API

> Prerequisite: Review the Segmentation API getting started guide for required headers and authentication details before making API calls.

Step 4: Connect the Dataset to CJA

Once the audience data is exported to the dataset:

1. In CJA, navigate to Connections.

2. Add the newly created dataset to your connection.

Step 5: Activate the Audience to Adobe Target

To deliver the audience to Adobe Target:

1. In AEP, navigate to Destinations.

2. Select the Adobe Target destination (available natively — no Audience Manager license required).

3. Activate your exported audience to the Target destination.

4. In Adobe Target, the audience will be available for use in A/B tests, Experience Targeting (XT), Automated Personalization, and other activity types.

Key Considerations

| Topic | Detail |

|---|---|

| Adobe Target destination | No Audience Manager license required |

| Export jobs | Asynchronous — allow time for completion before activating |

| Schema requirement | Must use `_xdm.context.profile__union` for profile-enabled datasets |

| Quick vs. Full segments | Quick segments are project-scoped; Full Segment Builder segments are reusable across CJA |

If you need more detail on a specific step — such as the Target destination configuration or the Segmentation API authentication — feel free to ask!
