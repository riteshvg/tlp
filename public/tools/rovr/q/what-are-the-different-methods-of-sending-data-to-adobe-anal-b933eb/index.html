What are the different methods of sending data to Adobe Analytics?

Methods of Sending Data to Adobe Analytics

Adobe Analytics provides several methods to send data from your website or application to Adobe's servers. Here are the main approaches:

Client-Side Implementation Methods

1. AppMeasurement

A JavaScript file hosted on your site and referenced on each page. Data is sent directly to Adobe Analytics. This is a traditional implementation method.

2. Adobe Experience Platform Web SDK

A JavaScript file hosted on your site and referenced on each page. Data is sent to the Adobe Experience Platform Edge Network, which can then forward data to Adobe Analytics along with other Adobe solutions.

3. Tags in Adobe Experience Platform Data Collection

A JavaScript file referenced on each page containing rules created within the Data Collection UI. This offers:

  • An Analytics extension as an easier way to implement AppMeasurement
  • A Web SDK extension as an easier way to implement the Web SDK

4. APIs

Programmatic methods to send data to Adobe:

  • AppMeasurement APIs:
  • Data Insertion API
  • Bulk Data Insertion API
  • Edge Network API:
  • Data Collection API

Data Flow

Before data reaches Adobe Analytics processing servers, it is typically compiled client-side using one of these methods. If you send data to the Edge Network, you can configure it to forward data to Adobe Analytics (as well as many other Adobe CX Enterprise solutions).

Key Consideration

If you're implementing Web SDK and plan to use Customer Journey Analytics in the future, Adobe recommends a clean Web SDK implementation using your own schema, rather than migrating from AppMeasurement.
