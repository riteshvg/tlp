The SMS/MMS channel in AJO follows a structured workflow similar to email but with SMS-specific configuration steps — particularly around third-party SMS provider integration since AJO does not have its own SMS sending infrastructure (unlike email).

STEP 1: Choose & Set Up SMS Provider

Why a Third-Party Provider is Needed

Unlike email (where AJO manages its own sending infrastructure), SMS messages are delivered through third-party telecom providers. AJO integrates with these providers via API.

```

AJO (Orchestration)

↓

SMS Provider API (Delivery Infrastructure)

↓

Telecom Networks (AT&T, Verizon, T-Mobile, etc.)

↓

Customer's Mobile Device (SMS received)

```

Supported SMS Providers

AJO natively supports four SMS providers:

| Provider | SMS | MMS | RCS | Best For |

|---|---|---|---|---|

| Sinch | ✅ | ✅ | ✅ | Global reach, RCS support |

| Twilio | ✅ | ✅ | ❌ | Developer-friendly, US focus |

| Infobip | ✅ | ✅ | ✅ | Enterprise, global coverage |

| Custom Provider | ✅ | ✅ | ✅ | Any provider via webhook |

Provider Setup Requirements

Before configuring in AJO, you must set up your account with the chosen provider:

For Sinch:

```

Required from Sinch Account:

├── Account SID (Service ID)

├── Auth Token

├── Sender Phone Number or Short Code

├── Service Plan ID (for messaging)

└── API Key (for RCS if needed)

```

For Twilio:

```

Required from Twilio Account:

├── Account SID

├── Auth Token

├── Twilio Phone Number (long code or short code)

├── Messaging Service SID (optional but recommended)

└── API Credentials

```

For Infobip:

```

Required from Infobip Account:

├── Base URL (your Infobip API endpoint)

├── API Key

├── Sender Number or Sender Name

└── Application & Entity IDs (for regulatory compliance)

```

For Custom Provider:

```

Required from Custom Provider:

├── API Endpoint URL

├── Authentication Method (API key, OAuth, etc.)

├── Inbound Message Webhook URL format

├── Outbound Message API format

└── Delivery Receipt format

```

Provider Account Prerequisites

```

Before Setting Up Your Provider Account:

├── Purchase SMS credits or plan ✅

├── Get a sending phone number or short code ✅

├── Verify your business identity ✅

├── Register your brand (for US 10DLC requirements) ✅

├── Register campaigns (for US compliance) ✅

└── Obtain API credentials ✅

```

US SMS Regulatory Requirements (10DLC)

> ⚠️ Important for US SMS Sending:

```

10DLC (10-Digit Long Code) Requirements:

├── BRAND REGISTRATION

│ ├── Register your company with The Campaign Registry (TCR)

│ ├── Provide: Company name, EIN, address, website

│ └── Status: Verified brand needed before sending

│

├── CAMPAIGN REGISTRATION

│ ├── Register each SMS use case (marketing, OTP, alerts, etc.)

│ ├── Provide message samples

│ └── Status: Campaign must be approved

│

└── NUMBER REGISTRATION

├── Associate phone numbers with registered campaign

└── Numbers must be verified before sending

```

Failure to register for 10DLC results in:

  • ❌ Messages filtered/blocked by carriers
  • ❌ Extremely poor deliverability
  • ❌ Potential account suspension

STEP 2: Configure API Credentials in AJO

Once you have your provider account set up, you connect it to AJO via API credentials.

How to Configure API Credentials

Prerequisites:

  • System Administrator role in AJO ✅
  • API credentials from your SMS provider ✅

Steps:

1. In AJO left navigation, go to Administration → Channels → SMS Settings → API Credentials

2. Click Create new API credentials

3. Select your SMS Provider:

```

Provider Options:

├── Sinch

├── Twilio

├── Infobip

└── Custom

```

Configuration for Sinch

```

Sinch API Credentials Setup:

Field: Name → "Sinch Production"

Field: Service SID → [Your Sinch Service ID]

Field: API Key ID → [Your Sinch API Key]

Field: API Key Secret → [Your Sinch API Secret]

Field: Inbound SMS Number → +1XXXXXXXXXX

Field: Inbound MMS Number → +1XXXXXXXXXX (if different)

```

Sinch-Specific Settings:

```

Additional Sinch Options:

├── Enable Fuzzy Opt-out → ✅ YES (recommended)

│ └── Detects near-matches of STOP keywords

│ (e.g., "CANCIL" is treated as "CANCEL")

│

└── Inbound Keyword Responses:

├── STOP → "You have been unsubscribed. Reply START to re-subscribe."

├── START → "You have been re-subscribed to our messages."

└── HELP → "For assistance call 1-800-XXX-XXXX or reply STOP to opt-out."

```

Configuration for Twilio

```

Twilio API Credentials Setup:

Field: Name → "Twilio Production"

Field: Account SID → [Your Twilio Account SID]

Field: Auth Token → [Your Twilio Auth Token]

Field: Phone Number → +1XXXXXXXXXX (Twilio number)

```

Twilio-Specific Settings:

```

Additional Twilio Options:

├── Messaging Service SID → [Optional but recommended]

│ └── Groups multiple numbers for better deliverability

│

├── Enable Fuzzy Opt-out → ✅ YES

│

└── Inbound Keyword Configuration:

├── Opt-out keywords: STOP, QUIT, CANCEL, END, UNSUBSCRIBE

├── Opt-in keywords: START, YES, UNSTOP

└── Help keywords: HELP, INFO

```

Configuration for Infobip

```

Infobip API Credentials Setup:

Field: Name → "Infobip Production"

Field: Base URL → https://XXXXX.api.infobip.com

Field: API Key → [Your Infobip API Key]

Field: Sender Number → +1XXXXXXXXXX

Field: Application ID → [Your App ID]

Field: Entity ID → [Your Entity ID]

```

Configuration for Custom Provider

```

Custom Provider API Credentials Setup:

Field: Name → "Custom SMS Provider"

Field: Outbound Endpoint → https://api.yourprovider.com/send

Field: Authentication Type:

├── API Key (Header-based)

├── Bearer Token

└── Basic Auth (username/password)

Field: Inbound Webhook URL → AJO generates this automatically

```

4. After filling in all fields, click Submit

5. AJO validates the credentials by making a test API call

6. Status shows ✅ Active when credentials are verified

STEP 3: Create Webhook (For Inbound Messages)

What is a Webhook?

A webhook is an endpoint URL that your SMS provider calls when:

  • A customer replies to an SMS (inbound message)
  • A delivery status update is received
  • An opt-out/opt-in keyword is received (STOP, START, HELP)

```

Customer replies "STOP"

↓

SMS Provider receives the reply

↓

Provider calls AJO Webhook URL

↓

AJO processes the inbound message

↓

AJO updates profile consent/preference

↓

AJO sends configured reply (if any)

```

How to Create a Webhook in AJO

1. In AJO, go to Administration → Channels → SMS Settings → Webhooks

2. Click Create Webhook

3. Fill in webhook details:

```

Webhook Configuration:

Name: "Sinch Inbound Webhook"

SMS Vendor: Sinch (or your provider)

Execution Encoding: UTF-8

Inbound Keywords & Responses:

├── STOP → Auto-reply: "You have been unsubscribed."

├── START → Auto-reply: "Welcome back! You are now subscribed."

└── HELP → Auto-reply: "Text STOP to unsubscribe."

```

4. AJO generates a Webhook URL:

```

Generated Webhook URL:

https://platform.adobe.io/ajo/webhooks/sms/sinch/XXXXXXXX

```

5. Copy this Webhook URL

6. Go to your SMS provider dashboard and add this URL as the callback/webhook endpoint

7. Back in AJO, click Save

Keyword Management

AJO automatically handles standard opt-out keywords:

```

Standard Opt-out Keywords (automatically handled):

├── STOP

├── STOPALL

├── UNSUBSCRIBE

├── CANCEL

├── END

└── QUIT

Standard Opt-in Keywords:

├── START

├── YES

└── UNSTOP

Standard Help Keywords:

├── HELP

└── INFO

```

Fuzzy Opt-out

Fuzzy Opt-out is a feature that detects misspelled opt-out keywords:

```

Example Fuzzy Opt-out Matching:

├── "CANCIL" → Detected as CANCEL ✅

├── "STOPP" → Detected as STOP ✅

├── "UNSIBSCRIBE" → Detected as UNSUBSCRIBE ✅

└── "QUTI" → Detected as QUIT ✅

How it Works:

1. Customer replies with misspelled keyword

2. AJO detects similarity to opt-out keyword

3. AJO sends confirmation message:

"Did you mean to unsubscribe? Reply YES to confirm."

4. Customer replies YES

5. AJO unsubscribes the customer

```

STEP 4: Create Channel Configuration (Surface)

What is a Channel Configuration?

A Channel Configuration for SMS defines the technical parameters for how SMS messages are sent — which provider, which phone number, which message type, and which compliance rules apply.

How to Create an SMS Channel Configuration

Prerequisites:

  • API Credentials configured ✅
  • Webhook created ✅
  • Manage Messages Presets permission ✅

Steps:

1. Go to Administration → Channels → Channel Configurations

2. Click Create channel configuration

3. Enter a Name (e.g., "Marketing SMS - US")

4. Select SMS as the channel type

5. Configure SMS-specific parameters:

```

SMS Channel Configuration Settings:

├── BASIC SETTINGS

│ ├── Name: "Marketing SMS - US"

│ ├── Description: "SMS for US marketing campaigns"

│ └── Channel: SMS ✅

│

├── SMS PROVIDER SETTINGS

│ ├── SMS Vendor Configuration: "Sinch Production" ← select API credentials

│ ├── Sender Number: +1XXXXXXXXXX

│ └── Sender Name (Alphanumeric): "LUMA" (if supported by region)

│

├── MESSAGE TYPE

│ ├── Marketing → Subject to opt-out management and frequency caps

│ └── Transactional → Bypasses marketing opt-out (order alerts, OTPs)

│

├── MESSAGE ENCODING

│ ├── GSM-7 (Standard) → 160 characters per SMS

│ └── Unicode (Extended) → 70 characters per SMS (for emojis, special chars)

│

├── OPT-OUT MANAGEMENT

│ ├── Enable opt-out: ✅ YES

│ ├── Opt-out keywords: STOP, CANCEL, QUIT, etc.

│ └── Opt-out reply: "You have been unsubscribed."

│

└── MARKETING ACTION (for data governance)

└── Select: "SMS Marketing"

```

6. Click Submit

7. Status moves to ✅ Active

STEP 5: Create Journey or Campaign

Option A: Journey (Event-Triggered SMS)

Use case: SMS sent in response to customer event

```

Example: Order Confirmation Journey

Event: Customer places an order

↓

Action: Send Order Confirmation SMS

"Hi {{firstName}}, your order #{{orderNumber}}

has been confirmed! Track: {{trackingLink}}"

↓

Wait: 2 days

↓

Action: Send Shipping Update SMS

"Your order is on the way!

Expected delivery: {{deliveryDate}}"

↓

Wait: Until delivery event received

↓

Action: Send Delivery Confirmation SMS

"Your order has been delivered!

How did we do? {{feedbackLink}}"

```

How to Create SMS Journey:

1. Go to Journey Management → Journeys

2. Click Create Journey

3. Add Event Trigger (e.g., `orderPlaced`)

4. Drag SMS action onto canvas

5. Select Channel Configuration created in Step 4

6. Click Edit Content to write SMS message

7. Add personalization tokens

8. Add subsequent journey steps as needed

Option B: Campaign (Scheduled SMS)

Use case: SMS sent to audience on a schedule

```

Example: Flash Sale Alert Campaign

Audience: "All SMS Subscribers - US"

↓

SMS Content:

"LUMA: Flash sale STARTS NOW!

25% off sitewide - TODAY ONLY.

Shop: {{saleLink}}

Reply STOP to opt-out"

↓

Schedule: Saturday 9:00 AM EST

↓

Execute: Send to all qualifying profiles

```

How to Create SMS Campaign:

1. Go to Journey Management → Campaigns

2. Click Create Campaign

3. Select Scheduled or API-Triggered

4. Select SMS as the action

5. Select Channel Configuration from Step 4

6. Define Audience

7. Set Schedule

8. Create SMS content
