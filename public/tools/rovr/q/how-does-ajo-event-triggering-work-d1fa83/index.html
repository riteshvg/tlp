How does AJO event triggering work?

What the documentation confirms

The CONTEXT shows that AJO supports three types of events that trigger journeys:

1. Unitary events — linked to a specific person's behavior (such as a purchase or loyalty milestone)

2. Business events — triggered by a global occurrence (such as a flight cancellation or stock update)

3. Audience qualification events — triggered when a profile enters or exits an audience

Events are described as "real-time triggers that launch your journeys" and require mandatory configuration by a Data engineer.

What it does NOT cover

The provided documentation does not explain the technical mechanics of how event triggering actually works — such as how events are detected, validated, matched to journey entry conditions, or processed through the system pipeline.

For additional details

Consult the official Journey Optimizer documentation on event configuration and journey guardrails, which should contain the technical triggering mechanisms and event processing details not covered here.
