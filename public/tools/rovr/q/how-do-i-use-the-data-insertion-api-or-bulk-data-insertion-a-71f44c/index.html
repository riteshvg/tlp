How do I use the Data Insertion API or Bulk Data Insertion API to send data to Adobe Analytics?

Using Data Insertion API and Bulk Data Insertion API for Adobe Analytics

Overview

Both APIs are methods to submit server-side collection data to Adobe Analytics. Here's how they differ and when to use each:

| Feature | Data Insertion API | Bulk Data Insertion API (BDIA) |

|---|---|---|

| Data format | One event at a time | CSV files, one event per row |

| Best for | High-frequency, real-time events | Larger batches sent less frequently |

| Throughput | No upper bound on throughput rate | Optimized for larger, less frequent files |

| Recommendation | Use if sending more than 1 file per second | Recommended for new server-side implementations |

When to Use Each API

Use Bulk Data Insertion API (BDIA) when:

  • Ingesting historical data from a previous analytics system
  • You have an internal analytics collection system that makes it unfeasible to use AppMeasurement — you can use Extract-Transform-Load (ETL) processes to put data into batch files and upload them via BDIA
  • Collecting data from devices with intermittent internet connectivity — devices store up interactions and upload all at once when connected
  • You are working on any new implementation of server-side collection (Adobe strongly recommends BDIA for new implementations)

Use Data Insertion API (Legacy) when:

  • You need to bring data into Adobe Analytics and cannot use AppMeasurement, Web SDK, or the Bulk Data Insertion API
  • Your use case requires sending more than 1 file per second

Data Latency

Once data is submitted via either API:

  • Data with timestamps less than 24 hours old typically appears within 20–50 minutes (some data like page views can appear within minutes)
  • Data with timestamps older than 24 hours can take 2 hours or longer depending on the age of the data

Supported Variables in BDIA

BDIA supports a wide range of variables, including:

  • `aamlh` — Adobe Audience Manager location hint
  • `contextData.key` — Context data variables
  • `customerID` — Visitor ID Service variables (`id`, `authState`, `isMCSeed`)
  • `hints` — Client hint variables (`bitness`, `brands`, `mobile`, `model`, `platform`, `platformversion`, `wow64`)
  • `ipaddress` — IP address dimension
  • `language` — Language dimension
  • `list1` – `list3` — List variables
  • `marketingCloudVisitorID` — Experience Cloud ID
  • `tnta` — Target data payload for Analytics for Target (A4T) integrations
  • `trackingServer` — Tracking server variable
  • `transactionID` — Transaction ID variable
  • `userAgent` — Device user agent string

Variables NOT Supported in BDIA:

  • `charSet` — BDIA only supports UTF-8
  • `timezone` — Visitor's time zone offset from GMT
  • `clickAction`, `clickActionType`, `clickContext`, `clickContextType`, `clickSourceID`, `clickTag` — Activity Map data collection variables

Summary Recommendation

For any new server-side implementation, Adobe strongly recommends using the Bulk Data Insertion API over the legacy Data Insertion API, as it is a more scalable successor designed for modern server-side collection needs.
