54. How do Data Governance and Consent Policies prevent activation to destinations?

How Data Governance and Consent Policies Prevent Activation to Destinations

The Two-Layer Enforcement System

AEP has two independent policy types that can block activation to destinations. Both must pass for activation to succeed.

```

Activation to Destination Initiated

↓

╔═══════════════════════════════════════╗

║ Layer 1: DATA GOVERNANCE POLICIES ║

║ (Labels + Marketing Actions) ║

╚═══════════════════════════════════════╝

↓

✅ Pass or ❌ BLOCKED?

↓

╔═══════════════════════════════════════╗

║ Layer 2: CONSENT POLICIES ║

║ (Customer Consent Preferences) ║

╚═══════════════════════════════════════╝

↓

✅ Pass or ❌ BLOCKED?

↓

Both Layers Pass? → ✅ ACTIVATION PROCEEDS

Either Layer Fails? → ❌ ACTIVATION BLOCKED

```

Layer 1: Data Governance Policy Enforcement

How Data Governance Works

Data governance policies prevent activation based on what the data is (labels) and how you want to use it (marketing action).

The Logic:

```

Does the audience or mapped field have a DATA USAGE LABEL?

↓

YES → Check if there's a POLICY restricting that label + marketing action

↓

Policy Found? → Is this marketing action allowed for this label?

↓

❌ NOT ALLOWED → BLOCK ACTIVATION (Policy Violation)

✅ ALLOWED → PROCEED to Consent Policy check

```

Example 1: PII Restrictions

Scenario: Marketing team wants to activate an audience to a behavioral advertising platform (Facebook).

The Data:

  • Audience contains customer email addresses
  • Email field has I1 label ("Directly Identifiable")

The Policy:

```

Policy Name: "No PII for Behavioral Advertising"

IF: Data has label I1 (Directly Identifiable)

AND Marketing action = Behavioral Advertising

THEN: BLOCK activation

```

What Happens:

1. Marketer clicks "Activate to Facebook"

2. System checks: "Does this audience/field have I1 label?" → ✅ YES

3. System checks: "Is there a policy blocking I1 + Behavioral Advertising?" → ✅ YES

4. Result: ❌ ACTIVATION BLOCKED

Error Message:

```

⚠️ Policy Violation: Data Policy Violation

The following policies are violated:

  • "No PII for Behavioral Advertising"

Field 'personalEmail.address' contains label 'I1'

and cannot be used for the marketing action: 'Behavioral Advertising'

Resolution: Remove the email field from activation or change destination type.

```

Example 2: Contractual Export Restrictions

Scenario: Data analyst wants to export customer data to a data warehouse (Snowflake).

The Data:

  • Dataset has C2 label ("Data cannot be exported outside of AEP")

The Policy:

```

Policy Name: "No Data Export Outside AEP"

IF: Data has label C2

AND Marketing action = Data Export

THEN: BLOCK activation

```

What Happens:

1. Analyst clicks "Activate to Snowflake"

2. System checks: "Does this data have C2 label?" → ✅ YES

3. System checks: "Is there a policy blocking C2 + Data Export?" → ✅ YES

4. Result: ❌ ACTIVATION BLOCKED

Error Message:

```

⚠️ Policy Violation: Data cannot be exported

Dataset 'Customer_Master' has contractual restrictions (C2 label)

and cannot be exported to external destinations.

Resolution: Use only internal AEP segments or contact Legal.

```

Example 3: Combined Label Restrictions

Scenario: Marketer wants to use customer data for on-site personalization (Adobe Target).

The Data:

  • Customer name: C1 label ("Cannot combine with directly identifiable data")
  • Customer email: I1 label ("Directly Identifiable")

The Policy:

```

Policy Name: "C1 + I1 Cannot Be Combined for Personalization"

IF: Data has label C1

AND Data has label I1

AND Marketing action = On-Site Personalization

THEN: BLOCK activation

```

What Happens:

1. Marketer clicks "Activate to Target"

2. System checks: "Does audience have both C1 and I1 labels?" → ✅ YES

3. System checks: "Is there a policy blocking this combination?" → ✅ YES

4. Result: ❌ ACTIVATION BLOCKED

Resolution Options:

  • ✅ Remove the email field (I1) from the activation
  • ✅ Remove the name field (C1) from the activation
  • ✅ Update the data governance policy (if legal allows)

Layer 2: Consent Policy Enforcement

How Consent Policies Work

Consent policies prevent activation of profiles that haven't given consent for the intended use.

The Logic:

```

Does an active Consent Policy apply to this destination's marketing action?

↓

YES → Check if each profile has the required consent attributes

↓

Profile Missing Required Consent? → EXCLUDE profile from activation

↓

Final activated audience = [Original] - [Non-consenting profiles]

```

Example 1: Email Marketing Consent

Scenario: Marketer wants to activate an audience to email marketing platform (Adobe Campaign).

The Policy:

```

Policy Name: "Email Marketing - Consent Required"

IF: Marketing action = Email Marketing

THEN: Profile must have consents.marketing.email.val = true

```

The Audience: 10,000 profiles

  • 7,500 with email consent = true
  • 2,000 with email consent = false
  • 500 with email consent = unknown/missing

What Happens:

1. Marketer clicks "Activate to Campaign"

2. System checks: "Is there a consent policy for Email Marketing?" → ✅ YES

3. System evaluates each profile:

  • Profile A: email consent = true → ✅ INCLUDE
  • Profile B: email consent = false → ❌ EXCLUDE
  • Profile C: email consent = unknown → ❌ EXCLUDE (conservative)

4. Result: ✅ ACTIVATION PROCEEDS (but only 7,500 profiles sent)

Summary:

```

Original Audience: 10,000 profiles

After Consent Filter: 7,500 profiles

Profiles Excluded: 2,500 (no consent)

```

Example 2: SMS Marketing with Consent

Scenario: Team wants to activate an audience for SMS campaigns.

The Policy:

```

Policy Name: "SMS Marketing - Consent Required"

IF: Marketing action = SMS Marketing

THEN: Profile must have consents.marketing.sms.val = true

```

The Audience: 50,000 profiles breakdown:

  • 35,000 with SMS consent = true
  • 10,000 with SMS consent = false
  • 5,000 with SMS consent = unknown

Enforcement:

```

SMS Audience Activation

├── Original: 50,000 profiles

├── Consent Check Applied

├── Consents Given (true): 35,000 → ✅ INCLUDE

├── Consents Denied (false): 10,000 → ❌ EXCLUDE

├── Consents Unknown: 5,000 → ❌ EXCLUDE

└── Final Activation: 35,000 profiles

```

Business Impact:

  • 🟢 Only consenting customers receive SMS
  • 🟢 No violations, no blocks
  • 🟢 Compliant with consent regulations
