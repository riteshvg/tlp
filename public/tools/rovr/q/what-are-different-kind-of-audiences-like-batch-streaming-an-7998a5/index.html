Audience Evaluation Methods in Adobe Experience Platform

Adobe Experience Platform supports three distinct audience evaluation methods, each designed for different use cases and real-time requirements:

1. Batch Segmentation

What it is:

Batch segmentation processes profile data all at once to create corresponding audiences. It's the traditional, scheduled approach to audience creation.

Key characteristics:

  • Runs on a daily schedule (can be toggled off if needed)
  • Evaluates audiences periodically rather than in real-time
  • Best for audiences where immediate evaluation isn't critical
  • Can handle complex logic and large-scale data processing

Use cases:

  • General audience segmentation
  • Historical analysis and reporting
  • Audiences with complex criteria that don't require real-time updates

Important note:

If an audience definition combines both batch and streaming data sources, it will automatically be evaluated using batch segmentation. If you need streaming or edge evaluation with mixed data sources, you must explicitly create separate batch and streaming rulesets and combine them using segment of segments (inSegment).

2. Streaming Segmentation

What it is:

Streaming segmentation evaluates audiences in near real-time to segment streaming data as soon as it arrives in Experience Platform. This allows profiles to qualify for audiences immediately when they meet the criteria.

Key characteristics:

  • Near real-time evaluation (up to 350 milliseconds processing time after an event enters the Edge Network)
  • Customers qualify as soon as they meet criteria
  • Supports continuous, event-based evaluation
  • Maximum 500 active streaming audiences per sandbox

Throughput:

  • Combined peak throughput of 1500 events per second across production and development sandboxes
  • Additional streaming segmentation support up to 13,500 records per second can be purchased

Limitations:

  • Platform defaults to streaming segmentation whenever possible, but automatically converts to batch if criteria are too complex
  • Look-back windows greater than 7 days may cause automatic conversion to batch segmentation
  • Cannot directly combine batch and streaming data sources in a single audience definition

Use cases:

  • Real-time personalization
  • Immediate customer responses to behavioral triggers
  • Time-sensitive marketing campaigns

3. Edge Segmentation

What it is:

Edge segmentation evaluates audiences in near real-time at the Adobe Experience Platform Edge Network for same-page personalization and real-time use cases.

Key characteristics:

  • Processes events directly on the edge network with minimal latency
  • Ideal for same-page and cross-site personalization
  • Up to 350 milliseconds to process an inbound event after it enters the Edge Network
  • Maximum 150 active edge audiences per sandbox
  • Can have more than 150 edge audiences across the organization, as long as each sandbox has less than 150

Throughput:

  • Combined peak throughput of 1500 requests per second (RPS) across all sandboxes for edge segmentation

Use cases:

  • Real-time website personalization
  • Same-page targeting
  • Next-best-action recommendations
  • Dynamic content delivery

How Platform Chooses Evaluation Method

Platform defaults to streaming segmentation whenever possible. Here's the decision flow:

1. ✅ Simple criteria → Streaming segmentation

2. ⚠️ Complex criteria or look-back window > 7 days → Automatically converts to batch

3. 🚀 Same-page/cross-site personalization use cases → Edge segmentation

Important Considerations for Mixed Data Sources

When combining batch and streaming data sources, follow this pattern:

Example scenario:

  • Audience A (Batch): California residents using profile schema
  • Audience B (Streaming): Recent checkouts using experience event schema

To combine them for streaming evaluation:

```

inSegment("e3be6d7f-1727-401f-a41e-c296b45f607a") and

CHAIN(xEvent, timestamp, [C0: WHAT(eventType.equals("commerce.checkouts", false))

WHEN(<= 24 hours before now)])

```

Key rule: Keep batch data only within batch audiences and reference it in streaming audiences using `inSegment`. This ensures streaming segmentation accuracy.

Summary

Choose your audience evaluation method based on:

  • Batch: When daily updates are acceptable and logic is complex
  • Streaming: When near real-time qualification is needed and rules are simpler
  • Edge: When same-page personalization and minimal latency are critical

All three methods work together within Adobe Experience Platform to support diverse marketing and personalization needs.
