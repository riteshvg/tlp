How does Adobe Target integrate with Adobe Analytics and Experience Platform?

Adobe Target Integration with Adobe Analytics and Adobe Experience Platform

Integration with Adobe Analytics (A4T)

Analytics for Target (A4T) is a cross-solution integration that allows you to use Adobe Analytics as the reporting source for Adobe Target activities.

What A4T Provides

  • Use Analytics metrics and segments as optimization goals in Target activities
  • Full Analytics reporting capabilities applied to Target activity data
  • A dedicated Analytics for Target (A4T) panel is available directly in Analysis Workspace
  • Eliminates data discrepancies by using a single reporting source

Supported Activity Types with A4T

  • A/B Testing
  • Auto-Allocate
  • Auto-Target
  • Experience Targeting (XT)
  • Multivariate Testing (MVT)
  • Automated Personalization has limited A4T support

How to Access A4T Reporting in Analysis Workspace

The Analytics for Target panel in Analysis Workspace lets you analyze your Adobe Target activities and experiences directly:

1. Open Analysis Workspace

2. Create or open a project

3. Add the Analytics for Target (A4T) panel

4. Select your Target activity and experience breakdown

5. Choose your success metrics and analysis dimensions

Prerequisites for A4T

  • Active Adobe Analytics and Adobe Target contracts
  • Implemented Experience Cloud ID Service
  • Properly configured report suite
  • Matching visitor identification between both solutions

Integration with Adobe Experience Platform (AEP)

Web SDK Integration

The recommended modern implementation approach uses the Adobe Experience Platform Web SDK, which enables Target to work seamlessly within the broader AEP ecosystem.

Key benefits of Web SDK integration:

  • Single tag deployment across multiple Adobe solutions
  • Data sent through the Edge Network to multiple products simultaneously
  • Reduced page load impact compared to legacy implementations
  • Future-proof architecture aligned with AEP

Implementation Overview via Edge Network

The Edge Network acts as the central hub, routing data to the appropriate Adobe solutions:

High-level setup steps:

1. Create a Datastream in Adobe Experience Platform Data Collection

  • Navigate to Data Collection in the Adobe Experience Cloud
  • Select Datastreams from the left navigation
  • Click New Datastream and configure it
  • Add Adobe Target as a service to the datastream

2. Install the Web SDK extension in your Tag property

  • Navigate to Tags in Adobe Experience Platform
  • Select your tag property
  • Go to Extensions > Catalog
  • Search for and install the Adobe Experience Platform Web SDK extension

3. Configure the Web SDK extension

  • Point the extension to your configured datastream
  • Set your environment-specific datastream IDs

4. Implement decisioning rules

  • Configure Target activities to use on-device or edge decisioning as appropriate

Real-Time Customer Profile Integration

When using AEP, Target can leverage:

  • Real-Time Customer Profile data for richer audience targeting
  • Audience segments built in AEP and shared to Target
  • Identity resolution across devices and channels for consistent personalization

AEP Schemas and XDM

  • Target data can be mapped to Experience Data Model (XDM) schemas
  • Standardized data collection enables consistent data across the entire Adobe Experience Cloud
  • XDM data collected via Web SDK flows into AEP and can inform Target personalization

How the Three Solutions Work Together

When all three are integrated, you get a powerful ecosystem:

| Layer | Solution | Role |

|-------|----------|------|

| Data Collection | AEP Web SDK + Edge Network | Centralized data collection and routing |

| Personalization | Adobe Target | Activity delivery and optimization |

| Reporting & Analysis | Adobe Analytics (A4T) | Deep reporting and insights |

| Audience Management | AEP Real-Time CDP | Unified profiles and audience sharing |

Typical Combined Workflow

1. AEP collects and unifies customer data into Real-Time Customer Profiles

2. Target uses those profiles to deliver personalized experiences

3. Analytics captures the results via A4T for deep reporting and insight

4. Insights feed back into AEP and Target to continuously improve personalization

Key Considerations

  • Using the Web SDK is the recommended and most future-proof approach for integrating all three solutions
  • A4T requires that both Analytics and Target share the same visitor identification via the Experience Cloud ID Service
  • Datastream configuration is the central control point for routing data between solutions
  • Legacy implementations using at.js and AppMeasurement can still leverage A4T but lack the unified data collection benefits of Web SDK
