How does AJO event triggering work?

How AJO Event Triggering Works

In Adobe Journey Optimizer, events are the real-time triggers that launch journeys. When an event is received, AJO delivers messages individually to each user as they enter the journey.

The Three Types of Events

1. Unitary Events

  • Linked to a specific person's behavior — such as a purchase, a loyalty milestone, or a product view
  • Triggers a journey for a single profile at a time
  • Best for real-time, individual-level responses

2. Business Events

  • Triggered by a global occurrence — such as a flight cancellation or a stock update
  • Not tied to a single person's behavior
  • Can trigger journeys for many profiles simultaneously based on the business occurrence

3. Audience Qualification Events

  • Triggered when a profile enters or exits an audience
  • Allows journeys to react to audience membership changes in real time

How Events Are Configured

  • Event configuration is mandatory and must be performed by a Data Engineer
  • Events are configured in the event configuration section of AJO before they can be used in journeys
  • Incoming event data is normalized following Adobe Experience Data Model (XDM) standards
  • Events come from Streaming Ingestion APIs for both authenticated and unauthenticated events (such as Adobe Mobile SDK events)

Flexibility of Event Usage

  • You can use multiple events in different steps of a single journey
  • Several journeys can use the same event — meaning one event can trigger multiple journeys simultaneously

Important Guardrails

There are requirements and limitations around events related to:

  • Streaming ingestion
  • Query Service
  • Batch ingestion

These guardrails define the boundaries within which event triggering operates reliably in AJO.

Key Takeaway

The event triggering model in AJO is designed around real-time, profile-level orchestration. Each event type serves a distinct use case — unitary for individual behaviors, business for global occurrences, and audience qualification for segment-based triggers — giving you flexible control over when and how journeys are launched.
