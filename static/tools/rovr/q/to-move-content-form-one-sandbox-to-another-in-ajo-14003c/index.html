Copying Objects Between Sandboxes in Adobe Journey Optimizer

Adobe Journey Optimizer uses AEP Sandbox Tooling (package export and import) to move objects such as journeys, campaigns, custom actions, content templates, and fragments between sandboxes.

Prerequisites

Before you begin, ensure you have the following Sandbox administration permissions:

  • Manage sandboxes (or View sandboxes)
  • Manage packages

Overview of the Process

The copy process follows three main stages:

1. Add the object to a package in the source sandbox

2. Publish the package

3. Import the package in the target sandbox

Step-by-Step Process

Step 1 — Add the Object to a Package (Source Sandbox)

1. Navigate to the object you want to copy (e.g., a journey, campaign, content template, or fragment) in your source sandbox.

2. Use the Add to Package option on the object.

3. Either create a new package or add it to an existing package.

4. You can include multiple objects in a single package, but all objects must come from the same sandbox.

Step 2 — Publish the Package

1. Once all desired objects are added, publish the package from the source sandbox.

2. Publishing makes the package available for import into a target sandbox.

Step 3 — Import the Package (Target Sandbox)

1. Switch to your target sandbox.

2. Locate the published package and initiate the import.

3. The system will automatically detect and reuse any existing matching objects (e.g., channel configurations) in the target sandbox.

4. If no matching channel configuration is found, it will be skipped — you will need to manually update channel settings in the target sandbox after import.

Key Behaviors & Guardrails

| Behavior | Detail |

|---|---|

| Channel configurations | Automatically reused if an identical one exists in the target sandbox; otherwise skipped and must be manually configured |

| Dependent objects | Most dependencies are resolved automatically during import |

| Data usage labels & language settings | Not copied — must be recreated manually |

| Existing experiments & audiences | Can be reused in the target sandbox as dependent objects |

| Decision items (offer items) | Not included automatically — must be added manually using Add to Package |

| AI model ranking formulas | Copying is not currently supported |

| Decision items copy order | Decision items must be copied before any other related Decisioning objects |

Special Case: Decisioning Objects

For migrating Decision Management objects to Decisioning, use the dedicated Decisioning Migration API instead of sandbox tooling. It provides:

  • Automated dependency resolution
  • Rollback capabilities
  • Purpose-built for decisioning entity migration
