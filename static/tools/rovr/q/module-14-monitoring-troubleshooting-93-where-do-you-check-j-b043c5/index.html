Where Do You Check Journey Execution Logs in AJO?

Overview

Adobe Journey Optimizer provides multiple locations and methods to access journey execution logs, helping you monitor performance, diagnose issues, and optimize customer experiences.

1. Journey Live Reports

Live Reports provide real-time monitoring of journey execution data for the last 24 hours.

How to Access:

1. Navigate to Journeys in the left menu

2. Open your journey

3. Click the Reports tab at the top

4. Select Last 24 hrs to view live execution data

What You Can See:

  • Profiles entered – Number of profiles that entered the journey
  • Profiles exited – Number of profiles that completed or exited
  • Errors – Actions or steps that failed
  • Messages sent – Delivery counts per channel
  • Real-time interactions – Active customer interactions

2. Journey Global Reports

Global Reports provide comprehensive journey performance metrics across all time or a selected date range.

How to Access:

1. Navigate to Journeys in the left menu

2. Open your journey

3. Click the Reports tab

4. Select All time or a custom date range

What You Can See:

  • Journey performance metrics – Entry, exit, and completion rates
  • Channel performance – Email, SMS, push delivery metrics
  • Step-by-step analysis – How profiles move through journey steps
  • Error rates – Failed actions and error details

3. Journey Step Events

Journey Step Events provide the most detailed, granular view of journey execution — tracking every step a profile takes through a journey.

How to Access:

1. Navigate to Adobe Experience Platform

2. Go to Datasets

3. Look for the AJO Journey Step Events dataset

4. Query the dataset using Query Service

What You Can See:

  • Step-by-step execution details – Every node a profile passes through
  • Timestamps – Exact timing of each step
  • Error messages – Detailed error information per step
  • Profile identifiers – Which profiles triggered which steps
  • Action results – Success or failure of each action

Sample Query:

```sql

SELECT

journeyName,

profileId,

stepName,

stepStatus,

errorMessage,

timestamp

FROM

ajo_journey_step_events

WHERE

journeyName = 'Your Journey Name'

AND timestamp > NOW() - INTERVAL '7 days'

ORDER BY

timestamp DESC

```

4. Audit Logs

Audit Logs track all actions performed by users in the system, providing visibility into configuration changes and administrative activities.

How to Access:

1. Navigate to Adobe Experience Platform

2. Go to Governance → Audit Logs

3. Filter by resource type (Journey, Message, etc.)

4. Filter by date range and user

What You Can See:

  • User actions – Who made what changes and when
  • Configuration changes – Journey edits, publishes, and deletions
  • System events – Administrative actions across AJO services

> Important: To view and export audit logs, you must have the View User Activity Log permission granted.

5. AJO System Datasets in AEP

Journey Optimizer automatically generates system datasets in Adobe Experience Platform that contain detailed execution logs:

| Dataset | Contents |

|---|---|

| AJO Journey Step Events | Step-by-step journey execution details |

| AJO Message Feedback Event Dataset | Email/SMS delivery logs and ISP feedback |

| AJO Email Tracking Experience Event Dataset | Email open, click, and tracking events |

| AJO Push Tracking Experience Event Dataset | Push notification interaction events |

| AJO BCC Feedback Event Dataset | BCC email tracking records |

How to Access:

1. Navigate to Adobe Experience Platform

2. Go to Data Management → Datasets

3. Search for AJO to filter Journey Optimizer datasets

4. Click on a dataset to explore its data

5. Use Query Service for detailed analysis

6. Deliverability Monitoring

For email-specific delivery issues, use the Deliverability Monitoring tools:

How to Access:

1. Navigate to Administration in the left menu

2. Click on Channels → Email Settings

3. Select Deliverability to view delivery metrics

What You Can See:

  • Bounce rates – Hard and soft bounce tracking
  • Spam complaints – ISP feedback on spam reports
  • Delivery rates – Success rates across ISPs
  • IP warm-up status – IP reputation monitoring

7. Inbound Action Logs

For inbound actions (in-app, web personalization), access specific troubleshooting tools:

1. Navigate to Journeys

2. Open your journey with inbound actions

3. Click More actions → View inbound action logs

4. Review execution details for inbound interactions

Quick Reference: Which Log to Use?

| Scenario | Best Log Source |

|---|---|

| Real-time journey monitoring | Journey Live Reports |

| Historical performance analysis | Journey Global Reports |

| Detailed step-by-step troubleshooting | Journey Step Events Dataset |

| Email delivery failures | Message Feedback Event Dataset |

| User configuration changes | Audit Logs |

| Email tracking (opens/clicks) | Email Tracking Dataset |

| Push notification tracking | Push Tracking Dataset |

| Inbound action issues | Inbound Action Logs |

Best Practices

✓ Start with Live Reports – Quick overview of real-time issues

✓ Use Journey Step Events – For deep-dive troubleshooting of specific profiles

✓ Query AEP Datasets – For custom analysis and reporting

✓ Monitor Audit Logs – Track configuration changes that may impact journeys

✓ Set up Alerts – Configure Adobe Experience Platform alerts for journey failures

✓ Cross-reference datasets – Combine step events with message feedback for complete picture
