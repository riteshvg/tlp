Overview

The `configure` and `sendEvent` commands are the two most fundamental commands in the Adobe Experience Platform Web SDK. They serve completely different purposes and are used at different points in the page lifecycle.

> Simple Analogy:

> - `configure` = Setting up a phone (one-time setup, hardware settings, SIM card, network)

> - `sendEvent` = Making a phone call (repeated action, each call carries different information)

Side-by-Side Comparison

| Aspect | `configure` | `sendEvent` |

|---|---|---|

| Purpose | Initialize and configure the SDK | Send data/events to Edge Network |

| When called | Once per page load (first) | Multiple times per page/interaction |

| Required | ✅ Always required first | ✅ Required to collect data |

| Returns data | ❌ No response data | ✅ Returns personalization response |

| Network request | ❌ No network request | ✅ Makes HTTPS POST to Edge Network |

| Parameters | SDK settings (orgId, datastreamId) | XDM data, identity, options |

| Timing | Before any other SDK command | After configure, on events/interactions |

| Frequency | Once per page | Unlimited (as needed) |

| Modifiable | ❌ Cannot be changed after init | ✅ Different data per call |

The `configure` Command

What It Does

The `configure` command initializes the Web SDK with all required settings. It must be called before any other Web SDK command and only once per page load.

Think of `configure` as the foundation — without it, no other Web SDK command can work.

Syntax

```javascript

alloy("configure", {

// Required settings

datastreamId: "YOUR_DATASTREAM_ID",

orgId: "YOUR_ORG_ID@AdobeOrg",

// Optional settings

edgeDomain: "data.example.com",

edgeBasePath: "ee",

defaultConsent: "in",

idMigrationEnabled: true,

thirdPartyCookiesEnabled: false,

prehidingStyle: ".hero-banner { display: none; }",

targetMigrationEnabled: false,

debugEnabled: false,

onBeforeEventSend: function(options) { return options; },

onBeforeLinkClickSend: function(options) { return options; }

});

```

Required Parameters

| Parameter | Type | Description | Example |

|---|---|---|---|

| datastreamId | String | ID of your Edge Network datastream | `"abc123def456"` |

| orgId | String | Your Adobe IMS Organization ID | `"ORGID@AdobeOrg"` |

Optional Parameters

Identity & Privacy

```javascript

alloy("configure", {

datastreamId: "abc123def456",

orgId: "ORGID@AdobeOrg",

// Identity Migration

idMigrationEnabled: true,

// Migrates ECID from legacy Visitor API (VisitorAPI.js)

// Set to TRUE when migrating from AppMeasurement + VisitorAPI

// Set to FALSE for new implementations

// Third-Party Cookies

thirdPartyCookiesEnabled: false,

// Controls whether third-party cookies are used

// Set to FALSE (recommended for privacy compliance)

// Third-party cookies being deprecated in major browsers

// Target Migration

targetMigrationEnabled: false,

// Migrates from legacy at.js to Web SDK

// Set to TRUE when migrating from at.js

// Maintains Target mbox compatibility during migration

});

```

Consent

```javascript

alloy("configure", {

datastreamId: "abc123def456",

orgId: "ORGID@AdobeOrg",

// Default Consent

defaultConsent: "in",

// "in" = Collect data by default (opt-in markets)

// "out" = Do NOT collect data by default (opt-out markets)

// "pending" = Wait for explicit consent before collecting

// Object = { general: "in" } for granular consent

});

```

Network Configuration

```javascript

alloy("configure", {

datastreamId: "abc123def456",

orgId: "ORGID@AdobeOrg",

// Custom Edge Domain (First-Party Data Collection)

edgeDomain: "data.retailstore.com",

// Use your own subdomain instead of adobedc.net

// Improves data collection in browser tracking prevention scenarios

// Requires CNAME setup with Adobe

// Edge Base Path

edgeBasePath: "ee",

// Default is "ee" - rarely needs to change

// Only modify if explicitly instructed by Adobe

});

```

Personalization

```javascript

alloy("configure", {

datastreamId: "abc123def456",

orgId: "ORGID@AdobeOrg",

// Prehiding Style (Flicker Prevention)

prehidingStyle: `

.hero-banner,

.product-recommendations,

#promo-banner {

display: none !important;

}

`,

// Hides elements before Target/AJO personalization loads

// Prevents "flicker" where default content shows before personalized content

// Only hide elements that will be personalized

});

```

Callbacks

```javascript

alloy("configure", {

datastreamId: "abc123def456",

orgId: "ORGID@AdobeOrg",

// On Before Event Send Callback

onBeforeEventSend: function(options) {

// Modify XDM data before every sendEvent call

// Runs before EVERY event is sent to Edge Network

// Return false to cancel the event send

// Example: Add global context to all events

options.xdm._retail = {

environment: "production",

deploymentVersion: "2.1.0"

};

// Example: Remove PII before sending

if (options.xdm.personalEmail) {

delete options.xdm.personalEmail.address;

}

// Example: Cancel event based on condition

if (options.xdm.eventType === "internal.test") {

return false; // Cancel this event

}

return options; // Return modified options

},

// On Before Link Click Send Callback

onBeforeLinkClickSend: function(options) {

// Modify data before link click events are sent

// Similar to onBeforeEventSend but only for link clicks

// Example: Add link context

options.xdm.web.webInteraction.type = "other";

return options;

}

});

```

Debugging

```javascript

alloy("configure", {

datastreamId: "abc123def456",

orgId: "ORGID@AdobeOrg",

// Debug Mode

debugEnabled: true,

// Enables verbose console logging

// Set to FALSE in production

// Can also be enabled via chrome extension (Adobe Experience Platform Debugger)

// or via: alloy("setDebug", { enabled: true })

});

```

What configure Does NOT Do

```

configure does NOT:

❌ Send any network requests

❌ Create cookies

❌ Collect any data

❌ Contact Edge Network

❌ Set visitor identities

❌ Return any response

configure ONLY:

✅ Stores SDK configuration settings in memory

✅ Initializes internal SDK state

✅ Sets up callback handlers

✅ Prepares SDK for subsequent commands

```

Configure in Adobe Tags

When using the Web SDK Tags extension, `configure` is handled automatically:

1. Navigate to Extensions → Adobe Experience Platform Web SDK

2. Click Configure

3. Fill in the UI fields (no code required):

```

UI Field: IMS Org ID

Maps to: orgId: "ORGID@AdobeOrg"

UI Field: Datastream (Production)

Maps to: datastreamId: "prod-datastream-id"

UI Field: Edge Domain

Maps to: edgeDomain: "data.example.com"

UI Field: Default Consent

Maps to: defaultConsent: "in"

UI Field: Migrate ECID from VisitorAPI

Maps to: idMigrationEnabled: true

```

Tags automatically calls `configure` when the library loads — no manual code required.

The `sendEvent` Command

What It Does

The `sendEvent` command sends XDM-formatted event data to the Edge Network. It can be called multiple times throughout a user session — once for each interaction or data collection point.

Syntax

```javascript

alloy("sendEvent", {

// XDM Data (primary data object)

xdm: { },

// Non-XDM Data (free-form data for server-side use)

data: { },

// Event-specific options

type: "web.webpagedetails.pageViews",

renderDecisions: true,

personalization: { },

documentUnloading: false,

edgeConfigOverrides: { }

})

.then(function(result) {

// Handle Edge Network response

})

.catch(function(error) {

// Handle errors

});

```

Parameters Explained

xdm (Primary Parameter)

```javascript

alloy("sendEvent", {

xdm: {

// eventType: Describes what happened (required for most events)

eventType: "commerce.productViews",

// Web context (auto-populated if not specified)

web: {

webPageDetails: {

name: "Product Detail Page",

URL: "https://www.retailstore.com/products/shoe-001"

},

webReferrer: {

URL: "https://www.google.com"

}

},

// Commerce data

commerce: {

productViews: { value: 1 }

},

// Product data

productListItems: [{

SKU: "SHOE-RUN-001",

name: "Pro Runner X1",

priceTotal: 129.99,

quantity: 1,

category: "Running Shoes"

}],

// Identity data

identityMap: {

Email: [{

id: "customer@email.com",

authenticatedState: "authenticated",

primary: true

}]

},

// Custom fields (tenant-specific)

_retailstore: {

loyaltyTier: "Gold",

storeLocation: "San Francisco"

}

}

});

```

data (Non-XDM Data)

```javascript

alloy("sendEvent", {

xdm: {

eventType: "web.webpagedetails.pageViews"

},

// data: For non-XDM data (server-side processing only)

// Used for data that doesn't conform to XDM schema

// Accessible in event forwarding rules

data: {

__adobe: {

analytics: {

// Legacy Analytics variables (migration use case)

prop1: "custom value",

eVar5: "product category"

},

target: {

// Target-specific parameters

mbox: "homepage-hero",

profile: {

membershipStatus: "Gold"

}

}

},

// Custom free-form data

internalData: {

deploymentVersion: "2.1.0",

featureFlags: ["new-checkout", "loyalty-v2"]

}

}

});

```

renderDecisions

```javascript

alloy("sendEvent", {

xdm: {

eventType: "web.webpagedetails.pageViews"

},

// renderDecisions: Controls automatic personalization rendering

renderDecisions: true,

// true = Web SDK automatically renders Target/AJO personalization

// decisions returned by Edge Network

// false = You manually handle and render personalization decisions

// (for SPAs or custom rendering logic)

});

```

type

```javascript

alloy("sendEvent", {

// type: Sets the eventType at the top level

// Shorthand for xdm.eventType

type: "commerce.productViews",

xdm: {

// eventType will be auto-populated from type above

commerce: {

productViews: { value: 1 }

}

}

});

```

documentUnloading

```javascript

// Used for tracking navigation away from page

alloy("sendEvent", {

xdm: {

eventType: "web.webpagedetails.pageUnload"

},

// documentUnloading: true = Uses sendBeacon API instead of fetch

// Ensures data is sent even as page unloads

documentUnloading: true

});

```

edgeConfigOverrides

```javascript

alloy("sendEvent", {

xdm: {

eventType: "commerce.purchases"

},

// Override datastream settings for this specific event

edgeConfigOverrides: {

com_adobe_experience_platform: {

datasets: {

event: {

datasetId: "commerce-specific-dataset-id"

}

}

},

com_adobe_analytics: {

reportSuites: ["commerce-specific-rs"]

},

com_adobe_target: {

propertyToken: "different-property-token"

}

}

});

```

sendEvent Response Handling

Unlike `configure`, `sendEvent` returns a Promise with Edge Network response data:

```javascript

alloy("sendEvent", {

xdm: {

eventType: "web.webpagedetails.pageViews"

},

renderDecisions: false // Get decisions without auto-rendering

})

.then(function(result) {

// result.decisions: Personalization decisions from Target/AJO

var decisions = result.decisions;

decisions.forEach(function(decision) {

console.log("Decision ID:", decision.id);

console.log("Scope:", decision.scope);

console.log("Items:", decision.items);

// Manually render personalization

decision.items.forEach(function(item) {

if (item.schema === "https://ns.adobe.com/personalization/html-content-item") {

document.querySelector("#" + decision.scope).innerHTML = item.data.content;

}

});

});

// result.propositions: AJO/Target propositions

var propositions = result.propositions;

// result.inferences: AI/ML model results

var inferences = result.inferences;

})

.catch(function(error) {

console.error("sendEvent failed:", error);

});

```
