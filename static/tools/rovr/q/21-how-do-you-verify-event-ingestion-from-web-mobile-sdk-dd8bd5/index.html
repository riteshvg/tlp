21. How Do You Verify Event Ingestion from Web/Mobile SDK?

Verifying Event Ingestion from Web/Mobile SDK

There are several ways to verify that events are being properly ingested from your Web SDK or Mobile SDK implementations:

Using Adobe Experience Platform Assurance

Adobe Experience Platform Assurance (formerly Project Griffon) is the primary tool for validating event collection from your SDKs:

What Assurance Does

  • Inspects raw SDK events generated by the Adobe Experience Platform Mobile SDK or Web SDK
  • Displays all collected events in a list view, sorted by time
  • Provides detailed views for each event with comprehensive information
  • Shows SDK configuration, data elements, Shared States, and SDK extension versions
  • Helps you simulate and validate how data is collected

How to Use Assurance

1. Confirm access — Ensure your organization has access to Assurance and you're added to the appropriate profile

2. Set up your base URL — Configure the connection parameters for your app

3. Add SDK-specific code — Implement required code for iOS, Android, or Web

4. Connect to a session — Establish a connection to monitor events in real-time

Validation Process

When you Deploy and Validate your Tags/SDK configuration:

  • Create an environment where you can iterate on your development
  • Use Assurance to inspect events as they're collected
  • Verify data is being sent to your configured datastream
  • Once everything is validated, publish to your production environment

What You Can Verify

  • Raw event data structure and content
  • Whether events are properly formatted according to your schema
  • Data flow through the Edge Network
  • Datastream routing to your Adobe Experience Platform dataset
  • SDK extension configuration and versions

This validation step is critical before moving your implementation to production, ensuring that data collection works as expected across all your tracking implementations.
