Using Profile Attributes from Delivery API to Exclude Recommendations in Custom Design

You can leverage profile attributes passed via the Delivery API directly inside your Recommendations custom design template (Velocity-based) to filter out/exclude items that match those profile attribute values.

Overview of the Approach

When you make a Delivery API call, you can pass profile attributes in the request payload. Those profile attributes are then accessible inside your Velocity design template using the `$profile` object. You can use conditional logic in Velocity to skip rendering any recommended entity that matches the value(s) stored in the profile attribute.

Step 1: Pass Profile Attributes via Delivery API

In your Delivery API request payload, include the profile attributes you want to use for exclusion:

```json

{

"context": {

"channel": "web"

},

"execute": {

"mboxes": [

{

"name": "your-recs-mbox",

"index": 0,

"profileParameters": {

"excludedProductIds": "SKU-001,SKU-002,SKU-003",

"lastPurchasedId": "SKU-001"

}

}

]

}

}

```

> Note: `excludedProductIds` here is a comma-separated string of product IDs you want to exclude. You can also pass a single value like `lastPurchasedId` depending on your use case.

Step 2: Update the Custom Velocity Design Template

In your Recommendations custom design, use Velocity scripting to read the profile attribute and conditionally skip the matching entity.

Example 1: Exclude a Single Product ID (e.g., last purchased item)

```velocity

#set($excludedId = $profile.get('lastPurchasedId'))

<ul class="recommendations-list">

#foreach($entity in $entities)

#if($entity.id != $excludedId)

<li>

<a href="$entity.pageUrl">

<img src="$entity.thumbnailUrl" alt="$entity.name"/>

<span>$entity.name</span>

<span>$entity.value</span>

</a>

</li>

#end

#end

</ul>

```

Example 2: Exclude Multiple Product IDs (comma-separated string)

```velocity

Get the comma-separated excluded IDs from profile attribute

#set($excludedIdsRaw = $profile.get('excludedProductIds'))

Split the string into a list

#set($excludedList = $excludedIdsRaw.split(','))

<ul class="recommendations-list">

#foreach($entity in $entities)

Flag to track if this entity should be excluded

#set($isExcluded = false)

Loop through excluded IDs and check for a match

#foreach($exId in $excludedList)

#set($exId = $exId.trim())

#if($entity.id == $exId)

#set($isExcluded = true)

#end

#end

Only render if not excluded

#if($isExcluded == false)

<li>

<a href="$entity.pageUrl">

<img src="$entity.thumbnailUrl" alt="$entity.name"/>

<span>$entity.name</span>

<span>$$entity.value</span>

</a>

</li>

#end

#end

</ul>

```

Example 3: Exclude by a Custom Attribute (e.g., category already viewed)

```velocity

#set($excludedCategory = $profile.get('lastViewedCategory'))

<ul class="recommendations-list">

#foreach($entity in $entities)

#if($entity.categoryId != $excludedCategory)

<li>

<a href="$entity.pageUrl">

<img src="$entity.thumbnailUrl" alt="$entity.name"/>

<span>$entity.name</span>

</a>

</li>

#end

#end

</ul>

```

Key Points to Remember

| Aspect | Detail |

|---|---|

| Profile attribute access in Velocity | Use `$profile.get('attributeName')` to read profile attributes inside the design |

| Delivery API parameter key | Pass attributes under `profileParameters` in the mbox object |

| Data type | Profile attributes are strings; use `.split(',')` for multi-value exclusion lists |

| `.trim()` | Always trim whitespace when splitting comma-separated values to avoid mismatches |

| Limit | Up to 100 concurrent Target content delivery requests per user session apply (HTTP 420 if exceeded) |

| Batch mbox limit | Maximum 50 mboxes per batch mbox request |

Important Caveat

The `profileParameters` passed in a Delivery API call are session-scoped by default. If you need these exclusions to persist across sessions, you should also store them as persistent profile attributes using the `profile.` prefix in script profile rules, or ensure they are passed on every Delivery API call.

This approach gives you full runtime control inside your Velocity design to dynamically exclude any recommended items based on what was passed through the Delivery API call.
