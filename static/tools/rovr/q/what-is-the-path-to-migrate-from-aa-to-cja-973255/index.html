Migrating from Adobe Analytics to Customer Journey Analytics

There are multiple upgrade paths available, and the best choice depends on your current Adobe Analytics implementation. Here's a comprehensive breakdown:

Available Upgrade Paths

If you're using AppMeasurement or the Adobe Analytics Extension (Tags):

1. New implementation of the Experience Platform Web SDK (high effort, highest long-term viability)

2. Migrate Adobe Analytics to the Web SDK (medium effort)

3. Analytics Source Connector (lower effort, lower long-term viability)

4. ✅ (Recommended) New implementation of the Experience Platform Web SDK + Analytics Source Connector

If you're already using the Web SDK:

1. Configure your existing Web SDK implementation to send data to Platform

2. ✅ (Recommended) New implementation of the Experience Platform Web SDK + Analytics Source Connector

Step-by-Step: High-Level Migration Process

Prerequisites

  • Adobe Analytics administrator access
  • Access to Adobe Experience Platform and Customer Journey Analytics

Steps

1. Ingest your data into Adobe Experience Platform using one of these methods:

  • Analytics Source Connector:
  • Set up your report suites for ingestion into AEP and CJA
  • Ingest and use the data via the source connector
  • Web SDK:
  • Set up your report suites for ingestion into AEP and CJA
  • Ingest data via the Adobe Experience Platform Web SDK
  • Note: All schema fields must be manually mapped when using Web SDK

2. Prepare for component migration:

  • Perform an Adobe Analytics Inventory using the tool available to Analytics administrators
  • Review migration preparation guidelines before migrating any projects

3. Migrate projects and components:

  • Re-create Adobe Analytics projects in Customer Journey Analytics
  • Map dimensions and metrics from Adobe Analytics report suites to CJA data views
  • Some dimensions/metrics are automatically mapped
  • Others require manual mapping
  • Segments are migrated automatically and do not require mapping

4. Review the migration summary:

  • All migrated components are displayed in the migration summary upon completion

> 💡 Tip: Migration can be performed either through the UI or via the Adobe Analytics 2.0 APIs, depending on your preference and scale of migration.

Key Recommendation

Adobe's recommended path for most organizations is:

> New implementation of the Experience Platform Web SDK combined with the Analytics Source Connector for historical data continuity.

Contact your Adobe representative if you need more specific guidance tailored to your organization's implementation.
