what is data mirror?

Data Mirror Overview

Data Mirror is an Adobe Experience Platform capability that enables row-level change ingestion from external databases directly into the data lake using relational schemas. It synchronizes data changes from your external data warehouse without requiring upstream extract, transform, and load (ETL) processes.

Key Capabilities

Data Mirror allows you to:

  • Synchronize data changes (inserts, updates, and deletes) from external data warehouse native solutions like Snowflake, Azure Databricks, or Google BigQuery
  • Preserve database relationships and enforce data integrity during ingestion
  • Maintain your existing database model structure within the Experience Platform data lake
  • Track precise changes with upserts and deletes using change data capture

How It Works

Data Mirror operates through relational schemas, which are flexible data structures with enforced constraints. Unlike standard XDM schemas, relational schemas:

  • Do not require classes or field groups
  • Enforce primary keys and support version identifiers
  • Define schema-to-schema relationships using primary and foreign keys
  • Are optimized for change data capture ingestion workflows

Requirements

Your setup must include:

  • Primary key and version descriptor in all schemas
  • Timestamp descriptor if working with time-series data
  • Unique identifiers in source data (single field or composite primary key)
  • Version information so the system applies updates in the correct order
  • A `_change_request_type` column to identify whether records are upserts or deletes

When to Use Data Mirror

Use Data Mirror when you need to synchronize data changes from external systems while preserving database relationships and supporting advanced analytics and journey orchestration.
