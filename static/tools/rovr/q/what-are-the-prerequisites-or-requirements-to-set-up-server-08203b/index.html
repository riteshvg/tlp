What are the prerequisites or requirements to set up server-side forwarding in Adobe Analytics?

Prerequisites & Requirements for Server-Side Forwarding in Adobe Analytics

Solution Requirements

Server-side forwarding requires the following Adobe solutions:

  • Adobe Analytics (active license)
  • Adobe Audience Manager and/or Adobe Audiences (via Experience Cloud)

Service Requirements

You must have the following services properly configured:

  • Experience Cloud Identity Service (ECID) — Must be properly deployed on your website. This is critical; if the Identity Service is not correctly implemented, server-side forwarding will not work even if all other settings are correct.
  • Adobe Experience Cloud Organization — Your Analytics report suite must be mapped to the correct Experience Cloud Organization ID.

Code Requirements

AppMeasurement Library

  • You must have an updated version of AppMeasurement (the Analytics JavaScript library) deployed on your site.
  • You can check your current version by inspecting your Analytics tracking request.

Audience Management Module

  • You must install the Audience Management Module page code.
  • Important: If you are an existing Audience Manager customer using DIL (Data Integration Library) code, you must remove DIL code before enabling server-side forwarding. Running both simultaneously will cause:
  • Duplicate data
  • Double billing in Audience Manager

Admin Configuration Requirements

Analytics Admin UI

  • Server-side forwarding must be enabled in the Analytics Admin Tools at the report suite level.
  • The applicable report suite must be mapped to the correct Experience Cloud Organization ID.

Verifying Your Current Implementation State

Before setting up, check whether server-side forwarding is already implemented by inspecting the HTTP response from your Analytics tracking request in your browser's developer tools (filter for paths containing `b/ss`):

| Response Received | What It Means |

|---|---|

| JSON response with items like `"postbacks"` or `"dcs_region"` | Server-side forwarding is already enabled |

| `"status":"SUCCESS"` | Audience Management Module is implemented, but SSF is not properly configured |

| A 2×2 image | Neither server-side forwarding nor the Audience Management Module is implemented |

Billing Considerations

  • Analytics: No additional billing occurs for server-side forwarding.
  • Audience Manager: Forwarded hits are treated like any other hits and are billed accordingly.
  • This is why it is critical to not have DIL and server-side forwarding enabled at the same time — doing so results in double billing and data duplication.
