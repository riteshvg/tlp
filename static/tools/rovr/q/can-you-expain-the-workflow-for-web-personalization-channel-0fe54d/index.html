Overview

The Web Personalization channel in AJO allows you to deliver real-time, personalized experiences directly on your website — modifying content, layouts, images, and text for different audience segments without requiring constant developer involvement.

Understanding Web Personalization Architecture

Before configuring, it is important to understand how web personalization flows from AJO to your website visitor.

```

WEB PERSONALIZATION FLOW:

Visitor lands on your website

↓

Web SDK (alloy.js) fires sendEvent

↓

Adobe Edge Network receives request

↓

AJO evaluates:

├── Is visitor in a personalized audience?

├── Is there an active web campaign/journey?

├── What content modifications apply?

└── What offers are eligible?

↓

AJO returns personalization decisions

↓

Web SDK applies DOM modifications

↓

Visitor sees personalized experience

```

Supported Implementation Types

AJO supports two implementation modes for web personalization:

| Mode | Description | Best For |

|---|---|---|

| Client-Side Only | Web SDK on page requests and renders personalization | Most websites |

| Hybrid Mode | Server-side API fetches decisions, Web SDK renders | Performance-critical sites |

> ⚠️ Server-side only implementation is NOT supported for the Web channel. If you have server-side only implementation, use the Code-based Experience channel instead.

STEP 1: Implement Web SDK on Your Website

What is the Web SDK?

The Adobe Experience Platform Web SDK (also called `alloy.js`) is a JavaScript library that:

  • ✅ Sends visitor behavioral events to AEP
  • ✅ Requests personalization decisions from AJO
  • ✅ Renders DOM modifications on the page
  • ✅ Tracks interactions (clicks, impressions, conversions)
  • ✅ Manages visitor identity (ECID)

> ⚠️ Important: Your Web SDK version must be 2.16 or above for web channel support.

Implementation Option A: Via Adobe Tags (Recommended)

Prerequisites:

  • Adobe Data Collection access ✅
  • Access to your website's `<head>` tag ✅

Steps:

1. Go to Adobe Experience Platform Data Collection (launch.adobe.com)

2. Click Tags → Properties

3. Click New Property

4. Enter:

```

Property Settings:

├── Name: "Luma Website - Production"

├── Platform: Web

├── Domain: www.yourbrand.com

└── Advanced Settings: default

```

5. Click Save

6. Open the property → click Extensions

7. Click Catalog → find Adobe Experience Platform Web SDK

8. Click Install

9. Configure Web SDK Extension:

```

Web SDK Extension Settings:

├── Name: "AEP Web SDK"

├── IMS Org ID: [Your Org ID - auto populated]

├── Edge Domain: edge.adobedc.net (default)

├── Datastream:

│ ├── Production: [Your Production Datastream]

│ ├── Staging: [Your Staging Datastream]

│ └── Development: [Your Dev Datastream]

└── Privacy:

├── Default Consent: In (or Pending for consent-first)

└── IDFA: Enabled

```

10. Click Save

11. Create a Rule for page load:

```

Rule Name: "Send Page View Event"

Event:

├── Extension: Core

├── Event Type: Page Bottom (or Library Loaded)

└── Order: 50

Action:

├── Extension: AEP Web SDK

├── Action Type: Send Event

├── Type: web.webpagedetails.pageViews

├── XDM Data:

│ ├── web.webPageDetails.name: {{Page Name}}

│ ├── web.webPageDetails.URL: {{Page URL}}

│ └── web.webReferrer.URL: {{Referring URL}}

└── Render Decisions: ✅ TRUE (enables personalization rendering)

```

12. Click Save → Build → Publish

13. Copy the Tag embed code and paste into your website's `<head>`:

```html

STEP 2: Configure Datastream

Why is a Datastream Needed?

The Datastream tells Adobe's Edge Network how to route data from your website to the correct AEP services.

```

Website (Web SDK)

↓

Adobe Edge Network

↓

Datastream (routing configuration)

├── Adobe Experience Platform → Profile + Events

├── Adobe Journey Optimizer → Personalization decisions

└── Adobe Analytics → Reporting (optional)

```

How to Configure Datastream for Web Personalization

1. Go to Adobe Data Collection → Datastreams

2. Click New Datastream (or edit existing)

3. Enter:

```

Datastream Settings:

├── Name: "Luma Website - Production"

├── Event Schema: [Your XDM Web Schema]

└── Description: "Production datastream for website"

```

4. Click Save

5. Add Adobe Experience Platform service:

```

AEP Service Settings:

├── Enabled: ✅ YES

├── Sandbox: [Your sandbox name]

├── Event Dataset: [Your web events dataset]

├── Profile Dataset: [Your profile dataset]

├── Edge Segmentation: ✅ YES (enables real-time segment evaluation)

├── Personalization Destinations: ✅ YES

└── Adobe Journey Optimizer: ✅ YES (CRITICAL for web personalization)

```

> ⚠️ Critical: You MUST enable Adobe Journey Optimizer in the datastream service for web personalization to work.

```

STEP 3: Install Visual Editing Helper Extension

What is the Visual Editing Helper?

The Adobe Experience Cloud Visual Editing Helper is a Chrome/Edge browser extension that enables the visual web designer in AJO to load your website inside the composer and make visual modifications.

Why is it needed?

Without the extension:

  • ❌ AJO cannot load your website inside the visual composer
  • ❌ Cross-origin restrictions prevent DOM inspection
  • ❌ Web SDK interactions cannot be simulated

With the extension:

  • ✅ Your website loads inside the AJO visual composer
  • ✅ You can click on any element to modify it
  • ✅ Changes preview in real-time
  • ✅ Web SDK can be bypassed for authoring purposes

How to Install

1. Open Google Chrome or Microsoft Edge

2. Go to the Chrome Web Store

3. Search for "Adobe Experience Cloud Visual Editing Helper"

4. Click Add to Chrome (or Add to Edge)

5. Click Add Extension in the confirmation dialog

6. The extension icon appears in your browser toolbar ✅

Extension Configuration

After installing:

```

Extension Settings:

├── Allow on all sites: ✅ YES (recommended for authoring)

├── Enable on specific sites:

│ └── yourbrand.com ✅

└── Disable on: (none)

```

> ⚠️ Note: The Visual Editing Helper is only needed for authoring — it is not required for production delivery of personalization.

STEP 4: Configure Web Channel Surface

What is a Web Channel Surface?

A Web Channel Surface (Channel Configuration) defines:

  • Which pages the web personalization applies to (URL matching rules)
  • What marketing action the surface uses (for governance)
  • Web property settings for the configuration

How to Create a Web Channel Surface

Prerequisites:

  • Web SDK implemented on website ✅
  • Datastream configured with AJO enabled ✅
  • Manage Messages Presets permission ✅

Steps:

1. Go to Administration → Channels → Channel Configurations

2. Click Create channel configuration

3. Enter a Name (e.g., "Luma Website - All Pages")

4. Select Web as the channel type

5. Configure Web Settings:

URL Matching Rules

URL matching rules define which pages the web experience is delivered on:

```

URL Matching Rule Types:

1. EQUALS (Exact Match)

├── Rule: URL equals "https://www.yourbrand.com/home"

└── Matches: Only exactly that URL

2. STARTS WITH

├── Rule: URL starts with "https://www.yourbrand.com/products"

└── Matches: All product pages

3. ENDS WITH

├── Rule: URL ends with "/checkout"

└── Matches: All checkout pages regardless of path

4. CONTAINS

├── Rule: URL contains "category=shoes"

└── Matches: All pages with shoes category parameter

5. DOES NOT EQUAL

├── Rule: URL does not equal "https://www.yourbrand.com/admin"

└── Excludes: Admin pages from personalization

6. DOES NOT CONTAIN

├── Rule: URL does not contain "?preview=true"

└── Excludes: Preview pages from personalization

7. REGULAR EXPRESSION

├── Rule: URL matches regex "^https://www\.yourbrand\.com/products/[0-9]+"

└── Matches: All product detail pages with numeric IDs

```

Example URL Matching Configurations:

```

Configuration 1: Homepage Only

└── Rule: URL equals "https://www.yourbrand.com/"

Configuration 2: All Product Pages

└── Rule: URL starts with "https://www.yourbrand.com/products"

Configuration 3: Entire Website (except admin)

├── Rule: URL starts with "https://www.yourbrand.com"

└── Rule: URL does not contain "/admin"

Configuration 4: Campaign Landing Pages

└── Rule: URL contains "campaign=summer2024"

```

Web Property Settings

```

Web Property Settings:

├── Page Match: [URL Matching Rules above]

├── Enable Single Page Application (SPA) mode:

│ ├── YES → For React, Angular, Vue.js apps

│ └── NO → For traditional multi-page websites

└── Marketing Action: [Select for data governance]

└── Options: "Web Personalization", "On-Site Personalization"

```

6. Assign Marketing Action for data governance

7. Click Submit

8. Status shows ✅ Active
