In-App Messages Channel Workflow in Adobe Journey Optimizer

In-app messages in AJO allow you to deliver contextual notifications to users while they are actively using your web or mobile application. Here is a complete breakdown of the workflow:

Overview

The workflow consists of three main phases:

1. Channel Configuration (prerequisites)

2. Campaign or Journey Setup

3. Design, Test, and Publish

Phase 1: Channel Configuration (Prerequisites)

Before creating any in-app message, you must set up the channel infrastructure.

For Mobile (iOS / Android)

  • Ensure the Adobe Mobile SDK is installed and configured in your app.
  • Register lifecycle events, geolocation events, and core tracking APIs so that triggers are available.

For Web

  • Install the Web SDK tag extension to support Web In-App Messaging.
  • Configure your Datastream to route data to Adobe Experience Platform.

Create a Channel Configuration

1. In the left menu, navigate to Channels → Channel configurations.

2. Click Create channel configuration.

3. Enter a name (e.g., `MyOrg_In-app_Messages`).

4. Select In-app messaging as the channel.

5. Enable the target platforms: Web, iOS, and/or Android.

6. For Web, enter the URL of your website.

7. For iOS/Android, enter your app's bundle/package ID.

8. Click Submit.

Phase 2: Campaign or Journey Setup

You can deliver in-app messages through either a Campaign or a Journey.

Option A — Campaign (Scheduled or API-triggered)

1. Navigate to Campaigns in the left rail and click Create campaign.

2. Choose the execution type: Scheduled - Marketing or API-triggered.

3. From the Actions drop-down, select In-app message.

4. Choose or create your app configuration (the channel configuration created above).

5. Define your campaign properties (name, description, audience).

6. Configure your trigger — in-app messages can be triggered by:

  • Data sent to Adobe Experience Platform
  • Manual triggers
  • Mobile Core generic API events (actions, states, PII collection)
  • Application lifecycle events (launch, install, upgrade, close, crash)
  • Geolocation events (entering or exiting a point of interest)

Option B — Journey

1. Create or open a Journey in Journey Optimizer.

2. Add an In-app message action node at the appropriate step in the journey.

3. Configure the message content and trigger conditions within the journey flow.

Phase 3: Design, Test, and Publish

Design the Message

  • Use the in-app message designer to build your message content (text, images, buttons, layout).
  • Personalize content using profile attributes and contextual data.

Content Experimentation (Optional)

  • Run A/B tests on your in-app messages to compare variants and optimize performance.

▶ Watch: Configure and Analyze Content Experiments for In-App A/B Testing

Test and Publish

  • Use the preview and test tools to validate message rendering on target devices/browsers.
  • Once validated, publish the campaign or activate the journey.

How Triggers Work (Mobile SDK)

On mobile, the SDK event hub plays a central role:

  • It publishes and receives event data from registered extensions.
  • Events generated by Mobile Core generic APIs (tracking actions, states, PII) are published to the event hub.
  • The in-app messaging extension listens for these events and displays the message when the configured trigger condition is met.

This architecture means in-app messages are fully event-driven and respond in real time to user behavior within your app.

Summary Flow

```

Channel Config → Campaign/Journey Creation → Trigger Definition

→ Message Design → A/B Test (optional) → Test → Publish
```
