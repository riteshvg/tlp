My streaming ingestion in AEP is failing. What logs should I check?

Troubleshooting Streaming Ingestion Failures in AEP

When your streaming ingestion is failing, here are the key areas and logs you should check:

Data Validation Issues

Check for validation errors by reviewing:

  • Synchronous validation responses - If you're using synchronous validation mode, the API will return validation errors immediately in the response. These will indicate schema violations, missing required fields, or data type mismatches.
  • Asynchronous validation results - If using asynchronous validation, check the validation status after submission. Errors may appear in the response payload indicating what failed validation.

Common Failure Points to Investigate

XDM Schema Compliance

  • Verify your data conforms to the XDM schema associated with your streaming dataset
  • Check that all required fields are included
  • Ensure data types match the schema definition

Authentication & Data Inlet Configuration

  • Confirm your data inlet is properly configured and authenticated
  • Verify the data inlet is enabled to receive streaming data
  • Check that trusted source authentication is properly set up

Data Lake & Dataset Issues

  • Verify the target dataset is streaming-enabled
  • Check that the dataset hasn't reached capacity or encountered storage issues
  • Confirm the dataset connection is active

What to Review

The streaming ingestion validation documentation outlines both synchronous and asynchronous validation modes - review which mode you're using and check the corresponding error responses.
