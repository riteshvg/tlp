What specific components (segments, calculated metrics, classifications) can be migrated and which ones need to be recreated?

Component Migration: What Migrates vs. What Must Be Recreated

Overview

When migrating from Adobe Analytics to Customer Journey Analytics, components fall into three categories:

  • ✅ Automatically migrated
  • ⚠️ Partially migrated (some attributes are not carried over)
  • ❌ Must be manually recreated

Detailed Component Breakdown

📊 Dimensions & Metrics

| Attribute | Migrated? |

|---|---|

| The dimension/metric itself | ⚠️ Partially — some auto-mapped, some require manual mapping |

| Owner | ❌ Not migrated |

| Sharing settings | ❌ Not migrated |

| Descriptions | ❌ Not migrated |

| Tags | ❌ Not migrated |

| Attribution (on dimensions) | ❌ Not migrated |

> ⚠️ After migration, you will need to manually re-apply descriptions, tags, sharing settings, attribution models, and ownership in CJA.

🔍 Segments

| Attribute | Migrated? |

|---|---|

| Segment definition | ✅ Yes — re-created in CJA |

| Owner | ✅ Yes |

| Descriptions | ✅ Yes |

| Sharing settings | ❌ Not migrated |

| Tags | ❌ Not migrated |

| Attribution | ❌ Not migrated |

> ✅ Segments are automatically re-created in CJA based on the mapped dimensions and metrics — no manual mapping required.

🧮 Calculated Metrics

| Attribute | Migrated? |

|---|---|

| Calculated metric definition | ✅ Yes — re-created in CJA |

| Owner | ❌ Not migrated |

| Sharing settings | ❌ Not migrated |

| Descriptions | ❌ Not migrated |

| Tags | ❌ Not migrated |

> ✅ Calculated metrics are automatically re-created in CJA based on the mapped dimensions and metrics.

> ❌ However, owner, descriptions, tags, and sharing must be manually re-applied in CJA after migration.

📅 Date Ranges

| Attribute | Migrated? |

|---|---|

| Date range definition | ✅ Yes — re-created in CJA |

| Owner | ✅ Yes |

| Descriptions | ✅ Yes |

| Sharing settings | ❌ Not migrated |

| Tags | ❌ Not migrated |

> ✅ Date ranges are automatically re-created in CJA.

> ❌ Sharing settings and tags must be manually re-applied.

🗂️ Classifications

| Attribute | Migrated? |

|---|---|

| Classification sets | ❌ Not migrated |

| Classification rules | ❌ Not migrated |

| Classification importer data | ❌ Not migrated |

> ❌ Classifications are not migrated as part of the component migration process. They must be manually recreated in CJA using lookup datasets in Adobe Experience Platform.

> ⚠️ Important deprecation notice:

> - Classification Rules Builder will be deprecated after February 28, 2027

> - Classification Importer (including FTP imports) will be deprecated after August 31, 2026

>

> Adobe recommends transitioning to Classification Sets, the primary and simplified interface for managing classifications going forward.

🗃️ Analysis Workspace Projects

| Attribute | Migrated? |

|---|---|

| Project structure & visualizations | ✅ Yes — re-created in CJA |

| Panels and freeform tables | ✅ Yes |

| Associated segments | ✅ Yes |

| Associated calculated metrics | ✅ Yes |

| Associated date ranges | ✅ Yes |

| Dimension/metric mappings | ⚠️ Requires prior mapping setup |

Summary Table

| Component | Migrates? | Manual Action Required |

|---|---|---|

| Workspace Projects | ✅ Yes | Map dimensions/metrics first |

| Segments | ✅ Yes (definition, owner, description) | Re-apply tags & sharing |

| Calculated Metrics | ✅ Yes (definition only) | Re-apply owner, description, tags, sharing |

| Date Ranges | ✅ Yes (definition, owner, description) | Re-apply tags & sharing |

| Dimensions | ⚠️ Partial | Manual mapping + re-apply all attributes |

| Metrics | ⚠️ Partial | Manual mapping + re-apply all attributes |

| Classifications | ❌ No | Fully recreate as AEP lookup datasets |

| Sharing settings | ❌ No | Manually re-apply for all components |

| Tags | ❌ No | Manually re-apply for all components |

| Attribution models | ❌ No | Manually re-apply in CJA data view |

Post-Migration Checklist

After completing the migration, manually address the following in CJA:

1. ✅ Re-apply ownership to calculated metrics

2. ✅ Re-add descriptions to dimensions, metrics, and calculated metrics

3. ✅ Re-apply tags to all component types

4. ✅ Re-configure sharing settings for all components

5. ✅ Re-apply attribution models to dimensions in your CJA data view

6. ✅ Recreate classifications as lookup datasets in AEP

7. ✅ Validate all migrated projects in CJA Analysis Workspace for accuracy
