How to Test Personalization Before Sending

Adobe Journey Optimizer provides multiple tools and workflows to validate that your personalization renders correctly before sending messages to your audience.

Video Walkthrough

▶ Watch: Test Email Rendering and Personalization

This video demonstrates how to:

  • Use test profiles to test email rendering across different inboxes
  • Preview personalized emails against test profiles
  • Send proofs to validate content before launch

Testing Methods

1. Use the Personalization Playground

Before building your content:

  • Access the personalization playground to learn syntax and test expressions with sample data
  • Experiment with personalization logic in a safe environment
  • Validate that your expressions work as expected before applying them to your message

2. Preview with Test Profiles

During content creation:

  • Use test profiles to preview how your personalized content renders for specific customers
  • Test multiple scenarios by selecting different test profiles
  • Verify that personalization variables (names, attributes, conditional logic) display correctly

3. Test Multiple Variations with Sample Input Data

  • Use sample input data to test up to 30 variants without creating individual test profiles
  • Upload CSV/JSON files containing different customer attributes
  • Add values manually or auto-generate variants with AI
  • Preview all variations at once to see how content adapts to different profiles

Important Testing Considerations

  • Personalization with context data – When previewing a message or sending proofs, only profile personalization data is displayed. Personalization based on context data (such as event information) can only be tested in the context of a journey.
  • Preview content with multiple conditional variants – When simulating or rendering proofs for emails containing multiple conditional variants, Journey Optimizer may require more processing time. If you experience timeouts or error messages, consider reducing the total number of variants or simplifying conditional rules.

Required Permissions

  • Preview/simulate content – You need the [!DNL Manage Simulate Content] permission in the [!DNL Content Library Manager] product profile
  • Send proofs – You need Approve and publish permissions for the specific resource (campaign or journey), plus Publish journey permission if sending proofs in a journey
