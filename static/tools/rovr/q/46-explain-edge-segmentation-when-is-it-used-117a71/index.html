46. Explain Edge Segmentation. When is it Used?

Edge Segmentation

What is Edge Segmentation?

Edge segmentation is the ability to evaluate segment definitions in Adobe Experience Platform instantaneously on the edge, enabling same-page and next-page personalization use cases.

Rather than waiting for batch processing or streaming evaluation, edge segmentation processes audience evaluations at the edge servers closest to where data was collected, delivering real-time results for immediate personalization decisions.

Key Characteristics

Real-Time Evaluation

  • Segments are evaluated in near real-time as requests come in
  • Results are immediately available for personalization decisions
  • No delay waiting for batch jobs or streaming pipelines

Edge Processing

  • Data is processed on edge servers located closest to data collection points
  • Edge servers dynamically load the metadata required for evaluation
  • Data may be stored in edge locations rather than the hub/principal Adobe Experience Platform data center

Identity Requirements

  • The edge segmentation engine honors requests where there is one primary marked identity
  • This is consistent with non-edge-based primary identity requirements
  • Proper identity resolution is critical for accurate segmentation

When to Use Edge Segmentation

Edge segmentation is specifically designed for:

1. Same-page personalization — Personalize content on the current page immediately

2. Next-page personalization — Segment users for the next page they visit

3. Real-time decisioning — Make instant audience qualification decisions at request time

4. High-volume, low-latency use cases — Process requests at scale with minimal delay

Important Considerations

Cold-Start Latency

Since edge servers dynamically load metadata on-demand, the first calls may experience "cold-start" latency regardless of sandbox type. During this window, initial evaluation calls may result in timeouts.

Recommendation: Run a brief pre-warm burst or realistic load before testing to eliminate false-positive failures.

Comparison with Other Methods

  • Streaming Segmentation — Focuses on data richness; evaluates as data lands in Experience Platform
  • Edge Segmentation — Focuses on same-page/next-page personalization; evaluates at the edge for immediate decisions

For more detailed setup and configuration, refer to the edge segmentation UI guide and documentation.
