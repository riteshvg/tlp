How do I change the color theme of the Analysis Workspace UI?

Changing the Color Theme in Analysis Workspace

There are two types of color customization available in Analysis Workspace — the UI theme (dark/light mode) and the visualization color palette. Here's how to configure both.

1. UI Theme (Dark/Light Mode)

You can toggle the Analysis Workspace interface theme through user preferences.

Steps:

1. In Analysis Workspace, navigate to your user preferences

2. Locate the UI theme toggle setting

3. Switch between light and dark mode as preferred

▶ Watch: Customize Analysis Workspace User Preferences

2. Visualization Color Palette

You can also change the color palette applied to visualizations across your project.

Steps:

1. Open your project in Analysis Workspace

2. Click Project in the top menu

3. Select Project Settings

4. Locate the Color Palette section

5. Choose from one of the available options:

| Palette Type | Where It Applies | Options |

|---|---|---|

| Categorical palette | Most visualizations in Workspace | Adobe-provided presets or custom comma-delimited hex values |

| Divergent palette | Cohort tables | Two color extremes with a neutral baseline in the middle |

| Sequential palette | Frequency trends (stacked bar) guided analysis | Light to dark numeric progression |

Pre-set Palette Options

  • There are six pre-set color palettes available
  • The default palette and the second listed palette have been optimized for color contrast and are more accessible for users who are color blind
  • The remaining palettes are optimized for color harmony

Custom Palette

  • You can specify your own palette to match your company's branding colors
  • Enter custom colors as comma-delimited hex values

Key Limitations

  • Color palette changes do not affect the following:
  • Summary Change visualizations
  • Conditional formatting in Freeform tables
  • Map visualizations
  • Color palette support is not enabled for Internet Explorer 11

Accessibility Note

Analysis Workspace strives for WCAG 2.1 AA conformance, including requirements for color contrast. The default and second pre-set palettes are specifically designed with color-blind accessibility in mind. User-level color palette preferences set under Project Settings apply per project.
