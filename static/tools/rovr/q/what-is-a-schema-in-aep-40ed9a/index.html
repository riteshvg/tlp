What is a Schema in Adobe Experience Platform?

A schema is a set of rules that represent and validate the structure and format of data in Adobe Experience Platform (AEP). It provides an abstract definition of a real-world object (such as a person) and outlines what data should be included in each instance of that object.

Key Characteristics

Structure and Validation

  • Schemas define the structure and format of data
  • They apply constraints and expectations to data for validation as it moves between systems
  • They enable data to be interpreted consistently, regardless of origin

Composition

Schemas are comprised of:

  • A base class (required)
  • Zero or more schema field groups (optional)
  • Individual fields with defined data types
  • Behavioral attributes, timestamps, identities, attribute definitions, and relationships

Why Schemas Matter in Experience Platform

Data Normalization: Schemas maintain semantic normalization by using standardized definitions, allowing data to be reused across your organization without conflicts

Reusability: All data conforming to schemas can be reused across an organization and even shared between multiple organizations

Foundation for Workflows: Before data can be ingested into Experience Platform, a schema must be composed to describe the data's structure and define what types of data each field can contain

Schema Resources

You can access schemas through the [!UICONTROL Schemas] workspace in the Experience Platform UI. This workspace allows you to:

  • View and manage schemas, classes, field groups, data types, and relationships
  • Access Adobe-provided standard resources
  • Create and manage custom resources for your organization

Schemas are stored in the Schema Library, which contains both Adobe's industry-standard XDM resources and custom resources defined by your organization.
