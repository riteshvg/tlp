Event Forwarding in Adobe Experience Platform

What is Event Forwarding?

Event forwarding is a feature in Adobe Data Collection that allows you to send event-level data to the Adobe Experience Platform Edge Network, then transform, enrich, and forward that data to non-Adobe third-party destinations — all from Adobe's servers rather than the client-side browser.

This provides:

  • Lower latency — processing happens server-side
  • Reduced client-side code — fewer tags and scripts on your web/mobile app
  • Centralized data collection — single point of control for data routing

How Data Gets to the Edge Network

Before event forwarding can work, data must first be sent to the Adobe Experience Platform Edge Network using one of these three methods:

1. Adobe Experience Platform Web SDK (`alloy.js`)

2. Adobe Experience Platform Mobile SDK

3. Edge Network Server API (server-to-server)

> Note: While Web SDK and Mobile SDK don't require deployment through Tags, using Tags to deploy them is recommended.

High-Level Setup Process

Step 1: Create an Event Forwarding Property

  • Navigate to Data Collection in the Adobe Experience Platform left navigation
  • Create a new Event Forwarding property (separate from your Tags/client-side property)

Step 2: Update Your Datastream

  • Update your existing Datastream to make data collected by your Web SDK (client-side Tags property) available to your Event Forwarding (server-side) property
  • This links the two properties together through the Edge Network

Step 3: Configure Secrets (for Authentication)

Event forwarding allows you to create, manage, and store secrets for authenticating to destination servers. To create a secret:

1. Select Event Forwarding in the left navigation

2. Open the event forwarding property you want to add the secret to

3. Select Secrets in the left navigation

4. Click Create New Secret

5. Select the Target Environment from the dropdown

6. Provide a Secret Name (must be unique across all secrets in the property)

> Note: A secret must be assigned to an existing environment before it can be used. If no environments exist yet, configure them first before creating secrets.

Step 4: Create Data Elements and Rules

  • Data Elements — Define which pieces of XDM data from the Edge Network payload you want to use
  • Rules — Configure conditions and actions that determine when and where data gets forwarded

> Key Difference from Tags: Event forwarding rules are triggered by server-side events from the Edge Network, not browser-side interactions.

Step 5: Configure a Destination Extension

Install and configure an extension for your target destination. Available extensions include:

| Destination | Extension Type |

|---|---|

| Custom Webhook | HTTP send action |

| Google Cloud Pub/Sub | GCP extension |

| AWS Kinesis & AWS S3 | AWS extension |

| Splunk | Splunk extension |

| Zendesk | Zendesk extension |

| Snapchat Conversions API | Snap Conversions API extension |

Supported Destination Examples

From the documentation, the following forwarding destinations are covered:

  • Custom Webhooks — Create and configure a custom webhook to receive forwarded data
  • Google Cloud Pub/Sub — Create a Google Cloud Function and forward Web SDK data to Google
  • AWS Kinesis & AWS S3 — Configure AWS IAM, Kinesis, Firehose, and S3 to receive event data
  • Splunk — Send data using the Splunk event forwarding extension
  • Zendesk — Send data using the Zendesk event forwarding extension

Key Architectural Concept

The flow works as follows:

```

Browser/App → Web SDK / Mobile SDK → Edge Network → Event Forwarding Rules → Third-Party Destination

```

All routing and transformation logic executes on Adobe's servers, meaning:

  • No additional client-side scripts needed for each destination
  • Data is not exposed to the browser before forwarding
  • Faster and more secure than client-side tag-based forwarding

Learning Objectives

After setting up event forwarding, you will be able to:

  • Create an event forwarding property
  • Link it to a Platform Web SDK datastream
  • Understand the differences between tag property data elements/rules and event forwarding property data elements/rules
  • Create event forwarding data elements
  • Configure event forwarding rules
  • Validate that the event forwarding property is successfully sending data
