Overview

Consent management in Web SDK + Tags implementations involves three key layers:

1. Default Consent – What happens before a visitor provides explicit consent

2. Consent Collection – How visitor consent preferences are captured

3. Consent Enforcement – How the SDK respects and acts on those preferences

> Legal Context:

> - GDPR (Europe): Requires explicit opt-in consent BEFORE collecting data

> - CCPA (California): Requires opt-out mechanism — data collected by default unless opted out

> - Other Regulations: LGPD (Brazil), PDPA (Thailand), PIPL (China) — each with specific requirements

Consent Architecture Overview

```

┌─────────────────────────────────────────────────────────────────┐

│ CONSENT MANAGEMENT FLOW │

│ │

│ 1. VISITOR ARRIVES │

│ ↓ │

│ 2. DEFAULT CONSENT APPLIED (configured in Web SDK/Tags) │

│ → "in" = collect immediately │

│ → "out" = block all collection │

│ → "pending" = queue events until decision │

│ ↓ │

│ 3. CONSENT DIALOG SHOWN (CMP or custom) │

│ ↓ │

│ 4. VISITOR MAKES CHOICE │

│ → Accept All → setConsent("in") │

│ → Reject All → setConsent("out") │

│ → Partial → setConsent(granular preferences) │

│ ↓ │

│ 5. WEB SDK PROCESSES CONSENT │

│ → "in": Send queued events + collect going forward │

│ → "out": Discard queued events + block collection │

│ → Persists preference in consent cookie │

│ ↓ │

│ 6. SUBSEQUENT VISITS │

│ → Consent cookie read automatically │

│ → Previous preference immediately applied │

│ → No dialog shown (already consented) │

└─────────────────────────────────────────────────────────────────┘

```

Layer 1: Default Consent Configuration

What is Default Consent?

Default consent is the SDK behavior before a visitor explicitly provides their preference. It is configured once in the Web SDK extension and applies to all visitors on their first visit.

> Important: Default consent level is NOT saved to user profiles — it only controls SDK behavior during the pre-consent window.

Configuring Default Consent in Tags

1. Log in to experience.adobe.com

2. Navigate to Data Collection → Tags

3. Open your Tag Property

4. Go to Extensions → Adobe Experience Platform Web SDK → Configure

5. Scroll to the Privacy section

Default Consent Options

```

Option 1: IN (Opt-In Default)

Behavior: Collect ALL events immediately

Use Case: Regions where data collection is legal by default

(e.g., USA under CCPA — collect unless opted out)

Risk: May violate GDPR if used in European markets

Option 2: OUT (Opt-Out Default)

Behavior: DROP all events until explicit consent given

Use Case: Regions requiring explicit consent (e.g., EU under GDPR)

Impact: You lose ALL data for visitors who never interact with consent dialog

Recommendation: Best for strict privacy compliance

Option 3: PENDING (Queue Default)

Behavior: QUEUE all events until explicit consent given

→ If visitor consents: queued events are SENT

→ If visitor declines: queued events are DISCARDED

Use Case: When you want to capture pre-consent interactions

IF the visitor ultimately consents

Recommendation: Best balance of compliance and data capture

Option 4: Data Element

Behavior: Resolve consent level dynamically from a data element

Use Case: Different default per region or visitor type

Example: EU visitors → "pending", US visitors → "in"

```

Configuring Default Consent via Code

```javascript

// Option 1: Static default consent

alloy("configure", {

datastreamId: "YOUR_DATASTREAM_ID",

orgId: "YOUR_ORG_ID@AdobeOrg",

defaultConsent: "pending" // "in", "out", or "pending"

});

// Option 2: Dynamic default consent based on region

alloy("configure", {

datastreamId: "YOUR_DATASTREAM_ID",

orgId: "YOUR_ORG_ID@AdobeOrg",

defaultConsent: function() {

// Determine consent based on visitor's region

var region = getVisitorRegion(); // Your custom function

if (region === "EU" || region === "UK" || region === "CA") {

return "pending"; // GDPR regions: queue events

} else {

return "in"; // Other regions: collect by default

}

}

});

// Option 3: Object-based consent (Adobe 2.0 standard)

alloy("configure", {

datastreamId: "YOUR_DATASTREAM_ID",

orgId: "YOUR_ORG_ID@AdobeOrg",

defaultConsent: {

general: "pending"

}

});

```

Layer 2: Consent Standards

Web SDK supports two consent standards:

Standard 1: Adobe Consent Standard

Version 1.0 (Simple)

```javascript

// Adobe 1.0: Simple in/out consent

alloy("setConsent", {

consent: [{

standard: "Adobe",

version: "1.0",

value: {

general: "in" // or "out"

}

}]

});

```

Consent Values:

  • `"in"` = Visitor has opted IN → collect data
  • `"out"` = Visitor has opted OUT → block data collection
Version 2.0 (Granular)

```javascript

// Adobe 2.0: Granular consent per purpose

alloy("setConsent", {

consent: [{

standard: "Adobe",

version: "2.0",

value: {

collect: {

val: "y" // "y" = yes, "n" = no

},

share: {

val: "y"

},

personalize: {

content: {

val: "y"

}

},

marketing: {

preferred: "email",

any: {

val: "y"

},

email: {

val: "y"

},

push: {

val: "n"

},

sms: {

val: "n"

}

},

idSpecific: {

ECID: {

"98765432109876543210": {

collect: { val: "y" },

personalize: { val: "y" }

}

}

},

metadata: {

time: new Date().toISOString()

}

}

}]

});

```

Standard 2: IAB Transparency & Consent Framework (TCF 2.0)

For organizations using an IAB-compliant CMP (Consent Management Platform):

```javascript

// IAB TCF 2.0 consent string from CMP

alloy("setConsent", {

consent: [{

standard: "IAB TCF",

version: "2.0",

value: "CQABBkAQABBkAGAAAENCAAAAAAAAAAAAAAAAAAAAAAA",

// TC String from your CMP (OneTrust, TrustArc, etc.)

gdprApplies: true

// true if GDPR applies to this user

// false if outside EU

}]

});

```

Requirements for IAB TCF 2.0 to update profiles:

1. XDM Individual Profile schema must include IAB TCF 2.0 Consent field group

2. Experience Event schema must include IAB TCF 2.0 Consent field group

3. IAB consent information must be included in the event XDM object

> Note: Web SDK does NOT automatically include IAB consent information when sending event data — you must explicitly include it in the XDM object.

Layer 3: Consent Collection via Consent Management Platforms (CMPs)

Using OneTrust (Most Common CMP)

```javascript

// OneTrust Integration with Web SDK

// OneTrust fires OptanonConsent event when visitor makes choice

window.addEventListener("OptanonConsent", function(event) {

// Get consent object from OneTrust

var consentObject = window.OnetrustActiveGroups;

// Map OneTrust categories to Adobe consent

var analyticsConsented = consentObject.includes("C0002"); // Analytics category

var personalizationConsented = consentObject.includes("C0003"); // Targeting category

var marketingConsented = consentObject.includes("C0004"); // Marketing category

// Set consent in Web SDK

alloy("setConsent", {

consent: [{

standard: "Adobe",

version: "2.0",

value: {

collect: {

val: analyticsConsented ? "y" : "n"

},

personalize: {

content: {

val: personalizationConsented ? "y" : "n"

}

},

marketing: {

any: {

val: marketingConsented ? "y" : "n"

},

email: {

val: marketingConsented ? "y" : "n"

}

},

metadata: {

time: new Date().toISOString()

}

}

}]

});

});

```

Using TrustArc

```javascript

// TrustArc Integration

window.addEventListener("trustarc-consent-updated", function(event) {

var consentData = event.detail;

alloy("setConsent", {

consent: [{

standard: "Adobe",

version: "2.0",

value: {

collect: {

val: consentData.analytics ? "y" : "n"

},

marketing: {

any: {

val: consentData.marketing ? "y" : "n"

}

}

}

}]

});

});

```

Custom Consent Dialog (No CMP)

```javascript

// Custom consent implementation

function showConsentBanner() {

// Show consent banner HTML

document.getElementById("consent-banner").style.display = "block";

}

// Handle "Accept All" button click

document.getElementById("accept-all").addEventListener("click", function() {

// Set full consent in Web SDK

alloy("setConsent", {

consent: [{

standard: "Adobe",

version: "1.0",

value: { general: "in" }

}]

});

// Store preference locally

localStorage.setItem("consentChoice", "in");

// Hide banner

document.getElementById("consent-banner").style.display = "none";

});

// Handle "Reject All" button click

document.getElementById("reject-all").addEventListener("click", function() {

// Set opt-out in Web SDK

alloy("setConsent", {

consent: [{

standard: "Adobe",

version: "1.0",

value: { general: "out" }

}]

});

// Store preference locally

localStorage.setItem("consentChoice", "out");

// Hide banner

document.getElementById("consent-banner").style.display = "none";

});

// On page load: check for existing consent

document.addEventListener("DOMContentLoaded", function() {

var existingConsent = localStorage.getItem("consentChoice");

if (existingConsent) {

// Apply previously stored consent

alloy("setConsent", {

consent: [{

standard: "Adobe",

version: "1.0",

value: { general: existingConsent }

}]

});

} else {

// No previous consent — show banner

showConsentBanner();

}

});

```

Layer 4: Implementing Consent in Adobe Tags

Step 1: Configure Default Consent in Web SDK Extension

1. Navigate to Extensions → Adobe Experience Platform Web SDK → Configure

2. Scroll to Privacy section

3. Set Default Consent Level:

```

For GDPR regions (EU):

→ Select: Pending

For CCPA regions (US):

→ Select: In

For global deployments:

→ Select: Provide a data element

→ Create data element that returns correct value per region

```

Step 2: Create Consent Data Elements

Data Element: Consent Value from CMP

```

Name: "CMP - Consent Value"

Extension: Core

Type: Custom Code

Code:

// Return consent value from your CMP

if (window.OnetrustActiveGroups) {

// OneTrust example

var groups = window.OnetrustActiveGroups;

return groups.includes("C0002") ? "in" : "out";

} else if (window.Optanon) {

// Check Optanon cookie

return Optanon.IsAlertBoxClosed() ? "in" : "pending";

} else {

// Default if CMP not loaded

return "pending";

}

```

Data Element: Adobe 2.0 Consent Object

```

Name: "XDM - Consent Object"

Extension: Adobe Experience Platform Web SDK

Type: XDM Object

{

"consentStrings": [{

"consentStandard": "Adobe",

"consentStandardVersion": "2.0",

"consentStringValue": {

"collect": {

"val": "%CMP - Analytics Consent%"

},

"personalize": {

"content": {

"val": "%CMP - Personalization Consent%"

}

},

"marketing": {

"any": {

"val": "%CMP - Marketing Consent%"

},

"email": {

"val": "%CMP - Email Consent%"

}

}

}

}]

}

```
