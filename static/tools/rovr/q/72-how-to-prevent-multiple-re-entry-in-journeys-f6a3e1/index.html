72. How to prevent multiple re-entry in journeys?

Preventing Multiple Re-Entry in Journeys

Adobe Journey Optimizer provides several mechanisms to control and prevent profiles from re-entering a journey multiple times.

Option 1: Disable Re-entrance Entirely

For "one-shot" journeys (e.g., a one-time welcome gift or single promotional offer), you can fully disable re-entry:

1. Open your journey and navigate to the Journey Properties panel.

2. Locate the Allow reentrance toggle.

3. Uncheck the Allow reentrance option.

Once disabled, a profile cannot enter the journey again within the global journey timeout period (up to 90 days).

> Example use case: Offering a one-time gift when a person enters a shop — you would disable re-entrance so the gift is only triggered once per profile.

Option 2: Configure a Reentrance Wait Period

If you want to allow re-entry but only after a certain amount of time has passed:

1. Open Journey Properties.

2. Ensure Allow reentrance is enabled.

3. Set a duration in the Reentrance wait period field that appears.

Key details:

  • Default wait period: 5 minutes
  • Maximum duration: 90 days
  • This prevents journeys from being erroneously triggered multiple times for the same event
  • Once the wait period elapses, the profile is eligible to re-enter

Option 3: Add a Condition Using Profile or Audience Data

Even after the reentrance wait period expires, you can add an extra layer of protection by using a Condition activity inside the journey:

  • Add a Condition node at the beginning of the journey canvas.
  • Configure it to check whether the profile has already entered the journey (using profile attributes or audience membership data).
  • Route profiles that have already entered down a separate path (e.g., end the journey for them).

> This is especially useful when you want to permanently block re-entry beyond what the 90-day global timeout enforces.

Option 4: Use Frequency Capping Rules

To prevent over-communication across multiple journeys, configure frequency capping rules and journey capping and arbitration. These enforce cross-journey message limits at the profile level.

Summary Table

| Method | Best For | Max Duration |

|--------|----------|--------------|

| Disable re-entrance | One-time journeys | Global timeout (90 days) |

| Reentrance wait period | Temporary blocks with eventual re-entry | Up to 90 days |

| Condition node (profile/audience check) | Permanent or custom logic-based blocking | No limit |

| Frequency capping rules | Cross-journey over-messaging prevention | Configurable |

> Note: These re-entry controls apply to Unitary event and Audience qualification journeys. For Read Audience journeys, entry behavior is governed differently.
