Audience Activation in Adobe Experience Platform (AEP)

What is Audience Activation?

Audience Activation is the process of taking the audiences (segments) you have built in Adobe Experience Platform and delivering them to external destinations — advertising platforms, email systems, CRMs, cloud storage, or any other endpoint — so that those audiences can be acted upon in real-world marketing campaigns.

In simple terms:

> Build → Enrich → Activate

> You collect data → unify profiles → build audiences → activate them where they matter

Activation is the last mile of the Real-Time CDP workflow. Without activation, your audiences remain inside AEP and produce no business value.

The End-to-End Activation Workflow

The overall flow looks like this:

1. Data Ingestion — Raw data flows into AEP from various sources

2. Profile Unification — Data is merged into Real-Time Customer Profiles

3. Audience Building — Segments are created using Segment Builder or Audience Composition

4. Destination Configuration — A destination connection is established

5. Audience Mapping & Activation — Audiences are mapped to the destination and activated

Step-by-Step: How Audience Activation Works in AEP

Prerequisites

Before activating audiences, ensure:

  • You have Real-Time CDP (or AEP with destinations access)
  • Your account has the developer and user roles enabled for the AEP API product profile
  • You have built at least one audience/segment in the Audience Portal
  • You have configured at least one destination connection

Step 1: Navigate to Destinations

1. Log in to Adobe Experience Platform

2. In the left navigation, click Destinations

3. Click the Catalog tab to browse available destination connectors

The Destinations Catalog is organized by category:

  • Advertising (Google, Meta, The Trade Desk, Amazon Ads, etc.)
  • Analytics (Adobe Analytics, Mixpanel, etc.)
  • Cloud Storage (Amazon S3, Azure Blob, Google Cloud Storage, SFTP)
  • CRM (Salesforce CRM, HubSpot, etc.)
  • Email Marketing (Mailchimp, Adobe Campaign, etc.)
  • Social (Facebook, LinkedIn, TikTok, etc.)
  • Personalization (Adobe Target, Custom Personalization)
  • Data Warehouse (Snowflake, BigQuery, etc.)

Step 2: Configure a Destination Connection

1. In the Catalog, find your desired destination

2. Click Set up or Activate audiences on the destination card

3. Provide:

  • Authentication credentials (API keys, OAuth tokens, S3 bucket info, etc.)
  • Connection name and optional description
  • Marketing use case tags (for data governance compliance)

4. Click Next to proceed to audience selection

Step 3: Select Audiences to Activate

Once your destination is configured, you select which audiences to send:

1. In the Activate audiences workflow, navigate to the Select audiences step

2. Browse your Audience Portal — you will see:

  • Experience Platform audiences (built with Segment Builder or Audience Composition)
  • Custom uploads (externally generated CSV audiences)
  • Audience Manager segments (if integrated via Expanded Activation)

3. Check the box next to each audience you want to activate

4. Click Next

Step 4: Attribute & Identity Mapping

This step defines what data fields are sent to the destination:

For Streaming Destinations:
  • Map profile attributes (e.g., email, phone, first name) to the destination's expected field names
  • Map identity namespaces (e.g., `Email`, `GAID`, `IDFA`) to the destination's identity fields
  • You must always map at least one mandatory identity field
For Batch (File-Based) Destinations:
  • Map XDM schema fields to destination column names
  • Optionally add calculated fields using transformation functions (e.g., to hash an email address before export)
  • Choose whether to include the segment membership column in the exported file

Example mapping for a batch destination:

| Source Field (AEP) | Destination Field |

|---|---|

| `person.name.firstName` | `first_name` |

| `personalEmail.address` | `email` |

| `mobilePhone.number` | `phone` |

| `segmentMembership.status` | `audience_status` |

Step 5: Configure Scheduling

For Batch Destinations:
  • Set the export frequency (e.g., Once, Daily, Weekly)
  • Set the start date and time
  • Choose between:
  • Full file export — All qualifying profiles exported every run
  • Incremental file export — Only profiles that changed since last export
For Streaming Destinations:
  • Scheduling is event-driven — no manual schedule needed
  • Exports happen automatically when:
  • A profile qualifies (`realized`) for a mapped segment
  • A profile exits (`exited`) a mapped segment
  • A mapped attribute changes in value

Step 6: Review & Finish

1. Review the summary screen showing:

  • Destination connection name and type
  • Selected audiences
  • Mapped fields
  • Export schedule

2. If everything looks correct, click Finish

3. The destination moves to Active status and begins exporting

Audience States During Activation

When audiences are activated to a destination, each profile carries a segment membership status:

| Status | Meaning |

|---|---|

| `realized` | Profile has just qualified for the segment — include in export |

| `existing` | Profile was already in the segment on prior exports — continue sending |

| `exited` | Profile has left the segment — suppress or remove at the destination |

> For batch destinations, `realized` + `existing` profiles are exported as active members. `exited` profiles are excluded.

> For streaming destinations, both `realized` and `exited` events are sent in real-time so the destination can act immediately.

Ad-Hoc (On-Demand) Activation

Beyond scheduled activation, AEP also supports Ad-Hoc Activation via API — allowing you to trigger an immediate audience export outside the normal schedule.

This is useful when:

  • You need to backfill a destination with a large audience immediately
  • A destination missed a scheduled export due to a technical issue
  • You need to test an activation flow without waiting for the next scheduled run

The ad-hoc activation API requires the same authentication headers as all AEP API calls:

  • `Authorization: Bearer {ACCESS_TOKEN}`
  • `x-api-key: {API_KEY}`
  • `x-gw-ims-org-id: {ORG_ID}`
  • `x-sandbox-name: {SANDBOX_NAME}`

Audience Manager Expanded Activation

If your organization uses Adobe Audience Manager, you can activate Audience Manager segments to social and advertising destinations through Expanded Activation — without needing a full Real-Time CDP license.

Key Requirements for Expanded Activation:

  • Audience Manager audiences must be based on hashed email addresses
  • Audiences are imported into AEP as custom uploads (externally generated audiences)
  • Supported destination types: Advertising and Social connectors (e.g., Facebook, Google, LinkedIn)

Expanded Activation Workflow:

1. Configure the Audience Manager source connection in AEP

2. Audience Manager segments flow into the Audience Portal as custom uploads

3. Configure a supported advertising or social destination

4. Map and activate the Audience Manager audiences to the destination

Data Governance During Activation

AEP enforces data governance policies automatically during the activation process:

  • Every dataset and field can carry data usage labels (e.g., `C1` = no advertising, `C3` = no combining with directly identifiable data)
  • Marketing action policies are evaluated at activation time
  • If an audience or mapped field violates a governance policy, AEP will block the activation and display a policy violation warning
  • You must resolve the conflict (e.g., remove the violating field or change the marketing action) before the activation can proceed
