How does CJA handle event deduplication for duplicate records?

Based on the Adobe Experience League documentation, Customer Journey Analytics handles event deduplication through a Metric Deduplication feature with several configurable options:

Enabling Deduplication

Metric deduplication is available as a checkbox setting but is disabled by default. You need to explicitly enable it when you want to prevent duplicate counting.

Deduplication Scope

Once enabled, you can determine how far back the unique check goes by selecting from these scopes:

  • Global account: Only the first metric occurrence in the reporting window is counted
  • Account: Only the first metric occurrence in the reporting window is counted
  • Opportunity: Only the first metric occurrence in the reporting window is counted
  • Buying group: Only the first metric occurrence in the reporting window is counted
  • Person: Only the first metric occurrence in the reporting window is counted
  • Session: Only the first metric occurrence of the session is counted

Deduplication ID

Instead of deduplicating based on the metric itself, you can apply deduplication based on a specific dimension. This is particularly valuable for cases like Purchase ID, where you want to deduplicate based on that identifier rather than the metric event itself.

Value to Keep

You can choose which instance to count:

  • Keep first instance: Use when the initial occurrence is the valid one (e.g., purchase confirmations where page reloads might create duplicates)
  • Keep last instance: Use when the most recent occurrence is more relevant (e.g., profile updates where you only want the final version from a session)
