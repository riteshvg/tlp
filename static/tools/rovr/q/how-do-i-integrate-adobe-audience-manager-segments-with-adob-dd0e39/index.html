How do I integrate Adobe Audience Manager segments with Adobe Experience Platform?

Integrating Adobe Audience Manager Segments with Adobe Experience Platform

This guide walks you through the complete end-to-end process of connecting AAM to AEP, ingesting traits and segments, and using that data within the platform.

Overview

The Adobe Audience Manager Source Connector in AEP streams first-party data collected in Audience Manager directly into Adobe Experience Platform. This includes:

  • AAM Traits → ingested as XDM fields
  • AAM Segments → ingested as AEP Audiences
  • AAM Signals & Declared IDs → mapped to Identity namespaces

Prerequisites

Before you begin, ensure you have the following:

  • ✅ An active Adobe Audience Manager contract
  • ✅ An active Adobe Experience Platform provisioned org
  • ✅ System Administrator or Data Engineer role in AEP
  • ✅ AAM and AEP must be provisioned under the same IMS Organization
  • ✅ At least one Trait and one Segment already created in AAM
  • ✅ Familiarity with XDM (Experience Data Model) schemas in AEP

Phase 1: Setting Up the AAM Source Connection in AEP

Step 1: Access the Sources Workspace

1. Log in to Adobe Experience Platform at experience.adobe.com.

2. From the left navigation, click Sources.

3. The Sources Catalog workspace opens, displaying all available source connectors.

Step 2: Locate the Adobe Audience Manager Source

1. In the Sources Catalog, look at the left-hand category panel.

2. Select the Adobe Applications category.

3. Locate the Adobe Audience Manager tile in the catalog.

4. Hover over the tile and click Set up.

> Note: If a connection already exists, the button will display Add data instead of Set up.

Step 3: Select Traits and Segments to Ingest

Once you click Set up, the AAM source configuration flow begins:

1. The Select traits and segments screen appears, listing all available AAM traits and segments tied to your IMS Org.

2. Review the list:

  • Traits are listed with their trait ID, name, and type (Rule-based, Onboarded, Algorithmic)
  • Segments are listed with segment ID, name, and population count

3. Use the checkboxes to select which traits and segments you want to ingest into AEP.

4. Use the search bar to filter by name if you have a large number of traits/segments.

5. Click Next to proceed.

> Important: By default, AAM automatically shares all realized traits and segments at the profile level. You do not need to manually re-select every time — only the initial setup requires this selection.

Phase 2: Dataflow Configuration

Step 4: Map to an XDM Schema

1. On the Dataflow detail screen, you need to select or create a target dataset in AEP.

2. Click New dataset or select an existing dataset.

3. Choose an XDM Schema that aligns with your AAM data structure:

  • For segment membership data → use the XDM Individual Profile class with the Segment Membership Details field group
  • For behavioral/trait data → use the XDM ExperienceEvent class

4. Assign a Dataset Name and optionally a Description.

5. Click Next.

Step 5: Configure Dataflow Settings

1. Give your Dataflow a meaningful name (e.g., `AAM Segments to AEP - Production`).

2. Optionally enable Error Diagnostics — this helps flag ingestion errors at the row level.

3. Optionally enable Partial Ingestion — allows AEP to ingest valid records even if some records fail.

4. Set an Ingestion Schedule:

| Schedule Option | Description |

|---|---|

| Once | One-time full backfill ingestion |

| Recurring | Ongoing scheduled ingestion (daily recommended) |

5. Click Next.

Step 6: Review and Finish

1. The Review screen summarizes your entire configuration:

  • Source: Adobe Audience Manager
  • Selected Traits and Segments
  • Target Dataset and Schema
  • Dataflow Name and Schedule

2. Carefully review all details.

3. If everything is correct, click Finish.

4. AEP will now begin creating the dataflow and start the initial ingestion run.

Phase 3: Data Ingestion & Monitoring

Step 7: Monitor the Dataflow

1. After finishing setup, navigate to Sources > Dataflows in the left navigation.

2. Locate your AAM dataflow by name.

3. Click on the dataflow to view:

  • Run History — list of all ingestion runs with timestamps
  • Records Ingested — count of successfully ingested profiles/events
  • Records Failed — count of failed records with error messages
  • Status — Active, Failed, or Disabled

4. For each run, click View Details to see granular ingestion metrics.

Step 8: Verify Data in AEP Datasets

1. Navigate to Data Management > Datasets in the left navigation.

2. Search for and click on your AAM target dataset.

3. Click Preview Dataset to see a sample of ingested records.

4. Verify that:

  • AAM Segment IDs are present in the `segmentMembership` field
  • AAM Trait data is mapped to the correct XDM fields
  • Identity fields (ECID, AAM UUID) are correctly populated

Phase 4: Using AAM Data in AEP

Step 9: View AAM Audiences in AEP Segmentation

Once ingested, AAM segments are available as Audiences in AEP:

1. Navigate to Audiences in the left navigation.

2. Click Browse to see all available audiences.

3. Filter by Origin: Audience Manager to see only AAM-sourced audiences.

4. Click on any audience to view:

  • Audience size and trend
  • Profile samples
  • Composition details
  • Activation history

Step 10: Activate AAM Audiences to Destinations

You can now activate AAM-ingested audiences to any AEP-supported destination:

1. Navigate to Destinations in the left navigation.

2. Click Catalog and select your desired destination (e.g., LinkedIn, Google, Meta Ads).

3. Click Set up or Activate audiences.

4. In the activation workflow:

  • Select audiences — choose your AAM-sourced audiences
  • Map attributes — map XDM profile fields to destination fields
  • Schedule — set activation frequency and start/end dates

5. Click Finish to activate.

Step 11: Use AAM Audiences in AEP Real-Time Customer Profile

AAM segment membership data automatically enriches Real-Time Customer Profiles in AEP:

1. Navigate to Profiles in the left navigation.

2. Search for a profile using an Identity (e.g., ECID, email).

3. Click on the profile to open the Profile Detail view.

4. Navigate to the Segment Membership tab.

5. You will see:

  • AAM segment names and IDs
  • Membership status (Realized, Existing, Exited)
  • Last qualification timestamp

Step 12: Use AAM Data in Customer Journey Analytics (CJA)

1. In AEP, confirm your AAM dataset is successfully ingested and available.

2. Open Customer Journey Analytics.

3. Navigate to Connections and click Create new connection.

4. Click Add datasets and search for your AAM target dataset.

5. Select the dataset and configure:

  • Person ID — map to your primary identity field (e.g., ECID)
  • Timestamp — map to the event timestamp field

6. Save the connection.

7. Create a Data View on top of this connection.

8. AAM segment membership data now appears as dimensions in Analysis Workspace within CJA.

Phase 5: Governance & Compliance

Step 13: Apply Data Usage Labels

Ensure your AAM data complies with your organization's data governance policies:

1. Navigate to Data Management > Datasets.

2. Select your AAM dataset.

3. Click Data Governance.

4. Apply appropriate Data Usage Labels such as:

| Label | Meaning |

|---|---|

| `C1` | Cannot be exported outside Adobe |

| `C2` | Cannot be used for cross-site targeting |

| `C5` | Cannot be used for interest-based targeting |

| `I1` | Directly identifiable personal data |

5. Save labels.

6. AEP's Policy Enforcement engine will automatically block any activation that violates these labels.

Complete Integration Summary

| Phase | Step | Action | Tool |

|---|---|---|---|

| Setup | 1–3 | Access Sources, find AAM connector, select traits/segments | AEP Sources UI |

| Configuration | 4–6 | Map to XDM schema, configure dataflow, review & finish | AEP Sources UI |

| Ingestion | 7–8 | Monitor dataflow runs, verify dataset records | AEP Monitoring |

| Activation | 9–10 | View audiences, activate to destinations | AEP Audiences & Destinations |

| Profile Enrichment | 11 | View AAM data in Real-Time Customer Profile | AEP Profiles |

| Analytics | 12 | Use AAM audiences as dimensions in CJA | Customer Journey Analytics |

| Governance | 13 | Apply data usage labels and enforce policies | AEP Data Governance |

Key Things to Know

  • 🔄 AAM data is ingested on a daily batch cadence into AEP by default
  • 👤 AAM segment membership enriches Real-Time Customer Profiles automatically
  • 🏷️ Only segments complying with AAM Data Export Controls are shared to AEP
  • 🔗 AAM and AEP must share the same IMS Organization for the connector to work
  • 📊 Once in AEP, AAM audiences can be activated to 600+ destinations in the AEP destination catalog
  • 🔒 All data flows through AEP's Data Governance framework, ensuring policy compliance at every step
