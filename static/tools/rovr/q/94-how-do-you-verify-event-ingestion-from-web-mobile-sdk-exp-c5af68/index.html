Overview

Verifying that events are being correctly ingested from your Web SDK or Mobile SDK into Adobe Experience Platform is critical for ensuring data quality and journey execution. AJO and AEP provide multiple tools and methods to validate event ingestion.

1. Using Adobe Experience Platform Assurance

Adobe Experience Platform Assurance is the primary tool for validating SDK event ingestion in real-time.

▶ Watch: Set Up Assurance for Mobile SDK Validation

How to Access Assurance:

1. Navigate to Adobe Experience Platform

2. Go to Data Collection → Assurance

3. Click Create Session

4. Select your property (web or mobile)

5. Generate a session URL or QR code

6. Open the URL or scan the QR code on your device/app

What You Can Monitor:

  • Event validation – See every event fired by your SDK in real-time
  • Event payload structure – View complete JSON payload for each event
  • XDM schema compliance – Verify events match your XDM schema
  • Network requests – Monitor HTTP calls to Edge Network
  • Response data – See what Edge Network returns to your app
  • SDK configuration – Verify SDK settings and initialization

Real-Time Event Monitoring:

Once connected to an Assurance session:

1. Interact with your web/mobile app (click buttons, navigate pages, etc.)

2. Events appear in real-time in the Assurance interface

3. Click on each event to expand and view full payload details

4. Verify field mapping – Confirm data is captured as expected

2. Datastream Debugging

Datastream Debugging allows you to capture and inspect Edge Network traffic for events sent from your SDK.

How to Enable:

1. Navigate to Data Collection → Datastreams

2. Select your datastream

3. Click Edit

4. Enable Debugging

5. Save your changes

View Datastream Logs:

1. Go to Data Collection → Edge Network Debugger

2. Filter by your datastream

3. View all events sent to Edge Network within the last 24 hours

4. Inspect request/response payloads

3. Adobe Experience Platform Datasets

Real-time validation of event ingestion by checking if data arrives in your AEP datasets.

How to Check:

1. Navigate to Adobe Experience Platform

2. Go to Data Management → Datasets

3. Select your event dataset (created for SDK events)

4. Click the Dataset Activity tab

5. View Ingestion metrics:

  • Batches ingested – Number of data batches received
  • Records ingested – Total event records
  • Last batch timestamp – When the latest data arrived
  • Failed records – Any ingestion errors

Real-Time Monitoring:

  • Refresh the page to see updated ingestion metrics
  • Monitor ingestion in near real-time (typically 15-30 second latency)
  • Check for failed batches or validation errors

4. Query Service for Event Verification

Query Service allows you to run SQL queries against ingested events to verify data quality.

How to Access:

1. Navigate to Adobe Experience Platform

2. Go to Data Management → Query Service

3. Click Create Query

4. Write a SQL query to inspect your event data

Example Queries:

Check latest events ingested:

```sql

SELECT

timestamp,

eventType,

identityMap,

_id

FROM

your_event_dataset

ORDER BY

timestamp DESC

LIMIT 10

```

Verify event count by type:

```sql

SELECT

eventType,

COUNT(*) as event_count

FROM

your_event_dataset

WHERE

timestamp > NOW() - INTERVAL '1 hour'

GROUP BY

eventType

ORDER BY

event_count DESC

```

5. Profile Data Verification

Verify that ingested events are creating/updating customer profiles in Real-time Customer Profile.

How to Check:

1. Navigate to Adobe Experience Platform

2. Go to Customer → Profiles

3. Use Identity namespace to search for a test profile

4. Click on the profile to view:

  • Profile attributes – Data from ingested events
  • Experience events – Timeline of events associated with this profile
  • Segment membership – Audiences the profile qualifies for

6. Journey Validation

Verify events are triggering journeys correctly by checking journey execution logs.

How to Check:

1. Navigate to Journeys

2. Open a journey that uses your events as entry triggers

3. Click the Reports tab

4. Check Live Reports (last 24 hours):

  • Profiles entered the journey
  • Actions executed
  • Events triggered the journey step

Quick Verification Checklist

| Step | Tool | What to Verify |

|---|---|---|

| 1. SDK events firing | Assurance | Events appear in real-time session |

| 2. Event payload structure | Assurance | Full JSON payload matches XDM schema |

| 3. Edge Network delivery | Datastream Debugging | Events reach Edge Network successfully |

| 4. Dataset ingestion | Dataset Activity | Batch ingestion metrics increasing |

| 5. Data quality | Query Service | Required fields are populated |

| 6. Profile updates | Profile UI | Profile attributes reflect ingested data |

| 7. Journey triggering | Journey Reports | Profiles entering journey as expected |

Common Issues & Solutions

| Issue | Solution |

|---|---|

| No events in Assurance | Verify Assurance session is active; check SDK initialization |

| Events not in dataset | Check datastream configuration; verify event mapping to schema |

| Profile not updating | Verify identity namespace used; check data ingestion lag (15-30 sec) |

| Journey not triggering | Verify event entry condition matches ingested event structure |

| Failed ingestion batches | Check XDM schema compliance; review error logs in dataset |

| High event latency | Normal delay is 15-30 seconds; check system performance |

Best Practices

✓ Use Assurance first – Start with real-time validation in Assurance

✓ Enable Datastream Debugging – Monitor all Edge Network traffic during testing

✓ Check multiple profiles – Test with different customer segments

✓ Query datasets regularly – Use Query Service for data quality audits

✓ Monitor ingestion metrics – Set up alerts for failed batches

✓ Test end-to-end – Verify events flow all the way to journeys

✓ Document test results – Keep records of validation tests for compliance
