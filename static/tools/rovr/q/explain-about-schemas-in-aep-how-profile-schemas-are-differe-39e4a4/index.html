Schemas in Adobe Experience Platform

Schemas are fundamental to AEP's data management. Let me explain what they are and how profile schemas differ.

What is a Schema?

In Adobe Experience Platform, schemas are standardized frameworks based on the Experience Data Model (XDM) that define the structure of data being ingested. All XDM schema resources are stored in the Schema Library, which includes:

  • Standard resources provided by Adobe
  • Custom resources defined by your organization

Schemas provide:

  • A consistent structure for data across the platform
  • Information on expected data types for each field
  • Documentation of field use cases
  • Guidance for data preparation and ingestion planning

Exploring Schemas in the UI

In the Experience Platform UI, you can view and explore:

  • Existing schemas – their structure and fields
  • Classes – foundational building blocks of schemas
  • Field groups – reusable collections of fields
  • Data types – field-level data structures
  • Relationships – connections between schemas

Profile Schemas vs. Other Schemas

How Profile Schemas Differ:

Profile schemas are specifically designed for Real-Time Customer Profile in AEP. Key differences include:

  • Purpose: Profile schemas enable individual customer profiles by unifying data from multiple sources
  • Profile Enablement: Profile schemas must be explicitly enabled to contribute data to the Real-Time Customer Profile
  • Identity Configuration: Profile schemas require identity fields to be configured so records can be merged and linked to create unified customer profiles
  • Use Case: Used for creating comprehensive customer 360 views combining behavioral, transactional, and demographic data

Other Schema Types:

  • Event schemas – capture time-series data and customer interactions (events that happened)
  • Standard schemas – general-purpose schemas for various data types
  • Lookup schemas – reference data that provides context

Schema Components

Every schema is built from:

1. Class – defines the basic structure (e.g., XDM Individual Profile, XDM ExperienceEvent)

2. Field Groups – add specific fields relevant to your use case

3. Data Types – define individual field structures

4. Fields – the actual attributes storing data

Benefits for Data Preparation

Understanding schemas helps you:

  • Plan data mapping before ingestion
  • Validate that incoming data matches expected structure
  • Identify required vs. optional fields
  • Understand field data types and constraints
  • Prepare datasets that align with organizational standards

The Schema Library in AEP ensures consistency, reduces errors, and accelerates your ability to activate data across the platform for analytics, personalization, and customer engagement.
