What is the global mbox in Adobe Target and how do I configure it?

Global Mbox in Adobe Target

What Is the Global Mbox?

The global mbox is a special mbox (measurement box) in Adobe Target that fires automatically on every page of your website. Unlike regular mboxes that you manually place on specific pages, the global mbox enables Target to deliver personalized content and collect data site-wide without requiring individual mbox calls on each page.

It is the default mechanism through which Target activities (A/B tests, Experience Targeting, Automated Personalization, Recommendations, etc.) are delivered when using the standard JavaScript implementation.

How the Global Mbox Works

  • It fires on every page load by default when the Target JavaScript library (at.js or Web SDK) is deployed.
  • It allows you to run activities that target any page on your site.
  • It is used for Visual Experience Composer (VEC)-based activities by default, since the VEC targets page elements delivered through the global mbox.
  • It can also carry profile parameters and entity parameters passed at page load.

Configuration Options

1. Configuring the Global Mbox Name (at.js)

The default global mbox name is `target-global-mbox`. You can customize this:

Via the Target UI:

1. Navigate to Administration > Implementation in the Target interface.

2. Under the at.js settings section, locate the Global Mbox Name field.

3. Enter your desired custom name.

4. Click Save.

Via at.js `targetGlobalSettings()`:

You can override it programmatically before at.js loads:

```javascript

window.targetGlobalSettings = {

globalMboxName: "my-custom-global-mbox"

};

```

2. Enabling or Disabling Auto-Creation of the Global Mbox

By default, at.js automatically creates and fires the global mbox on page load. You can control this behavior:

Via the Target UI:

1. Go to Administration > Implementation.

2. Find the Auto-create global mbox toggle.

3. Enable or disable as needed.

Via `targetGlobalSettings()`:

```javascript

window.targetGlobalSettings = {

autoCreateGlobalMbox: true // set to false to disable

};

```

3. Global Mbox with Web SDK (Datastream / AEP)

If you are using the Adobe Experience Platform Web SDK instead of at.js:

  • The concept of a named global mbox is replaced by the `__view__` decision scope, which fires automatically on each page view event.
  • You configure Target delivery through a Datastream in Adobe Data Collection, where you enable the Adobe Target service and optionally set a Target Property Token and Environment ID.
  • The Web SDK's `sendEvent` command with `renderDecisions: true` serves the equivalent function of the global mbox auto-fire.

Passing Data to the Global Mbox

You can attach additional data to the global mbox call:

  • Profile parameters (persisted to the visitor profile):

```javascript

targetPageParams = function() {

return {

"profile.membershipLevel": "gold",

"profile.age": 35

};

};

```

  • mbox parameters (available only for the current request):

```javascript

targetPageParams = function() {

return {

"pageName": "homepage",

"categoryId": "electronics"

};

};

```

The `targetPageParams` function is specifically designed to pass parameters into the global mbox on every page load.

Key Considerations

  • VEC activities require the global mbox — activities created with the Visual Experience Composer use the global mbox by default to deliver experiences.
  • Naming consistency — if you customize the global mbox name, ensure it matches across your Target UI configuration and your `targetPageParams` / `targetGlobalSettings` implementation.
  • One global mbox per page — only one global mbox fires per page load; additional mboxes must be placed manually using `adobe.target.getOffer()` or `applyOffer()`.
  • Flicker management — the global mbox fires synchronously (with at.js in synchronous mode) or requires a pre-hiding snippet to prevent flicker when content is swapped by Target.
