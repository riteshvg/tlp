Direct Mail Channel Workflow in Adobe Journey Optimizer

Direct mail is an offline channel that allows you to personalize and generate extraction files that third-party direct mail providers use to send physical mail to your customers. Here is the complete workflow:

How It Works — High Level

When a direct mail campaign or journey is activated, Journey Optimizer automatically:

1. Generates an extraction file containing all targeted profiles and selected data (postal addresses, profile attributes, decision items, etc.)

2. Exports that file to a server of your choice

3. Your third-party direct mail provider picks up the file and handles the physical mailing

> Important: Adobe does not control or manage the physical mailing process. You must work with your chosen third-party provider to obtain any required consents from your customers.

Phase 1: Prerequisites & Configuration

Step 1 — Configure File Routing

File routing defines where the extraction file will be exported (i.e., your server or the direct mail provider's server).

1. Navigate to Administration → Channels → General settings → File routing configurations.

2. Create a new file routing configuration, specifying:

  • The server type and connection credentials
  • The destination path where files will be deposited

Step 2 — Create a Direct Mail Channel Configuration

1. Navigate to Administration → Channels → General settings → Channel configurations.

2. Click Create channel configuration.

3. Enter a name for the configuration.

4. Select Direct mail as the channel.

5. Define the direct mail settings, including:

  • File format settings for the extraction file
  • File split threshold — the maximum number of records per file (between 1 and 200,000 records). When the threshold is reached, a new file is created for the remaining records.

6. Select the file routing configuration you created in Step 1.

7. Click Submit.

Phase 2: Create the Direct Mail Message

You can deliver direct mail through either a Campaign or a Journey.

Option A — Campaign

1. Navigate to Campaigns and click Create campaign.

2. Select the campaign type and choose Direct mail as the action.

3. Select the channel configuration created in Phase 1.

4. Define your audience (the targeted profiles).

5. Set the schedule for when the extraction file should be generated.

6. Click Edit content to configure the extraction file.

Option B — Journey

1. Open or create a Journey.

2. Drag and drop a Direct mail activity from the Actions section of the palette onto the journey canvas.

3. Provide basic information:

  • Label and description
  • Category
  • Channel configuration to use (pre-filled with the last used configuration by default)

4. Click Edit content to configure the extraction file.

Phase 3: Configure the Extraction File

The extraction file is the core deliverable of the direct mail channel. You configure:

  • Filename — the name of the generated file
  • Columns to include — profile attributes such as:
  • First name, last name
  • Postal address, city, zip code, country
  • Any other profile attributes relevant to your mailing
  • Sort order — how records are sorted within the file
  • Personalization — using profile attributes and contextual data to personalize field values per recipient

Optional: Batch Decisioning

If you are using Decisioning, you can include personalized decision items (e.g., best offer, recommended image) in the extraction file:

  • Set the Number of items in the decision policy to return multiple items per profile.
  • This supports two use cases:
  • Direct mail personalization — include offer attributes (e.g., image URLs, offer IDs) in the file for your print provider to use.
  • Batch export to downstream systems — export profiles and decisioning results to a third-party system for their own campaigns.

Phase 4: Preview, Validate & Send

Preview the Extraction File

  • Before activating, preview the extraction file to verify the data and formatting look correct.
  • Check that all required columns are present and profile data is populating as expected.

Manage Postal Consent

  • Ensure that targeted profiles have the appropriate postal mail consent recorded.
  • Journey Optimizer respects consent policies — profiles without consent will be excluded from the extraction file.

Activate the Campaign or Journey

1. Review the message summary and extraction file settings.

2. Click Review to activate (for campaigns) or activate the journey.

3. Journey Optimizer generates the extraction file and exports it to the configured server.

4. Your third-party direct mail provider retrieves the file and proceeds with the physical mailing.

Summary Flow

```

File Routing Config → Channel Config → Campaign/Journey Setup

→ Extraction File Configuration → (Optional) Batch Decisioning

→ Preview & Validate → Activate → File Exported to Server

→ Third-Party Provider Sends Physical Mail

```

Key Considerations

| Topic | Detail |

|---|---|

| File split | Files are automatically split when the record threshold (max 200,000) is reached |

| Consent | Postal consent must be captured; non-consenting profiles are excluded |

| Third-party responsibility | Adobe generates the file only — the provider handles physical delivery |

| Personalization | Profile attributes and decision items can be included as columns |

| Channels | Can be used in both Campaigns and Journeys |
