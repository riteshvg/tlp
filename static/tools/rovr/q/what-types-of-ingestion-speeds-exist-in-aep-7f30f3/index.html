Data Ingestion Speeds in Adobe Experience Platform

Adobe Experience Platform supports two main types of data ingestion, which differ in their speed and approach:

1. Streaming Ingestion (Real-time)

Streaming ingestion allows you to send data from client and server-side devices to Experience Platform in real-time. Key characteristics include:

  • Data is sent and persisted immediately in streaming-enabled datasets within the data lake
  • Uses data inlets that can be configured to automatically authenticate incoming data
  • Ideal for time-sensitive use cases where you need immediate data availability
  • Supports continuous flow of experience data from various sources

2. Batch Ingestion (Scheduled/Periodic)

Batch ingestion allows you to insert your data in bulk at scheduled intervals. Key characteristics include:

  • Supports various file types (such as CSVs and other formats)
  • Data is processed and ingested in batches rather than continuously
  • Better for large volumes of historical or non-time-critical data
  • More efficient for processing larger datasets at once

Summary

The choice between streaming and batch ingestion depends on your use case:

  • Choose streaming when you need real-time data availability and immediate processing
  • Choose batch when dealing with large volumes of data or when real-time processing isn't critical

Both methods allow you to bring data from multiple sources into Experience Platform for analysis and use by downstream services.
