What is the difference between field based and graph based stitching?

Field-Based vs Graph-Based Stitching

Both are stitching methods in Customer Journey Analytics (CJA), but they differ significantly in approach, scope, and capabilities.

Field-Based Stitching

How it works:

  • Uses a single prop or eVar that you specify to contain a person identifier (e.g., hashed login ID, email)
  • Adobe reads the values in that field across hits and links them together based on matching IDs
  • Case-sensitive matching — "Bob" and "BOB" are treated as two different people
  • Operates within a single report suite or dataset

Best for:

  • Organizations with high user authentication/identification rates
  • Datasets where you already have a consistent person identifier captured
  • Simpler implementations with fewer identity types

Limitations:

  • Only one prop or eVar can be used for stitching
  • Cannot combine or concatenate multiple fields
  • Does not transform or normalize the ID (case-sensitive)
  • Limited to a single identifier type per field (not a mix of login IDs and emails)
  • Does not work across multiple datasets with different identifier schemes

Graph-Based Stitching

How it works:

  • Harnesses the Identity Graph from Adobe Experience Platform (AEP)
  • Uses a common person identifier to align identities across multiple Experience Platform applications
  • Allows multiple datasets to be joined in CJA using different identifier types
  • Profiles created in Adobe Real-Time CDP and Adobe Journey Optimizer can be aligned with people in CJA
  • More intelligent identity resolution across systems

Best for:

  • Organizations leveraging Adobe Experience Platform, Real-Time CDP, or Adobe Journey Optimizer
  • Complex multi-dataset scenarios where different identity types exist across systems
  • Need for cross-application identity alignment
  • Better coverage and accuracy across multiple data sources

Advantages:

  • Works across multiple datasets without manual ETL
  • Improves identity coverage by sharing identity information across datasets
  • Aligns CJA with CDP and Journey Optimizer profiles
  • Handles multiple identity types intelligently
  • More sophisticated identity resolution

Side-by-Side Comparison

| Feature | Field-Based | Graph-Based |

|---|---|---|

| Data source | Single report suite/dataset | Multiple datasets via AEP |

| Identifier | Single prop/eVar | Identity Graph with multiple types |

| Cross-application support | No | Yes (CDP, AJO, AEP) |

| Case sensitivity | Case-sensitive | No |

| Field concatenation | Not supported | Supported through Identity Graph |

| Multiple identity types | No | Yes |

| Setup complexity | Simpler | More complex (requires AEP) |

| Identity coverage | Limited to what you capture | Better coverage through graph |

| Best use case | High auth rate, simple needs | Multi-dataset, multi-app ecosystem |

Which Should You Use?

Choose Field-Based Stitching if:

  • You're in Adobe Analytics or CJA without AEP
  • You have a high authentication rate
  • Your identity needs are straightforward
  • You want simplicity and predictability

Choose Graph-Based Stitching if:

  • You're using Adobe Experience Platform, Real-Time CDP, or Journey Optimizer
  • You have multiple datasets with different identifier schemes
  • You need cross-application identity alignment
  • You want more sophisticated identity coverage and resolution
