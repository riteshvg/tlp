How do I set up a data element in Launch/Tags?

Setting Up a Data Element in Adobe Tags (Launch)

Data elements are Tags' version of a data layer — they store and reference values from query string parameters, cookies, JavaScript variables, DOM elements, local storage, and more. Once created, a data element acts as a reusable alias that rules and extensions can reference throughout your tag property.

▶ Watch: Create Tag Data Elements

Prerequisites

  • An existing Tag Property in Adobe Data Collection (Experience Platform UI)
  • Appropriate permissions to edit the property
  • Knowledge of where the source data lives (e.g., a data layer object, cookie name, query string key, etc.)

Steps to Create a Data Element

1. Navigate to your Tag Property in the Adobe Data Collection UI.

2. Click Data Elements in the left navigation bar.

3. Click the Create New Data Element button.

4. Name your data element (e.g., `Page Name`). Use a clear, descriptive name since this is how rules and extensions will reference it.

5. Select the appropriate Extension from the dropdown. The available Data Element Type options will populate based on the extension you choose (e.g., Core, Adobe Client Data Layer, etc.).

6. Choose the Data Element Type that matches your data source. Common types include:

  • JavaScript Variable — points to a value in your data layer (e.g., `digitalData.page.pageInfo.pageName`)
  • Cookie — reads a named cookie value
  • Query String Parameter — reads a URL parameter
  • Local Storage — reads a named local storage item
  • DOM Attribute — reads an attribute from a page element

7. Enter the source value for the data element type selected (e.g., the JavaScript variable path or cookie name).

8. Configure optional settings as needed:

  • Force Lowercase Value — standardizes casing
  • Clean Text — removes extraneous whitespace
  • Storage Duration — choose None, Session, or Visitor depending on how long the value should persist. If the value changes page-to-page, leave it as None.

9. Click Save to save the data element.

How Data Elements Are Used

Once created, data elements can be referenced in:

  • Rules — as trigger conditions (e.g., Data Element Change Event) or within rule actions
  • Extensions — passed as configuration values
  • Other data elements — composed together for derived values

> Key benefit: If your underlying data source ever changes (e.g., a site redesign renames a data layer variable), you only need to update the mapping once in the data element — all rules and extensions referencing it automatically use the new source value.
