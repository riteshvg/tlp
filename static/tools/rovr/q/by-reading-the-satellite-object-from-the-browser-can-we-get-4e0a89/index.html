By reading the satellite object from the browser, can we get full list of data elements and rules from the property?

Yes! The `_satellite` object exposed in the browser by the Adobe Tags (Launch) runtime library gives you access to the underlying data elements and rules configured in your published Tag property. Here's how:

Accessing Data Elements via `_satellite`

View All Data Element Definitions

You can inspect the internal data element registry directly in the browser console:

```javascript

_satellite.property.settings

```

or more specifically:

```javascript

Object.keys(_satellite._container.dataElements)

```

This returns an array of all data element names defined in the published library.

To see the full definition (type, settings, etc.) of all data elements:

```javascript

_satellite._container.dataElements

```

Each key is the data element name, and the value contains its configuration (extension, type, settings, etc.).

Retrieve a Specific Data Element's Value at Runtime

```javascript

_satellite.getVar('Your Data Element Name')

```

Accessing Rules via `_satellite`

To view all rules bundled in the published library:

```javascript

_satellite._container.rules

```

This returns an array of rule objects, each containing:

  • name — the rule name
  • conditions — the conditions configured on the rule
  • events — the events that trigger the rule
  • actions — the actions the rule executes

Important Caveats

| Consideration | Detail |

|---|---|

| Published library only | The `_satellite._container` object only reflects what has been built and published (or staged/dev, depending on the environment loaded). Unpublished changes are not visible. |

| `_satellite._container` is private | The leading underscore signals this is an internal/undocumented API. Adobe does not guarantee stability across library versions — it could change without notice. |

| Minification | In production builds, property values may be minified, making some settings harder to read. |

| No sensitive data exposure concern | Be aware that this means anyone visiting your site can see your rule logic and data element configurations, so avoid storing secrets in Tag configurations. |

Practical Summary

| What you want | Command |

|---|---|

| List all data element names | `Object.keys(_satellite._container.dataElements)` |

| Full data element definitions | `_satellite._container.dataElements` |

| Get a data element's current value | `_satellite.getVar('DE Name')` |

| List all rules | `_satellite._container.rules` |

| Property metadata | `_satellite._container.property` |

This is a very useful technique for debugging and auditing a deployed Tags property directly from the browser DevTools console.
