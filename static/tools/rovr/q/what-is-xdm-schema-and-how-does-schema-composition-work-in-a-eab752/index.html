XDM Schema and Schema Composition in Adobe Experience Platform

What is XDM Schema?

Experience Data Model (XDM) is a publicly documented specification designed to standardize and improve the power of digital experiences. It provides the foundational framework for Adobe Experience Platform.

Core Purpose

XDM schemas define the structure of data in a consistent and reusable way across Experience Platform. By defining data consistently across systems, it becomes easier to retain meaning and gain value from data.

Key benefits include:

  • Better data governance and data minimization (especially important for privacy regulations)
  • Out-of-the-box insights and use of AI/ML services with minimal customizations
  • Infrastructure for data sharing, insights, and efficient orchestration
  • Ability to deliver the right message to the right person, on the right channel, at exactly the right moment

Why Schemas Matter

Before data can be ingested into Experience Platform, a schema must be composed to:

  • Describe the data's structure
  • Provide constraints on the type of data that can be contained within each field
  • Ensure all customer experience data can be incorporated into a common representation
  • Enable data to be interpreted consistently, regardless of origin

How Schema Composition Works

Schemas consist of two main building blocks:

1. A base class — Defines the fundamental structure

2. Zero or more schema field groups — Add additional fields and capabilities

Key Schema Components

Data Types

Data types are reusable field components that represent multi-field objects. For example, a "Phone Number" data type might contain fields for country code, area code, and number.

Field Groups

Field groups are sets of fields that define specific concepts or functionality. When you add a field group to a schema, all the fields within that group are added to the schema.

Union Schemas

When a schema is enabled for use in Real-Time Customer Profile, it becomes part of a union. Union schemas combine the fields of all schemas within the same class into a single representation.

Planning Your Schema

Before building a schema, follow these steps:

1. Identify the concept — Determine the real-world object you're trying to capture (e.g., a person, a purchase, an event).

2. Think about your data — Consider:

  • What type of data you need to store
  • Which fields might serve as identity fields
  • How the schema may evolve in the future

3. Consider data behaviors — Understand how your data will behave within Experience Platform (streaming vs. batch, real-time vs. historical).

Standard XDM Components

Experience Platform provides standard XDM components — Adobe-built classes, field groups, and data types that are available out-of-the-box. These standardized components allow you to:

  • Leverage pre-built, tested structures
  • Ensure compatibility across Adobe solutions
  • Share data more easily across your organization
  • Reduce customization work

Schema-Based Workflows

The XDM System facilitates schema-based workflows and includes:

  • Schema Registry — Stores all your schemas
  • Schema Editor — Tool for building and managing schemas
  • Schema Metadata — Tracks schema versions, relationships, and usage
  • Service Consumption Patterns — How various Experience Platform services use schemas

By using standardized schemas, all data that conforms to XDM can be reused across your organization without conflicts—and even shared between multiple organizations.
