Where Do You Configure Custom SMS Payloads in AJO?

Overview

In Adobe Journey Optimizer, custom SMS payloads are configured through Custom Actions — which allow you to connect to third-party SMS providers (such as Twilio, Sinch, Infobip, etc.) via REST API calls with JSON-formatted payloads.

Prerequisites

Before configuring custom SMS payloads:

  • You must have a technical user role or Developer/Admin permissions
  • Your third-party SMS provider must support REST API with JSON payloads
  • You need the SMS provider's API endpoint URL, authentication credentials, and payload structure

Configuration Steps

Step 1: Access Custom Actions

1. Navigate to Administration in the left menu

2. Click on Configurations

3. Select Actions

4. Click Create Action to define a new custom action

Step 2: Define the Action Settings

1. Enter a Name for your custom SMS action

2. Add a Description (optional but recommended)

3. Select the Action type as Custom

4. Enter your SMS provider's API endpoint URL

Step 3: Configure Authentication

Choose the appropriate authentication method for your SMS provider:

  • No authentication – For open endpoints (not recommended)
  • Basic authentication – Username and password
  • Custom authentication – API keys or token-based authentication

Step 4: Configure the SMS Payload

1. Set the Request method (typically POST for SMS sends)

2. Set Content-Type header to `application/json`

3. Define the JSON payload structure for your SMS provider

Example payload structure:

```json

{

"to": "{{profile.mobilePhone.number}}",

"from": "YourSenderID",

"message": "{{message.body}}",

"channel": "SMS"

}

```

Step 5: Define Payload Parameters

In the payload editor:

  • Static fields – Fixed values that don't change per recipient
  • Dynamic fields – Personalization fields pulled from profile attributes or journey context
  • Mark fields as constant (fixed values) or variable (set by marketers at journey level)

Step 6: Validate and Save

1. Test the payload using sample data to ensure correct formatting

2. Review all field mappings

3. Click Save to store the custom action configuration

Using Custom SMS Action in a Journey

Once configured, the custom SMS action becomes available in your journey canvas:

1. Open your journey in the Journey Designer

2. In the left palette, locate the Action category

3. Drag and drop your custom SMS action onto the canvas

4. Configure the action parameters at the journey level:

  • Map dynamic fields to profile attributes
  • Set message content with personalization
  • Configure any additional required parameters

Supported Payload Features

| Feature | Supported |

|---|---|

| JSON payload | ✅ Yes |

| Scalar arrays | ✅ Yes |

| Dynamic personalization | ✅ Yes |

| Profile attributes | ✅ Yes |

| Contextual event data | ✅ Yes |

| Heterogeneous arrays | ❌ Not supported |

Supported Array Example (Scalar)

```json

"messageTypes": [

"promotional",

"transactional"

]

```

Not Supported Array Example (Heterogeneous)

```json

"dummyRandomArray": [

20,

"aafw",

false

]

```

Third-Party SMS Providers You Can Connect

Using custom actions, you can connect to any REST API-based SMS provider including:

  • Twilio
  • Sinch
  • Infobip
  • Epsilon
  • Firebase
  • Adobe Developer
  • Any other provider with a REST API endpoint

Key Considerations

> Important: Custom actions should only be used with private or internal endpoints and used with an appropriate capping or throttling limit to avoid overwhelming external SMS endpoints.

  • The configuration of custom actions must be performed by a technical user
  • Once configured, custom actions appear in the journey palette under the Action category
  • Marketers can then use these actions without needing to understand the underlying API configuration
  • Each custom action supports one API endpoint — create separate actions for different SMS operations if needed

Best Practices

✓ Use descriptive action names – Make it clear what SMS provider and operation the action performs

✓ Mark sensitive fields as constant – Keep API keys and sender IDs fixed at configuration level

✓ Expose only necessary fields – Give marketers access only to the fields they need to customize

✓ Test thoroughly – Validate payload structure with your SMS provider before going live

✓ Set appropriate capping limits – Protect your SMS provider endpoints from excessive calls

✓ Document payload structure – Maintain clear records of required fields and formats
