Overview

This end-to-end flow represents the complete customer experience lifecycle — from a customer interaction on a website, all the way through data processing, profile building, journey orchestration, and final message delivery back to the customer.

```

┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐

│ WEB SDK │──▶│ EDGE │──▶│ AEP │──▶│ PROFILE │──▶│ AJO │──▶│ CUSTOMER │

│(Browser) │ │ NETWORK │ │(Platform)│ │ STORE │ │(Journey) │ │(Channel) │

└──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘

5ms 100ms 1-5 sec Real-time Seconds Seconds-Min

```

The Real-World Scenario

To make this flow tangible, we'll use a real-world retail scenario throughout:

> Jane Smith visits an e-commerce website, browses running shoes, adds a pair to her cart but abandons without purchasing. AJO detects this behavior and sends her a personalized cart abandonment email with the exact shoes she viewed, followed by an SMS reminder if she doesn't open the email within 24 hours.

Stage 1: Web SDK (Browser)

What Happens

Jane visits the website. The Web SDK (alloy.js) is loaded and captures her interactions in real-time.

Web SDK Initialization

```javascript

// Web SDK configured via Adobe Tags or direct implementation

alloy("configure", {

datastreamId: "retail-prod-datastream-id",

orgId: "ADOBE_ORG_ID@AdobeOrg",

debugEnabled: false,

defaultConsent: "in",

onBeforeEventSend: function(options) {

// Enrich XDM with data layer values

options.xdm.commerce = window.dataLayer.commerce;

return options;

}

});

```

Event 5: Session Abandonment (Jane leaves without purchasing)
  • Jane closes the browser tab
  • No `commerce.purchases` event is fired
  • Cart abandonment is detected via absence of purchase event within a defined time window

What Web SDK Does With Each Event

```

Jane's Interaction

↓

Web SDK captures DOM event (click, page load, etc.)

↓

Tags rules fire (if using Adobe Tags)

↓

XDM payload constructed from data layer

↓

Identity map populated (ECID auto-managed by Web SDK)

↓

HTTPS POST to adobedc.net (Edge Network)

↓

Web SDK awaits response for personalization

```

Stage 2: Edge Network (Global Processing)

What Happens

Each Web SDK event hits the nearest Edge Network node (e.g., San Francisco for a US West Coast visitor). The Edge Network processes the request in ~100ms.

Edge Network Processing Flow

```

Jane's Web SDK Event Arrives

↓

┌─────────────────────────────────────────────────────┐

│ EDGE NETWORK PROCESSING │

│ │

│ ① Validate request & datastream ID │

│ ② Apply datastream configuration │

│ ③ Bot detection & filtering │

│ ④ Data Prep field transformations │

│ ⑤ Identity resolution (ECID lookup) │

│ ⑥ Edge profile lookup │

│ ⑦ Edge segmentation evaluation │

│ ⑧ Target/AJO decisioning (if applicable) │

│ ⑨ Route data to AEP Hub (async) │

│ ⑩ Return response to Web SDK │

└─────────────────────────────────────────────────────┘

```

Datastream Configuration

The datastream routes Jane's events to:

| Service | Configuration | Purpose |

|---|---|---|

| Adobe Experience Platform | Prod Sandbox, Event Dataset | Store events for profile building |

| Adobe Analytics | Report Suite: retail-prod | Real-time analytics reporting |

| Adobe Target | Property token: xyz | Real-time personalization |

| Adobe Journey Optimizer | AJO enabled | Journey triggering |

Data Prep Transformations

Edge Network transforms XDM fields:

```

XDM Field → Mapped To

────────────────────────────────────────────────────────

productListItems[0].SKU → Analytics eVar10

commerce.productListAdds.value → Analytics event2

web.webPageDetails.name → Analytics pageName

productListItems[0].priceTotal → Analytics currency event

identityMap.Email[0].id → Analytics visitorID

```

Identity Resolution at Edge

For Jane's login event:

```

Login event arrives with:

Email: jane.smith@email.com (authenticated)

ECID: 98765432109876543210 (ambiguous)

CRM_ID: CRM-JANE-12345 (authenticated)

Edge Network Identity Resolution:

1. Looks up ECID: 98765432109876543210 → Found in edge profile

2. Looks up Email: jane.smith@email.com → Found in profile store

3. Links all three identities → Single identity graph updated

4. Anonymous session data now linked to Jane's authenticated profile

```

Edge Segmentation

For Jane's "add to cart" event, Edge Network evaluates:

```

Edge-enabled segment: "Active Cart - No Purchase"

Criteria: productListAdds event in last 30 minutes

AND no commerce.purchases event

Result: Jane QUALIFIES → Audience membership updated in edge profile

```

Stage 3: Adobe Experience Platform (Data Processing)

What Happens

Data routed from Edge Network to AEP Hub undergoes comprehensive ingestion, storage, and processing.

3a. Streaming Ingestion

Jane's events flow from Edge Network to AEP:

```

Edge Network

↓ (Async, ~1 second latency)

AEP Streaming Ingestion API

↓

XDM Schema Validation

↓

Data Lake Storage (Raw Events)

↓

Identity Service Processing

↓

Real-Time Profile Update

```

3b. Identity Service Processing

Identity Service processes Jane's login event:

```

Before Login Event:

─────────────────

Identity Graph:

[ECID: 98765432109876543210] ← Standalone anonymous node

After Login Event:

─────────────────

Identity Graph:

[Email: jane.smith@email.com]

↕ linked

[ECID: 98765432109876543210]

↕ linked

[CRM_ID: CRM-JANE-12345]

Result: All three identities linked in single graph

Anonymous session data now associated with Jane

```

3c. Data Lake Storage

Jane's events stored in AEP datasets:

| Dataset | Records Stored | Query Example |

|---|---|---|

| Web Events Dataset | Page views, product views, cart events | All of Jane's session events |

| Profile Dataset | CRM attributes from batch ingestion | Jane's name, loyalty tier, preferences |

| Identity Dataset | Identity link records | Email ↔ ECID ↔ CRM_ID connections |

Query Jane's events in Query Service:

```sql

SELECT

timestamp,

eventType,

productListItems[0].SKU as sku,

productListItems[0].name as product_name,

commerce.productListAdds.value as added_to_cart,

commerce.purchases.value as purchased

FROM

web_events_dataset

WHERE

identityMap['Email'][0].id = 'jane.smith@email.com'

AND timestamp > NOW() - INTERVAL '24 hours'

ORDER BY

timestamp ASC

```

