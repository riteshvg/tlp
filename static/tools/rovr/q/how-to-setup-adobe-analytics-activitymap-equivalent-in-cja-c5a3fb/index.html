Setting Up an Activity Map Equivalent in Customer Journey Analytics (CJA)

Overview

Customer Journey Analytics does not have a native Activity Map overlay like Adobe Analytics does. However, you can replicate Activity Map-style link click analysis in CJA by leveraging the Activity Map context data variables that are already collected through your Adobe Analytics or Web SDK implementation, and then building visualizations in CJA Workspace.

How Activity Map Data Works

Activity Map data is captured via context data variables during your implementation:

  • `c.a.activitymap.page` — The page name when a link was clicked (the Activity Map Page dimension)
  • The data is typically sent on the next hit after a link was clicked

The Activity Map Page dimension collects the value from the Page dimension at the time of the click. If no Page dimension value exists, it falls back to the Page URL dimension.

Prerequisites

  • An existing Adobe Analytics or Web SDK implementation with Activity Map enabled
  • Activity Map data flowing into Adobe Experience Platform (AEP) via a configured datastream
  • A CJA Connection mapped to the relevant AEP dataset
  • A CJA Data View exposing the Activity Map-related dimensions and metrics

Steps to Enable and Analyze Activity Map Data in CJA

Step 1: Enable Activity Map in Your Implementation

If not already enabled, Activity Map must be turned on in your Adobe Analytics Tags extension:

▶ Watch: Enable, debug, and customize the Adobe Analytics Activity Map

1. In Adobe Experience Platform Tags, open your property.

2. Navigate to the Adobe Analytics extension.

3. Enable the Activity Map module within the extension settings.

4. Publish your library changes.

Step 2: Verify Context Data Variables Are Collected

Confirm that the following context data variables are being captured in your dataset:

  • `c.a.activitymap.page` → maps to the Activity Map Page dimension
  • Additional Activity Map variables (link name, region, etc.) if applicable

Step 3: Configure Your CJA Data View

1. Open Customer Journey Analytics and navigate to Data Views.

2. Edit the Data View associated with your Analytics dataset.

3. Add the Activity Map Page dimension (sourced from `c.a.activitymap.page`) as a component.

4. Add any other Activity Map dimensions (e.g., link name, link region) as needed.

5. Save the Data View.

Step 4: Build Link Click Analysis in CJA Workspace

1. Open a new or existing Workspace project in CJA.

2. Use a Freeform Table to analyze:

  • Activity Map Page — to see which pages had the most link clicks
  • Link Name / Link ID — to identify the most clicked links
  • Occurrences or Events — as your metric

3. Use Filters to isolate specific pages or link regions.

4. Use a Bar chart or Freeform table visualization to replicate the ranked link reporting that Activity Map provides.

Key Difference vs. Adobe Analytics Activity Map

| Capability | Adobe Analytics Activity Map | CJA Equivalent |

|---|---|---|

| Visual overlay on page | ✅ Yes | ❌ Not available |

| Link click dimension analysis | ✅ Yes | ✅ Via Activity Map Page dimension |

| Page-level link ranking | ✅ Yes | ✅ Via Freeform Table |

| Custom link region analysis | ✅ Yes | ✅ If region variable is collected |

> Note: The visual bubble overlay on your webpage is a feature exclusive to Adobe Analytics and is not replicated in CJA. CJA provides the underlying data analysis for link clicks but not the graphical page overlay experience.

Summary

To get an Activity Map equivalent in CJA:

1. Ensure Activity Map is enabled in your Tags implementation

2. Confirm context data variables are flowing into AEP

3. Expose the dimensions in your CJA Data View

4. Build Freeform Table analyses in CJA Workspace to rank pages and links by click activity
