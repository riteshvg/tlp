What is data validation agent and how does it work?

Data Validation with the Data Engineering Agent

The Data Validation skill is part of the Data Engineering Agent, an AI-powered copilot for data teams in Adobe Experience Platform. It helps you quickly profile and validate your datasets to ensure data quality before building reports and journeys.

What Data Validation Does

Data Validation enables you to:

  • Detect invalid values based on available metadata and actual data values
  • Understand field-level patterns including null counts, unique value counts, and frequency distributions
  • Leverage AI-powered insights to identify data quality issues before downstream use
  • Validate specific fields or entire datasets with automated quality assessments

Two Main Validation Types

Field Validation

Validates a specific field in a dataset and provides:

  • Null count and unique value count
  • Top unique values and their corresponding frequencies
  • AI-assisted semantic validation (detecting invalid values based on metadata and actual data)

Example prompts:

  • Validate the email field in the Customers_2024 dataset
  • Validate field status for the dataset customer_events_2024
  • Validate field person.address.city for Customer Data dataset

Dataset Validation

Validates up to five (5) fields in a dataset at once, summarizing overall quality and key issues. The agent can automatically determine the most relevant fields to analyze.

Example prompts:

  • Validate Customer Data 2024 dataset
  • Validate fields email, phone for Customers_2024
  • Summarize firstName, lastName, birthDate for Customer Data

How It Works

The Data Validation skill performs comprehensive assessments including:

  • Identifying anomalies in your data
  • Reporting valid/invalid record counts
  • Recommending data quality improvements such as type normalization, currency formatting, and ID deduplication
  • Providing AI-powered insights to improve data quality

This validation happens within your Customer Journey Analytics workflow, allowing you to troubleshoot data quality issues directly in your analysis processes without leaving your workflow.
