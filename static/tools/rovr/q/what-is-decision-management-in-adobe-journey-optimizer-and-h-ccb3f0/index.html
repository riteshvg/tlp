What is decision management in Adobe Journey Optimizer and how do I set up an offer library?

Decision Management in Adobe Journey Optimizer

What Is Decision Management?

Decision Management is an established feature in Adobe Journey Optimizer that empowers marketers to deliver the right personalized offers to customers at the perfect moment, across multiple channels. It combines two core components:

  • A centralized offer library — a single repository for managing all your offers, collections, and fallback options
  • A decision engine — applies rules and constraints to real-time customer profiles, leveraging Adobe Experience Platform data to select and deliver the right offer

Supported Channels

Decision Management supports the following channels:

| Channel | Supported |

|---|---|

| Email | ✓ |

| In-App Messaging | ✓ |

| Push Notifications | ✓ |

| SMS | ✓ |

| Direct Mail | ✓ |

> Note: Decision Management is labeled as Legacy in the AJO interface. Adobe's next-generation decisioning framework is called Decisioning, available via code-based experience and email channels. If you are starting fresh, consider evaluating Decisioning as the forward-looking capability.

Key Components of Decision Management

Before setting up your offer library, it helps to understand the building blocks:

| Component | Description |

|---|---|

| Placements | Locations where offers will appear (e.g., email banner, web hero image) |

| Decision Rules | Eligibility criteria that determine which customers qualify for which offers |

| Personalized Offers | Tailored offers targeted to specific customer segments |

| Fallback Offers | Default offers shown when no personalized offer qualifies |

| Collections | Organized groups of offers (static or dynamic) |

| Decisions | The logic container that brings placements, offers, and rules together |

Navigating the Decision Management Interface

The Decision Management section in the AJO left navigation rail contains two menus:

Offers Menu

| Menu Item | Purpose |

|---|---|

| Overview | Onboarding steps for new users; summary of recent offers and decisions for existing users |

| Offers | Create and manage personalized offers and fallback offers |

| Collections | Organize offers into static and dynamic collections |

| Decisions | Create and manage decisions to deliver offers |

| Batch Decisioning | Deliver offer decisions to all profiles in a given AEP audience |

| Simulation | Validate decisioning logic by simulating which offers are delivered to a test profile |

Components Menu

Used to create and manage the foundational components required to build offers and decisions, such as placements and decision rules.

Setting Up Your Offer Library

Prerequisites

Before creating offers, ensure you have:

  • Access to Adobe Journey Optimizer with appropriate permissions
  • At least one placement configured (defines where the offer will appear)
  • Decision rules defined if you need eligibility criteria
  • AEP audience or profile data available for targeting

Step 1: Create Placements

Placements define where and how your offers will be displayed.

1. In the AJO left navigation, go to Decision Management > Components

2. Select Placements

3. Click Create Placement

4. Define:

  • Name — a recognizable label
  • Channel type — e.g., Email, Web, Mobile
  • Content type — e.g., JSON, HTML, Image

5. Click Save

Step 2: Create Decision Rules (Eligibility Criteria)

Decision rules determine which customer profiles qualify for an offer.

1. Navigate to Decision Management > Components > Rules

2. Click Create Rule

3. Define your eligibility logic using profile attributes, events, or audience membership

4. Click Save

Step 3: Create Personalized Offers

1. Navigate to Decision Management > Offers

2. Click Create Offer and select Personalized Offer

3. Configure the offer:

  • Name — descriptive label
  • Start and end dates — offer validity window
  • Representations — define the offer content for each placement (e.g., image URL, text, HTML)
  • Eligibility — assign decision rules or audience criteria
  • Priority — numeric score used when multiple offers qualify (higher = more likely to be selected)
  • Capping — optionally limit how many times an offer can be shown overall or per profile

4. Click Save and Approve to make the offer available for use in decisions

Step 4: Create a Fallback Offer

A fallback offer is shown when no personalized offer qualifies for a given profile.

1. In Decision Management > Offers, click Create Offer

2. Select Fallback Offer

3. Provide:

  • Name
  • Representations for each relevant placement

4. Click Save and Approve

Step 5: Create Collections

Collections group offers together for easier management and use in decisions.

1. Navigate to Decision Management > Collections

2. Click Create Collection

3. Choose collection type:

  • Static — manually select specific offers
  • Dynamic — automatically includes offers matching a defined tag or filter

4. Name the collection and add your offers

5. Click Save

Step 6: Create a Decision

A decision contains the logic that selects the right offer for each customer.

1. Navigate to Decision Management > Decisions

2. Click Create Decision

3. Configure:

  • Name and description
  • Start and end dates
  • Decision scopes — combine a placement with a collection of offers
  • Ranking — define how offers are ranked (by priority score, AI ranking model, or custom formula)
  • Fallback offer — assign the fallback for when no offers qualify

4. Click Finish then Activate to make the decision live

Step 7: Validate with Simulation

Before going live, use the Simulation tool to test your logic:

1. Navigate to Decision Management > Simulation

2. Select a test profile

3. Select the decision to simulate

4. Review which offer would be delivered and why — confirming your eligibility rules and ranking are working as expected

Using Decisions in Campaigns and Journeys

Once your decision is active, you can reference it in:

  • AJO Email campaigns — insert offer content dynamically using the offer decision component in the email designer
  • AJO Journeys — add an offer decision action within a journey step
  • Batch Decisioning — run decisions against an entire AEP audience and export results

Decision Management vs. Decisioning (Next Generation)

| Capability | Decision Management (Legacy) | Decisioning (New) |

|---|---|---|

| Channels | Email, In-App, Push, SMS, Direct Mail | Code-based Experience, Email, Push, SMS |

| Item Catalog | Offer Library | Schema-based item catalog with custom metadata |

| Reusability | Limited | Updated decision policy and selection strategy configuration |

| Experimentation | Not native | Built-in experimentation capabilities |

| Status | Legacy (fully supported) | Strategic direction |

> If you are migrating from Decision Management to Decisioning, Adobe provides dedicated migration documentation and a Migration API guide to assist with the transition.
