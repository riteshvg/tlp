What is decision management in Adobe Journey Optimizer and how do I set up an offer library?

Decision Management in Adobe Journey Optimizer

What is Decision Management?

Decision Management in Adobe Journey Optimizer empowers marketers to deliver the right personalized offers to customers at the perfect moment, across multiple channels. It combines a centralized offer library with a powerful decision engine to tailor messages based on unique customer profiles and business rules.

> 💡 Note: Adobe Journey Optimizer now also offers Decisioning — a newer decisioning capability available via the code-based experience and email channels. Decision Management (covered here) is the legacy capability, still fully supported.

Core Components of Decision Management

Before setting up your offer library, understand the key building blocks:

| Component | Description |

|---|---|

| Placements | Locations or channels where offers are displayed (e.g., email hero image, web banner) |

| Decision Rules | Eligibility criteria that determine who qualifies for an offer |

| Tags/Collections Qualifiers | Labels used to organize and filter offers |

| Personalized Offers | Tailored offers shown to eligible customers |

| Fallback Offers | Default offers shown when no personalized offer qualifies |

| Collections | Organized groups of offers (static or dynamic) |

| Decisions | The logic container that selects the best offer for a given placement |

| Rankings | Priority rules or AI models that determine offer order |

Setting Up Your Offer Library: Step-by-Step

Prerequisites

  • Access to Adobe Journey Optimizer with Decision Management permissions
  • At least one audience segment defined in AEP
  • XDM schemas configured for profile attributes you want to use in eligibility rules

Step 1: Create Placements

Placements define where and how offers are rendered. Create at least one before building offers.

1. In the left rail, navigate to [!UICONTROL Components] → [!UICONTROL Placements]

2. Click [!UICONTROL Create Placement]

3. Configure the placement:

  • Name — e.g., "Email Hero Banner"
  • Channel type — Email, Web, Mobile, etc.
  • Content type — JSON, HTML, Image URL, Text

4. Click [!UICONTROL Save]

> ✅ Create separate placements for each channel and content format combination you intend to use.

Step 2: Create Decision Rules

Decision Rules define the eligibility criteria a customer must meet to receive an offer.

1. Navigate to [!UICONTROL Components] → [!UICONTROL Decision Rules]

2. Click [!UICONTROL Create Rule]

3. Enter a Name and optional Description

4. Use the expression editor to build your rule logic:

  • Profile attributes (e.g., `loyalty tier = Gold`)
  • Audience membership
  • Contextual data (e.g., device type, location)
  • Event-based conditions

5. Click [!UICONTROL Save]

> 💡 Tip: You can also pass context data directly in Decisioning API requests to create dynamic eligibility rules based on real-time conditions like device type, location, or user preferences.

Step 3: Create Personalized Offers

Personalized Offers are the tailored marketing messages delivered to eligible customers.

1. Navigate to [!UICONTROL Offers] → [!UICONTROL Offers]

2. Click [!UICONTROL Create Offer] → select [!UICONTROL Personalized Offer]

3. Configure the offer:

General settings:

  • Name — Descriptive internal name
  • Start and end dates — Offer validity window
  • Tags — Add collection qualifiers for organization

Representations:

  • Add one or more representations (one per placement)
  • For each representation, select the Placement and provide the content (HTML, JSON, image URL, or text)

Eligibility:

  • Select [!UICONTROL All visitors] for no restrictions, or
  • Choose a specific Audience segment, or
  • Apply a Decision Rule for attribute-based eligibility

Priority:

  • Set a numeric priority score (higher = more likely to be selected when multiple offers qualify)
  • Optionally apply an AI ranking model (requires Adobe Sensei configuration)

Capping:

  • Set frequency capping to limit how many times an offer is shown per customer
  • Configure total cap across all customers

4. Click [!UICONTROL Save and Approve] to make the offer available

Step 4: Create a Fallback Offer

Fallback Offers are shown when no personalized offer qualifies for a customer. Every decision requires one.

1. Navigate to [!UICONTROL Offers] → [!UICONTROL Offers]

2. Click [!UICONTROL Create Offer] → select [!UICONTROL Fallback Offer]

3. Configure:

  • Name
  • Representations — Add content for each relevant placement
  • Tags — Optional organizational labels

4. Click [!UICONTROL Save and Approve]

Step 5: Create Collections

Collections organize your offers into logical groups for use in decisions.

1. Navigate to [!UICONTROL Offers] → [!UICONTROL Collections]

2. Click [!UICONTROL Create Collection]

3. Choose collection type:

  • Static Collection — Manually select specific offers
  • Dynamic Collection — Automatically includes offers matching selected tags/qualifiers

4. Name the collection and configure its contents

5. Click [!UICONTROL Save]

Step 6: Create a Decision

A Decision ties everything together — it evaluates eligible offers from a collection against a placement and returns the best offer for each customer.

1. Navigate to [!UICONTROL Offers] → [!UICONTROL Decisions]

2. Click [!UICONTROL Create Decision]

3. Configure:

  • Name and Description
  • Start and end dates

4. Add Decision Scopes:

  • Select a Placement
  • Select a Collection of offers to evaluate
  • Configure Ranking — how offers are ranked (priority score, formula, or AI model)

5. Select a Fallback Offer for when no personalized offer qualifies

6. Click [!UICONTROL Finish] then [!UICONTROL Save and Approve]

Step 7: Validate with Simulation

Before going live, test your decisioning logic:

1. Navigate to [!UICONTROL Offers] → [!UICONTROL Simulation]

2. Select a test profile

3. Choose a Decision to simulate

4. Review which offers would be delivered to that profile and why

5. Adjust eligibility rules, priorities, or collections based on results

Delivering Offers via API

For advanced workflows, you can deliver offers programmatically:

1. Make a POST request to the Decisioning API with:

  • A unique placement ID
  • A unique decision ID
  • Optional context data for dynamic eligibility rules

2. The API returns the best offer for the given profile and placement

Delivering Offers via Batch Decisioning

For campaigns targeting large audiences:

1. Navigate to [!UICONTROL Offers] → [!UICONTROL Batch Decisioning]

2. Select the Decision and AEP Audience to target

3. Run the batch job — offer decisions are delivered to all profiles in the selected audience

Key Tips for a Successful Offer Library

✅ Plan your placements first — Every offer needs at least one representation tied to a placement

✅ Always create a fallback offer — Decisions cannot be activated without one

✅ Use tags consistently — They power dynamic collections and make offer management scalable

✅ Simulate before activating — Use the Simulation tool to validate logic before impacting real customers

✅ Set frequency caps — Prevent offer fatigue by limiting how often customers see the same offer

✅ Use context data — Pass real-time signals in API requests for more precise, dynamic personalization
