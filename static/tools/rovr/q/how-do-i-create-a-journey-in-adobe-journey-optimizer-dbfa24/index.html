How do I create a journey in Adobe Journey Optimizer?

Creating a Journey in Adobe Journey Optimizer

Prerequisites

Before building your journey, ensure you have:

  • Access to Adobe Journey Optimizer with appropriate permissions
  • At least one channel configuration set up (for email, SMS, push, etc.)
  • An audience segment or event defined to trigger the journey
  • XDM schemas and datasets configured in Adobe Experience Platform

Step-by-Step: Creating a Journey

Step 1: Access the Journey Canvas

1. Log in to Adobe Journey Optimizer

2. In the left navigation, select [!UICONTROL Journeys]

3. Click [!UICONTROL Create Journey] in the upper right corner

Step 2: Define Journey Properties

Before designing your journey, configure its core properties:

1. Enter a [!UICONTROL Name] for your journey

2. Add an optional [!UICONTROL Description]

3. Set the [!UICONTROL Start date and time] for when the journey becomes active

4. Set an [!UICONTROL End date and time] — profiles cannot enter the journey after this date

5. Configure the [!UICONTROL Timeout and error] duration for actions

6. Select a [!UICONTROL Timezone]

7. Enable [!UICONTROL Re-entrance] if you want profiles to re-enter the journey after completing it

Step 3: Choose Your Journey Type

Adobe Journey Optimizer supports four journey types. Select the right one based on your use case:

| Journey Type | Best For | Entry Mechanism |

|---|---|---|

| Unitary (Event-triggered) | Cart abandonment, welcome flows | Real-time event |

| Read Audience | Newsletters, promotional campaigns | Scheduled batch |

| Audience Qualification | VIP upgrades, re-engagement | Audience membership change |

| Business Event | Inventory alerts, flash sales | Business condition |

Step 4: Add an Entry Point

Drag your chosen entry activity onto the canvas from the [!UICONTROL Palette] on the left:

  • [!UICONTROL Event] — For event-triggered (unitary) journeys; select the event you have pre-configured
  • [!UICONTROL Read Audience] — For batch journeys; select your target audience segment
  • [!UICONTROL Audience Qualification] — For real-time audience membership responses

Step 5: Design the Journey Flow

With your entry point set, build out the journey by dragging activities from the palette:

Action Activities (deliver messages)
  • [!UICONTROL Email] — Send personalized email messages
  • [!UICONTROL SMS] — Send text messages
  • [!UICONTROL Push Notification] — Send mobile push messages
  • [!UICONTROL In-App] — Display in-app messages
  • [!UICONTROL Web] — Web personalization
  • [!UICONTROL Code-based Experience] — Custom developer-driven experiences
  • [!UICONTROL Content Card] — Persistent content surfaces
  • [!UICONTROL Custom Action] — Integrate with external systems via API
Orchestration Activities (control flow)
  • [!UICONTROL Condition] — Split the journey based on profile attributes, events, or scores
  • [!UICONTROL Wait] — Pause before the next step (fixed duration or until a specific date)
  • [!UICONTROL End] — Mark the end of a journey path
Data Activities
  • [!UICONTROL Read Audience] — Pull in audience membership data mid-journey

Step 6: Configure Each Activity

For each activity you add:

1. Click the activity on the canvas to open its properties panel on the right

2. For message actions: select or create the message content, configure personalization, and set delivery settings

3. For conditions: define the expression logic using the expression editor

4. For wait activities: set the duration or specify a date/time condition

Step 7: Test Your Journey

Before going live, validate your journey using Test Mode:

1. Toggle [!UICONTROL Test] in the upper right of the journey canvas

2. Select [!UICONTROL Trigger an event] or use test profiles to simulate entry

3. Follow the visual flow indicators to confirm profiles move through the journey as expected

4. Check for errors flagged in the canvas — resolve all errors before publishing

> ⚠️ Important: Test mode does not support custom upload audience attribute enrichment. Use standard test profiles for validation.

Step 8: Publish the Journey

Once testing is complete:

1. Click [!UICONTROL Publish] in the upper right corner

2. If your organization has configured an approval policy, submit the journey for approval first — explicit approval is required before publishing

3. Confirm publication — the journey status changes to [!UICONTROL Live]

4. Profiles will begin entering the journey based on your configured entry point and start date

Managing a Live Journey

Once published, keep these constraints in mind:

| Action | Available? |

|---|---|

| Edit journey activities | ❌ Journey becomes read-only |

| Create a new version | ✅ Create a new version to modify |

| Stop the journey permanently | ✅ Via the journey menu |

| Monitor performance | ✅ Via the journey dashboard |

  • When you publish a new version, the previous version enters [!UICONTROL Closed] status — no new profiles can enter the closed version, but profiles already in progress continue through it
  • The [!UICONTROL Overview] tab displays a dashboard with key metrics related to your journeys
  • The [!UICONTROL Browse] tab displays the list of all existing journeys

Tips for Success

✅ Start simple — Begin with a linear flow and add branching logic once the core journey is validated

✅ Use descriptive names — Name activities clearly so collaborators understand the flow at a glance

✅ Test thoroughly — Use multiple test profiles representing different customer scenarios

✅ Monitor after launch — Review performance metrics regularly and create new versions to optimize

✅ Plan for re-entrance — Decide upfront whether customers should be able to re-enter the journey
