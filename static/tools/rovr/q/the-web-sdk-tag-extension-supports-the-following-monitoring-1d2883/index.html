The Web SDK tag extension supports the following monitoring hooks:

Web SDK Tag Extension Monitoring Hooks

Available Monitoring Hooks

The Web SDK tag extension supports the following monitoring hooks that allow you to capture and monitor all SDK system events:

| Hook Event | Triggered When | Use Case |

|---|---|---|

| onBeforeConfigure | Before SDK configure command | Modify configuration before initialization |

| onAfterConfigure | After SDK configure command completes | Verify configuration was applied |

| onBeforeSendEvent | Before sendEvent is processed | Modify XDM data before sending |

| onAfterSendEvent | After sendEvent completes | Monitor event success/failure |

| onBeforeDatastreamConfiguration | Before datastream config applied | Inspect datastream settings |

| onAfterDatastreamConfiguration | After datastream config applied | Verify routing configuration |

| onSendEventResponse | When Edge Network response received | Inspect personalization decisions |

| onNetworkRequest | When network request made | Monitor all Edge Network calls |

| onNetworkResponse | When network response received | Inspect response data |

| onNetworkError | When network error occurs | Debug connection issues |

| onConsentChange | When consent preference changes | Track consent changes |

| onRequestStart | Request processing begins | Time request duration |

| onRequestComplete | Request processing ends | Calculate processing time |

Accessing Monitoring Hooks

Monitoring hooks are accessed via the global `__alloyMonitors` object:

```javascript

// Define monitoring hooks BEFORE Web SDK loads

window.__alloyMonitors = window.__alloyMonitors || [];

window.__alloyMonitors.push({

// Hook for sendEvent

onBeforeSendEvent: function(options) {

console.log("sendEvent called with:", options);

console.log("XDM Data:", options.xdm);

},

// Hook for network requests

onNetworkRequest: function(request) {

console.log("Network request:", request.url);

console.log("Request payload:", request.body);

},

// Hook for network responses

onNetworkResponse: function(response) {

console.log("Network response status:", response.status);

console.log("Response data:", response.body);

},

// Hook for errors

onNetworkError: function(error) {

console.error("Network error:", error.message);

}

});

```

Practical Monitoring Hook Examples

Example 1: Monitor All sendEvent Calls

```javascript

window.__alloyMonitors = window.__alloyMonitors || [];

window.__alloyMonitors.push({

onBeforeSendEvent: function(options) {

console.group("📤 Web SDK sendEvent");

console.log("Event Type:", options.xdm?.eventType);

console.log("Full Payload:", JSON.stringify(options, null, 2));

console.groupEnd();

// Track in custom analytics

if (window.myCustomAnalytics) {

myCustomAnalytics.trackEvent({

type: "web-sdk-event",

eventType: options.xdm?.eventType

});

}

},

onAfterSendEvent: function(result) {

console.log("✅ sendEvent succeeded:", result);

}

});

```

Example 2: Monitor Consent Changes

```javascript

window.__alloyMonitors = window.__alloyMonitors || [];

window.__alloyMonitors.push({

onConsentChange: function(consent) {

console.group("🔐 Consent Changed");

console.log("New Consent Value:", consent);

console.log("Timestamp:", new Date().toISOString());

console.groupEnd();

// Log to your CMP/consent tracking system

if (window.consentLogger) {

consentLogger.logConsentChange({

consentValue: consent,

timestamp: new Date().toISOString()

});

}

}

});

```

Example 3: Monitor Network Performance

```javascript

window.__alloyMonitors = window.__alloyMonitors || [];

window.__alloyMonitors.push({

onRequestStart: function(request) {

window._sdkRequestStart = performance.now();

},

onRequestComplete: function(response) {

var duration = performance.now() - window._sdkRequestStart;

console.log("🚀 Web SDK Request Duration:", duration + "ms");

// Alert if response slow

if (duration > 500) {

console.warn("⚠️ Slow Web SDK response:", duration + "ms");

}

},

onNetworkError: function(error) {

console.error("❌ Network Error:", error);

// Send to error tracking service

if (window.errorTracker) {

errorTracker.captureError({

type: "web-sdk-network-error",

error: error.message

});

}

}

});

```

Example 4: Monitor Personalization Decisions

```javascript

window.__alloyMonitors = window.__alloyMonitors || [];

window.__alloyMonitors.push({

onSendEventResponse: function(response) {

console.group("🎯 Personalization Decisions");

if (response.decisions && response.decisions.length > 0) {

response.decisions.forEach(function(decision, index) {

console.log(

"Decision " + (index + 1) + ":",

decision.id,

"Scope:",

decision.scope

);

});

} else {

console.log("No personalization decisions returned");

}

console.groupEnd();

}

});

```

Tool 6: Query Service Validation

After debugging Web SDK in the browser, validate data reached AEP correctly:

Querying Ingested Events

```sql

-- Check if events reached AEP

SELECT

timestamp,

eventType,

identityMap,

web.webPageDetails.name as page_name,

_id

FROM

web_events_dataset

WHERE

timestamp > NOW() - INTERVAL '1 hour'

AND eventType = 'commerce.productViews'

ORDER BY

timestamp DESC

LIMIT 10;

```

Expected Results:

```

timestamp eventType page_name _id

────────────────────────────────────────────────────────────────────────────

2024-01-15 10:05:23 commerce.productViews Product Detail abc123

2024-01-15 10:03:45 commerce.productViews Product Detail def456

2024-01-15 10:01:12 commerce.productViews Product Detail ghi789

```

Checking for Data Quality Issues

```sql

-- Find events missing required fields

SELECT

timestamp,

eventType,

CASE

WHEN productListItems IS NULL THEN 'Missing productListItems'

WHEN productListItems[0].SKU IS NULL THEN 'Missing SKU'

WHEN productListItems[0].name IS NULL THEN 'Missing product name'

WHEN productListItems[0].priceTotal IS NULL THEN 'Missing price'

ELSE 'Complete'

END as data_quality

FROM

web_events_dataset

WHERE

timestamp > NOW() - INTERVAL '24 hours'

AND eventType = 'commerce.productViews'

GROUP BY

data_quality

ORDER BY

COUNT(*) DESC;

```

Production Debugging Checklist

Use this checklist when debugging Web SDK implementations in production:

| Step | Tool | What to Check | Expected Result |

|---|---|---|---|

| 1. Verify Tags Load | Browser Console | Check embed code present | No 404 errors on script |

| 2. Check SDK Initialize | AEP Debugger | Web SDK tab shows events | configure event visible |

| 3. Monitor Events | AEP Assurance | Real-time event stream | Events appearing in real-time |

| 4. Inspect XDM Data | AEP Debugger | Expand event details | XDM
