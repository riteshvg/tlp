Overview

Adobe Experience Platform is the most powerful, flexible, and open system for building and managing complete solutions that drive customer experience. It enables organizations to:

  • Centralize and standardize customer data from any system
  • Apply data science and machine learning to improve experience delivery
  • Activate audiences to external destinations for personalization and engagement

AEP Architecture: The Big Picture

AEP architecture is organized into four major layers:

```

┌─────────────────────────────────────────────────────────┐
```

Layer 1: Data Sources

AEP ingests data from virtually any source system:

Online Sources

  • Web browsers – via Web SDK (alloy.js)
  • Mobile apps – via Mobile SDK (iOS/Android)
  • Single Page Applications – via Web SDK or API
  • IoT devices – via streaming APIs

Offline Sources

  • CRM systems – Salesforce, Microsoft Dynamics
  • ERP systems – SAP, Oracle
  • Data warehouses – Snowflake, BigQuery, Redshift
  • Marketing platforms – Marketo, Campaign
  • Point of Sale systems – Retail transaction data
  • Call center data – Customer service interactions

Partner Sources

  • Adobe Source Connectors – Pre-built connectors for 200+ sources
  • Streaming sources – Kafka, HTTP API
  • Batch sources – SFTP, S3, Azure Blob Storage

Layer 2: Data Collection Layer

Adobe Tags (Launch)

Tags is the tag management system that deploys and manages data collection code:

  • Web Properties – JavaScript tags for websites
  • Mobile Properties – SDK configuration for mobile apps
  • Extensions – Pre-built integrations (Web SDK, Analytics, Target, etc.)
  • Rules – Event-driven logic for data collection
  • Data Elements – Variables that capture data layer values

Web SDK (alloy.js)

The primary data collection library for web-based implementations:

  • Single library – Replaces AppMeasurement, AT.js, and Visitor API
  • Edge Network communication – Sends data directly to Edge Network
  • XDM data format – All data structured in XDM schema format
  • Real-time personalization – Supports Target and AJO decisioning
  • Identity management – Handles ECID and identity stitching

Mobile SDK

The mobile data collection framework for iOS and Android:

  • Modular architecture – Core + extension-based design
  • Edge Network integration – Direct communication with Edge Network
  • Lifecycle tracking – App launches, upgrades, crashes
  • Assurance integration – Real-time debugging capability

Edge Network

The globally distributed data processing layer:

  • Low latency – Processes data close to the user's location
  • Real-time processing – Events processed in milliseconds
  • Data routing – Routes data to appropriate AEP services
  • Personalization responses – Returns Target/AJO decisions in real-time
  • Identity resolution – Applies identity stitching at the edge

Datastreams

Configuration layer for the Edge Network:

  • Defines which AEP services receive data
  • Controls data routing to Analytics, Target, AJO, etc.
  • Manages data preparation and field mapping
  • Configures bot detection and filtering

Layer 3: Adobe Experience Platform Core

3a. Data Ingestion

AEP supports two primary ingestion methods:

| Method | Description | Use Case |

|---|---|---|

| Streaming Ingestion | Real-time data via HTTP API or SDK | Web events, mobile events, real-time CRM updates |

| Batch Ingestion | File-based data in scheduled batches | CRM exports, offline transactions, historical data |

3b. Data Lake

The central repository for all raw data in AEP:

  • Stores all ingested data regardless of format
  • XDM-structured – All data conforms to Experience Data Model
  • Queryable – Accessible via Query Service
  • Historical data retention – Configurable retention periods
  • Dataset management – Organized into datasets per data type

3c. Experience Data Model (XDM)

The standardized data model that powers AEP:

  • Schemas – Define the structure of data
  • Field Groups – Reusable collections of fields
  • Data Types – Custom field type definitions
  • Class-based – Profile, ExperienceEvent, Lookup, Summary classes
  • Extensible – Custom fields can be added to any schema

3d. Identity Service

Manages customer identity resolution across all data sources:

  • Identity Graph – Links multiple identities to a single customer
  • Namespace management – Organizes identities by type (Email, ECID, CRM ID, etc.)
  • Cross-device linking – Connects authenticated and anonymous identities
  • Graph Simulation – Test identity linking rules safely
  • Real-time resolution – Identity graphs updated as new data arrives

3e. Real-Time Customer Profile

Creates a unified, real-time view of each customer:

  • Profile store – Optimized database for fast profile lookups
  • Merge policies – Rules for combining data from multiple datasets
  • Profile attributes – Current state of customer data
  • Experience events – Time-series behavioral data
  • Computed attributes – Pre-calculated values based on event history
  • Real-time updates – Profile updated within seconds of new data

3f. Segmentation Service

Builds audience segments from profile data:

| Segmentation Type | Description | Latency |

|---|---|---|

| Batch Segmentation | Scheduled evaluation of all profiles | Daily |

| Streaming Segmentation | Real-time evaluation as events arrive | Seconds |

| Edge Segmentation | On-device evaluation at Edge Network | Milliseconds |

3g. Query Service

SQL-based analytics on all AEP data:

  • Standard SQL – Query data lake using familiar SQL syntax
  • Dataset exploration – Explore and validate ingested data
  • Data quality checks – Identify missing or incorrect data
  • Audience creation – Build audiences using SQL queries
  • BI tool integration – Connect Tableau, Power BI, Looker via PSQL

3h. Data Governance

Privacy and compliance management across all data:

  • Data Usage Labels – Tag data with usage restrictions
  • Data Usage Policies – Define rules for how data can be used
  • Policy enforcement – Automatically block policy violations
  • Consent management – Respect customer consent preferences
  • Audit Logs – Track all data access and modifications

3i. AI/ML Services

Intelligent insights powered by machine learning:

| Service | Description |

|---|---|

| Attribution AI | Multi-touch attribution modeling |

| Customer AI | Propensity scoring (churn, conversion) |

| Content AI | Automated content tagging and similarity |

Layer 4: Activation Layer

Adobe Journey Optimizer (AJO)

Orchestrates customer journeys across all channels:

  • Journey Designer – Visual drag-and-drop journey canvas
  • Campaigns – Scheduled and API-triggered messages
  • Decision Management – AI-powered offer selection
  • Channels – Email, SMS, Push, In-App, Web, Direct Mail
  • Real-time triggering – Journeys triggered by AEP events

Adobe Real-Time CDP (RTCDP)

Activates audiences to external destinations:

  • Destinations catalog – 200+ pre-built destination connectors
  • Audience activation – Send segments to ad platforms, CRMs, etc.
  • Real-time activation – Stream audiences as profiles qualify
  • B2B Edition – Account-based marketing capabilities

Adobe Target

Delivers personalization on web and mobile:

  • A/B Testing – Experiment with different experiences
  • Multivariate Testing – Test multiple elements simultaneously
  • Automated Personalization – AI-powered content delivery
  • AEP integration – Uses AEP segments for targeting

Adobe Analytics

Analyzes customer behavior across digital touchpoints:

  • Real-time reporting – Live visitor data
  • Analysis Workspace – Drag-and-drop analytics interface
  • AEP data sharing – Analytics data flows into AEP datasets
  • CJA integration – Cross-channel analysis with AEP data

Destinations

Exports data and audiences to external systems:

| Destination Type | Examples |

|---|---|

| Advertising | Google, Facebook, LinkedIn, The Trade Desk |

| Email Marketing | Mailchimp, Salesforce Marketing Cloud |

| CRM | Salesforce, Microsoft Dynamics |

| Data Warehouses | Snowflake, BigQuery, Azure Data Lake |

| Streaming | Kafka, HTTP API |

End-to-End Data Flow

Here is the complete data flow through AEP architecture:

```

1. CUSTOMER INTERACTION

Customer visits website / uses mobile app / makes purchase

2. DATA COLLECTION

Web SDK / Mobile SDK captures interaction

Tags rules fire and send XDM-formatted events

Edge Network receives and routes the data

3. DATA INGESTION

Streaming: Events arrive in AEP in real-time (<1 second)

Batch: Offline data arrives in scheduled batches

4. DATA PROCESSING

Data Lake stores raw event data

Identity Service links identities in the identity graph

Real-Time Profile is updated with new attributes and events

Segmentation evaluates profile against audience criteria

5. INTELLIGENCE

AI/ML services calculate propensity scores

Computed attributes aggregate behavioral data

Insights generated for marketing decisions

6. ACTIVATION

AJO triggers journeys based on profile events

RTCDP activates audiences to ad platforms

Target delivers personalized web experiences

Analytics reports on customer behavior

```

Key Architectural Principles

✓ Open and API-first – Every AEP capability is accessible via RESTful APIs

✓ Real-time processing – Data is processed and profiles updated in seconds

✓ Standardized data model – XDM ensures consistency across all data sources

✓ Privacy by design – Data governance and consent built into the foundation

✓ Scalable infrastructure – Cloud-native architecture scales to any data volume

✓ Interoperable – Integrates with any Adobe or third-party solution

✓ Intelligent – AI/ML services embedded throughout the platform
