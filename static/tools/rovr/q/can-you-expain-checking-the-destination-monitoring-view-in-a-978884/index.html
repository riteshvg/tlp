Destination Monitoring in AEP — Complete Guide

The Destination Monitoring view in AEP gives you full visibility into the health and status of your audience activation flows — helping you confirm successful exports and troubleshoot any issues.

How to Access the Monitoring Dashboard

Prerequisites

  • You need View Destinations access control permission
  • You must have at least one active destination connection with audiences activated

Steps to Navigate

1. Log in to Adobe Experience Platform

2. In the left navigation panel, select Monitoring (the monitoring icon)

3. On the Monitoring page, select Destinations from the top menu

4. The Destinations Monitoring Dashboard will load

What the Monitoring Dashboard Shows

The dashboard provides two levels of visibility:

1. Aggregated Overview (Top Level)

At the top level, you get an overall picture of all your destination flows:

  • All batch destinations — summary metrics
  • All streaming destinations — summary metrics
  • Overall health status of activation flows

2. Drill-Down Views (Detailed Level)

You can drill deeper into:

| Level | What You See |

|---|---|

| Dataflow level | Status, audience count, identities processed |

| Dataflow run level | Individual run timestamps, success/failure status |

| Activated audience level | Per-audience export metrics and status |

Key Metrics Available in the Dashboard

The monitoring dashboard surfaces the following metrics for each activation run:

| Metric | Description |

|---|---|

| Identities activated | Number of identities successfully exported |

| Identities excluded | Identities skipped due to missing attributes or consent violations |

| Identities failed | Identities that failed to export |

| Activation rate | Percentage of profiles successfully activated |

| Last dataflow run | Timestamp of the most recent export attempt |

| Dataflow run status | Success, Failed, or Processing |

Filtering Options

You can filter the monitoring dashboard by:

  • Data type:
  • Customers — standard customer profile data
  • Accounts — for Real-Time CDP B2B edition only
  • Prospects — prospect profile data
  • Account enrichment — B2B account enrichment data

Streaming vs Batch Destinations — What to Expect

Streaming Destinations (e.g., Facebook Custom Audiences)

  • Dataflow runs appear near real-time
  • You can see individual streaming runs with timestamps
  • Each run shows identities processed per micro-batch

Batch Destinations (e.g., cloud storage, email platforms)

  • Dataflow runs appear based on your configured schedule (daily, weekly, etc.)
  • Each run shows the full file export status

Setting Up Alerts for Destination Monitoring

You can also subscribe to alerts so you are notified proactively when something goes wrong:

1. Navigate to your destination connection

2. In the destination details, locate the Alerts section

3. Select an alert type from the list:

  • Destination flow run succeeded
  • Destination flow run failed

4. Click Subscribe to start receiving notifications on the status of your dataflow

Important Note on Adobe Target

> ⚠️ Destination monitoring is currently supported for ALL destinations in Experience Platform EXCEPT the Adobe Target destination. If you are using Adobe Target, the monitoring dashboard will not show data for that specific destination.

Quick Summary — What to Check for Facebook Export Confirmation

When validating a Facebook audience export specifically, follow this sequence in the Monitoring dashboard:

1. ✅ Check dataflow run status — should show Success

2. ✅ Check identities activated count — should match your expected audience size

3. ✅ Check identities failed — should be 0 or minimal

4. ✅ Check last dataflow run timestamp — confirms when AEP last pushed data

5. ✅ Check identities excluded — review if count is unexpectedly high (may indicate mapping or consent issues)

Once AEP confirms a successful export on its side, any remaining delay is on Facebook's processing side.
