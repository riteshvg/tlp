How does marketing channel work in Customer Journey Analytics?

Marketing Channels in Customer Journey Analytics

Marketing channels in Customer Journey Analytics (CJA) work differently from traditional Adobe Analytics — there are no built-in channel groupings. Instead, you have two primary approaches to implement and report on marketing channel dimensions:

Approach 1: Derived Fields (Recommended Native Method)

CJA provides a Marketing Channels function template within Derived Fields that lets you define your own channel groupings directly inside a Data View.

How it works:

  • Navigate to your Data View and create a new Derived Field
  • Use the built-in Marketing Channels function template
  • Define rules that categorize how customers arrive on your site (e.g., paid search, email, organic, direct, etc.)
  • The derived field then becomes available as a dimension in Analysis Workspace

This approach gives you full flexibility to define channel logic based on any XDM field available in your datasets — including referrer, UTM parameters, tracking codes, and more.

> Note on UTM parameters (e.g., `utm_source`, `utm_medium`): These are not collected automatically. Each must be mapped to an XDM field during implementation before they can be used in channel rules. If campaign values arrive as a tracking code, use the Tracking Code dimension.

Approach 2: Analytics Source Connector (For Existing Adobe Analytics Customers)

If your organization uses the Analytics source connector to bring Adobe Analytics report suite data into CJA, you can carry over your existing Marketing Channel processing rules from Adobe Analytics into CJA.

How it works:

  • Configure a connection in CJA pointing to the dataset created by the Analytics source connector
  • The Marketing Channel dimensions (e.g., Last Touch Channel, First Touch Channel) processed by Adobe Analytics are brought in as fields
  • These dimensions are then available to use in your Data View and subsequently in Analysis Workspace

Key considerations:

  • The channel values are pre-processed by Adobe Analytics's Marketing Channel processing rules — they are not re-evaluated in CJA
  • Related dimensions available include Referring Domain and Referrer Type for source/medium analysis

Key Differences from Adobe Analytics

| Feature | Adobe Analytics | Customer Journey Analytics |

|---|---|---|

| Built-in channel groupings | Yes | No — must be defined manually |

| Processing rules | Defined at report suite level | Defined as derived fields in Data Views |

| UTM parameters | Partially auto-collected | Must be mapped to XDM fields at implementation |

| Cross-channel data | Limited | Combine online + offline data sources |

Reporting on Marketing Channels in Analysis Workspace

Once your marketing channel derived field (or Analytics source connector dimensions) is configured, you can use it in Analysis Workspace just like any other dimension — in freeform tables, flow visualizations, attribution panels, etc. CJA's flexible attribution models also allow you to apply first touch, last touch, or custom attribution to your channel dimension, going beyond the fixed models in traditional Analytics.
