How can i migrate components and projects from AA to CJA?

Migrating Components and Projects from Adobe Analytics to Customer Journey Analytics

Prerequisites

Before starting the migration, ensure the following are in place:

1. Data is flowing into AEP — via either the Analytics Source Connector or the Web SDK

2. A CJA Data View is configured — mapped to the dataset containing your Analytics data

3. Adobe Analytics Administrator access — only admins can perform migrations

4. Complete an Analytics Inventory — use the inventory tool available to Analytics admins to audit your existing components and projects

What Gets Migrated?

| Component | Auto-Mapped | Manually Mapped | Notes |

|---|---|---|---|

| Dimensions | Some | Some | Requires mapping to CJA data view |

| Metrics | Some | Some | Requires mapping to CJA data view |

| Segments | ✅ Yes | ❌ Not required | Migrated automatically |

| Calculated Metrics | ✅ Yes | ❌ Not required | Migrated with project |

| Date Ranges | ✅ Yes | ❌ Not required | Migrated with project |

| Analysis Workspace Projects | ✅ Yes | ❌ Not required | Re-created in CJA |

Migration Methods

There are two ways to perform the migration:

Option 1: UI-Based Migration (Recommended for most users)

Option 2: API-Based Migration (For advanced users or large-scale migrations)

Option 1: UI-Based Migration — Step by Step

Step 1 — Map Dimensions and Metrics

Before migrating projects, you must map your Adobe Analytics dimensions and metrics to their CJA equivalents in your data view.

1. Navigate to Adobe Analytics > Admin > Component Migration

2. Select the report suite you want to migrate

3. Review the list of dimensions and metrics:

  • Auto-mapped components are already matched — review for accuracy
  • Manually map any unmatched dimensions and metrics by selecting the corresponding CJA data view component from the drop-down

4. Once all mappings are complete, save your mappings

Step 2 — Migrate Projects

1. In Adobe Analytics, go to Admin > Component Migration

2. Select the project(s) you want to migrate

3. Select the target CJA Data View to map the project to

4. Confirm the migration — the tool will:

  • Re-create the Workspace project in CJA
  • Migrate associated segments, calculated metrics, and date ranges
  • Apply your previously saved dimension/metric mappings

Step 3 — Review Migration Summary

  • Once migration completes, a migration summary is displayed
  • All successfully migrated components are listed
  • If any components failed to migrate, they will be flagged:

  • Address any failed components manually in CJA

Option 2: API-Based Migration

For large-scale or automated migrations, you can use the Adobe Analytics 2.0 APIs. You will need the following identifiers:

  • Adobe Analytics Project ID
  • Report Suite ID
  • CJA Data View ID

API Migration Order

> ⚠️ Important: If you plan to migrate both dimensions/metrics AND projects, always use the mappings APIs first so the project ID is associated with those mappings before migrating the project.

1. Use the Mappings API — to map dimensions and metrics from your report suite to CJA data view components

2. Use the Migration API — to migrate the project and its associated components (segments, calculated metrics, date ranges)

Key Considerations

  • Segments do not need to be manually mapped — they are migrated automatically as part of the project migration
  • Web SDK implementations require all schema fields to be manually mapped, as there is no automatic field mapping
  • Analytics Source Connector implementations benefit from more automatic mappings out of the box
  • All migrated components are visible in CJA under Components > the respective component type (Segments, Calculated Metrics, Date Ranges)
