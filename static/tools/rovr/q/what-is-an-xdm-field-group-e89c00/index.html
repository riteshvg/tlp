What is an XDM Field Group?

An XDM field group (also called a schema field group) is a reusable component in Adobe Experience Platform that defines a collection of related fields. Think of it as a template or building block for XDM schemas.

Key Characteristics

  • Reusable: A single field group can be applied to multiple schemas, promoting consistency across your data structures
  • Modular: Field groups contain logically grouped fields that represent related data concepts
  • Standardized: XDM provides pre-built field groups for common business use cases, or you can create custom ones
  • Composable: You combine multiple field groups together with an XDM class to create a complete schema

How Field Groups Work

Field groups are added to schemas that are based on XDM classes (like the XDM Individual Profile class). The class defines the base structure, and field groups add specialized fields on top of it.

Examples of Field Groups

Common Adobe-provided field groups include:

  • IdentityMap – Manages user identities and identity resolution
  • XDM Business Person Details – Business-specific profile information
  • XDM Business Person Components – Additional business person attributes

Custom vs. Standard Field Groups

  • Standard/Pre-built: Adobe provides tested field groups for common scenarios
  • Custom: You can create your own field groups tailored to your organization's specific data needs

Field groups are essential for structuring how data flows into AEP and ensuring consistency across your schemas and datasets.
