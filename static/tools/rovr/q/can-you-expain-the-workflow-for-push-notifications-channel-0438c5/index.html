Push Notification Channel Workflow in Adobe Journey Optimizer

Overview

The Push Notification channel in AJO enables you to send targeted, personalized messages directly to customers' mobile devices — even when they are not actively using your app. Unlike SMS, push notifications require a mobile app with Adobe Mobile SDK integration.

▶ Watch: Push Notifications Overview

↓

STEP 2: CONFIGURE PUSH CREDENTIALS (APNs & FCM)

↓

↓

STEP 4: CREATE DATASTREAM & TAG PROPERTY

↓

↓

↓

Understanding Push Notification Architecture

Before diving into configuration, it is important to understand how push notifications flow from AJO to a customer's device.

```

PUSH NOTIFICATION FLOW:

AJO sends push payload

↓

Adobe Experience Edge Network

↓

Push Service Gateway

├── APNs (Apple Push Notification Service) → iOS Devices

└── FCM (Firebase Cloud Messaging) → Android Devices

↓

Customer's Mobile Device

↓

Push Notification Displayed

```

Key Infrastructure Components

| Component | Purpose | Platform |

|---|---|---|

| APNs | Apple's push delivery service | iOS, iPadOS, macOS, watchOS |

| FCM | Google's push delivery service | Android, Chrome |

| Adobe Mobile SDK | Collects device tokens, sends events to AEP | iOS & Android |

| Datastream | Routes data from SDK to AEP/AJO | Both platforms |

| App Surface | Registers app with push credentials in AJO | Both platforms |

STEP 1: Set Up Mobile App with Adobe Mobile SDK

Why is this needed?

The Adobe Mobile SDK is the bridge between your mobile app and Adobe Experience Platform. It:

  • ✅ Collects and registers device push tokens
  • ✅ Sends behavioral events to AEP (app opens, purchases, etc.)
  • ✅ Enables in-app messaging alongside push
  • ✅ Tracks push notification interactions (opens, dismissals)
  • ✅ Handles foreground/background notification delivery

Prerequisites for Mobile App Setup

```

Mobile App Prerequisites:

├── iOS App → Xcode project with iOS 14+ target ✅

├── Android App → Android Studio project with API level 21+ ✅

├── Apple Developer Account (for APNs certificates) ✅

├── Google Firebase Project (for FCM credentials) ✅

├── AEP Datastream configured ✅

└── AEP Tag Property configured ✅

```

```

Step 2: Initialize SDK in AppDelegate

```swift

import AEPCore

import AEPLifecycle

import AEPIdentity

import AEPEdge

import AEPEdgeIdentity

import AEPMessaging

import UserNotifications

@main

class AppDelegate: UIResponder, UIApplicationDelegate {

func application(

_ application: UIApplication,

didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?

) -> Bool {

// Register Adobe Extensions

MobileCore.registerExtensions([

Lifecycle.self,

Identity.self,

Edge.self,

AEPEdgeIdentity.Identity.self,

Messaging.self // Required for Push Notifications

]) {

// Set your Datastream ID (from AEP Data Collection)

MobileCore.configureWith(appId: "YOUR_DATASTREAM_ID")

// Set Environment (DEV/STAGING/PROD)

MobileCore.updateConfigurationWith(configDict: [

"global.privacy": "optedin"

])

}

// Request Push Notification Permission

UNUserNotificationCenter.current().requestAuthorization(

options: [.alert, .badge, .sound]

) { granted, error in

if granted {

DispatchQueue.main.async {

application.registerForRemoteNotifications()

}

}

}

return true

}

// Register Device Token with Adobe SDK

func application(

_ application: UIApplication,

didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data

) {

MobileCore.setPushIdentifier(deviceToken)

// This sends the device token to AEP for push targeting

}

// Handle Push Notification Receipt

func application(

_ application: UIApplication,

didReceiveRemoteNotification userInfo: [AnyHashable: Any],

fetchCompletionHandler completionHandler: @escaping (UIBackgroundFetchResult) -> Void

) {

Messaging.handleNotificationResponse(...)

completionHandler(.newData)

}

}

```

Step 3: Handle FCM Token Registration

```kotlin

class MyFirebaseMessagingService : FirebaseMessagingService() {

// Called when FCM token is generated or refreshed

override fun onNewToken(token: String) {

super.onNewToken(token)

// Register FCM token with Adobe SDK

MobileCore.setPushIdentifier(token)

// This sends the FCM token to AEP for push targeting

}

// Handle incoming push messages when app is in foreground

override fun onMessageReceived(remoteMessage: RemoteMessage) {

super.onMessageReceived(remoteMessage)

STEP 2: Configure Push Credentials (APNs & FCM)

iOS: Apple Push Notification Service (APNs)

You need APNs credentials to send push notifications to iOS devices.

Option A: APNs Authentication Key (Recommended)

```

APNs Key Setup in Apple Developer Portal:

1. Go to developer.apple.com

2. Navigate to Certificates, IDs & Profiles

3. Click Keys → + (Add Key)

4. Enable Apple Push Notifications service (APNs)

5. Click Continue → Register

6. Download the .p8 key file (SAVE THIS - you can only download once)

7. Note the Key ID (10-character string)

8. Note your Team ID (from Membership section)

You will need:

├── .p8 Key File ← Authentication Key

├── Key ID ← 10-character identifier

├── Team ID ← Your Apple Developer Team ID

└── Bundle ID ← Your app's bundle identifier

(e.g., com.yourbrand.app)

```

Option B: APNs Certificate (Legacy)

```

APNs Certificate Setup:

1. Go to developer.apple.com

2. Navigate to Certificates, IDs & Profiles

3. Click Certificates → + (Add Certificate)

4. Select Apple Push Notification service SSL

5. Select your App ID

6. Upload a Certificate Signing Request (CSR)

7. Download the generated .p12 certificate

8. Set a certificate password

You will need:

├── .p12 Certificate File

├── Certificate Password

└── Certificate Type (Development or Production)

```

Android: Firebase Cloud Messaging (FCM)

You need FCM credentials to send push notifications to Android devices.

```

FCM Setup in Firebase Console:

1. Go to console.firebase.google.com

2. Create a Firebase Project (or use existing)

3. Add your Android app to the project:

├── Enter Android package name (e.g., com.yourbrand.app)

├── Download google-services.json

└── Add to your Android project root

4. Navigate to Project Settings → Cloud Messaging

5. Get FCM Credentials:

├── Option A: Server Key (Legacy - deprecated but still works)

└── Option B: Service Account JSON (Recommended - V1 API)

For Service Account (V1 API - Recommended):

1. Go to Project Settings → Service Accounts

2. Click Generate new private key

3. Download the JSON file

4. This JSON contains:

├── Project ID

├── Private Key ID

├── Private Key

├── Client Email

└── Client ID

```

STEP 3: Create App Surface (App Configuration)

What is an App Surface?

An App Surface is the configuration in AJO that registers your mobile app with AJO and links it to your APNs and FCM credentials. It is the bridge between your mobile app and AJO's push sending infrastructure.

```

App Surface connects:

Mobile App (iOS/Android)

↓

APNs/FCM Credentials (Push delivery)

↓

AJO Sending Infrastructure

```

How to Create an App Surface

Prerequisites:

  • APNs credentials (for iOS) ✅
  • FCM credentials (for Android) ✅
  • System Administrator role ✅

Steps:

1. In AJO left navigation, go to Administration → Channels → App Surfaces

> ⚠️ Note: In newer versions of AJO, App Surfaces have been moved to Adobe Experience Platform Data Collection → App Configurations

2. Click Create App Surface

3. Enter a Name (e.g., "Luma Mobile App - Production")

4. Enter your App Bundle ID (iOS) and/or App Package Name (Android)

iOS APNs Configuration

```

iOS Push Credentials:

App ID (Bundle ID): com.yourbrand.app

APNs Authentication:

├── Type: Authentication Key (recommended)

│ ├── Key File: [Upload .p8 file]

│ ├── Key ID: XXXXXXXXXX

│ ├── Team ID: XXXXXXXXXX

│ └── Environment:

│ ├── Sandbox (Development/Testing)

│ └── Production (Live App)

│

└── Type: Certificate (legacy)

├── Certificate File: [Upload .p12 file]

├── Certificate Password: [Enter password]

└── Certificate Type: Production/Development

```

Android FCM Configuration

```

Android Push Credentials:

App Package Name: com.yourbrand.app

FCM Authentication:

├── Type: Service Account JSON (V1 API - Recommended)

│ └── Service Account JSON: [Upload JSON file]

│

└── Type: Server Key (Legacy)

```

5. Click Save

6. App Surface is created and shows ✅ Active

STEP 4: Create Datastream & Tag Property

What is a Datastream?

A Datastream is the server-side configuration that tells Adobe's Edge Network where to route data from your mobile app.

```

Mobile App (Adobe Mobile SDK)

↓

Adobe Edge Network

↓

Datastream (routing rules)

├── Route to AEP → Profile updates, event data

├── Route to AJO → Push token registration, journey triggers

└── Route to Analytics → Behavioral tracking

```

How to Create a Datastream

1. Go to Adobe Experience Platform Data Collection (launch.adobe.com)

2. Click Datastreams in the left navigation

3. Click New Datastream

4. Enter:

```

Datastream Settings:

├── Name: "Luma Mobile App - Production"

├── Description: "Datastream for Luma iOS and Android apps"

└── Event Schema: Select your XDM Mobile Schema

```

5. Click Save

6. Add Services to the datastream:

```

Services to Enable:

├── Adobe Experience Platform

│ ├── Sandbox: [Your sandbox]

│ ├── Event Dataset: [Your mobile events dataset]

│ └── Profile Dataset: [Your profile dataset]

│

└── Adobe Journey Optimizer

└── Dataset: [Your AJO dataset for push]

```

7. Click Save

How to Create a Tag Property (Mobile)

1. In Adobe Data Collection, click Tags → Properties

2. Click New Property

3. Select Mobile as platform

4. Enter:

```

Property Settings:

├── Name: "Luma Mobile App"

└── Platform: Mobile

```

5. Click Save

7. Install required extensions:

```

Required Extensions:

├── Mobile Core (installed by default)

├── Adobe Experience Platform Edge Network

│ └── Configure: Select your Datastream

```
