How do I configure frequency capping and suppression rules in Adobe Journey Optimizer?

Configuring Frequency Capping and Suppression Rules in Adobe Journey Optimizer

Frequency Capping with Business Rules (Rule Sets)

Frequency capping controls how often customers receive communications, preventing over-solicitation and managing customer fatigue. Rules are grouped into rule sets and applied to campaigns or journeys.

▶ Watch: Work with Rule Sets

Step 1: Create a Rule Set

1. Navigate to Conflict Management & Prioritization in the left rail.

2. Select Rule Sets and click Create Rule Set.

3. Give the rule set a descriptive name (e.g., Promotional Email Cap or Overall Marketing Capping).

Step 2: Add Capping Rules to the Rule Set

1. Within the rule set, click Add Rule.

2. Configure the rule parameters:

  • Channel scope – Select which channels the rule applies to (e.g., Email, Push, SMS, or all channels).
  • Communication type – Optionally restrict to marketing vs. transactional messages.
  • Capping limit – Set the maximum number of messages allowed within a time window (e.g., 1 per day, 4 per month).
  • Time window – Define the rolling period (daily, weekly, monthly).

3. Repeat to add as many rules as needed within the rule set.

> Example combinations you can build:

> - A global rule: All channels – max 12 messages per month

> - A channel-specific rule: Push notifications – max 4 per month

> When combined, a profile is excluded from push notifications after 4, even if the global cap of 12 hasn't been reached yet.

Step 3: Activate the Rule Set

1. Once all rules are configured, activate the rule and the rule set it belongs to.

2. A rule set must be in an active state before it can be applied to journeys or campaigns.

Step 4: Apply Capping Rules to Campaigns

1. Open the campaign you want to apply the rule to.

2. In the campaign configuration, locate the Capping Rules or Business Rules section.

3. Select the relevant rule set from the dropdown.

4. The rules will take effect once the campaign is activated.

Step 5: Apply Capping Rules to Journeys

1. Open the journey and access its Properties.

2. In the Capping Rules dropdown, select the relevant rule set.

3. Activate the journey — the capping rules will take effect immediately.

> Important: If a journey is activated immediately (without a scheduled future start time), it can take up to 10 minutes for the system to begin suppressing customers. A warning message will display if the start time is less than 10 minutes away.

Step 6: Monitor Rule Set Exclusions

Once a journey is live, you can track exclusions in the journey report:

1. Open the Journey Report.

2. Locate the Journey Exclusions table — this provides detailed breakdowns of exclusions by rule set name and rule name.

3. The discard sub-reason field will show either:

  • `CAP_REACHED` – profile was excluded due to hitting the frequency cap
  • `LOWER_PRIORITY` – profile was excluded due to priority scoring

You can also use the Adobe Experience Platform Query Service to build custom queries identifying which rule caused a profile to not enter a given journey.

Suppression Rules (Suppression List)

The suppression list controls which email addresses and domains are automatically excluded from all sending within your organization's sandbox environment.

How the Suppression List Works

Journey Optimizer automatically collects and maintains a suppression list of email addresses excluded from journeys and campaigns, including:

  • Hard bounces – permanent delivery failures
  • Soft bounces – temporary delivery failures
  • Spam complaints – recipients who marked messages as spam

These addresses are suppressed across all mailings in a single client environment (specific to an organization ID associated with a sandbox ID).

> Note: Adobe also maintains a global suppression list of known bad addresses that is shared across all Adobe customers. Addresses on the global list are hidden — only the count of excluded recipients appears in delivery reports.

Accessing and Managing the Suppression List

1. Navigate to Administration in Journey Optimizer.

2. Open the Suppression List section.

3. From here you can:

  • Filter entries by address, domain, or suppression reason
  • Add specific addresses or domains manually
  • Remove entries when appropriate (e.g., after a re-permission campaign)
  • Download the suppression list for offline review

Using the Suppression REST API

For programmatic control, Journey Optimizer provides a Suppression REST API that allows you to:

  • Query suppressed addresses
  • Add addresses or domains to the suppression list
  • Remove addresses from the suppression list
  • Manage allow lists

This is especially useful for external systems that need to sync suppression logic with Journey Optimizer without manual UI intervention.

Frequency Capping for Offers (AJO Decisioning)

If you are delivering offers via AJO Decisioning, frequency capping works slightly differently:

1. Enable capping on the offer itself within the offer library.

2. Capture impression and interaction events via the Adobe Web SDK:

  • `decisioning.propositionDisplay` – tracks when an offer is shown
  • `decisioning.propositionInteract` – tracks when a user interacts with an offer

3. These events are mapped to XDM schemas in Adobe Experience Platform, allowing Journey Optimizer to accurately track how many times a user has seen or interacted with a given offer.

4. Once the cap is reached, the offer is automatically suppressed for that profile.

> Prerequisite: You must have a valid AJO campaign using Decisioning that is actively serving offers before configuring offer-level frequency capping.
