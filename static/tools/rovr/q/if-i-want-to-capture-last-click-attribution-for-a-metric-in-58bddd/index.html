Configuring Last Touch (Last Click) Attribution for a Metric in CJA

There are two ways to apply last touch attribution to a metric in Customer Journey Analytics:

Option 1: Set It in the Data View (Default for All Reports)

This is the recommended approach if you want last touch attribution to be the default for a metric across all reports.

Prerequisites: You must be a CJA Admin with access to edit Data Views.

1. Navigate to [!UICONTROL Data Management] > [!UICONTROL Data Views] in CJA.

2. Open the Data View you want to edit.

3. Locate the metric component you want to configure in the [!UICONTROL Components] tab.

4. Click on the metric to open its component settings.

5. Scroll to the [!UICONTROL Attribution] section.

6. Set the [!UICONTROL Attribution Model] to [!UICONTROL Last Touch].

7. Set the [!UICONTROL Lookback Window] as appropriate (e.g., Person, Session, or a custom time window).

8. [!UICONTROL Save] the Data View.

> This setting becomes the default for that metric in every report using this Data View. You can also duplicate the metric and assign different attribution models to each copy for side-by-side comparison.

Option 2: Override Attribution in a Freeform Table (Per-Report Basis)

This is useful when you want to apply last touch attribution only for a specific analysis without changing the Data View default.

1. Open your project in [!UICONTROL Analysis Workspace].

2. Drag the metric into a [!UICONTROL Freeform Table].

3. Right-click the metric's column header.

4. Select [!UICONTROL Use non-default attribution model].

5. In the attribution dialog, choose [!UICONTROL Last Touch] as the model.

6. Set the desired [!UICONTROL Lookback Window].

7. Click [!UICONTROL Apply].

Key Considerations

  • No default model is applied in CJA unless you explicitly configure one. This is different from GA4, which defaults to data-driven attribution.
  • Attribution in CJA is applied at report time, meaning any model change applies retroactively to all historical data without reprocessing.
  • If you need to compare Last Touch against other models (e.g., First Touch, Algorithmic), use the [!UICONTROL Duplicate] feature in the Data View to create multiple copies of the same metric, each with a different attribution setting — for example, naming them `Total Revenue (Last Touch)` and `Total Revenue (First Touch)`.
