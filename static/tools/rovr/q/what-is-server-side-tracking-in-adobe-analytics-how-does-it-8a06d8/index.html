What is Server Side tracking in Adobe Analytics? How does it work?

Server-Side Forwarding in Adobe Analytics

Server-side forwarding is a data collection feature that automatically sends data from Adobe Analytics to other Adobe Experience Cloud solutions (like Audience Manager) in real time, without requiring additional page calls.

How It Works

When server-side forwarding is enabled, Analytics processes your data on Adobe's servers and forwards it to other CX Enterprise solutions during the data collection process. This happens automatically after your Analytics hit is sent — no additional client-side code is needed to pass data to these other solutions.

Key Benefits

Server-side forwarding improves your implementation in several ways:

  • Reduces page load time — Eliminates the need for separate data collection calls (like DIL calls in Audience Manager). Fewer calls mean faster page performance and a better customer experience.
  • Enables real-time data sharing — Allows Analytics to push data to other CX Enterprise solutions, and those solutions can push data back to Analytics during the same collection process.
  • Follows best practices — Represents the recommended approach for implementing code across Adobe's CX Enterprise solutions.
  • Simplifies implementation — Removes the need to manually forward data through additional tags or code.

Important Considerations

GDPR & Privacy Compliance: If you need to comply with EU cookie regulations, Analytics can restrict pre-consent data from being forwarded to Audience Manager until proper consent is received. You can flag hits where consent hasn't been received to prevent them from being server-side forwarded.

Legacy vs. New Methods: Adobe continues to support the legacy tracking server-based server-side forwarding method, but the newer report suite-based server-side forwarding is recommended — especially if you want to share Audience Manager segments back into Analytics.
