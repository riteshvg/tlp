What tools or features does Adobe provide to help automate the migration process from Adobe Analytics to Customer Journey Analytics?

Automation Tools and Features for Migrating from Adobe Analytics to Customer Journey Analytics

Based on the retrieved documentation, here is what Adobe provides to help automate or streamline the migration process:

1. Component Migration UI Tool

Adobe provides a built-in migration tool within the Adobe Analytics Admin interface that automates several parts of the migration:

  • Automatic re-creation of Adobe Analytics projects in Customer Journey Analytics
  • Automatic mapping of certain dimensions and metrics from Adobe Analytics report suites to CJA data views (not everything requires manual mapping)
  • Migration summary report — automatically generated at the end of migration, listing all migrated components

2. Adobe Analytics 2.0 Migration APIs

For organizations that prefer a programmatic approach, Adobe provides two dedicated migration API endpoints via the Adobe Analytics 2.0 APIs. These can automate migration of:

  • Segments
  • Calculated metrics
  • Date ranges
  • Dimensions and metrics (when mappings are pre-configured)
  • Full Analysis Workspace projects

To use the APIs, you need:

  • Adobe Analytics project ID
  • Adobe Analytics report suite ID
  • Customer Journey Analytics data view ID

> ⚠️ Important sequencing note: If you plan to migrate dimensions and metrics via API, you must run the mappings API endpoints first before running the project migration endpoints, so the project ID is correctly associated with those mappings.

All API definitions are available in the Select a definition drop-down menu within the Adobe Analytics APIs reference.

3. Adobe Analytics Inventory Tool

Available to Analytics administrators, the Analytics Inventory tool helps automate the discovery and audit phase of migration by:

  • Cataloging existing projects, components, and assets across your Adobe Analytics implementation
  • Helping you understand the scope of what needs to be migrated before you begin

This is a recommended first step before using either the UI migration tool or the APIs.

What Is Not Automated

The documentation makes clear that some aspects still require manual effort:

| Task | Automation Level |

|---|---|

| Dimension & metric mapping (some) | 🔶 Partially automated — some auto-map, others require manual mapping |

| Web SDK schema field mapping | ❌ Must be done entirely manually |

| User onboarding and training | ❌ Manual — plan for 3–6 months |

| Connection and data view setup in CJA | ❌ Manual prerequisite |

Recommended Sequence for Automation

1. Run the Analytics Inventory Tool to understand your scope

2. Set up your CJA connection and data view (manual prerequisite)

3. Use the Component Migration UI for a guided, partially automated migration

4. Optionally use the Adobe Analytics 2.0 Migration APIs for bulk or programmatic migration, running mappings APIs before project migration APIs

The documentation does not describe any additional third-party or AI-assisted automation tools beyond what is listed above. If you need details on specific API endpoint structures or the inventory tool's capabilities, I'd recommend exploring the Adobe Analytics 2.0 APIs reference directly.
