Can I use datastream overrides for specific user segments or conditions?

What the documentation confirms

Yes, datastream overrides can be applied conditionally. The CONTEXT confirms that datastream overrides allow you to define additional configurations for your datastreams and conditionally trigger different datastream behaviors without creating a new datastream. The documentation provides examples of overrides being used for specific business scenarios:

  • Sending data to country-specific datastreams based on customer actions (e.g., placing an order)
  • Routing data to different datastreams for different business units
  • Overriding specific configurations like Analytics report suites, Target property tokens, Experience Platform datasets, and Audience Manager ID sync containers

The CONTEXT indicates these overrides are configured via the Web SDK tag extension and datastreams UI, and can be applied on a per-environment basis (Development, Staging, Production).

What it does NOT cover

The documentation does not specify:

  • How to define conditions based on user segment membership or audience qualification
  • Whether Real-Time CDP audiences or Analytics segments can directly trigger overrides
  • The specific conditional logic syntax or operators available
  • How to apply overrides based on visitor characteristics or behavioral segments
