Prerequisites for Configuring the Email Channel in Adobe Journey Optimizer

Overview

Before you can send any emails through AJO, several prerequisites must be met at the organizational, technical, and administrative levels. These prerequisites ensure deliverability, security, compliance, and proper sender reputation.

1. 🏢 Organizational & Licensing Prerequisites

Adobe Journey Optimizer License

  • ✅ Your organization must have Adobe Journey Optimizer (AJO) purchased and activated
  • ✅ Alternatively, Adobe Real-Time CDP with Journey Optimizer add-on
  • ❌ Email channels are NOT available in standalone Adobe Analytics or Adobe Target

Adobe Experience Platform (AEP)

  • ✅ AJO is built on AEP, so AEP must be active in your organization
  • ✅ You need access to AEP Admin Console for user provisioning
  • ✅ Real-Time Customer Profiles must be enabled (enabled by default in most orgs)

Required Feature Flags

  • ✅ Email channel must be enabled for your organization
  • ✅ Contact Adobe Support if email channel is not visible in the UI
  • ✅ This is typically enabled by default but can be disabled for regulatory reasons

2. 👤 User Roles & Permissions

Required Adobe Admin Console Roles

Before anyone can configure email, they need the correct roles in Adobe Admin Console.

For Email Channel Configuration:

| Role | Permissions | Required For |

|---|---|---|

| System Administrator | Full AJO access, can assign roles | Managing all configurations |

| Journey Administrator | Can create/manage channels, email configs | Subdomain delegation, IP pools |

| Journey Approver | Can approve journeys/campaigns | Reviewing email before publish |

| Journey Manager | Can create journeys/campaigns | Building email journeys |

| Journey Viewer | Read-only access to journeys/campaigns | Viewing reports only |

Specific Permissions Needed:

For Subdomain Delegation:

```

Required Permissions:

├── Manage Email Channel ✅

├── Manage Message Presets (Channel Configurations) ✅

├── Administer Delivery ✅

└── System Administrator (recommended) ✅

```

For IP Pool Creation:

```

Required Permissions:

├── Manage IP Pools ✅

├── Administer Delivery ✅

└── System Administrator (recommended) ✅

```

For Email Content Creation:

```

Required Permissions:

├── Manage Email Messages ✅

├── Publish Messages ✅

└── Edit Messages ✅

```

How to Assign Roles in Adobe Admin Console:

1. Go to Adobe Admin Console (adminconsole.adobe.com)

2. Select your Organization

3. Click Products → Adobe Journey Optimizer

4. Click Product Profiles

5. Select or create a profile (e.g., "Email Marketers")

6. Click Edit Permissions

7. Enable required permissions for email:

```

✅ Channel - Email

✅ Message Presets

✅ Email Content

✅ Journeys

✅ Campaigns

```

8. Click Save

9. Add users to the product profile

3. 🌐 Domain & DNS Prerequisites

Own a Sending Domain

You must own or have authorization to use a domain for sending emails.

```

Examples of valid domains:

├── yourbrand.com ✅

├── marketing.yourbrand.com ✅

├── email.yourbrand.com ✅

└── notifications.yourbrand.com ✅

Invalid:

├── gmail.com ❌ (not your domain)

├── someoneelses.com ❌ (unauthorized)

└── no-domain-just-email ❌ (not a domain)

```

DNS Access & Control

You must have access to your domain's DNS records to set up email authentication.

```

Who Needs DNS Access:

├── IT/Network team

├── Domain registrar admin

└── DNS provider administrator

```

DNS Provider

Your domain must be registered with a DNS provider that allows you to manage:

  • SPF records
  • DKIM records
  • DMARC records
  • CNAME records (for subdomain delegation)

Common DNS Providers:

```

├── GoDaddy

├── Cloudflare

├── AWS Route 53

├── Azure DNS

├── Google Domains

├── Namecheap

├── Bluehost

└── Any registrar with DNS management

```

Email Authentication Standards

You must understand these email authentication protocols:

| Protocol | Purpose | Who Sets It Up |

|---|---|---|

| SPF (Sender Policy Framework) | Specifies which servers can send from your domain | Adobe (via delegation) |

| DKIM (DomainKeys Identified Mail) | Cryptographically signs emails from your domain | Adobe (via delegation) |

| DMARC (Domain-based Message Authentication) | Provides authentication policy and reporting | Adobe (via delegation) |

> ℹ️ Good news: When you delegate a subdomain to AJO, Adobe automatically sets up SPF, DKIM, and DMARC. You just add the DNS records provided by AJO to your DNS provider.

4. 📊 Data & Profile Prerequisites

Real-Time Customer Profiles

Real-Time Customer Profiles must be enabled in your AEP organization.

```

Check if Enabled:

1. Go to AEP → Data Management → Profiles

2. You should see "Real-time profiles are enabled"

3. If not enabled, contact Adobe Support

```

What this means:

  • ✅ Customer data is unified across datasets
  • ✅ Profiles can be personalized in real-time
  • ✅ Segment membership is evaluated in real-time

Identity Graph (Identity Service)

Identity Service must be configured to resolve customer identities.

```

Required Identity Namespaces:

├── Email (required for email sends) ✅

├── Phone (if using SMS) ✅

├── ECID (Experience Cloud ID) ✅

└── Any custom identifiers ✅

```

Ensure Email Identity Namespace is Configured:

```

1. Go to AEP → Data Management → Identities

2. Verify "Email" namespace exists

3. If not, create it:

├── Name: Email

├── Type: Email

└── Primary: YES (for email channel)

```

AEP Datasets & Schemas

At least one dataset with the following must exist:

```

Required XDM Schema Fields:

├── identityMap (for identifying profiles) ✅

├── personalEmail.address (for email sending) ✅

├── person.name (for personalization) ✅

├── consentPreferences (for consent checking) ✅

└── Any custom fields for personalization ✅

```

Audience Segments

You should have at least one audience/segment created before building journeys.

```

Audiences can be created via:

├── Segment Builder (rule-based)

├── Audience Composition (multi-step)

└── Audience Portal (custom uploads)

```

5. 🔐 Security & Compliance Prerequisites

Data Governance

Data Governance policies must be configured:

```

Required Governance Setup:

├── Data Usage Labels applied to datasets ✅

│ (C labels, I labels, S labels)

├── Marketing Action Policies defined ✅

│ ("Email Marketing" action exists)

└── Policy enforcement enabled ✅

```

Why it matters:

  • ✅ Ensures sensitive data isn't accidentally used for unauthorized purposes
  • ✅ Required for GDPR, CCPA, HIPAA compliance
  • ✅ Prevents sending emails with restricted data fields

Consent Management

Consent preferences must be captured in customer profiles
