However, I should clarify: identityMap is not an Adobe Journey Optimizer (AJO) feature. It's actually part of Adobe Experience Platform (AEP) and Adobe Data Collection.

If you're asking about identityMap in the context of AEP or Data Collection, I can help! Here's what identityMap is used for:

Purpose of identityMap

identityMap is a standard schema field group used to manage and organize user identities in Adobe Experience Platform. It serves these key purposes:

  • Consolidates Multiple Identities: Allows you to include one or more user identifiers in a single event payload, each keyed by namespace (like `CRMID`, `Email`, `Phone`, `LoyaltyId`, or `ECID`)
  • Identity Resolution: Helps AEP's Identity Service link and unify different identifiers that belong to the same person across devices and channels
  • Primary Identity Designation: Lets you mark which identity is the primary identifier for an event (exactly one must be marked `primary: true`)
  • Authentication Context: Captures how confidently you know an identity belongs to the visitor through the `authenticatedState` field (`authenticated`, `ambiguous`, or `loggedOut`)

Example Structure

```json

{

"identityMap": {

"CRMID": [

{

"id": "abc-123-xyz",

"authenticatedState": "authenticated",

"primary": true

}

],

"Email": [

{

"id": "user@example.com",

"authenticatedState": "authenticated",

"primary": false

}

]

}

}

```
