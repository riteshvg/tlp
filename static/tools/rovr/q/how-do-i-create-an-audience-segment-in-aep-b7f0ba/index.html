How do I create an audience segment in AEP?

Creating an Audience Segment in Adobe Experience Platform

There are two primary methods to create an audience segment in AEP: using the Segment Builder UI or the Segmentation Service API.

Prerequisites

  • Access to Adobe Experience Platform with appropriate permissions
  • Profile data ingested and available in your sandbox
  • Familiarity with your XDM schema structure

Option 1: Using the Segment Builder UI

The Segment Builder provides a rich, intuitive workspace with drag-and-drop controls for building segment definitions against your Profile data.

1. Navigate to Audiences in the left navigation of the AEP UI.

2. Click Create Audience.

3. Select Build rule to open the Segment Builder workspace.

4. In the left panel, browse or search for attributes, events, or audiences to use as building blocks.

5. Drag and drop the desired data properties into the rule builder canvas.

6. Configure conditions, operators, and values for each rule.

7. Use AND/OR logic to combine multiple rules as needed.

8. Give your segment a Name and optionally a Description.

9. Review the estimated audience size in the Audience Properties panel on the right.

10. Click Save to finalize and publish your segment definition.

Option 2: Using the Segmentation Service API

You can develop, test, preview, and save segment definitions programmatically using the Segmentation Service API.

Step 1: Develop the Segment Definition

1. Set up your API authentication via the Adobe Developer Console.

2. Make a POST request to the segment definitions endpoint:

`POST /segment/definitions`

3. Include your segment definition in PQL (Profile Query Language) format in the request body. For example:

```json

{

"name": "My Segment",

"description": "Customers who viewed a product in the last 30 days",

"expression": {

"type": "PQL",

"format": "pql/text",

"value": "chain(xEvent, timestamp, [A: WHAT(eventType = 'productView') WHEN(< 30 days before now)])"

},

"schema": {

"name": "_xdm.context.profile"

}

}

```

Step 2: Preview and Test the Segment

1. Make a POST request to create a preview job:

`POST /segment/previews`

2. Pass the `segmentDefinitionId` returned from Step 1.

3. Poll the preview job status with a GET request:

`GET /segment/previews/{previewId}`

4. Review the estimated profile count and sample profiles returned.

Step 3: Save and Activate

1. Once satisfied with the preview, confirm the segment definition is saved by making a GET request:

`GET /segment/definitions/{segmentDefinitionId}`

2. Activate the segment to destinations as needed via the Destinations workflow.

Additional Audience Creation Methods

Beyond Segment Builder and the API, AEP also supports:

  • Audience Composition — a drag-and-drop workflow for combining, enriching, and splitting existing audiences. Access via Compose audiences in the Audiences portal.
  • Federated Audience Composition — build new audiences from enterprise data warehouse datasets without copying underlying data, then store them in AEP Audience Portal.
  • Data Distiller — use SQL extensions to build audiences directly from the data lake, including dimension entities like customer attributes or product information.

Key Tips

  • Use dynamic segments when you want audience membership to update automatically as new profile data arrives.
  • The Segment Builder left panel shows available attributes and events based on your XDM schemas — familiarity with your schema structure helps greatly.
  • Segment definitions use the XDM Segment Definition class to capture details like the segment ID, name, and other attributes.
