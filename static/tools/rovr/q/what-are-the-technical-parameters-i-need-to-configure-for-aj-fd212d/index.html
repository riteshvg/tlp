What are the technical parameters I need to configure for AJO email channels?

Technical Parameters for AJO Email Channel Configuration

Before configuring, ensure you have these prerequisites in place:

1. Subdomains delegated to Adobe

2. At least one IP pool created

3. The Manage messages presets permission assigned to your role

Step-by-Step Configuration Process

Step 1 — Delegate Subdomains

Delegate to Adobe the subdomains you will use to send emails. These subdomains determine elements such as:

  • Web pages to be tracked
  • Mirror page URLs

Step 2 — Create IP Pools

Group together the IP addresses provisioned with your instance into IP pools.

Step 3 — Create a Channel Configuration

Create a channel configuration and select the Email channel.

Step 4 — Configure Technical Parameters

Within each email channel configuration, the following technical parameters must be defined:

| Parameter | Description |

|---|---|

| Email Type | Define whether the email is marketing or transactional |

| Subdomain | Select the delegated subdomain for sending |

| IP Pool | Assign the IP pool to use for delivery |

| List Unsubscribe | Configure one-click unsubscribe links for compliance |

| Header Parameters | Set sender name, sender email, reply-to, and error routing addresses |

| BCC | Configure blind carbon copy for archiving/auditing purposes |

| Seed List | Add seed addresses for testing and monitoring |

| Retry Logic | Define retry settings for failed deliveries |

| URL Tracking | Enable and configure URL tracking for analytics |

| Execution Address | Set the execution fields used when sending |

How Configuration Changes Are Applied

Once a configuration is updated, it is picked up automatically in journeys and campaigns using it, following this logic:

  • Batch journeys — Changes apply at the next recurrence or new execution (not to already-started batch executions)
  • Transactional messages — Changes are picked up immediately for the next communication (with up to a 5-minute delay)
