What are some common use cases for datastream overrides?

Common Use Cases for Datastream Overrides

Multi-Region Data Collection

One of the primary use cases is handling data collection across different geographic regions or countries. For example:

  • International companies with separate websites for different countries can dynamically route data to region-specific datastreams
  • You can send data to a global datastream for important actions (like purchases or profile updates) while also routing regional data to country-specific datastreams
  • This eliminates the need to create and maintain separate implementations for each region

Multiple Business Units

Companies with distinct business units or divisions can leverage datastream overrides to:

  • Ensure each business unit receives data in its own datastream without mixing data
  • Avoid sending all data to a single default datastream
  • Maintain separate data flows for different departments or product lines
  • Keep data isolated and properly organized by business unit

Other Strategic Applications

Datastream overrides are useful when you need:

  • Different data destinations for different user segments or contexts
  • Conditional routing based on user behavior, attributes, or conditions
  • Dynamic configuration changes without modifying your website or app code
  • A/B testing or phased rollouts of new datastream configurations

Important Limitations

Keep in mind that datastream overrides should NOT be used for:

  • Personalization use cases
  • Consent data handling

For these scenarios, you should use other Adobe solutions designed specifically for those purposes.
