How do I configure a destination in Real-Time CDP?

Configuring a Destination in Real-Time CDP

Destinations are pre-built integrations with destination platforms that allow for the seamless activation of data from Adobe Experience Platform. You can use destinations to activate your known and unknown data for cross-channel marketing campaigns, email campaigns, targeted advertising, and many other use cases.

Prerequisites

  • Access to Real-Time Customer Data Platform in Adobe Experience Platform
  • Appropriate permissions to manage destinations in your organization
  • An existing audience or segment ready to activate
  • Any destination-specific credentials or account details required (e.g., advertiser account ID, API keys, etc.)

Step 1: Navigate to the Destinations Catalog

1. Log in to Adobe Experience Platform.

2. In the left navigation, click Destinations.

3. Click the Catalog tab to browse all available destinations.

4. Use the search bar or filter by category (Advertising, Social, Email, Cloud Storage, etc.) to find your desired destination.

Step 2: Connect to a Destination

1. Click on your desired destination card in the catalog.

2. Click Set up or Activate audiences (if already configured by your organization).

3. You will be taken to the destination's configuration screen.

4. Enter the required authentication credentials for the destination (e.g., API key, OAuth login, advertiser account ID).

5. Click Connect to destination.

6. Provide a Name and optionally a Description for this destination connection.

7. Configure any destination-specific settings (e.g., data export frequency, identity namespace mappings).

8. Click Next to proceed.

Step 3: Configure Activation Settings

Select Audiences

1. On the Activate audiences screen, select the audiences you want to send to this destination.

2. Click Next.

Map Identities (if required)

Some destinations — particularly advertising and social platforms — require identity mapping so AEP knows which identifier to use for matching (e.g., email, phone, IDFA):

1. In the Mapping step, click Add new mapping.

2. Select the Source field (the identity or attribute from your profile).

3. Select the Target field (the identifier the destination expects).

4. Repeat for any additional identity mappings required.

5. Click Next.

Schedule Activation

1. Set the Start date (and optionally an End date) for the audience export.

2. Choose the export frequency:

  • Streaming — sends updates in real time as profiles qualify/exit audiences
  • Batch — sends full audience exports on a scheduled cadence (daily, etc.)

3. Click Next.

Step 4: Review and Finish

1. On the Review screen, verify all your configuration settings — destination name, selected audiences, identity mappings, and schedule.

2. If everything looks correct, click Finish.

Your destination is now active and audiences will begin flowing to the connected platform according to your configured schedule.

Available Destination Types

Real-Time CDP supports more than 60 native destinations, including:

| Category | Examples |

|---|---|

| Social | Facebook, LinkedIn, TikTok, Snapchat |

| Advertising | Google Customer Match, The Trade Desk, Amazon Ads, Pinterest |

| Email | Marketo, Salesforce Marketing Cloud |

| Cloud Storage | Amazon S3, Azure Blob, Google Cloud Storage |

| Analytics | Adobe Analytics, Customer Journey Analytics |

Over 20 of these destinations are advertising or social destinations that specifically support first-party audience matching — critical for cookieless targeting strategies.

People-Based Destinations Note

If you are migrating from Adobe Audience Manager and were previously using people-based destinations (Facebook, Google Customer Match, LinkedIn), Adobe strongly recommends:

1. Push those segments to Real-Time CDP via the Audience Manager Source Connector.

2. Re-activate them through Real-Time CDP destinations rather than continuing to use Audience Manager directly.

All people-based destinations available in Audience Manager are also available natively in Real-Time CDP.
