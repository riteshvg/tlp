How to configure a consent policy in RTCDP to evaluate IAB consent string to activate profiles to downstream destinations?

Configuring IAB TCF 2.0 Consent Policy in Real-Time CDP for Downstream Destination Activation

Here is a comprehensive guide to configuring consent policy enforcement using IAB TCF 2.0 consent strings in Real-Time CDP, from data ingestion through to destination activation.

Prerequisites

  • Adobe Healthcare Shield or Privacy and Security Shield add-on (required for custom consent policies)
  • A configured Consent Management Platform (CMP) on your website (e.g., OneTrust, Sourcepoint)
  • IAB TCF 2.0 vendor registration (Adobe Experience Platform vendor ID = 565)
  • Adobe Experience Platform Web SDK or Mobile SDK deployed

Step 1: Capture IAB Consent Strings via Your CMP

Your CMP dialog must allow customers to opt in or out of the following before a TCF consent string is generated:

| Consent Option | Requirement |

|---|---|

| Purpose 1 | Store and/or access information on a device |

| Purpose 10 | Develop and improve products |

| Vendor 565 | Adobe Experience Platform vendor permission |

Step 2: Collect and Ingest Consent Data into Experience Platform

Option A: Web SDK (Recommended)

Use Adobe Tags (Launch) to listen for CMP consent-change events and send them to AEP via the Web SDK.

1. In your Tag property, create a Rule (e.g., `setConsent – IAB`).

2. Add a Custom Code event named `tcfapi addEventListener` with the following code:

```js

function addEventListener() {

if (window.__tcfapi) {

window.__tcfapi("addEventListener", 2, function (tcData, success) {

if (success && (tcData.eventStatus === "useractioncomplete" || tcData.eventStatus === "tcloaded")) {

// Capture the consent string and GDPR applicability into data elements

_satellite.setVar("IAB TCF Consent String", tcData.tcString);

_satellite.setVar("IAB TCF Consent GDPR", tcData.gdprApplies);

trigger();

}

});

} else {

setTimeout(addEventListener, 100);

}

}

addEventListener();

```

3. Configure the Set Consent action in the rule to pass `%IAB TCF Consent String%` and `%IAB TCF Consent GDPR%` to the Web SDK.

> Note: You must type the data element names with percent signs manually — the data element selector cannot be used for custom code variables.

Option B: Mobile SDK

Integrate the Consent for Edge Network mobile extension and use the Consent API to send updates whenever consent changes occur in your mobile app.

Option C: Batch Ingestion

Map collected IAB TCF consent data to your consent dataset's XDM schema and ingest via batch upload.

Step 3: Configure the Consent Schema and Dataset for Real-Time Customer Profile

1. Open Schemas in the AEP UI and select your consent schema.

2. Enable the schema for Real-Time Customer Profile by toggling the Profile switch in the Schema properties panel.

3. When prompted about a missing primary identity:

  • Use the `identityMap` field as the primary identity (recommended for IAB use cases).
  • Do not use directly identifiable fields (e.g., email address) as primary identity for interest-based advertising contexts — consult your legal counsel.

4. Select Save.

> Each identity value in the schema must include an `identityIABConsent` property conforming to the Consent String XDM data type.

Step 4: Understand How AEP Processes IAB Consent Data

Once ingested, AEP enforces consent at two levels:

1. Profile level: Real-Time Customer Profile updates stored consent data for that customer's profile.

2. Identity cluster level: AEP processes customer IDs only if vendor permission for AEP (565) is granted for every ID in a cluster.

3. Destination activation level: When exporting audiences, AEP includes a profile only if vendor permissions for both:

  • Adobe Experience Platform (565), AND
  • The individual destination vendor

…are granted for every ID in the cluster.

Step 5: Create a Consent Policy in AEP

> Requirement: Healthcare Shield or Privacy & Security Shield license.

1. Navigate to Experience Platform > Data Governance > Policies.

2. Select Create Policy and choose Consent Policy as the policy type.

3. In the If (condition) section, configure the marketing action you want to govern (e.g., email targeting, paid media).

4. In the Then (result) section:

  • Select the consent/preference attribute from the union schema (e.g., the IAB TCF consent field).
  • Define which attribute value qualifies a profile for inclusion.

5. Use Add result to stack multiple consent attributes if needed.

6. Enter a Name and optional Description, then select Save.

7. In the right rail, toggle the Status switch to Enabled.

Step 6: Apply Consent Policies in Downstream Activation

For Destinations (Real-Time CDP)

Consent policies are automatically enforced during audience activation workflows — no manual attribute tagging by marketers is needed.

To preview enforcement before activation:

1. Begin the audience activation workflow to your destination.

2. At the Review step, select View applied policies.

3. The UI will show which consent policies are applied and display the Estimated Qualified Profiles count — reflecting only profiles that meet all consent requirements.

Continuous Enforcement (Batch & Streaming Destinations)

For both batch and streaming destinations (including paid media), consent enforcement is proactive:

  • Profiles are automatically removed from destination audiences when consent status changes.
  • Consent changes propagate immediately, ensuring only consented profiles are targeted at all times.

Summary Flow

```

CMP Dialog → IAB TCF Consent String Generated

↓

Web SDK / Mobile SDK / Batch → Ingested into AEP Dataset

↓

Real-Time Customer Profile Updated with Consent Data

↓

Only Qualifying Profiles Activated to Downstream Destinations
```
