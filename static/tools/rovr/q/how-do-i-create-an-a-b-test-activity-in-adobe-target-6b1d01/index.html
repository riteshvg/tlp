How do I create an A/B test activity in Adobe Target?

Creating an A/B Test Activity in Adobe Target

Adobe Target provides three types of A/B Test activities: Manual (Default), Auto-Allocate, and Auto-Target. The steps below cover creating a Manual A/B Test using the Visual Experience Composer (VEC).

Prerequisites

  • Access to an active Adobe Target account
  • A Target-enabled page with the Target library implemented
  • (For Auto-Target) A Target Premium license — Auto-Target is not available in Target Standard

Step-by-Step: Create a Manual A/B Test

Step 1 — Start the Activity

1. From the Activities list, click Create Activity > A/B Test

> Note: Available activity types depend on your Target account. Some types may not appear in your list. For example, Recommendations is a Target Premium feature.

Step 2 — Choose Your Composer

From the Create A/B Test Activity dialog, select your preferred composer:

  • Visual (Default) — Use the Visual Experience Composer (VEC) to make changes directly on your page
  • Form — Use the Form-Based Experience Composer for non-visual implementations (email, IoT, etc.)

> In addition to the VEC and Form-Based Experience Composer, Target also offers a Single Page Application (SPA) VEC for SPA frameworks.

Step 3 — Select a Workspace (Target Premium Only)

If you are a Target Premium customer, use the Choose Workspace drop-down to select the appropriate workspace for this activity.

Step 4 — Enter Your Activity URL

1. Specify your activity URL — the page where the test will run

2. Click Next (or Create depending on your UI version)

Your account's default URL will pre-populate if configured. You can change it to any other URL.

The Visual Experience Composer opens displaying the page at that URL.

Step 5 — Name Your Activity

Type a descriptive name for the activity in the name field provided.

> Important: The activity name cannot begin with any of these characters:

| Character | Description |

|-----------|-------------|

| `=` | Equals to |

| `+` | Plus |

| `-` | Minus |

| `@` | At sign |

Step 6 — Create Your Experiences

The VEC displays two tabs by default:

  • Experience A — The control (default) experience
  • Experience B — The alternate experience you modify

Key actions at this step:

  • Modify Experience B by clicking and changing elements directly on the page
  • Click Add to add additional experiences beyond A and B
  • Optionally delete Experience A if you don't want the default site experience as a test option

Step 7 — Configure Targeting

Click Targeting at the top of the VEC to move to the next step in the three-step guided workflow.

At this step you will:

  • Choose your audience — Who will see this test
  • Set traffic allocation — Choose between:
  • Manual split — You define the percentage per experience
  • Auto-Allocate — Target automatically sends more traffic to the winning experience
  • Auto-Target — Target uses machine learning to serve the most tailored experience to each visitor (requires Target Premium)

When satisfied with your audience and traffic allocation, click Next.

Step 8 — Set Goals and Settings

Specify the goals and settings for the activity, including:

  • Primary goal metric — What defines success (e.g., conversion, revenue)
  • Additional metrics — Optional secondary metrics to track
  • Reporting settings — Including whether to use Analytics for Target (A4T)

> Note: If using Analytics for Target (A4T) with Auto-Allocate or Auto-Target, review the specific A4T support considerations for those activity types before proceeding.

Step 9 — Save the Activity

Click either:

  • Save & Close — Saves and returns you to the Activities list
  • Save — Saves and keeps you in the activity editor

After saving, the Overview tab displays a summary and diagram of your activity.
