Moving Packages Between Sandboxes in Adobe Experience Platform

Here's a complete guide on how to create a package and import it into another sandbox using Sandbox Tooling.

▶ Watch: Sandbox Tooling Overview

Prerequisites

  • You must have access to the Sandboxes section in Adobe Experience Platform UI.
  • The package must be published before it can be imported into a target sandbox.
  • Only sandboxes within your organization are available for import (unless sharing across orgs).

Part 1: Create a Package

1. Navigate to the Sandboxes menu and select the Packages tab.

2. Click Create package.

3. In the Create package dialog, select the option to Paste package payload, then select Select.

4. Paste the copied package payload into the text field and select Create.

> You can monitor the sharing/creation status under the Sharing status tab, where the current status is shown in the Sharing status column.

Part 2: Publish the Package

Before importing, the package must be published. You can do this via the API:

API Format:

```http

GET /packages/{PACKAGE_ID}/export

```

Example Request:

```shell

curl -X GET \

https://platform.adobe.io/data/foundation/exim/packages/{PACKAGE_ID}/export \

-H 'Authorization: Bearer {ACCESS_TOKEN}' \

-H 'x-api-key: {API_KEY}' \

-H 'x-gw-ims-org-id: {ORG_ID}' \

-H 'x-sandbox-name: {SANDBOX_NAME}'

```

> Note: By default, the package expires 90 days after publishing. You can customize this with the `expiryPeriod` property (in days).

Part 3: Import the Package into a Target Sandbox

1. Navigate to the Sandboxes menu and select the Browse tab.

2. Search for the target sandbox and click the + icon next to its name.

3. In the Target sandbox field, confirm the correct sandbox is selected.

4. From the Package name drop-down list, select the package you want to import.

5. Add a Job name for future monitoring.

6. Toggle Enable schemas for profile if you want unified profile enabled on imported schemas.

7. Click Next.

8. Review the Package objects and dependencies page — this lists all objects included such as:

  • Audiences
  • Schemas
  • Events
  • Actions
  • Journeys

> For each object, you can choose to create a new one or use an existing one in the target sandbox (helpful to avoid duplicates).

9. If there are missing attributes, they will be displayed at the top of the page. Select View details for a breakdown.

10. Click Finish (top-right corner) to start copying the package to the target sandbox.

Part 4: Monitor & Verify the Import

  • Click the import job to review the copy result.
  • Use View imported objects to see each individual object copied.
  • Use View import details to check the import results per object.

  • Finally, access your target sandbox and perform a thorough check of all copied objects.

> Note: All import actions are recorded in the audit logs.

Best Practices

  • Import a package containing an entire sandbox into an empty sandbox for cleanest results.
  • Regularly update your golden sandbox package as your configurations evolve.
  • You can also share packages across organizations using the package sharing feature.
